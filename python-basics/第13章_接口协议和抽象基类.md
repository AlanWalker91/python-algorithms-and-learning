## 3. 两种协议：动态协议 vs 静态协议

```python
# ---- 动态协议（非正式接口，隐式约定）----
# 只要实现了所需的方法，就满足协议
# 无须声明，Python 解释器自动识别

class Vowels:
    def __getitem__(self, i):
        return 'AEIOU'[i]     # 只实现了序列协议的一部分

v = Vowels()
v[0]       # 'A'    ← __getitem__ 支持
for c in v:        # ← 迭代（Python 用 __getitem__ 兜底）
    print(c)
'E' in v   # True  ← in 操作符（全面扫描，Python 自动处理）
# 没有 __len__，仍然"算是"序列！


# ---- 静态协议（Python 3.8+，需显式定义）----
from typing import Protocol

class Drawable(Protocol):
    def draw(self) -> None: ...   # 必须实现这个方法才满足协议

# 无须继承 Drawable，只要有 draw 方法就行
class Circle:
    def draw(self) -> None:
        print('O')

def render(obj: Drawable) -> None:  # 类型检查工具会验证
    obj.draw()

render(Circle())   # ✅ 运行时和静态检查都通过
```

**两种协议的关键区别**：

| | 动态协议 | 静态协议 |
|--|---------|---------|
| 定义方式 | 隐式，文档中描述 | 显式，typing.Protocol 子类 |
| 实现要求 | 可以只实现部分 | 必须实现所有声明的方法 |
| 检查方式 | 运行时（隐式）| 静态工具（Mypy、PyCharm 等）|
| 继承要求 | 不需要 | 不需要（结构类型）|
| 声明支持 | 不能用类型提示声明 | 可用于类型提示 |

## 4. 鸭子类型编程

### 4.1 序列协议：Python 的极致支持

```python
# FrenchDeck 没有继承任何序列 ABC
# 只实现了 __getitem__ 和 __len__
# 但 Python 自动为它提供迭代、in 操作、切片等功能

import collections

Card = collections.namedtuple('Card', ['rank', 'suit'])

class FrenchDeck:
    ranks = [str(n) for n in range(2, 11)] + list('JQKA')
    suits = 'spades diamonds clubs hearts'.split()

    def __init__(self):
        self._cards = [Card(rank, suit) for suit in self.suits
                       for rank in self.ranks]

    def __len__(self):
        return len(self._cards)

    def __getitem__(self, position):
        return self._cards[position]


deck = FrenchDeck()
len(deck)           # 52    ← __len__
deck[0]             # 第一张牌 ← __getitem__
deck[-1]            # 最后一张 ← __getitem__
deck[:3]            # 切片    ← __getitem__
for card in deck:   # 迭代    ← Python 用 __getitem__(0), [1], [2]... 实现
    pass
Card('Q', 'hearts') in deck  # True ← Python 扫描迭代实现
```

### 4.2 猴子补丁：运行时动态实现协议

```python
from random import shuffle

# 问题：FrenchDeck 不支持洗牌
shuffle(deck)  # TypeError: 'FrenchDeck' object does not support item assignment
# 原因：shuffle 需要可变序列（__setitem__），而 FrenchDeck 只有 __getitem__

# 解决方案：猴子补丁（在运行时给类添加方法）
def set_card(deck, position, card):
    """实现 __setitem__ 协议。"""
    deck._cards[position] = card

FrenchDeck.__setitem__ = set_card  # 动态给类添加方法

shuffle(deck)  # 现在可以洗牌了！
print(deck[:5])  # 随机的5张牌

# 猴子补丁的特点：
# ✅ 强大：无须修改源码就能添加功能
# ⚠️ 危险：与被打补丁的代码紧密耦合
# ⚠️ 需要了解类的内部实现（如 _cards 属性）
```

### 4.3 防御性编程："快速失败"原则

```python
# 原则：尽早抛出有意义的错误，而不是等到使用时才报错

class TombolaBase:
    def __init__(self, iterable):
        # ✅ 推荐：立即转换为 list，快速失败
        self._balls = list(iterable)
        # 好处：
        # 1. 如果 iterable 不可迭代，这里立即报错
        # 2. 创建副本，防止外部修改
        # 3. 之后的操作都基于 list，行为可预期

        # ❌ 不推荐：直接存引用
        # self._balls = iterable  # 等到实际使用时才可能报错


# 利用鸭子类型处理多种输入
def process_field_names(field_names):
    """处理 field_names：可以是字符串或字符串列表。"""
    try:
        field_names = field_names.replace(',', ' ').split()  # 假设是字符串
    except AttributeError:
        pass                    # 不是字符串，假设是可迭代对象
    field_names = tuple(field_names)  # 转为元组，防止修改
    if not all(s.isidentifier() for s in field_names):
        raise ValueError('field_names must all be valid identifiers')
    return field_names

# 使用 EAFP（取得原谅比获得许可容易）原则
# 先尝试，失败时处理，而不是先检查类型
```

## 5. 大鹅类型：抽象基类（ABC）

### 5.1 Alex Martelli 的核心观点（"水禽"比喻）

> 只要 `cls` 是抽象基类（元类是 `abc.ABCMeta`），就可以安全使用 `isinstance(obj, cls)` 检查。相比具体类的 `isinstance` 检查，这更灵活。

```python
from collections import abc

# 大鹅类型的使用方式：
# 1. 继承抽象基类（最明确）
class MySeq(abc.Sequence):
    def __getitem__(self, i): ...
    def __len__(self): ...

# 2. isinstance 检查使用 ABC（不是具体类）
def process(seq):
    if isinstance(seq, abc.MutableSequence):
        seq.append(42)           # 安全！确保有 append
    elif isinstance(seq, abc.Sequence):
        pass                     # 只读操作

# 3. 无须显式继承，有正确方法就能被识别
class Struggle:
    def __len__(self): return 23

isinstance(Struggle(), abc.Sized)  # True！不用继承，有 __len__ 就行
# 背后：abc.Sized 实现了 __subclasshook__，检查是否有 __len__ 方法

# ⚠️ Alex 的建议：
# - 不要在生产代码中随意定义抽象基类
# - 只要正确使用现有的 ABC，就能获得 99.9% 的好处
```

### 5.2 子类化内置 ABC

```python
from collections import namedtuple, abc

Card = namedtuple('Card', ['rank', 'suit'])


class FrenchDeck2(abc.MutableSequence):
    """继承 MutableSequence ABC，获得洗牌等功能。"""
    ranks = [str(n) for n in range(2, 11)] + list('JQKA')
    suits = 'spades diamonds clubs hearts'.split()

    def __init__(self):
        self._cards = [Card(rank, suit) for suit in self.suits
                       for rank in self.ranks]

    def __len__(self):
        return len(self._cards)

    def __getitem__(self, position):
        return self._cards[position]

    def __setitem__(self, position, value):  # ← 支持洗牌的关键
        self._cards[position] = value

    def __delitem__(self, position):         # ← MutableSequence 要求的抽象方法
        del self._cards[position]

    def insert(self, position, value):       # ← MutableSequence 要求的抽象方法
        self._cards.insert(position, value)


# 继承 MutableSequence 的收益：
# 必须实现：__getitem__, __setitem__, __delitem__, __len__, insert（5个）
# 免费获得：__contains__, __iter__, __reversed__, index, count（来自Sequence）
#           append, reverse, extend, pop, remove, __iadd__（来自MutableSequence）

from random import shuffle
deck2 = FrenchDeck2()
shuffle(deck2)  # ✅ 现在可以洗牌了
print(deck2[:5])
```

### 5.3 `collections.abc` 中的主要抽象基类

| 抽象基类 | 关键抽象方法 | 继承自 | 说明 |
|---------|------------|------|------|
| `Container` | `__contains__` | — | 支持 `in` |
| `Iterable` | `__iter__` | — | 支持 `for` 循环 |
| `Sized` | `__len__` | — | 支持 `len()` |
| `Collection` | （组合三者）| Container, Iterable, Sized | Python 3.6+ |
| `Sequence` | `__getitem__`, `__len__` | Reversible, Collection | 有序序列 |
| `MutableSequence` | +`__setitem__`, `__delitem__`, `insert` | Sequence | 可变序列 |
| `Mapping` | `__getitem__`, `__len__`, `__iter__` | Collection | 字典类 |
| `MutableMapping` | +`__setitem__`, `__delitem__` | Mapping | 可变字典 |
| `Set` | `__contains__`, `__iter__`, `__len__` | Collection | 集合 |
| `Callable` | `__call__` | — | 可调用 |
| `Hashable` | `__hash__` | — | 可哈希 |
| `Iterator` | `__next__` | Iterable | 迭代器 |

## 6. 定义和使用自定义抽象基类

### 6.1 定义抽象基类 Tombola

```python
import abc


class Tombola(abc.ABC):
    """随机摸球机抽象基类。

    子类必须实现 load 和 pick 两个抽象方法。
    loaded 和 inspect 是由抽象方法构建的具体方法，子类可选择覆盖。
    """

    @abc.abstractmethod
    def load(self, iterable):
        """把可迭代对象中的元素放入容器。"""
        # 抽象方法可以有实现，但子类仍必须覆盖
        # 子类中可用 super().load() 调用这里的实现

    @abc.abstractmethod
    def pick(self):
        """随机摸出一个元素并返回。

        容器为空时应该抛出 LookupError（IndexError 或 KeyError 的父类）。
        """

    def loaded(self):
        """容器中至少有一个元素时返回 True。

        具体方法：只依赖抽象方法定义的接口（pick 和 load）。
        """
        return bool(self.inspect())

    def inspect(self):
        """返回由当前元素构成的有序元组（不改变容器内容）。"""
        items = []
        while True:
            try:
                items.append(self.pick())   # 依赖抽象方法 pick
            except LookupError:
                break
        self.load(items)                    # 依赖抽象方法 load（放回去）
        return tuple(items)


# 验证：未实现所有抽象方法的子类无法实例化
class Fake(Tombola):
    def pick(self):
        return 13       # 只实现了 pick，没实现 load

# Fake()  # TypeError: Can't instantiate abstract class Fake with abstract method load
```

### 6.2 实现具体子类

```python
import random
from tombola import Tombola


class BingoCage(Tombola):
    """Tombola 的具体实现，使用加密级随机数。"""

    def __init__(self, items):
        self._randomizer = random.SystemRandom()  # 使用 os.urandom() 实现的随机器
        self._items = []
        self.load(items)

    def load(self, items):
        self._items.extend(items)
        self._randomizer.shuffle(self._items)     # 加入时立即打乱

    def pick(self):
        try:
            return self._items.pop()
        except IndexError:
            raise LookupError('pick from empty BingoCage')  # 遵守接口契约

    def __call__(self):
        self.pick()     # 额外功能：允许直接调用实例


class LottoBlower(Tombola):
    """Tombola 的另一种实现，随机取出任意位置的球。"""

    def __init__(self, iterable):
        # ✅ 好习惯：转换为 list，而不是保存引用
        self._balls = list(iterable)

    def load(self, iterable):
        self._balls.extend(iterable)

    def pick(self):
        try:
            position = random.randrange(len(self._balls))
        except ValueError:
            raise LookupError('pick from empty LottoBlower')
        return self._balls.pop(position)   # 随机取出一个（不总是最后一个）

    def loaded(self):
        """覆盖父类的低效实现，直接检查列表。"""
        return bool(self._balls)

    def inspect(self):
        """覆盖父类的迂回实现，直接返回元组。"""
        return tuple(self._balls)
```

### 6.3 抽象基类句法细节

```python
import abc


class MyABC(abc.ABC):
    """自定义抽象基类的标准写法。"""

    # ① 抽象方法：子类必须覆盖
    @abc.abstractmethod
    def abstract_method(self): ...

    # ② 抽象类方法：叠放顺序：@classmethod 在上，@abstractmethod 在下
    @classmethod
    @abc.abstractmethod
    def abstract_class_method(cls, ...): ...

    # ③ 抽象静态方法
    @staticmethod
    @abc.abstractmethod
    def abstract_static_method(...): ...

    # ④ 抽象特性
    @property
    @abc.abstractmethod
    def abstract_property(self): ...

    # 注意：@abstractmethod 必须在最靠近 def 的位置
    # 即：在 @abstractmethod 和 def 之间不能有其他装饰器
```

## 7. 抽象基类的虚拟子类

> 无须继承 ABC，只需注册，就能通过 `isinstance`/`issubclass` 检查。

### 7.1 用 `register` 注册虚拟子类

```python
from random import randrange
from tombola import Tombola


@Tombola.register                   # ← 把 TomboList 注册为 Tombola 的虚拟子类
class TomboList(list):              # ← 继承 list，不继承 Tombola！
    """Tombola 的虚拟子类：继承 list，注册为 Tombola。

    注意：虚拟子类：
    - 不从 Tombola 继承任何方法
    - 实例化时不检查是否实现了 Tombola 的接口
    - 仅通过 register 建立名义上的"子类"关系
    """

    def pick(self):
        if self:            # list 本身的布尔值（非空为True）
            position = randrange(len(self))
            return self.pop(position)
        else:
            raise LookupError('pop from empty TomboList')

    load = list.extend      # 重用 list.extend 作为 load 方法

    def loaded(self):
        return bool(self)

    def inspect(self):
        return tuple(self)

    # Tombola.register(TomboList)   # 也可以在类外调用 register
```

**验证**：

```python
from tombola import Tombola
from tombolist import TomboList

issubclass(TomboList, Tombola)          # True（注册生效）
t = TomboList(range(100))
isinstance(t, Tombola)                   # True

# 但是 __mro__ 中没有 Tombola！
print(TomboList.__mro__)
# (<class 'TomboList'>, <class 'list'>, <class 'object'>)
# Tombola 不在 MRO 中，说明 TomboList 不从 Tombola 继承任何东西
```

### 7.2 标准库中 `register` 的使用

```python
# Python 内置类型通过 register 注册为 Sequence 的虚拟子类
# 来自 _collections_abc.py 源码
from collections.abc import Sequence

Sequence.register(tuple)      # tuple 是不可变序列
Sequence.register(str)        # str 是不可变序列
Sequence.register(range)      # range 也是序列
Sequence.register(memoryview) # memoryview 也是序列

# 因此：
from collections.abc import Sequence
isinstance((1, 2, 3), Sequence)  # True（tuple 已注册）
isinstance('abc', Sequence)      # True（str 已注册）
```

### 7.3 `__subclasshook__`：自动结构检查

```python
# Sized 抽象基类通过 __subclasshook__ 实现结构检查
# 来自标准库 _collections_abc.py：

class Sized(metaclass=ABCMeta):
    @classmethod
    def __subclasshook__(cls, C):
        if cls is Sized:
            # 如果 C 或其任一父类有 __len__ 方法
            if any('__len__' in B.__dict__ for B in C.__mro__):
                return True             # 认为是虚拟子类
        return NotImplemented           # 让正常的子类检查继续

class MyClass:
    def __len__(self): return 42

# 未注册，未继承，但有 __len__
isinstance(MyClass(), Sized)    # True！（__subclasshook__ 的功劳）
issubclass(MyClass, Sized)      # True！

# ⚠️ 自己定义的 ABC 通常不应实现 __subclasshook__
# 只有有充分把握时（如像 Sized 那样只有一个特殊方法）才这么做
```

## 8. 静态协议（`typing.Protocol`）

### 8.1 为 double 函数添加类型提示

```python
from typing import TypeVar, Protocol

T = TypeVar('T')


class Repeatable(Protocol):
    """可重复的协议：支持 * 操作符乘以整数。"""
    def __mul__(self: T, repeat_count: int) -> T: ...
    # 注意：self 用 T 类型变量，确保返回值类型与 self 相同


RT = TypeVar('RT', bound=Repeatable)   # RT 的上界是 Repeatable


def double(x: RT) -> RT:
    """把 x 重复一次。
    
    类型检查工具确保 x 实现了 Repeatable 协议（有 __mul__ 方法），
    且返回值与 x 类型相同。
    """
    return x * 2


# 测试：各种类型都满足 Repeatable 协议（有 __mul__ 方法）
double(1.5)                        # 3.0   （float * int → float）
double('A')                        # 'AA'  （str * int → str）
double([10, 20, 30])               # [10, 20, 30, 10, 20, 30]（list）
from fractions import Fraction
double(Fraction(2, 5))             # Fraction(4, 5)

# 这就是"静态鸭子类型"：
# - 不看类的继承关系（名义类型）
# - 看对象有没有所需方法（结构类型）
# - 在静态检查阶段完成（不是运行时）
```

### 8.2 运行时可检查的静态协议

```python
from typing import Protocol, runtime_checkable, Any
import random


@runtime_checkable                  # ← 使协议支持 isinstance/issubclass 运行时检查
class RandomPicker(Protocol):
    """可以随机取出元素的协议。"""
    def pick(self) -> Any: ...


class SimplePicker:
    """实现了 RandomPicker 协议，但不继承它。"""

    def __init__(self, items):
        self._items = list(items)
        random.shuffle(self._items)

    def pick(self) -> Any:
        return self._items.pop()


# 测试运行时检查
popper = SimplePicker([1, 2, 3])
isinstance(popper, RandomPicker)    # True！（有 pick 方法）

# ⚠️ 运行时检查的局限：
# 只检查方法名称，不检查签名或返回值类型
# 因此可能有误判（有 pick 方法但不符合语义）
```

### 8.3 运行时协议检查的局限性

```python
from typing import SupportsFloat

class FakeFloat:
    def __float__(self):
        raise TypeError("I'm not really a float!")

fake = FakeFloat()
isinstance(fake, SupportsFloat)   # True！（有 __float__ 方法）
float(fake)                        # TypeError（但运行时失败）

# 原因：isinstance 只检查方法名称，不检查方法行为
# 结论：运行时协议检查不能保证语义正确性
```

### 8.4 支持静态协议：给现有类添加方法

```python
from typing import SupportsComplex
import math


class Vector2d:
    """支持 SupportsComplex 协议的向量类。"""

    def __init__(self, x, y):
        self.x = float(x)
        self.y = float(y)

    def __abs__(self) -> float:
        return math.hypot(self.x, self.y)

    def __complex__(self) -> complex:
        """实现 SupportsComplex 协议的核心方法。"""
        return complex(self.x, self.y)

    @classmethod
    def fromcomplex(cls, datum: SupportsComplex) -> 'Vector2d':
        """从任何 SupportsComplex 对象创建 Vector2d。"""
        c = complex(datum)          # 显式转换（SupportsComplex 不保证有 .real/.imag）
        return cls(c.real, c.imag)


v = Vector2d(3, 4)
isinstance(v, SupportsComplex)  # True（有 __complex__ 方法）
complex(v)                       # (3+4j)
abs(v)                           # 5.0
Vector2d.fromcomplex(3+4j)       # Vector2d(3.0, 4.0)
```

### 8.5 扩展协议

```python
from typing import Protocol, runtime_checkable
from randompick import RandomPicker


@runtime_checkable
class LoadableRandomPicker(RandomPicker, Protocol):  # ← 必须显式写 Protocol
    """可加载的随机选取协议：扩展自 RandomPicker。"""
    def load(self, iterable) -> None: ...
    # 只需声明新增的方法，pick 从 RandomPicker 继承

# ⚠️ 注意：
# - 每个协议必须显式把 typing.Protocol 列为基类
# - @runtime_checkable 不会被继承，必须再次声明
# - 这与普通的 Python 继承不同
```

## 9. 协议设计最佳实践

### 9.1 窄协议（Narrow Protocol）

```python
# ✅ 好的设计：只有一个方法的协议，最灵活

from typing import Protocol, Any

class RandomPicker(Protocol):
    def pick(self) -> Any: ...   # 一个方法，非常专注


# ✅ 如果需要更多方法，通过扩展实现
class LoadableRandomPicker(RandomPicker, Protocol):
    def load(self, iterable) -> None: ...   # 新增方法


# ❌ 不好的设计：一个协议有太多方法
class HeavyProtocol(Protocol):
    def method1(self): ...
    def method2(self): ...
    def method3(self): ...
    # 太多方法 → 实现困难，降低灵活性
```

### 9.2 命名约定

```python
# typeshed 项目的命名建议：

# ① 清楚表明概念的名称（像 ABC 一样）
class Iterator(Protocol): ...     # 迭代器
class Container(Protocol): ...    # 容器

# ② SupportsX：表示提供可调用方法的协议
class SupportsRead(Protocol):
    def read(self) -> str: ...

class SupportsClose(Protocol):
    def close(self) -> None: ...

# ③ HasX：表示有可读/可写属性或属性方法的协议
class HasItems(Protocol):
    items: list

# ④ 动词 + er/or（Go 语言风格，单方法协议）
class Reader(Protocol):            # 而不是 SupportsRead
    def read(self) -> bytes: ...

class Formatter(Protocol):
    def format(self, data) -> str: ...
```

### 9.3 在客户代码中定义协议（而非库中）

```python
# ✅ 好的模式：协议定义在使用它的地方附近（客户代码）

# render.py（使用协议的模块）
from typing import Protocol

class Drawable(Protocol):          # ← 在客户代码中定义
    def draw(self) -> None: ...

def render_scene(objects: list[Drawable]) -> None:
    for obj in objects:
        obj.draw()

# 这样做的好处：
# - 方便测试（用 mock 实现协议）
# - 避免紧密耦合（库不需要知道协议的存在）
# - 符合"接口隔离原则"
```

## 10. 本章知识速查表

### 四种方式的选择指南

| 场景 | 推荐方式 | 理由 |
|------|---------|------|
| 函数参数支持多种类型 | 鸭子类型 + 快速失败 | 最灵活，EAFP 原则 |
| 运行时检查类型（框架）| 大鹅类型（isinstance + ABC）| 明确，支持虚拟子类 |
| 函数类型提示（静态检查）| 静态协议（typing.Protocol）| 精确表达结构要求 |
| 强制实现接口契约 | 抽象基类（继承 abc.ABC）| 实例化时自动检查 |

### 关键 ABC 选择

```python
from collections.abc import (
    Sequence,         # 只读有序序列（支持 __getitem__ 和 __len__）
    MutableSequence,  # 可变有序序列（另外需要 __setitem__, __delitem__, insert）
    Mapping,          # 只读映射
    MutableMapping,   # 可变映射
    Set,              # 不可变集合
    MutableSet,       # 可变集合
    Iterable,         # 可迭代
    Iterator,         # 迭代器
    Callable,         # 可调用
    Hashable,         # 可哈希
)
```

### 大鹅类型 vs 静态鸭子类型 对比

| | 大鹅类型（ABC）| 静态鸭子类型（Protocol）|
|--|---------------|----------------------|
| 运行时检查 | ✅（isinstance）| ✅（需 @runtime_checkable）|
| 静态检查 | 部分支持 | ✅（Mypy 等）|
| 虚拟子类 | ✅（register）| ❌ |
| 混入方法 | ✅（继承得到）| ❌ |
| 声明方式 | 继承 abc.ABC | 继承 typing.Protocol |
| 使用场景 | 框架、运行时类型检查 | 函数类型提示、静态检查 |
