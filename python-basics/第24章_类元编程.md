# 第24章：类元编程

> **核心思想**：类元编程指在运行时创建或定制类的技艺。Python 提供了四层递进的工具：`type` 工厂函数 → `__init_subclass__` → 类装饰器 → 元类。越高层越强大，也越复杂。**应用程序代码应优先使用简单方案，元类是框架开发者的工具。**

> ⚠️ **警告**：为了可读性和可维护性，应避免在应用程序代码中使用本章讲解的高级技术。这些工具是编写框架和库的利器。

## 2. 身为对象的类：类的标准属性

```python
# Python 中类是一等对象，拥有以下标准属性

class Widget(object):
    """示例基类"""
    pass

class Button(Widget):
    """示例子类"""
    category = 'UI'

class Icon(Widget):
    """另一个子类"""
    pass


# ---- cls.__bases__ ----
# 由类的直接基类构成的元组
print(Button.__bases__)          # (<class 'object'>,)... 等等，Widget 才是直接基类
print(Button.__bases__)          # (<class '__main__.Widget'>,)

# ---- cls.__qualname__ ----
# 从模块全局作用域到类的完整点分路径
class Ox:
    class Meta:                  # 嵌套类
        pass

print(Ox.Meta.__name__)          # 'Meta'（简单名称）
print(Ox.Meta.__qualname__)      # 'Ox.Meta'（限定名称，含路径）
# 在 Django 模型中，Meta 类的 __qualname__ 是 'ModelName.Meta'

# ---- cls.__subclasses__() ----
# 返回直接子类的列表（弱引用实现，不含未导入模块中的子类）
print(Widget.__subclasses__())   # [<class 'Button'>, <class 'Icon'>]

# ---- cls.mro() ----
# 返回方法解析顺序列表（与 __mro__ 属性内容相同）
print(Button.mro())
# [<class 'Button'>, <class 'Widget'>, <class 'object'>]

# ---- dir() 不显示这些属性 ----
# 以上属性不会出现在 dir(cls) 的结果中，需要直接访问
```

## 3. type：内置的类工厂函数

> **⚠️ 重点**：`type` 既是函数，又是类（元类）。它有两种完全不同的用法。

### 3.1 type 的双重身份

```python
# 用法一：type(obj) → 返回对象所属的类（单参数形式）
type(42)          # <class 'int'>
type('hello')     # <class 'str'>
type(int)         # <class 'type'>  ← int 是 type 的实例！
type(type)        # <class 'type'>  ← type 是自身的实例（特殊关系）

# 用法二：type(name, bases, dict) → 创建新类（三参数形式）
# 这才是 type 作为类工厂（元类）的用法


# ---- 两种等价的写法 ----

# 写法一：class 语句（常规方式）
class MyClass(MySuperClass, MyMixin):
    x = 42
    def x2(self):
        return self.x * 2

# 写法二：type() 调用（等价）
# Python 读取 class 语句时，内部就是这样调用 type 的
MyClass = type(
    'MyClass',                          # name:  类名字符串
    (MySuperClass, MyMixin),            # bases: 基类元组
    {'x': 42, 'x2': lambda self: self.x * 2}  # dict: 属性字典
)
```

### 3.2 type() 三参数的含义

```python
# type(name, bases, dict) 三个参数的详细说明：

# name（字符串）：
#   class 关键字后的标识符，成为新类的 __name__

# bases（元组）：
#   基类元组；如果 class 语句未指定基类，则为 (object,)

# dict（字典）：
#   属性名 → 值的映射
#   可调用对象自动成为方法，其他值成为类属性

# 示例：动态创建不同的类
def make_class(class_name, **attributes):
    """根据属性动态创建类的工厂函数。"""
    return type(class_name, (object,), attributes)

Dog = make_class('Dog', sound='woof', legs=4)
Cat = make_class('Cat', sound='meow', legs=4)

print(Dog.sound)          # 'woof'
print(type(Dog))          # <class 'type'>  ← Dog 是 type 的实例
print(isinstance(Dog(), Dog))  # True
```

## 4. 类工厂函数：record_factory

> 完整示例：用 `type()` 动态创建可迭代的简单记录类，类似 `collections.namedtuple`，但创建的是可变对象。

```python
from typing import Union, Any
from collections.abc import Iterable, Iterator

# 类型别名：字段名可以是字符串或字符串可迭代对象
FieldNames = Union[str, Iterable[str]]


def record_factory(cls_name: str, field_names: FieldNames) -> type[tuple]:
    """简单类工厂函数：动态创建带有指定字段的可变记录类。

    用法类似 collections.namedtuple，但创建的是可变对象（支持属性赋值）。
    使用 __slots__ 限制属性，节省内存。

    Args:
        cls_name:    新类的名称
        field_names: 字段名称，可以是空格/逗号分隔的字符串或字符串列表

    Returns:
        新创建的类（type[tuple] 的子类）
    """
    # 步骤1：解析字段名，转为字符串元组
    slots = parse_identifiers(field_names)

    # 步骤2：定义将成为新类方法的函数
    def __init__(self, *args, **kwargs) -> None:
        """支持位置参数和关键字参数的初始化方法。"""
        # 把位置参数按 __slots__ 顺序配对
        attrs = dict(zip(self.__slots__, args))
        attrs.update(kwargs)
        for name, value in attrs.items():
            setattr(self, name, value)

    def __iter__(self) -> Iterator[Any]:
        """按 __slots__ 顺序产出字段值，使实例可迭代。"""
        for name in self.__slots__:
            yield getattr(self, name)

    def __repr__(self) -> str:
        """返回友好的字符串表示形式，如 Dog(name='Rex', weight=30)。"""
        values = ', '.join(
            f'{name}={value!r}'
            for name, value in zip(self.__slots__, self)
        )
        cls_name = self.__class__.__name__
        return f'{cls_name}({values})'

    # 步骤3：组建类属性字典
    cls_attrs = dict(
        __slots__=slots,        # 限制实例属性，节省内存
        __init__=__init__,
        __iter__=__iter__,
        __repr__=__repr__,
    )

    # 步骤4：调用 type() 构建并返回新类
    # type(name, bases, dict) 三参数形式
    return type(cls_name, (object,), cls_attrs)


def parse_identifiers(names: FieldNames) -> tuple[str, ...]:
    """解析字段名称，转为合法标识符的元组。"""
    if isinstance(names, str):
        # '逗号 或 空格' 分隔的字符串 → 列表
        names = names.replace(',', ' ').split()
    if not all(s.isidentifier() for s in names):
        raise ValueError('names must all be valid identifiers')
    return tuple(names)


# ============================================================
# 测试
# ============================================================

# 用法一：空格分隔的字符串
Dog = record_factory('Dog', 'name weight owner')

rex = Dog('Rex', 30, 'Bob')
print(rex)                     # Dog(name='Rex', weight=30, owner='Bob')
print(rex.weight)              # 30

# 解包赋值（因为 __iter__ 使其可迭代）
name, weight, _ = rex
print(name, weight)            # Rex 30

# 字符串格式化（可迭代对象的拆包）
print("{2}'s dog weighs {1}kg".format(*rex))  # Bob's dog weighs 30kg

# 实例是可变的（不像 namedtuple）
rex.weight = 32
print(rex)                     # Dog(name='Rex', weight=32, owner='Bob')

# 新类继承自 object
print(Dog.__mro__)             # (<class 'factories.Dog'>, <class 'object'>)

# 用法二：逗号分隔的字符串
Point = record_factory('Point', 'x, y, z')
p = Point(1, 2, 3)
print(p)                       # Point(x=1, y=2, z=3)

# ⚠️ 局限：record_factory 创建的类的实例不能序列化（pickle）
# 原因：pickle 需要能在模块中找到该类，而动态创建的类不在模块中
```

## 5. `__init_subclass__`：定制子类创建

> **Python 3.6+**：当一个类的子类被创建时，自动调用父类中的 `__init_subclass__` 方法。这是"PEP 487—简化类创建定制"引入的新方式，替代了很多以前需要元类才能完成的任务。

### 5.1 工作原理

```python
class Validator:
    """当 Validator 的子类被创建时，自动执行验证逻辑注入。"""

    def __init_subclass__(subclass, **kwargs):
        """子类创建时自动调用。

        注意：
        - 第一个参数是"被新创建的子类"，不是 Validator 本身
        - 因此参数名用 subclass 而非 cls（避免误解）
        - 不需要 @classmethod 装饰器，但行为类似类方法
        - Python 传入的是子类，不是 Validator 自身

        ⚠️ 务必调用 super().__init_subclass__()，支持多继承中其他类的协作
        """
        super().__init_subclass__(**kwargs)
        print(f'  → {subclass.__name__} 已被创建并注入验证逻辑')
        # 在这里可以对子类做任何事：
        # - 注册子类到注册表
        # - 为子类添加/替换方法
        # - 验证子类的结构
        # - 为子类设置类属性


class Dog(Validator):      # 触发 Validator.__init_subclass__（subclass=Dog）
    pass

class Cat(Validator):      # 再次触发（subclass=Cat）
    pass

# 输出：
# → Dog 已被创建并注入验证逻辑
# → Cat 已被创建并注入验证逻辑
```

### 5.2 完整示例：Checked 类（运行时类型验证）

```python
from collections.abc import Callable
from typing import Any, NoReturn, get_type_hints


# ============================================================
# Field 描述符类：负责单个字段的类型验证
# ============================================================

class Field:
    """验证型描述符：通过构造函数对字段值进行类型验证和转换。

    用法：在 Checked 子类中通过类型提示声明字段类型，
    Field 实例由 Checked.__init_subclass__ 自动创建并注入。
    """

    def __init__(self, name: str, constructor: Callable) -> None:
        """初始化字段描述符。

        Args:
            name:        字段名称（对应类型提示的属性名）
            constructor: 类型构造函数，如 int, str, float
                         调用时无参数返回默认值，有参数则做类型转换
        """
        # constructor 必须可调用且不能是 NoneType
        # type(None) 得到 NoneType，虽然可调用，但只返回 None，无意义
        if not callable(constructor) or constructor is type(None):
            raise TypeError(f'{name!r} type hint must be callable')
        self.name = name
        self.constructor = constructor

    def __set__(self, instance: Any, value: Any) -> None:
        """拦截属性赋值，进行类型验证和转换。

        Args:
            instance: 托管实例（Checked 子类的实例）
            value:    要赋的值
                      特殊值 ...（Ellipsis）表示使用默认值
        """
        if value is ...:
            # Ellipsis 作为哨符：调用无参构造函数获取默认值
            # 例如 int() → 0, str() → '', float() → 0.0
            value = self.constructor()
        else:
            try:
                # 用构造函数转换值（同时实现类型检查）
                value = self.constructor(value)
            except (TypeError, ValueError) as e:
                # 例如 float('abc') → ValueError
                # 例如 float(None) → TypeError
                type_name = self.constructor.__name__
                msg = f'{value!r} is not compatible with {self.name}:{type_name}'
                raise TypeError(msg) from e
        # ✅ 存入托管实例的 __dict__（不是描述符实例的 __dict__）
        instance.__dict__[self.name] = value


# ============================================================
# Checked 基类：使用 __init_subclass__ 自动注入 Field 描述符
# ============================================================

class Checked:
    """带运行时类型验证的类构建器基类（使用 __init_subclass__ 实现）。

    用法：
        class Movie(Checked):
            title: str      # Field(name='title', constructor=str)
            year: int       # Field(name='year', constructor=int)
            box_office: float

        movie = Movie(title='The Godfather', year=1972, box_office=137)
        # → Movie(title='The Godfather', year=1972, box_office=137.0)
    """

    @classmethod
    def _fields(cls) -> dict[str, type]:
        """返回类的类型提示字典（字段名 → 构造函数类型）。

        封装 get_type_hints 调用，对类的其余部分隐藏实现细节。
        """
        return get_type_hints(cls)

    def __init_subclass__(subclass) -> None:
        """关键方法：子类被创建时自动调用，为子类注入 Field 描述符。

        ⚠️ 注意事项：
        - 第一个参数是新创建的子类（subclass），不是 Checked 本身
        - 不需要 @classmethod 装饰器
        - 务必调用 super().__init_subclass__()（多继承支持）
        - 在此不能配置 __slots__（类已创建，太晚了）
        """
        super().__init_subclass__()  # 协作调用，支持多继承

        # 遍历子类的类型提示，为每个提示创建 Field 描述符并注入
        for name, constructor in subclass._fields().items():
            # setattr(subclass, name, Field(name, constructor))
            # 将 Movie.title 从类型提示（str）替换为 Field('title', str)
            setattr(subclass, name, Field(name, constructor))

    def __init__(self, **kwargs: Any) -> None:
        """初始化实例：按字段逐一赋值（触发 Field.__set__ 验证）。"""
        for name in self._fields():
            # 从 kwargs 取值，默认用 ...（Ellipsis）→ Field 会调用无参构造函数
            value = kwargs.pop(name, ...)
            setattr(self, name, value)   # 触发 Field.__set__
        if kwargs:
            # 有多余参数 → 报错（快速失败原则）
            self.__flag_unknown_attrs(*kwargs)

    def __setattr__(self, name: str, value: Any) -> None:
        """拦截所有属性赋值，防止设置未声明的属性。

        因为 Field 是覆盖型描述符，需要显式调用 descriptor.__set__。
        """
        if name in self._fields():
            cls = self.__class__
            descriptor = getattr(cls, name)
            descriptor.__set__(self, value)  # 显式调用描述符
        else:
            self.__flag_unknown_attrs(name)

    def __flag_unknown_attrs(self, *names: str) -> NoReturn:
        """为未知属性抛出详细的 AttributeError。"""
        plural = 's' if len(names) > 1 else ''
        extra = ', '.join(f'{name!r}' for name in names)
        cls_name = repr(self.__class__.__name__)
        raise AttributeError(f'{cls_name} object has no attribute{plural} {extra}')

    def _asdict(self) -> dict[str, Any]:
        """返回字段名 → 字段值的字典。"""
        return {
            name: getattr(self, name)
            for name, attr in self.__class__.__dict__.items()
            if isinstance(attr, Field)
        }

    def __repr__(self) -> str:
        kwargs = ', '.join(
            f'{key}={value!r}' for key, value in self._asdict().items()
        )
        return f'{self.__class__.__name__}({kwargs})'


# ============================================================
# 使用示例
# ============================================================

class Movie(Checked):
    title: str          # → Field('title', str)
    year: int           # → Field('year', int)
    box_office: float   # → Field('box_office', float)


# 正常创建
movie = Movie(title='The Godfather', year=1972, box_office=137)
print(movie)            # Movie(title='The Godfather', year=1972, box_office=137.0)
print(movie.title)      # 'The Godfather'

# 缺少参数 → 使用默认值（构造函数无参调用）
m2 = Movie(title='Life of Brian')
print(m2)               # Movie(title='Life of Brian', year=0, box_office=0.0)

# 类型不兼容 → TypeError
try:
    Movie(title='Avatar', year=2009, box_office='billions')
except TypeError as e:
    print(e)            # 'billions' is not compatible with box_office:float

# 赋值时也验证
try:
    movie.year = 'MCMLXXII'
except TypeError as e:
    print(e)            # 'MCMLXXII' is not compatible with year:int

# 未知属性 → AttributeError
try:
    movie.director = 'Coppola'
except AttributeError as e:
    print(e)            # 'Movie' object has no attribute 'director'
```

### 5.3 `__init_subclass__` 的局限

```python
# ⚠️ 重要局限：__init_subclass__ 无法配置 __slots__
# 原因：__init_subclass__ 在类已经构建之后才被调用
#       而 __slots__ 必须在 type.__new__ 调用之前就存在于 cls_dict 中
#       此时再添加 __slots__ 只会创建一个普通类属性，不起作用

# 类装饰器也有同样的问题（比 __init_subclass__ 调用还晚）

# 要动态配置 __slots__，必须使用元类（在 type.__new__ 之前操作 cls_dict）
```

## 6. 类装饰器：@checked

> 类装饰器提供与 `__init_subclass__` 相似的功能，但更灵活：不强制继承关系，且可避免与元类、继承产生冲突。`@dataclass` 就是用类装饰器实现的。

```python
from collections.abc import Callable
from typing import Any, NoReturn, get_type_hints


def _fields(cls: type) -> dict[str, type]:
    """获取类的类型提示字典（模块级函数版）。"""
    return get_type_hints(cls)


def checked(cls: type) -> type:
    """类装饰器：接受一个类，为其注入 Field 描述符和验证方法，返回增强后的类。

    与 Checked 基类版本的区别：
    - 不需要继承 Checked，只需要 @checked 装饰
    - 装饰器在类创建之后运行（晚于 __init_subclass__）
    - 不能配置 __slots__（同样的局限）

    ⚠️ 类型注解：cls: type → type 表示这是一个类装饰器
    """
    # 步骤1：为每个类型提示的字段创建 Field 描述符
    for name, constructor in _fields(cls).items():
        setattr(cls, name, Field(name, constructor))

    # 步骤2：把 _fields 注入为类方法
    # type: ignore 是因为 Mypy 不知道 cls 有 _fields 属性
    cls._fields = classmethod(_fields)   # type: ignore

    # 步骤3：注入所有实例方法
    instance_methods = (
        __init__,
        __repr__,
        __setattr__,
        _asdict,
        __flag_unknown_attrs,
    )
    for method in instance_methods:
        setattr(cls, method.__name__, method)

    # 步骤4：返回增强后的类（满足装饰器合约）
    return cls


# 以下是要注入被装饰类的方法（模块级函数，名称以 _ 开头防止冲突）

def __init__(self: Any, **kwargs: Any) -> None:
    """注入的初始化方法。"""
    for name in self._fields():
        value = kwargs.pop(name, ...)
        setattr(self, name, value)
    if kwargs:
        self.__flag_unknown_attrs(*kwargs)

def __setattr__(self: Any, name: str, value: Any) -> None:
    """注入的赋值拦截方法。"""
    if name in self._fields():
        cls = self.__class__
        descriptor = getattr(cls, name)
        descriptor.__set__(self, value)
    else:
        self.__flag_unknown_attrs(name)

def __flag_unknown_attrs(self: Any, *names: str) -> NoReturn:
    """注入的错误报告方法。"""
    plural = 's' if len(names) > 1 else ''
    extra = ', '.join(f'{name!r}' for name in names)
    cls_name = repr(self.__class__.__name__)
    raise AttributeError(f'{cls_name} has no attribute{plural} {extra}')

def _asdict(self: Any) -> dict[str, Any]:
    """注入的字典转换方法。"""
    return {
        name: getattr(self, name)
        for name, attr in self.__class__.__dict__.items()
        if isinstance(attr, Field)
    }

def __repr__(self: Any) -> str:
    """注入的字符串表示方法。"""
    kwargs = ', '.join(
        f'{key}={value!r}' for key, value in self._asdict().items()
    )
    return f'{self.__class__.__name__}({kwargs})'

class Movie:
    title: str
    year: int
    box_office: float
    # 行为与继承 Checked 版完全相同

movie = Movie(title='The Godfather', year=1972, box_office=137)
print(movie)   # Movie(title='The Godfather', year=1972, box_office=137.0)
```

## 7. 导入时 vs 运行时：执行顺序实验

> **⚠️ 重点**：理解 Python 在"导入时"和"运行时"分别执行哪些代码，是掌握类元编程的基础。

### 7.1 import 语句的行为

```python
# ⚠️ import 语句不只是声明！
# 首次导入模块时，模块中的所有顶层代码都会立即执行

# 在导入时执行的内容：
# ① 解析源码（生成 AST）
# ② 编译字节码
# ③ 执行所有顶层代码，包括：
#    - class 语句（执行类主体）
#    - def 语句（创建函数对象，但不执行函数体）
#    - 模块级变量赋值
#    - import 语句

# 后续再次导入：使用缓存，只做名称绑定
```

### 7.2 类构建过程中各步骤的执行顺序

```
Python 处理 class Klass(Builder, metaclass=MetaKlass): ... 时的顺序：

导入时（import evaldemo_meta 时立即发生）：

  1. 调用 MetaKlass.__prepare__      ← 元类最早的干预机会
     → 返回命名空间对象（默认为 dict，可自定义）

  2. 执行 Klass 类主体
     → 按顺序执行所有语句：
       a. 描述符实例化（Descriptor.__init__）
       b. 把实例绑定到命名空间（触发 NosyDict.__setitem__）
       c. 定义 __init__、__repr__ 等方法

  3. 调用 MetaKlass.__new__           ← 创建类对象
     → 可以在这里修改 cls_dict（命名空间）
     → 调用 type.__new__ 实际创建类

  4. 调用描述符的 __set_name__        ← type.__new__ 返回后
     → 自动把属性名注入描述符

  5. 调用超类的 __init_subclass__     ← type.__new__ 返回后
     → 可以在这里为子类注入方法

  6. 应用类装饰器 @deco               ← 最晚执行（type.__new__ 之后）
     → 可以进一步修改类，但无法配置 __slots__

运行时（调用 Klass() 创建实例时发生）：

  7. 调用 Klass.__init__（及其超类的 __init__）
  8. 调用实例方法
  9. 触发描述符的 __set__（赋值时）
```

### 7.3 关键结论速查

```python
# ---- 哪些操作发生在"导入时"（无论是否调用实例）----
# ✅ 模块顶层的 class 语句           → 类主体执行
# ✅ 描述符实例化（在类主体中）       → Descriptor.__init__
# ✅ 描述符 __set_name__             → type.__new__ 返回后
# ✅ __init_subclass__               → type.__new__ 返回后
# ✅ 类装饰器                         → 最晚，在所有元类操作之后

# ---- 哪些操作发生在"运行时"（显式调用才发生）----
# ✅ 实例创建 Klass()               → __init__
# ✅ 实例方法调用 obj.method()
# ✅ 属性赋值 obj.attr = value     → 触发 __set__

# ---- 特殊：import 语句可以触发运行时行为 ----
# 导入模块时运行的顶层代码可以做任何事（如写日志、连接数据库）
# 这是导入时和运行时界线模糊的原因
```

### 8.1 元类的本质：构建类的类

```
 继承关系（__bases__）：
```

## 9. 元类如何定制类：`__new__` 方法

### 9.1 元类 `__new__` 的调用时机和参数

```python
class MetaKlass(type):
    """自定义元类示例：演示 __new__ 方法的签名和功能。"""

    def __new__(meta_cls, cls_name, bases, cls_dict):
        """元类的核心方法：在类创建时被调用。

        命名约定（推荐）：
          实例方法中的 self → cls（因为创建的实例是类）
          类方法中的  cls  → meta_cls（因为定义的类是元类）

        Args:
            meta_cls: 元类自身（MetaKlass）
            cls_name: 类名字符串（如 'Klass'）
            bases:    基类元组（如 (SuperKlass,)）
            cls_dict: 类命名空间映射（含类主体中定义的所有内容）
                      注意：这里已经是 __prepare__ 返回的对象了

        调用时机：
          1. 类主体读取完毕后
          2. 在 __set_name__、__init_subclass__、类装饰器之前
        """
        # ---- 检查/修改 cls_dict ----
        # 在这里可以向 cls_dict 添加/修改属性（方法注入）
        def inner_method(self):
            print('by MetaKlass')
        cls_dict['method_c'] = inner_method

        # ⚠️ 重要：__slots__ 必须在此处配置
        # 因为 __slots__ 只有在 type.__new__ 调用时 cls_dict 中已经包含它才有效
        # cls_dict['__slots__'] = ['_field1', '_field2']  # 示例

        # ---- 调用 super().__new__ 实际创建类 ----
        cls = super().__new__(meta_cls, cls_name, bases, cls_dict)

        # ---- super().__new__ 返回后，类已经创建，但可以继续修改 ----
        # cls.some_attr = value  # 此时修改也可以，但不如在 cls_dict 中修改早

        return cls


# 使用自定义元类

    x = 42

obj = MyClass()

```

## 10. MetaBunch：一个友好的元类示例

> 经典示例（Alex Martelli 2002年首创）：元类在 `type.__new__` 之前配置 `__slots__`，实现基于类属性默认值的轻量记录类。

```python
class MetaBunch(type):
    """元类：根据用户声明的类属性自动配置 __slots__ 和默认值。

    特点：
    - 把用户在类主体中定义的普通属性转成 __slots__
    - 生成带默认值的 __init__ 和简洁的 __repr__
    - 拒绝设置未声明的属性（快速失败）
    """

    def __new__(meta_cls, cls_name, bases, cls_dict):
        """在 type.__new__ 之前处理 cls_dict，配置 __slots__。

        这是使用元类而非 __init_subclass__ 或装饰器的核心原因：
        只有在 type.__new__ 调用之前才能正确配置 __slots__。
        """
        # 用于存储 {属性名: 默认值} 的映射（由闭包捕获）
        defaults = {}

        def __init__(self, **kwargs):
            """生成的初始化方法：按声明的字段设置属性，多余参数报错。"""
            for name, default in defaults.items():
                # 从 kwargs 获取值，缺少时使用默认值
                setattr(self, name, kwargs.pop(name, default))
            if kwargs:
                # 有多余参数：快速失败，列出所有多余的键
                extra = ', '.join(kwargs)
                raise AttributeError(f'No slots left for: {extra!r}')

        def __repr__(self):
            """生成的字符串表示：忽略等于默认值的属性，更简洁。"""
            rep = ', '.join(
                f'{name}={value!r}'
                for name, default in defaults.items()
                # 只显示与默认值不同的属性
                if (value := getattr(self, name)) != default
            )
            return f'{cls_name}({rep})'

        # 初始化新类的命名空间（包含 __slots__ 和注入的方法）
        new_dict = dict(
            __slots__=[],       # 槽列表，稍后填充
            __init__=__init__,
            __repr__=__repr__,
        )

        for name, value in cls_dict.items():
            if name.startswith('__') and name.endswith('__'):
                # 双下划线属性：原样复制（防止覆盖 __init__ 等）
                if name in new_dict:
                    raise AttributeError(f"Can't set {name!r} in {cls_name!r}")
                new_dict[name] = value
            else:
                # 普通属性：变成 __slots__ 中的槽，值作为默认值
                new_dict['__slots__'].append(name)
                defaults[name] = value          # 存入闭包捕获的 defaults

        # 关键：在调用 super().__new__ 之前，__slots__ 已经在 new_dict 中
        return super().__new__(meta_cls, cls_name, bases, new_dict)


class Bunch(metaclass=MetaBunch):
    """用户继承这个类，无须关心 MetaBunch。"""
    pass


# ============================================================
# 使用示例
# ============================================================

class Point(Bunch):
    x = 0.0     # 默认值 0.0，成为 __slots__ 中的槽
    y = 0.0     # 默认值 0.0
    color = 'gray'  # 默认值 'gray'


p1 = Point(x=1.2, y=3, color='green')
print(p1)           # Point(x=1.2, y=3, color='green')

p2 = Point()
print(p2.x, p2.color)  # 0.0 'gray'（使用默认值）
print(p2)           # Point()（所有属性等于默认值，repr 不显示）

p3 = Point(x=21)
p3.y = 42
print(p3)           # Point(x=21, y=42)

# 拒绝未声明的属性
try:
    Point(x=1, y=2, z=3)           # z 未声明
except AttributeError as e:
    print(e)        # No slots left for: 'z'

try:
    p1.flavor = 'banana'           # flavor 未声明
except AttributeError as e:
    print(e)        # 'Point' object has no attribute 'flavor'

# __slots__ 的效果：无 __dict__，内存更省
print(hasattr(p1, '__dict__'))     # False（因为有 __slots__）
```

## 11. 使用元类实现 Checked 类

> 在第5节的基础上，用元类重写 `Checked` 类，解决 `__slots__` 配置问题。元类在 `type.__new__` 之前配置 `__slots__`，使实例更省内存，且删除了 `__setattr__`（因为 `__slots__` 已经阻止了未声明属性）。

```python
from collections.abc import Callable
from typing import Any, NoReturn, get_type_hints


class Field:
    """带 storage_name 的改进版 Field 描述符（支持 __slots__）。

    与前一版的区别：
    - 添加了 storage_name 属性（字段值存储位置，如 '_title'）
    - 添加了 __get__ 方法（因为值不再存在同名属性中）
    - __set__ 改用 setattr(instance, self.storage_name, value)
    """

    def __init__(self, name: str, constructor: Callable) -> None:
        if not callable(constructor) or constructor is type(None):
            raise TypeError(f'{name!r} type hint must be callable')
        self.name = name
        # 为字段值选择不同的存储名称（加 _ 前缀避免与类属性冲突）
        # 例如：Field('title', str).storage_name = '_title'
        self.storage_name = '_' + name
        self.constructor = constructor

    def __get__(self, instance, owner=None):
        """读取字段值。"""
        if instance is None:
            return self     # 通过类访问时返回描述符本身
        # 从 _title 等实例属性读取（而非同名属性，避免与描述符冲突）
        return getattr(instance, self.storage_name)

    def __set__(self, instance: Any, value: Any) -> None:
        """写入字段值（包含验证）。"""
        if value is ...:
            value = self.constructor()
        else:
            try:
                value = self.constructor(value)
            except (TypeError, ValueError) as e:
                type_name = self.constructor.__name__
                msg = f'{value!r} is not compatible with {self.name}:{type_name}'
                raise TypeError(msg) from e
        # 存入 _title 等属性（storage_name），而非 title
        setattr(instance, self.storage_name, value)


class CheckedMeta(type):
    """元类：读取子类的类型提示，自动配置 __slots__ 和 Field 描述符。

    与 __init_subclass__ 版本的关键区别：
    - 在 type.__new__ 之前处理 cls_dict
    - 因此可以正确配置 __slots__（这是元类存在的核心价值）
    - 省略了 __setattr__ 方法（__slots__ 已阻止未声明属性）
    """

    def __new__(meta_cls, cls_name, bases, cls_dict):
        # 只处理没有 __slots__ 的类（即用户定义的子类，而非 Checked 基类本身）
        if '__slots__' not in cls_dict:
            slots = []
            # 直接从 cls_dict 获取类型提示（此时类还不存在，不能用 get_type_hints）
            type_hints = cls_dict.get('__annotations__', {})

            for name, constructor in type_hints.items():
                # 为每个字段创建 Field 描述符
                field = Field(name, constructor)
                # 把 Field 实例存入 cls_dict（替换类型提示）
                cls_dict[name] = field
                # field.storage_name = '_name'，加入 __slots__
                slots.append(field.storage_name)

            # 在 type.__new__ 调用之前设置 __slots__
            cls_dict['__slots__'] = slots

        return super().__new__(meta_cls, cls_name, bases, cls_dict)


class Checked(metaclass=CheckedMeta):
    """使用元类实现的运行时类型验证基类（最终版）。

    与 __init_subclass__ 版本的改进：
    - 使用 __slots__：实例无 __dict__，内存更省
    - 删除 __setattr__：__slots__ 已自动阻止未声明属性
    - 删除 __init_subclass__：工作由 CheckedMeta.__new__ 完成
    """
    __slots__ = ()   # 空 __slots__，告诉 CheckedMeta 不需要处理这个类

    @classmethod
    def _fields(cls) -> dict[str, type]:
        return get_type_hints(cls)

    def __init__(self, **kwargs: Any) -> None:
        for name in self._fields():
            value = kwargs.pop(name, ...)
            setattr(self, name, value)
        if kwargs:
            self.__flag_unknown_attrs(*kwargs)

    def __flag_unknown_attrs(self, *names: str) -> NoReturn:
        plural = 's' if len(names) > 1 else ''
        extra = ', '.join(f'{name!r}' for name in names)
        cls_name = repr(self.__class__.__name__)
        raise AttributeError(f'{cls_name} object has no attribute{plural} {extra}')

    def _asdict(self) -> dict[str, Any]:
        return {
            name: getattr(self, name)
            for name, attr in self.__class__.__dict__.items()
            if isinstance(attr, Field)
        }

    def __repr__(self) -> str:
        kwargs = ', '.join(
            f'{key}={value!r}' for key, value in self._asdict().items()
        )
        return f'{self.__class__.__name__}({kwargs})'


# ============================================================
# 使用示例（与前两个版本 API 完全兼容）
# ============================================================

class Movie(Checked):
    title: str
    year: int
    box_office: float

movie = Movie(title='The Godfather', year=1972, box_office=137)
print(movie)               # Movie(title='The Godfather', year=1972, box_office=137.0)

# 内存优化验证：使用 __slots__，没有 __dict__
print(hasattr(movie, '__dict__'))  # False

# 值存在 _title、_year、_box_office 这些 __slots__ 属性中
print(movie.__class__.__slots__)   # ['_title', '_year', '_box_office']
```

## 12. 元类的 `__prepare__` 方法：AutoConst 示例

> `__prepare__` 是 Python 只在元类上调用的类方法，在类主体读取之前被调用，返回用于存放类命名空间的映射对象。这是影响类创建过程的最早机会。

### 12.1 `__prepare__` 的调用时机

```python
class MetaKlass(type):
    @classmethod
    def __prepare__(meta_cls, cls_name, bases, **kwargs):
        """最早的干预机会：类主体执行之前调用。

        Args:
            meta_cls: 元类自身
            cls_name: 类名
            bases:    基类元组

        Returns:
            用于存放类命名空间的映射对象
            默认返回普通 dict；可以返回自定义映射
        """
        print(f'__prepare__ 被调用，类 {cls_name} 的命名空间即将创建')
        # 返回自定义映射，Python 会把类主体中的定义存入其中
        return {}   # 默认返回空 dict

    def __new__(meta_cls, cls_name, bases, cls_dict):
        # cls_dict 就是 __prepare__ 返回的对象（填充了类主体中的内容）
        return super().__new__(meta_cls, cls_name, bases, cls_dict)
```

### 12.2 AutoConst：`__prepare__` + `__missing__` 的创意组合

> 使用 `__prepare__` 返回一个实现了 `__missing__` 的自定义字典，实现类属性的自动常量枚举——只需在类主体中写属性名，即可自动赋值 0, 1, 2...

```python
class WilyDict(dict):
    """实现了 __missing__ 的特殊字典。

    当 Python 读取类主体时，尝试查找 banana、coconut 等名称，
    触发 __missing__，自动为其创建递增整数值。
    """

    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self.__next_value = 0   # 用于生成递增整数的计数器

    def __missing__(self, key):
        """当访问不存在的键时自动调用。

        核心机制：
        Python 读取类主体 'banana' 时，相当于在类命名空间中查找 'banana'。
        WilyDict 没有 'banana'，触发 __missing__，
        自动创建 'banana' → 0，返回 0。
        """
        if key.startswith('__') and key.endswith('__'):
            # 双下划线属性（如 __name__、__module__）不自动赋值
            # 必须抛出 KeyError，让 Python 按正常方式处理
            raise KeyError(key)
        # 为普通名称分配递增整数值
        self[key] = value = self.__next_value
        self.__next_value += 1
        return value


class AutoConstMeta(type):
    """元类：提供 WilyDict 作为类命名空间，实现自动常量。"""

    @classmethod
    def __prepare__(meta_cls, cls_name, bases, **kwargs):
        # 返回 WilyDict 实例作为类命名空间
        # Python 读取类主体时，所有名称查找都在这里进行
        return WilyDict()


class AutoConst(metaclass=AutoConstMeta):
    """基类：让用户无须关心元类的存在。"""
    pass


# ============================================================
# 使用示例：神奇的自动常量类
# ============================================================

class Flavor(AutoConst):
    # 这些"属性名"实际上只是被 Python 读取了一下
    # 触发 WilyDict.__missing__，自动赋值
    banana   # → 0（第一个，__missing__ 分配值 0）
    coconut  # → 1
    vanilla  # → 2

print(Flavor.vanilla)           # 2
print(Flavor.banana)            # 0
print(Flavor.coconut)           # 1

# 背后的执行过程：
# 1. AutoConstMeta.__prepare__ 返回 WilyDict 实例
# 2. Python 读取 class Flavor: 主体
# 3. 遇到 'banana' → 在 WilyDict 中查找 → 不存在 → __missing__ → 创建 {'banana': 0}
# 4. 遇到 'coconut' → 同上 → 创建 {'coconut': 1}
# 5. 遇到 'vanilla' → 同上 → 创建 {'vanilla': 2}
# 6. AutoConstMeta.__new__ 收到填充好的 WilyDict，构建类
```

## 13. 元类的实际运用建议

### 13.1 可简化/替代元类的现代功能

| 以前需要元类的场景 | 现代替代方案（Python 3.6+）|
|----------------|--------------------------|
| 自动为子类注入方法 | `__init_subclass__` |
| 自动设置描述符名称 | `__set_name__`（Python 3.6+）|
| 按顺序处理类属性 | 内置 `dict` 保留插入顺序（Python 3.7+）|
| 简单的类增强 | 类装饰器（更易理解）|

> **推荐原则**：先考虑 `__init_subclass__`，再考虑类装饰器，最后才考虑元类。

### 13.2 元类适用的场景（真正需要元类时）

```python
# 元类最不可替代的场景：在 type.__new__ 之前配置 __slots__

# ✅ 适合用元类的场景：
# 1. 需要在类创建之前修改 cls_dict（如配置 __slots__）
# 2. 需要定制 __prepare__（提供自定义命名空间）
# 3. 需要在元类层面定制类的 __repr__、__getitem__ 等特殊方法
# 4. 框架开发（Django ORM、SQLAlchemy、pydantic 等）
# 5. 实现类似 Enum 的功能

# ❌ 不适合用元类的场景（用更简单方案替代）：
# 1. 仅仅需要为子类注册 → __init_subclass__
# 2. 仅仅需要设置描述符名称 → __set_name__
# 3. 仅仅需要为类添加方法 → 类装饰器
```

### 13.3 元类冲突：一个类只能有一个元类

```python
import abc

# ❌ 错误：两个不同的元类产生冲突
class Record(abc.ABC, metaclass=SomeMeta):
    pass
# TypeError: metaclass conflict: the metaclass of a derived class
# must be a (non-strict) subclass of the metaclasses of all its bases
# 原因：abc.ABC 的元类是 ABCMeta，SomeMeta 与 ABCMeta 不是父子关系

# ✅ 解决方案：创建同时继承两个元类的新元类
class CombinedMeta(abc.ABCMeta, SomeMeta):
    pass

class Record(metaclass=CombinedMeta):
    pass
```

### 13.4 元类应作为实现细节

```python
# ✅ 好的做法：为用户提供普通的基类，元类隐藏在背后
class Checked(metaclass=CheckedMeta):
    """用户看到的是这个类，不需要知道 CheckedMeta 的存在。"""
    pass

# 用户只需：
class Movie(Checked):
    title: str
    year: int

# ❌ 不好的做法：要求用户直接指定元类
class Movie(metaclass=CheckedMeta):   # 用户需要知道 CheckedMeta
    title: str
    year: int
```

## 14. 四种方案对比速查表

### 四层工具的能力边界

| 能力 | type() | `__init_subclass__` | 类装饰器 | 元类 |
|------|:------:|:-------------------:|:--------:|:----:|
| 运行时创建类 | ✅ | — | — | — |
| 子类创建时回调 | — | ✅ | — | ✅ |
| 类创建后修改 | ✅ | ✅ | ✅ | ✅ |
| 类创建前修改 cls_dict | ❌ | ❌ | ❌ | ✅ |
| 配置 `__slots__` | ✅ | ❌ | ❌ | ✅ |
| 自定义类命名空间（`__prepare__`）| ❌ | ❌ | ❌ | ✅ |
| 与继承兼容 | ✅ | ✅ | ⚠️ | ⚠️ |
| 复杂度 | 低 | 低 | 中 | 高 |

### 执行顺序总结

```
class Klass(Builder, metaclass=MetaKlass):  处理顺序：

导入时（从早到晚）：
  1️⃣  MetaKlass.__prepare__          ← 元类，最早
  2️⃣  Klass 类主体执行（含描述符 __init__）
  3️⃣  MetaKlass.__new__              ← 元类创建类
  4️⃣  描述符 __set_name__            ← type.__new__ 返回后
  5️⃣  Builder.__init_subclass__      ← type.__new__ 返回后
  6️⃣  @deco 类装饰器                 ← 最晚

运行时（显式调用时）：
  7️⃣  Klass()  → __init__
  8️⃣  obj.method() → 实例方法
  9️⃣  obj.attr = v → 描述符 __set__
```

### 何时选择哪种方案

```
需求分析 → 方案推荐：

① 运行时根据数据创建类                    → type() 工厂函数
② 定义基类，子类自动获得某种行为           → __init_subclass__
③ 用 @decorator 增强任意类               → 类装饰器
④ 配置 __slots__ / 定制类命名空间         → 元类（唯一选择）
⑤ 现有框架要求（如 Django Model）         → 按框架规范继承基类

终极原则：先用最简单的方案，遇到限制再升级。
```
