## 2. 装饰器基础：3个核心性质

### 2.1 装饰器本质

```python
# 装饰器是可调用对象，参数是函数，返回函数（或其他可调用对象）

# 写法一：@ 语法（推荐）
@decorate
def target():
    print('running target()')

# 写法二：等价的直接调用
def target():
    print('running target()')
target = decorate(target)   # 完全等价

# ⚠️ 关键点：执行后 target 名称绑定的是 decorate(target) 返回的对象
# 可能是原来的 target，也可能是全新的函数
```

### 2.2 最简单的装饰器示例

```python
def deco(func):
    """最简装饰器：把被装饰的函数替换成另一个函数。"""
    def inner():
        print('running inner()')
    return inner    # ← 返回内部函数，而非 func

@deco
def target():
    print('running target()')

target()    # 实际运行的是 inner()，输出 'running inner()'
print(target)  # <function deco.<locals>.inner at 0x...>
```

### 2.3 注册装饰器（不替换函数）

```python
registry = []

def register(func):
    """注册装饰器：不替换被装饰的函数，只是记录它。"""
    print(f'running register({func})')
    registry.append(func)
    return func     # ← 返回原来的函数，不替换

@register
def f1():
    print('running f1()')

@register
def f2():
    print('running f2()')

def f3():           # 未被装饰
    print('running f3()')

# 使用场景：URL路由注册、命令行命令注册、测试发现等框架
# Flask、Django、Click 都大量使用这种注册装饰器模式
```

## 3. Python 何时执行装饰器

> **⚠️ 重点**：装饰器在**导入时（import time）**立即执行，被装饰的函数在**运行时（runtime）**才执行。

```python
# registration.py 模块示例
registry = []

def register(func):
    print(f'running register({func})')  # 导入时执行
    registry.append(func)
    return func

@register
def f1():
    print('running f1()')   # 只在显式调用时执行

@register
def f2():
    print('running f2()')

def f3():
    print('running f3()')

def main():
    print('running main()')
    print('registry ->', registry)
    f1()
    f2()
    f3()

if __name__ == '__main__':
    main()

# --- 当作脚本运行时的输出 ---
# running register(<function f1 ...>)   ← 导入时执行
# running register(<function f2 ...>)   ← 导入时执行
# running main()
# registry -> [<function f1 ...>, <function f2 ...>]
# running f1()
# running f2()
# running f3()

# --- 当作模块导入时的输出 ---
# running register(<function f1 ...>)   ← 导入时执行
# running register(<function f2 ...>)   ← 导入时执行
# （f1、f2、f3 不执行）
```

**两个关键发现**：

| | 导入时 | 运行时 |
|--|--------|--------|
| 装饰器本身 | ✅ 执行 | — |
| 被装饰的函数 | ❌ 不执行 | ✅ 显式调用才执行 |

## 4. 变量作用域规则

### 4.1 反直觉的 UnboundLocalError

```python
b = 6

def f2(a):
    print(a)
    print(b)    # ← 这里会报 UnboundLocalError！
    b = 9       # ← 因为这里给 b 赋值了

f2(3)
# 输出：
# 3
# UnboundLocalError: local variable 'b' referenced before assignment

# 原因：Python 编译函数主体时，发现 b 在函数内被赋值
# → 整个函数中 b 都被当作局部变量
# → print(b) 时局部的 b 还没赋值 → 报错
# 这不是 bug，是 Python 的设计选择
```

### 4.2 四种作用域（LEGB 规则）

```python
x = 'global'                    # G：全局作用域

def outer():
    x = 'enclosing'             # E：外层函数局部作用域（闭包的来源）

    def inner():
        x = 'local'             # L：内层函数局部作用域
        print(x)                # 先找 L，再找 E，再找 G，最后找 B

    inner()

# B：内置作用域（__builtins__，如 print、len 等）

# LEGB 查找顺序（从内到外）：
# L → E → G → B
# 赋值操作默认在 L（局部）作用域创建变量
```

### 4.3 `global` 声明

```python
b = 6

def f3(a):
    global b        # 声明 b 来自全局作用域，不是局部变量
    print(a)
    print(b)
    b = 9           # 修改的是全局变量 b

f3(3)
# 输出：3  6
print(b)  # 9（全局变量被修改了）
```

## 5. 闭包详解

> **定义**：闭包是一个函数，它**保留了定义时存在的自由变量的绑定**。调用函数时，虽然定义作用域不再存在，仍能使用那些绑定。

### 5.1 什么是自由变量

```python
def make_averager():
    series = []          # ← make_averager 的局部变量

    def averager(new_value):
        series.append(new_value)  # ← series 对 averager 来说是"自由变量"
        total = sum(series)
        return total / len(series)

    return averager      # 返回内部函数（带着对 series 的引用）

avg = make_averager()
# make_averager() 执行完毕，其局部作用域"消失"了
# 但 series 仍然活着，因为 averager 闭包持有它的引用

print(avg(10))  # 10.0
print(avg(11))  # 10.5
print(avg(12))  # 11.0
```

### 5.2 检查闭包的内部结构

```python
avg = make_averager()
avg(10); avg(11); avg(12)

# 查看自由变量名称
print(avg.__code__.co_freevars)    # ('series',)

# 查看自由变量的值（cell 对象）
print(avg.__closure__)             # (<cell at 0x...: list object ...>,)
print(avg.__closure__[0].cell_contents)  # [10, 11, 12]

# 总结：
# co_freevars  → 自由变量的名称
# __closure__  → 存储自由变量实际值的 cell 对象列表
```

### 5.3 类实现 vs 闭包实现（等价对比）

```python
# ---- 类实现（显式状态）----
class Averager:
    def __init__(self):
        self.series = []

    def __call__(self, new_value):
        self.series.append(new_value)
        return sum(self.series) / len(self.series)

avg_class = Averager()

# ---- 闭包实现（隐式状态）----
def make_averager():
    series = []
    def averager(new_value):
        series.append(new_value)
        return sum(series) / len(series)
    return averager

avg_closure = make_averager()

# 两种调用方式完全一样：
avg_class(10)   # 10.0
avg_closure(10) # 10.0
```

## 6. nonlocal 声明

> **必须使用 `nonlocal` 的场景**：在闭包内对**不可变类型**（int、str、tuple 等）重新绑定（赋值），否则会创建局部变量，遮盖自由变量。

### 6.1 不使用 nonlocal 的错误

```python
def make_averager_bad():
    count = 0
    total = 0

    def averager(new_value):
        count += 1      # ← 等价于 count = count + 1，赋值！
        total += new_value  # ← 同上
        return total / count

    return averager

avg = make_averager_bad()
avg(10)
# UnboundLocalError: local variable 'count' referenced before assignment
# 原因：count += 1 让 Python 把 count 当成局部变量
```

### 6.2 使用 nonlocal 的正确实现

```python
def make_averager():
    count = 0
    total = 0

    def averager(new_value):
        nonlocal count, total   # ← 声明 count 和 total 是自由变量
        count += 1              # 现在修改的是外层函数的 count
        total += new_value      # 现在修改的是外层函数的 total
        return total / count

    return averager

avg = make_averager()
print(avg(10))  # 10.0
print(avg(11))  # 10.5
print(avg(12))  # 11.0
```

### 6.3 为什么 series.append 不需要 nonlocal

```python
# 可变对象（list、dict、set）不需要 nonlocal
# 因为 series.append() 不是赋值操作，不会创建新的局部变量

def make_averager():
    series = []                 # 可变对象

    def averager(new_value):
        series.append(new_value)  # ← 修改对象内容，不是重新绑定变量
        # series = ... 这样才是赋值，才需要 nonlocal
        return sum(series) / len(series)

    return averager
```

### 6.4 变量查找完整规则总结

```
Python 字节码编译器对函数主体中变量 x 的处理规则：

1. 如果有 global x 声明
   → x 来自模块全局作用域

2. 如果有 nonlocal x 声明
   → x 来自最近一个定义它的外层函数

3. 如果 x 是参数，或在函数主体中被赋值
   → x 是局部变量

4. 如果引用了 x 但未赋值也不是参数
   → 依次查找：外层函数作用域（E）→ 全局（G）→ 内置（B）
```

## 7. 实现一个完整的装饰器

### 7.1 基础版（有缺陷）

```python
import time


def clock(func):
    """计时装饰器：记录函数运行时间和调用参数。"""
    def clocked(*args):              # ← 接受任意位置参数
        t0 = time.perf_counter()
        result = func(*args)         # ← 调用原始函数（闭包中的自由变量）
        elapsed = time.perf_counter() - t0
        name = func.__name__
        arg_str = ', '.join(repr(arg) for arg in args)
        print(f'[{elapsed:0.8f}s] {name}({arg_str}) -> {result!r}')
        return result
    return clocked                   # ← 返回包装函数，替换原始函数

@clock
def snooze(seconds):
    time.sleep(seconds)

@clock
def factorial(n):
    return 1 if n < 2 else n * factorial(n-1)

# 测试
factorial(6)
# 输出：
# [0.00000095s] factorial(1) -> 1
# [0.00002408s] factorial(2) -> 2
# ...
# [0.00008297s] factorial(6) -> 720

# 缺陷：遮盖了 __name__、__doc__
print(factorial.__name__)  # 'clocked'（应该是 'factorial'）
```

### 7.2 改进版（使用 `functools.wraps`）⭐

```python
import time
import functools


def clock(func):
    """计时装饰器（改进版）：支持关键字参数，正确保留函数元数据。"""
    @functools.wraps(func)           # ← 把 func 的元数据复制到 clocked
    def clocked(*args, **kwargs):    # ← 同时支持位置参数和关键字参数
        t0 = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - t0
        name = func.__name__
        arg_lst = [repr(arg) for arg in args]
        arg_lst.extend(f'{k}={v!r}' for k, v in kwargs.items())
        arg_str = ', '.join(arg_lst)
        print(f'[{elapsed:0.8f}s] {name}({arg_str}) -> {result!r}')
        return result
    return clocked


@clock
def factorial(n):
    return 1 if n < 2 else n * factorial(n-1)

# 改进后：
print(factorial.__name__)  # 'factorial'（正确！）
print(factorial.__doc__)   # 原始函数的文档字符串（正确！）
```

## 8. 标准库中的装饰器

### 8.1 `functools.cache`：无限缓存备忘（Python 3.9+）

```python
import functools
from clockdeco import clock


@functools.cache        # ← 无限缓存：所有参数组合的结果都缓存
@clock                  # ← 叠放装饰器：先应用 clock，再应用 cache
def fibonacci(n):
    """斐波那契数列（带缓存的递归实现）。"""
    if n < 2:
        return n
    return fibonacci(n - 2) + fibonacci(n - 1)


# 叠放装饰器等价于：
# fibonacci = functools.cache(clock(fibonacci))

# 不带缓存：fibonacci(30) 调用 2692537 次，耗时 12秒
# 带缓存：  fibonacci(30) 只调用 31 次，耗时 0.0002秒

# ⚠️ 注意：
# - 被装饰函数的参数必须可哈希（作为字典的键）
# - 缓存会持续增长，适合短期脚本
# - 长期运行的进程推荐使用 lru_cache 并设置 maxsize
```

### 8.2 `functools.lru_cache`：有限缓存（兼容旧版本）

```python
from functools import lru_cache


# 两种使用方式（Python 3.8+ 均可）：
@lru_cache              # 不加括号（3.8+）
def costly_function(a, b): ...

@lru_cache()            # 加括号（所有版本均可）
def costly_function(a, b): ...

# 自定义参数
@lru_cache(maxsize=2**20, typed=True)
def costly_function(a, b): ...

# maxsize 参数：
# - 默认 128，缓存满后淘汰最久未使用的条目（LRU）
# - 设为 None：禁用 LRU 淘汰（等同于 @cache）
# - 最佳性能：设为 2 的幂次方

# typed 参数：
# - 默认 False：f(1) 和 f(1.0) 共享一个缓存条目
# - True：f(1) 和 f(1.0) 分开缓存（结果可能不同）
```

### 8.3 `functools.singledispatch`：单分派泛化函数 ⭐

> 根据**第一个参数的类型**，把调用分派给不同的实现函数。

```python
from functools import singledispatch
from collections import abc
import fractions
import decimal
import html
import numbers


@singledispatch                         # ← 注册基函数（处理 object 类型）
def htmlize(obj: object) -> str:
    """默认实现：任何未特殊处理的类型都用这个。"""
    content = html.escape(repr(obj))
    return f'<pre>{content}</pre>'


@htmlize.register                       # ← 注册专门的实现（通过类型提示确定类型）
def _(text: str) -> str:
    """字符串类型：换行转为 <br/>，放在 <p> 中。"""
    content = html.escape(text).replace('\n', '<br/>\n')
    return f'<p>{content}</p>'


@htmlize.register
def _(seq: abc.Sequence) -> str:        # ← 使用抽象基类，支持 list/tuple 等
    """序列类型：渲染成 HTML 列表。"""
    inner = '</li>\n<li>'.join(htmlize(item) for item in seq)
    return '<ul>\n<li>' + inner + '</li>\n</ul>'


@htmlize.register
def _(n: numbers.Integral) -> str:      # ← 整数抽象基类（比 int 更通用）
    """整数类型：同时显示十进制和十六进制。"""
    return f'<pre>{n} (0x{n:x})</pre>'


@htmlize.register
def _(n: bool) -> str:                  # ← bool 是 Integral 子类，但优先匹配
    """布尔类型：特殊处理。"""
    return f'<pre>{n}</pre>'


# 一个装饰器支持多种类型（叠放 register）
@htmlize.register(decimal.Decimal)      # ← 不用类型提示，直接传类型
@htmlize.register(float)
def _(x) -> str:
    """浮点/Decimal：显示分数近似值。"""
    frac = fractions.Fraction(x).limit_denominator()
    return f'<pre>{x} ({frac.numerator}/{frac.denominator})</pre>'


# 测试
print(htmlize(42))          # <pre>42 (0x2a)</pre>
print(htmlize('hello\nworld'))  # <p>hello<br/>\nworld</p>
print(htmlize([1, 2, 3]))   # <ul><li>...</li></ul>
print(htmlize(True))        # <pre>True</pre>
print(htmlize(2/3))         # <pre>0.666... (2/3)</pre>

# singledispatch 的优势：
# - 模块化：各类型的实现分散在不同地方，而非一个大 if/elif
# - 可扩展：任何模块都可以为新类型注册专门实现
# - 推荐注册抽象基类（numbers.Integral）而非具体类（int），支持更广泛
```

## 9. 参数化装饰器（装饰器工厂）

### 9.1 参数化注册装饰器

```python
registry = set()


def register(active=True):
    """装饰器工厂函数：返回真正的装饰器。

    参数：
        active: 为 True 时注册函数，为 False 时取消注册
    """
    def decorate(func):             # ← 真正的装饰器
        print(f'running register(active={active})->decorate({func})')
        if active:
            registry.add(func)
        else:
            registry.discard(func)
        return func                 # ← 不替换函数，原样返回
    return decorate                 # ← 返回装饰器


# 使用方式：必须以函数调用形式使用，即使不传参数
@register(active=False)             # ← f1 不会被注册
def f1():
    print('running f1()')

@register()                         # ← 必须加括号！返回 decorate，再装饰 f2
def f2():
    print('running f2()')

def f3():
    print('running f3()')


# 不用 @ 语法，直接调用：
register()(f3)          # 等价于 @register()，把 f3 加入 registry
register(active=False)(f2)  # 把 f2 从 registry 中移除

# 关键理解：
# @register()   → register() 返回 decorate → decorate(f2) 装饰 f2
# @register     → 错误！register 直接作为装饰器，参数是被装饰的函数
```

### 9.2 参数化 clock 装饰器（三层嵌套）

```python
import time

DEFAULT_FMT = '[{elapsed:0.8f}s] {name}({args}) -> {result}'


def clock(fmt=DEFAULT_FMT):             # ← 第一层：装饰器工厂，接受参数
    def decorate(func):                 # ← 第二层：真正的装饰器
        def clocked(*_args):            # ← 第三层：包装函数，替换被装饰的函数
            t0 = time.perf_counter()
            _result = func(*_args)      # 调用原始函数
            elapsed = time.perf_counter() - t0
            name = func.__name__
            args = ', '.join(repr(arg) for arg in _args)
            result = repr(_result)
            # 使用 **locals() 把所有局部变量注入格式字符串
            print(fmt.format(**locals()))
            return _result
        return clocked                  # 返回包装函数
    return decorate                     # 返回装饰器


# 使用默认格式
@clock()
def snooze(seconds):
    time.sleep(seconds)

# 使用自定义格式
@clock('{name}: {elapsed}s')
def snooze_custom(seconds):
    time.sleep(seconds)

@clock('{name}({args}) dt={elapsed:0.3f}s')
def snooze_brief(seconds):
    time.sleep(seconds)


# 测试
snooze(.123)        # [0.12412500s] snooze(0.123) -> None
snooze_custom(.123) # snooze: 0.12414693832397461s
snooze_brief(.123)  # snooze(0.123) dt=0.124s

# 结构总结：
# clock(fmt)        → 执行立即返回 decorate
# decorate(func)    → 执行立即返回 clocked
# clocked(*_args)   → 每次调用被装饰函数时执行
```

## 10. 基于类的装饰器

> 用定义了 `__call__` 方法的类实现装饰器，比嵌套函数更清晰，适合复杂的装饰器。

```python
import time

DEFAULT_FMT = '[{elapsed:0.8f}s] {name}({args}) -> {result}'


class clock:
    """参数化计时装饰器（类实现版本）。

    等价于示例 9-24 中的函数实现，但结构更清晰。
    """

    def __init__(self, fmt=DEFAULT_FMT):
        """接收装饰器参数，存储在实例属性中。

        @clock('{name}(...)')
        等价于：clock('{name}(...)') 创建一个 clock 实例
        该实例作为装饰器被调用：instance(func)
        """
        self.fmt = fmt

    def __call__(self, func):
        """让实例成为可调用对象，作为真正的装饰器使用。

        clock 实例被调用时（decorating a function），
        返回包装函数 clocked。
        """
        def clocked(*_args):
            t0 = time.perf_counter()
            _result = func(*_args)
            elapsed = time.perf_counter() - t0
            name = func.__name__
            args = ', '.join(repr(arg) for arg in _args)
            result = repr(_result)
            print(self.fmt.format(**locals()))  # 从实例属性读取格式字符串
            return _result
        return clocked


# 使用方式完全一致
@clock()                        # clock() 创建实例，实例.__call__(snooze) 装饰
def snooze(seconds):
    time.sleep(seconds)

@clock('{name}({args}) dt={elapsed:0.3f}s')
def snooze_brief(seconds):
    time.sleep(seconds)
```

## 11. 常见误区与陷阱

### ❌ 误区一：以为装饰器在调用时执行

```python
# 装饰器在导入时立即执行，不是在调用被装饰函数时执行
@my_decorator
def my_func():
    pass

# ← 此时 my_decorator(my_func) 已经执行了！
# 即便从未调用 my_func()，装饰器也已经运行
```

### ❌ 误区二：忘记使用 `functools.wraps`

```python
def bad_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper  # ← 没有 @functools.wraps(func)

@bad_decorator
def my_func():
    """我的文档字符串"""
    pass

print(my_func.__name__)  # 'wrapper'（错误！应该是 'my_func'）
print(my_func.__doc__)   # None（文档字符串丢失！）

# ✅ 正确做法
import functools

def good_decorator(func):
    @functools.wraps(func)    # ← 复制 __name__、__doc__、__wrapped__ 等
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper
```

### ❌ 误区三：不可变类型的自由变量不用 nonlocal

```python
def make_counter():
    count = 0           # int 是不可变类型

    def counter():
        count += 1      # ← 错误！需要 nonlocal
        return count

    return counter

c = make_counter()
c()  # UnboundLocalError

# ✅ 正确
def make_counter():
    count = 0

    def counter():
        nonlocal count  # ← 声明 count 是自由变量
        count += 1
        return count

    return counter
```

### ❌ 误区四：参数化装饰器忘记调用工厂函数

```python
# ❌ 错误：register 需要参数时，不加括号就直接作为装饰器
@register       # 等价于 register(f1)，把 f1 当参数传给 register
def f1(): pass

# ✅ 正确：参数化装饰器必须以函数调用形式使用
@register()             # 使用默认参数
def f1(): pass

@register(active=True)  # 传入参数
def f2(): pass
```

### ❌ 误区五：叠放装饰器时搞错顺序

```python
# 叠放时，从下到上应用（离函数最近的先应用）
@alpha
@beta
def my_fn():
    pass

# 等价于：my_fn = alpha(beta(my_fn))
# 先应用 beta，再把结果传给 alpha

# 实际案例：
@functools.cache    # 后应用：缓存 clock 包装后的函数
@clock              # 先应用：给原始 fibonacci 加计时
def fibonacci(n):
    ...
# 等价于：fibonacci = functools.cache(clock(fibonacci))
```

## 12. 本章知识速查表

### 装饰器类型对比

| 装饰器类型 | 结构层数 | 语法示例 | 适用场景 |
|-----------|---------|---------|---------|
| 简单装饰器 | 2层（装饰器+包装函数）| `@deco` | 通用增强 |
| 注册装饰器 | 1层（只注册，不包装）| `@register` | URL路由、命令注册 |
| 参数化装饰器 | 3层（工厂+装饰器+包装）| `@clock(fmt='...')` | 可配置的增强 |
| 基于类的装饰器 | `__init__`+`__call__` | `@Clock(fmt='...')` | 复杂逻辑，易维护 |

### 标准库装饰器对比

| 装饰器 | 版本要求 | 特点 | 适用场景 |
|--------|---------|------|---------|
| `@functools.cache` | Python 3.9+ | 无限缓存，最简单 | 短期脚本 |
| `@functools.lru_cache` | Python 3.2+ | 可限制缓存大小 | 长期运行进程 |
| `@functools.singledispatch` | Python 3.4+ | 按第一参数类型分派 | 处理多种类型的函数 |
| `@functools.wraps` | Python 3.2+ | 复制函数元数据 | 构建装饰器时使用 |

### 作用域关键字对比

| 关键字 | 作用 | 必须使用的场景 |
|--------|------|--------------|
| `global` | 声明变量来自模块全局作用域 | 在函数内给全局变量赋新值 |
| `nonlocal` | 声明变量来自外层函数作用域 | 在闭包内给不可变类型的自由变量赋新值 |
