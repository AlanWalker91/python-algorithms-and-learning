## 2. 属性查找链（核心机制）

> **⚠️ 重点**：理解属性查找顺序是掌握本章所有内容的基础。

当你访问 `obj.attr` 时，Python 按以下顺序查找：

```
obj.attr 被访问
    │
    ▼
① type(obj).__mro__ 中有数据描述符（如 @property）？ → 最高优先级，直接调用
    │ 否
    ▼
② obj.__dict__ 中有该键？ → 直接返回
    │ 否
    ▼
③ type(obj).__mro__ 中有非数据描述符或普通类属性？ → 返回（函数/方法在这里）
    │ 否
    ▼
④ 调用 __getattr__(obj, 'attr') → 最后的兜底
    │ 仍找不到
    ▼
⑤ 抛出 AttributeError
```

**关键结论**：
- `@property`（数据描述符）**比实例 `__dict__` 优先级更高**（反直觉！）
- `__getattr__` 只在前三步都失败后才调用
- 直接操作 `obj.__dict__` 可以绕过所有特殊方法

## 3. 动态属性：`__getattr__` 与虚拟属性

### 3.1 `__getattr__` 的触发时机

```python
class MyClass:
    def __getattr__(self, name):
        # 仅当常规查找（__dict__、类属性、MRO）全部失败时才调用
        # name: 找不到的属性名
        print(f'找不到属性: {name}')
        raise AttributeError(f'{name} not found')
        # ⚠️ 应该抛 AttributeError，而非 KeyError 或其他异常
        # hasattr() 依赖 AttributeError 来判断属性是否存在
```

### 3.2 FrozenJSON：让 JSON 数据支持属性访问

> **场景**：JSON 数据要用 `feed['Schedule']['events'][40]['name']` 访问，太繁琐。
> 目标：实现 `feed.Schedule.events[40].name` 的属性式访问。

```python
from collections import abc
import keyword


class FrozenJSON:
    """只读接口，通过属性表示法访问 JSON 类数据。

    核心机制：__getattr__ 在属性找不到时，从内部字典中动态构建值。
    """

    def __init__(self, mapping):
        # 使用名称改写（双下划线前缀）保护内部字典
        # self.__data 实际存储为 self._FrozenJSON__data
        self.__data = dict(mapping)

    def __getattr__(self, name):
        """属性查找失败时的兜底方法。

        注意：只有在常规属性查找失败后才调用。
        self.__data 不会触发这里，因为它以 _FrozenJSON__data 存储在实例 __dict__ 中。
        """
        try:
            # 先尝试代理给内部字典的方法（如 .keys()、.items()）
            return getattr(self.__data, name)
        except AttributeError:
            # 再从字典中取值，递归包装
            return FrozenJSON.build(self.__data[name])
            # ⚠️ 若 name 不存在，会抛 KeyError 而非 AttributeError
            # 更好的写法（推荐）：
            # try:
            #     return FrozenJSON.build(self.__data[name])
            # except KeyError:
            #     raise AttributeError(name) from None

    def __dir__(self):
        """支持 Tab 自动补全。

        dir() 调用时返回可用属性列表，IDE 和交互式控制台依赖此方法。
        """
        return self.__data.keys()

    @classmethod
    def build(cls, obj):
        """备选构造方法，根据对象类型选择包装策略。"""
        if isinstance(obj, abc.Mapping):
            # 字典 → 创建 FrozenJSON 实例（递归支持嵌套）
            return cls(obj)
        elif isinstance(obj, abc.MutableSequence):
            # 列表 → 递归包装每个元素
            return [cls.build(item) for item in obj]
        else:
            # 基本类型（str、int、None 等）→ 原封不动返回
            return obj


# ---- 使用示例 ----
import json

raw = {
    "Schedule": {
        "events": [
            {"name": "Python Talk", "speakers": [3471, 5199]}
        ],
        "speakers": [
            {"serial": 3471, "name": "Anna Martelli"}
        ]
    }
}

feed = FrozenJSON(raw)

# 属性式访问（嵌套）
print(feed.Schedule.events[0].name)     # 'Python Talk'
print(feed.Schedule.speakers[0].name)  # 'Anna Martelli'

# 字典方法仍然可用（被代理给 self.__data）
print(sorted(feed.Schedule.keys()))    # ['events', 'speakers']
```

### 3.3 处理非法属性名

```python
import keyword


class FrozenJSON:
    def __init__(self, mapping):
        self.__data = {}
        for key, value in mapping.items():
            # 处理 Python 关键字（如 class、for、return）
            if keyword.iskeyword(key):
                key += '_'  # class → class_
            # 处理非法标识符（如以数字开头的键）
            if not key.isidentifier():
                key = f'_{key}'  # 2be → _2be
            self.__data[key] = value

    def __getattr__(self, name):
        try:
            return getattr(self.__data, name)
        except AttributeError:
            try:
                return FrozenJSON.build(self.__data[name])
            except KeyError:
                raise AttributeError(name) from None

    @classmethod
    def build(cls, obj):
        from collections import abc
        if isinstance(obj, abc.Mapping):
            return cls(obj)
        elif isinstance(obj, abc.MutableSequence):
            return [cls.build(item) for item in obj]
        else:
            return obj


# 测试关键字处理
student = FrozenJSON({'name': 'Jim Bo', 'class': 1982})
print(student.class_)   # 1982（自动加了下划线）
print(student.name)     # 'Jim Bo'
```

## 4. 对象创建：`__new__` 与 `__init__` 的分工

> **⚠️ 重点**：`__new__` 才是真正的构造方法，`__init__` 只是初始化方法。

### 4.1 对象创建的完整流程（伪代码）

```python
# 调用 Foo('bar') 等价于：
def make(the_class, some_arg):
    # 步骤1：__new__ 创建对象
    new_object = the_class.__new__(the_class, some_arg)

    # 步骤2：只有返回值是本类实例时，才调用 __init__
    if isinstance(new_object, the_class):
        the_class.__init__(new_object, some_arg)

    # 步骤3：返回对象
    return new_object

# 因此：
x = Foo('bar')
# 等价于：
x = make(Foo, 'bar')
```

### 4.2 `__new__` 实现灵活对象工厂

```python
from collections import abc
import keyword


class FrozenJSON:
    """使用 __new__ 替代 classmethod build，实现更自然的工厂逻辑。"""

    def __new__(cls, arg):
        """真正的构造方法。

        根据 arg 的类型决定返回什么：
        - 字典 → 返回 FrozenJSON 实例（正常创建，之后调用 __init__）
        - 列表 → 返回列表（不是本类实例，__init__ 不会被调用）
        - 其他 → 原样返回（同上）
        """
        if isinstance(arg, abc.Mapping):
            # 正常创建 FrozenJSON 实例
            # super().__new__(cls) 调用 object.__new__(FrozenJSON)
            return super().__new__(cls)
        elif isinstance(arg, abc.MutableSequence):
            # 返回列表——不是 FrozenJSON 实例，__init__ 不会执行
            return [cls(item) for item in arg]
        else:
            # 返回原始值——同上
            return arg

    def __init__(self, mapping):
        """初始化方法。

        只在 __new__ 返回 FrozenJSON 实例时才被调用。
        """
        self.__data = {}
        for key, value in mapping.items():
            if keyword.iskeyword(key):
                key += '_'
            self.__data[key] = value

    def __getattr__(self, name):
        try:
            return getattr(self.__data, name)
        except AttributeError:
            # 现在直接调用 FrozenJSON()，Python 会调用 __new__
            return FrozenJSON(self.__data[name])

    def __dir__(self):
        return self.__data.keys()


# ---- 测试 ----
obj = FrozenJSON({'a': 1, 'b': [10, 20]})
print(obj.a)       # 1
print(obj.b)       # [10, 20]（列表，每项是整数，原样返回）

nested = FrozenJSON({'inner': {'x': 42}})
print(nested.inner.x)  # 42（嵌套字典自动包装为 FrozenJSON）
```

## 5. 数据驱动属性：`__dict__.update` 技巧

> **场景**：把关键字参数批量转化为实例属性，常用于数据类或动态 ORM。

```python
class Record:
    """用关键字参数动态创建实例属性的简洁模式。"""

    def __init__(self, **kwargs):
        # 关键技巧：直接更新实例的 __dict__
        # 等价于逐一执行 self.key = value，但更简洁
        self.__dict__.update(kwargs)
        # ⚠️ 注意：这会绕过任何 @property 或 __setattr__
        # 若类中有描述符，应逐一赋值而非批量更新

    def __repr__(self):
        return f'<{self.__class__.__name__} serial={self.serial!r}>'


# ---- 使用示例 ----
speaker = Record(serial=3471, name='Anna Martelli', twitter='annaraven')
print(speaker.name)    # 'Anna Martelli'
print(speaker.serial)  # 3471
print(speaker)         # <Record serial=3471>

# 等价于 types.SimpleNamespace、argparse.Namespace 的功能
```

## 6. @property 特性详解

### 6.1 `property` 是类，不是语法糖

```python
# property 的完整签名：
# property(fget=None, fset=None, fdel=None, doc=None)

# 两种完全等价的写法：

# ---- 写法一：装饰器（推荐，可读性好）----
class LineItem:
    def __init__(self, description, weight, price):
        self.description = description
        self.weight = weight   # 触发 setter，会执行验证
        self.price = price

    @property
    def weight(self):
        """重量（千克），必须大于零。"""  # 文档字符串成为特性的文档
        return self.__weight   # 实际值存在私有属性 __weight 中

    @weight.setter
    def weight(self, value):
        if value > 0:
            self.__weight = value
        else:
            raise ValueError('weight must be > 0')

    @weight.deleter
    def weight(self):
        del self.__weight


# ---- 写法二：经典调用（工厂函数场景常用）----
class LineItem:
    def __init__(self, description, weight, price):
        self.description = description
        self.weight = weight
        self.price = price

    def get_weight(self):
        return self.__weight

    def set_weight(self, value):
        if value > 0:
            self.__weight = value
        else:
            raise ValueError('weight must be > 0')

    # property() 的四个参数：fget, fset, fdel, doc
    weight = property(get_weight, set_weight, doc='weight in kilograms')


# ---- 测试 ----
raisins = LineItem('Golden raisins', 10, 6.95)
print(raisins.weight)       # 10  → 调用 getter
raisins.weight = 15         # → 调用 setter，验证通过
raisins.weight = -1         # → ValueError: weight must be > 0
print(raisins.__dict__)     # {'description': ..., '_LineItem__weight': 10, ...}
# 注意：实际值存在 _LineItem__weight（名称改写后的私有属性）
```

### 6.2 只读特性

```python
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self):
        """半径（只读）。"""
        return self._radius

    @property
    def area(self):
        """面积（计算属性，只读）。"""
        import math
        return math.pi * self._radius ** 2

    # 没有 @radius.setter → 特性是只读的
    # 尝试赋值会抛出 AttributeError: can't set attribute


c = Circle(5)
print(c.radius)    # 5
print(c.area)      # 78.539...
c.radius = 10      # AttributeError: can't set attribute
```

## 7. 特性覆盖实例属性（重点）

> **⚠️ 最反直觉的规则**：`@property`（覆盖型描述符）的优先级**高于**同名实例属性。
> 这与普通类属性被实例属性遮盖的行为**正好相反**。

```python
class Demo:
    # 普通类属性
    data = 'class data'

    # 特性（覆盖型描述符）
    @property
    def prop(self):
        return 'prop value'


obj = Demo()

# ---- 普通类属性的行为：实例属性可以遮盖类属性 ----
print(obj.data)      # 'class data'（从类中读取）
obj.data = 'bar'     # 在实例 __dict__ 中创建同名属性
print(obj.data)      # 'bar'（实例属性遮盖了类属性）
print(Demo.data)     # 'class data'（类属性本身不变）

# ---- 特性的行为：实例属性无法遮盖特性 ----
print(obj.prop)      # 'prop value'（调用 getter）

# 尝试为 prop 赋值 → AttributeError（没有 setter）
# obj.prop = 'foo'  # AttributeError: can't set attribute

# 强行写入实例 __dict__（绕过特性）
obj.__dict__['prop'] = 'foo'
print(obj.__dict__)  # {'data': 'bar', 'prop': 'foo'}
print(obj.prop)      # 仍然是 'prop value'！特性胜出，实例属性被忽略

# 只有删除类上的特性，实例属性才能被"看见"
Demo.prop = 'baz'    # 用普通值替换类上的特性
print(obj.prop)      # 'foo'（现在读到实例 __dict__ 里的值了）
```

**实际应用场景**：

```python
class Event:
    def __init__(self, **kwargs):
        self.__dict__.update(kwargs)  # JSON 数据写入实例 __dict__
        # 假设 kwargs 包含 {'speakers': [3471, 5199]}
        # 实例 __dict__ 中有 speakers 键，存储原始编号列表

    @property
    def speakers(self):
        """返回 Speaker 对象列表，而非编号列表。"""
        # ⚠️ 必须用 self.__dict__['speakers']，而不是 self.speakers！
        # 否则会递归调用自身，导致 RecursionError
        spkr_serials = self.__dict__['speakers']
        return [Record.fetch(f'speaker.{key}') for key in spkr_serials]
        # 特性优先级高于实例属性，所以 self.speakers 总是调用这里
        # 但 self.__dict__['speakers'] 绕过了特性，直接读原始数据
```

## 8. 缓存特性：三种方案对比

### 8.1 方案一：手动实现缓存（旧式写法，不推荐）

```python
class Event:
    @property
    def speakers(self):
        # 用私有属性 __speaker_objs 缓存结果
        if not hasattr(self, '__speaker_objs'):
            spkr_serials = self.__dict__['speakers']
            # 计算并缓存（这会在初始化后创建属性，违背键共享优化）
            self.__speaker_objs = [
                Record.fetch(f'speaker.{key}') for key in spkr_serials
            ]
        return self.__speaker_objs
    # 问题：多线程场景下存在竞争条件，两个线程可能同时计算
```

### 8.2 方案二：`@functools.cached_property`（线程安全，但有限制）

```python
import functools


class Event:
    @functools.cached_property
    def venue(self):
        """首次访问时计算，之后缓存到同名实例属性。

        cached_property 是非覆盖型描述符（没有 __set__）。
        首次调用后，结果被写入实例 __dict__['venue']。
        之后读取 self.venue 直接从 __dict__ 拿，不再调用方法。
        """
        key = f'venue.{self.venue_serial}'
        return Record.fetch(key)

    # ⚠️ cached_property 的限制：
    # 1. 不能用于"同名实例属性已存在"的场景（如 speakers）
    #    因为它先读实例属性，若存在则不调用方法
    # 2. 不能用于定义了 __slots__ 的类
    # 3. 违背 PEP 412 键共享优化（初始化后才创建属性）
```

### 8.3 方案三：`@property + @cache`（推荐，万能）⭐

```python
import functools


class Event:
    @property           # ← 外层：把函数变成特性（覆盖型描述符）
    @functools.cache    # ← 内层：把函数结果缓存（首次调用后不再计算）
    def speakers(self):
        """缓存演讲者列表。

        装饰器叠放等价于：
        speakers = property(cache(speakers))

        @cache 先应用于函数，返回带缓存的新函数。
        @property 再应用，把带缓存的函数包装成特性。
        """
        # 从实例 __dict__ 直接取原始数据（绕过特性，防止递归）
        spkr_serials = self.__dict__['speakers']
        return [
            Record.fetch(f'speaker.{key}') for key in spkr_serials
        ]

    # ✅ 适合 speakers 这种"同名实例属性已存在"的场景
    # ✅ 线程安全（@cache 内部有锁）
    # ✅ 支持有 __slots__ 的类
    # ⚠️ 顺序很重要：@property 必须在上，@cache 必须在下
```

**三种方案对比表**：

| 方案 | 线程安全 | 允许写入 | 适用场景 |
|------|----------|----------|----------|
| 手动实现 | ❌ 竞争条件 | ✅ | 旧代码 |
| @cached_property | ✅ | ✅ | 无同名实例属性 |
| @property + @cache | ✅ | ❌ 只读 | 所有场景（推荐）|

## 9. 特性工厂函数

> **场景**：多个属性需要相同的验证逻辑时，避免重复写 `@property + @setter`。

```python
def quantity(storage_name):
    """特性工厂函数：生成验证"值必须 > 0"的特性对象。

    Args:
        storage_name: 在实例 __dict__ 中存储值的键名

    Returns:
        property: 带验证逻辑的特性对象
    """

    def qty_getter(instance):
        """读值函数。

        参数名用 instance 而非 self，因为这个函数不在类主体中。
        instance 指托管实例（如 LineItem 对象）。
        """
        # 直接从实例 __dict__ 读取，绕过特性，避免递归
        return instance.__dict__[storage_name]

    def qty_setter(instance, value):
        """写值函数，包含验证逻辑。"""
        if value > 0:
            # 直接写实例 __dict__，绕过特性，避免递归
            instance.__dict__[storage_name] = value
        else:
            raise ValueError(f'{storage_name} must be > 0')

    # 构建并返回特性对象
    return property(qty_getter, qty_setter)


class LineItem:
    # 每次调用 quantity() 返回一个独立的特性对象
    weight = quantity('weight')  # 特性对象作为类属性
    price = quantity('price')    # 另一个特性对象

    def __init__(self, description, weight, price):
        self.description = description
        self.weight = weight    # 触发 qty_setter，验证 > 0
        self.price = price      # 触发 qty_setter，验证 > 0

    def subtotal(self):
        return self.weight * self.price


# ---- 测试 ----
nutmeg = LineItem('Moluccan nutmeg', 8, 13.95)
print(nutmeg.weight, nutmeg.price)  # 8  13.95（通过 qty_getter 读取）
print(nutmeg.__dict__)              # {'description': 'Moluccan nutmeg', 'weight': 8, 'price': 13.95}
# 注意：值存在实例 __dict__ 中，但读取时被特性拦截

try:
    LineItem('bad item', -1, 10)
except ValueError as e:
    print(e)  # weight must be > 0
```

**工厂函数 vs 描述符类（第23章）的对比**：

| | 特性工厂函数 | 描述符类 |
|--|---|---|
| 代码复用 | 通过函数参数 | 通过继承 |
| 可扩展性 | 弱（难以子类化） | 强（可定义子类） |
| 属性名处理 | 需手动传入（重复写两次）| Python 3.6+ `__set_name__` 自动获取 |
| 适用场景 | 少量验证逻辑 | 多属性复用 / 框架开发 |

## 10. 属性删除：`@property.deleter`

```python
class BlackKnight:
    """演示 @property.deleter 的用法。"""

    def __init__(self):
        self.phrases = [
            ('an arm', "'Tis but a scratch."),
            ('another arm', "It's just a flesh wound."),
            ('a leg', "I'm invincible!"),
            ('another leg', "All right, we'll call it a draw.")
        ]

    @property
    def member(self):
        """读取下一个将失去的肢体名称。"""
        print('next member is:')
        return self.phrases[0][0]

    @member.deleter
    def member(self):
        """删除操作：失去一个肢体（弹出第一个元素）。"""
        member, text = self.phrases.pop(0)
        print(f'BLACK KNIGHT (loses {member}) -- {text}')

    # ---- 经典调用等价写法 ----
    # member = property(member_getter, fdel=member_deleter)


# ---- 测试 ----
knight = BlackKnight()
print(knight.member)    # 读取：'an arm'
del knight.member       # 删除：'BLACK KNIGHT (loses an arm) -- ...'
del knight.member       # 删除：'BLACK KNIGHT (loses another arm) -- ...'
```

## 11. 处理属性的完整 API 速查

### 11.1 影响属性处理的特殊属性

| 属性 | 说明 |
|------|------|
| `__dict__` | 存储对象可写属性的字典；直接操作可绕过所有特殊方法和描述符 |
| `__class__` | 对象所属类的引用；特殊方法从类中查找，而非实例 |
| `__slots__` | 限定允许的属性名，节省内存；定义后实例通常没有 `__dict__` |

### 11.2 内置函数

```python
class MyClass:
    @property
    def name(self):
        return self._name

    @name.setter
    def name(self, value):
        self._name = value


obj = MyClass()
obj._name = 'Alice'

# getattr(object, name[, default])：获取属性，可指定默认值
print(getattr(obj, 'name'))           # 'Alice'
print(getattr(obj, 'missing', None))  # None（不抛异常）

# setattr(object, name, value)：设置属性
setattr(obj, '_name', 'Bob')
print(obj._name)  # 'Bob'

# hasattr(object, name)：检查属性是否存在（内部调用 getattr，捕获 AttributeError）
print(hasattr(obj, 'name'))     # True
print(hasattr(obj, 'missing'))  # False

# vars([object])：返回对象的 __dict__
print(vars(obj))  # {'_name': 'Bob'}

# dir([object])：列出对象的"重要"属性（非完整列表）
print(dir(obj))   # 包括继承的属性和方法
```

### 11.3 处理属性的特殊方法

```python
class Traced:
    """演示各种属性处理特殊方法的触发时机。"""

    def __getattribute__(self, name):
        """每次属性访问都触发（包括读取 self.xxx）。

        ⚠️ 极难正确实现！内部访问任何属性都会再次触发，造成递归。
        必须用 super().__getattribute__(name) 委托给父类。
        """
        print(f'__getattribute__: {name}')
        return super().__getattribute__(name)  # 必须用 super() 防止无限递归

    def __getattr__(self, name):
        """仅当常规查找（包括 __getattribute__）失败后才调用。

        相对安全，适合实现虚拟属性。
        """
        print(f'__getattr__: {name}')
        raise AttributeError(f'{name} not found')  # 应该抛 AttributeError

    def __setattr__(self, name, value):
        """每次属性赋值都触发（包括 self.xxx = yyy）。

        ⚠️ 内部必须用 super().__setattr__() 或写 __dict__，否则无限递归。
        """
        print(f'__setattr__: {name} = {value}')
        super().__setattr__(name, value)  # 委托给父类实际执行赋值

    def __delattr__(self, name):
        """del obj.attr 时触发。"""
        print(f'__delattr__: {name}')
        super().__delattr__(name)

    def __dir__(self):
        """dir(obj) 调用时触发，可自定义显示的属性列表。"""
        return ['custom_attr_1', 'custom_attr_2']
```

> **建议**：`__getattribute__` 和 `__setattr__` 影响所有属性操作，极易出错。
> **优先选择 `@property` 或描述符，而非直接覆盖这两个方法。**

## 12. 常见误区与陷阱

### ❌ 误区一：认为 `__getattr__` 会在每次属性访问时触发

```python
class Bad:
    data = 'class data'

    def __getattr__(self, name):
        # ❌ 错误认知：这会在每次访问时都打印
        print(f'__getattr__ called for: {name}')
        return None


obj = Bad()
obj.data    # ✅ 不触发 __getattr__（data 在类属性中找到了）
obj.missing # ✅ 触发 __getattr__（missing 找不到，才会调用）
```

### ❌ 误区二：认为实例属性总是覆盖类属性

```python
class WithProperty:
    @property
    def value(self):
        return 'from property'


obj = WithProperty()
obj.__dict__['value'] = 'from instance'  # 直接写入 __dict__
print(obj.value)  # 'from property'（特性仍然胜出！）
# ⚠️ @property（覆盖型描述符）优先级高于实例 __dict__
```

### ❌ 误区三：在 `@property.deleter` 类中混用 `__delattr__`

```python
class Problematic:
    @property
    def x(self):
        return self.__x

    @x.setter
    def x(self, value):
        self.__x = value

    def __delattr__(self, name):
        # ⚠️ 如果 name == 'x'，__delattr__ 会被调用，而不是特性的 fdel
        # 如果同时定义了 __delattr__ 和 @x.deleter，__delattr__ 优先
        print(f'deleting {name}')
        super().__delattr__(name)
```

### ❌ 误区四：在特性工厂中把值存到错误的地方

```python
# ❌ 错误：存到描述符实例（self.__dict__），所有托管实例共享同一值
def bad_quantity(storage_name):
    def getter(instance):
        return getter.__dict__[storage_name]  # 错！存在函数对象里

    def setter(instance, value):
        if value > 0:
            setter.__dict__[storage_name] = value  # 错！

    return property(getter, setter)


# ✅ 正确：存到托管实例（instance.__dict__）
def quantity(storage_name):
    def getter(instance):
        return instance.__dict__[storage_name]  # 正确！每个实例独立

    def setter(instance, value):
        if value > 0:
            instance.__dict__[storage_name] = value  # 正确！

    return property(getter, setter)
```

## 13. 本章知识对比速查表

### 动态属性方法对比

| 方法 | 触发时机 | 适用场景 | 风险等级 |
|------|----------|----------|----------|
| `__getattr__` | 常规查找失败后（兜底） | 虚拟属性、代理对象 | 低 |
| `__getattribute__` | 每次属性访问（全量拦截）| 需要拦截所有读取 | 高（易递归）|
| `__setattr__` | 每次属性赋值 | 需要拦截所有写入 | 高（易递归）|
| `__delattr__` | `del obj.attr` | 拦截属性删除 | 中 |

### 属性优先级排序（高→低）

| 优先级 | 来源 | 说明 |
|--------|------|------|
| ① 最高 | 覆盖型描述符（`@property`） | 有 `__get__` + `__set__`，无论如何都会被调用 |
| ② | 实例 `__dict__` | `obj.__dict__` 中的普通属性 |
| ③ | 非覆盖型描述符 / 类属性 | 只有 `__get__`（如函数）或普通类变量 |
| ④ 最低 | `__getattr__` | 以上全部失败后才调用 |

### `__new__` vs `__init__` 对比

| | `__new__` | `__init__` |
|--|-----------|------------|
| 本质 | 构造方法（创建对象）| 初始化方法（配置对象）|
| 调用时机 | 实例创建之前 | 实例创建之后 |
| 返回值 | 新创建的对象（可以是任意类型）| 必须返回 `None` |
| `__init__` 是否调用 | 取决于 `__new__` 返回值是否为本类实例 | 总是被调用（若 `__new__` 返回本类实例）|
| 常用场景 | 单例模式、工厂方法、元类 | 通常这里做属性初始化 |
