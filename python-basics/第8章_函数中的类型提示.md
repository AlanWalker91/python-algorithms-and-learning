# 第8章：函数中的类型提示

> **核心思想**：Python 的类型提示系统是*渐进式*的——完全可选、不在运行时捕获类型错误、不改变性能。它为 IDE 和 CI 工具提供静态分析能力，协助开发者在代码运行前发现 bug。

## 2. 关于渐进式类型系统

```python
# =============================================
# 渐进式类型系统（Gradual Type System）：
# 1. 可选：没有类型提示的代码默认视为 Any 类型
# 2. 不在运行时捕获：类型错误只能由静态工具发现
# 3. 不能改善性能：仅用于静态分析
# =============================================

# 无类型提示的函数——Mypy 默认忽略（除非严格模式）
def show_count(count, word):
    if count == 1:
        return f'1 {word}'
    count_str = str(count) if count else 'no'
    return f'{count_str} {word}s'

# 运行时调用（不检查类型）：
print(show_count(99, 'bird'))   # '99 birds'
print(show_count(1, 'bird'))    # '1 bird'
print(show_count(0, 'bird'))    # 'no birds'

# 运行 Mypy 检查：
# $ mypy messages.py
# Success: no issues found（默认模式忽略无注解函数）

# 严格模式：
# $ mypy --disallow-incomplete-defs messages.py
# error: Function is missing a type annotation for one or more arguments

# Mypy 配置文件（mypy.ini）：
# [mypy]
# python_version = 3.10
# warn_unused_configs = True
# disallow_incomplete_defs = True
```

## 3. 渐进式类型实践（Mypy 初体验）

```python
# =============================================
# 逐步为函数添加类型提示
# =============================================

# ---- 第 1 步：只添加返回类型 ----
def show_count(count, word) -> str:
    if count == 1:
        return f'1 {word}'
    count_str = str(count) if count else 'no'
    return f'{count_str} {word}s'
# Mypy 严格模式报告：缺少参数类型注解

# ---- 第 2 步：完整的类型提示 ----
def show_count(count: int, word: str) -> str:
    if count == 1:
        return f'1 {word}'
    count_str = str(count) if count else 'no'
    return f'{count_str} {word}s'
# Mypy 严格模式：Success

# ---- 第 3 步：添加可选参数 ----
from typing import Optional

def show_count(count: int, singular: str, plural: str = '') -> str:
    """plural 是可选参数，默认为空字符串（自动加 's'）。"""
    if count == 1:
        return f'1 {singular}'
    count_str = str(count) if count else 'no'
    if not plural:
        plural = singular + 's'
    return f'{count_str} {plural}'

# 使用 None 作为默认值（更安全的可选参数）
def show_count_v2(count: int, singular: str,
                  plural: Optional[str] = None) -> str:
    if count == 1:
        return f'1 {singular}'
    count_str = str(count) if count else 'no'
    if not plural:
        plural = singular + 's'
    return f'{count_str} {plural}'

# ⚠️ 常见写法错误：
# def f(x=str):     ← 这是把 str 类作为默认值，不是类型注解！
# def f(x: str):    ← 这才是类型注解（冒号分隔）
# def f(x: str = ''):  ← 注解 + 默认值（等号前后有空格）

# PEP 8 代码风格：
# x: int    → 注解（: 后有空格）
# x = 0     → 默认值（= 两侧有空格，无注解时）
# x: int = 0 → 注解 + 默认值（: 后有空格，= 两侧有空格）
```

## 4. 类型由受支持的操作定义

```python
# =============================================
# 关键洞察：类型 = 受支持的操作集合
# Mypy 检查的是操作是否被注解的类型支持
# =============================================

# 无类型提示：x 可以是任何支持 * 操作的类型
def double(x):
    return x * 2

# 使用 abc.Sequence 注解：
from collections import abc

def double_seq(x: abc.Sequence):
    return x * 2     # Mypy 报错！Sequence 没有 __mul__ 方法

# Mypy 只看类型注解，不运行代码：
# - abc.Sequence 的接口里没有 __mul__，所以 Mypy 报错
# - 运行时，str/list/tuple 都有 __mul__，所以实际能运行

# ---- 正确做法：使用支持该操作的类型 ----
from typing import TypeVar

# 方法1：使用 Any（跳过类型检查）
def double_any(x) -> None:  # 等价于 x: Any
    return x * 2

# 方法2：使用 TypeVar + Protocol（见后文）
```

## 5. 鸭子类型 vs 名义类型

```python
# =============================================
# 鸭子类型：运行时检查，看对象"能做什么"
# 名义类型：静态检查，看变量的声明类型
# =============================================

class Bird:
    pass

class Duck(Bird):
    def quack(self):
        print('Quack!')

# 无类型提示（鸭子类型）：运行时灵活
def alert(birdie):
    birdie.quack()    # Mypy 忽略（无提示）

# 名义类型（Duck）：只接受 Duck 类型
def alert_duck(birdie: Duck) -> None:
    birdie.quack()    # ✅ Duck 有 quack 方法

# 名义类型（Bird）：Mypy 报错！
def alert_bird(birdie: Bird) -> None:
    birdie.quack()    # ❌ Bird 没有 quack 方法！
    # Mypy 报告：error: "Bird" has no attribute "quack"

# ---- 运行时 vs 静态检查的差异 ----
daffy = Duck()
alert(daffy)        # ✅ 运行时：鸭子类型，正常调用 quack
alert_duck(daffy)   # ✅ 运行时 + 静态：Duck 有 quack
alert_bird(daffy)   # ✅ 运行时（Duck 有 quack），但 Mypy 警告！

woody = Bird()
alert(woody)        # ❌ 运行时 AttributeError（Bird 没有 quack）
alert_duck(woody)   # ❌ 运行时错误 + Mypy 静态警告
alert_bird(woody)   # ❌ 运行时错误 + Mypy 警告

# 结论：名义类型可以在运行前捕获 alert_bird 的设计问题
```

## 6. 注解中可用的类型（概览）

| 类型 | 示例 | 说明 |
|------|------|------|
| `Any` | `x: Any` | 动态类型，关闭检查 |
| 简单类型 | `x: int`, `x: str` | 内置类型和类 |
| `Optional[X]` | `x: Optional[str]` | X 或 None |
| `Union[X, Y]` | `x: Union[int, str]` | X 或 Y |
| 泛化容器 | `list[int]`, `dict[str, int]` | Python 3.9+；或 `List[int]` |
| 元组 | `tuple[int, str]`, `tuple[int, ...]` | 固定长度或变长 |
| `Callable` | `Callable[[int], str]` | 可调用对象 |
| 抽象基类 | `Iterable[int]`, `Sequence[str]` | collections.abc 中的类 |
| `TypeVar` | `T = TypeVar('T')` | 参数化泛型 |
| `Protocol` | `class P(Protocol): ...` | 静态鸭子类型 |
| `NoReturn` | `-> NoReturn` | 永不正常返回 |

## 7. Any 类型

```python
# =============================================
# Any：渐进式类型的基础
# - 是最一般的类型（接受任何参数）
# - 也是最特定的类型（支持所有操作）
# - 对类型检查工具"关闭检查"
# =============================================

# 没有注解的函数等同于所有参数和返回值都是 Any：
def double(x):
    return x * 2
# 等价于：def double(x: Any) -> Any: ...

# Any 支持所有操作（不报错）：
from typing import Any
def process(x: Any) -> Any:
    return x * 2 + x[0] - len(x)  # 任意操作，Mypy 不检查

# ---- Any vs object 的区别 ----
# Any：支持所有操作（最宽松）
# object：不支持 __mul__ 等大多数操作（最严格的基类）

def double_object(x: object) -> object:
    return x * 2    # ❌ Mypy 报错：object 不支持 *

# 类型层次结构中的特殊地位：
# Any 既是最一般的类型（位于顶部，接受任何类型）
# 又是最特定的类型（位于底部，支持所有操作）
# 这是为了实现渐进式类型而引入的"魔法类型"
```

## 8. Optional 和 Union 类型

```python
from typing import Optional, Union

# =============================================
# Optional[X]：X 或 None（不是"可选参数"！）
# Union[X, Y]：X 或 Y
# =============================================

# Optional[str] 表示：值可以是 str 或 None
def show_count(count: int, singular: str,
               plural: Optional[str] = None) -> str:
    # ⚠️ Optional 不能让参数"可选"（那是 = 默认值的作用）
    # Optional[str] 等价于 Union[str, None]
    pass

# 使用 Union：接受多种类型
def parse_token(token: Union[str, float]) -> str:
    return str(token)

# Python 3.10+ 新语法（| 替代 Union）：
# def parse_token(token: str | float) -> str: ...

# Union 嵌套（扁平化）
from typing import Union
T = Union[int, Union[str, float]]   # 等价于 Union[int, str, float]

# ⚠️ Optional 名称的误导性：
# "Optional" 让人以为参数是可选的
# 实际上应该叫 "Maybe"（Haskell/Elm 的叫法更准确）
# Optional[str] 的真正含义：值类型是 str 或 NoneType
```

## 9. 泛化容器类型

```python
from typing import List, Tuple, Dict, Set  # Python 3.8 及以下
# Python 3.9+ 可以直接用 list[int], dict[str, int] 等

# =============================================
# 泛化容器：指定容器内元素的类型
# =============================================

# ---- list ----
def tokenize(text: str) -> list[str]:   # Python 3.9+
    return text.upper().split()

# ---- tuple ----
# 固定长度元组：每个位置指定类型
def get_coord() -> tuple[float, float]:
    return (1.0, 2.0)

# 可变长度元组（同类型元素）：用 ...
def get_coords() -> tuple[float, ...]:
    return (1.0, 2.0, 3.0)

# ---- dict ----
def get_mapping() -> dict[str, int]:
    return {'a': 1, 'b': 2}

# ---- 嵌套泛型 ----
def group_by_type(items: list[str | int]) -> dict[type, list]:
    result: dict[type, list] = {}
    for item in items:
        t = type(item)
        result.setdefault(t, []).append(item)
    return result

# ---- 早期写法（Python 3.8 及以下）----
from typing import List, Tuple, Dict

def old_style(x: List[int], y: Tuple[str, float]) -> Dict[str, int]:
    pass
# Python 3.9+ 推荐直接用 list[int], tuple[str, float], dict[str, int]
```

## 10. 抽象基类（ABC）作为类型

```python
from collections.abc import (
    Sequence, MutableSequence, Mapping, MutableMapping,
    Iterable, Iterator, Callable
)

# =============================================
# 使用 ABC 注解：比具体类型更灵活
# 接受 ABC 的任何具体实现
# =============================================

# ✅ 推荐：接受任何序列（list, tuple, str...）
def first(seq: Sequence) -> object:
    return seq[0]

# ❌ 过窄：只接受 list
def first_list(seq: list) -> object:
    return seq[0]

# 接受任何可映射类型（dict, OrderedDict...）
def get_value(mapping: Mapping[str, int], key: str) -> int:
    return mapping.get(key, 0)

# 接受任何可迭代对象
def process(items: Iterable[str]) -> list[str]:
    return [item.upper() for item in items]

# ---- ABC 层次结构 ----
# Iterable
# └── Collection
#     ├── Sequence（__getitem__, __len__）
#     │   ├── MutableSequence（+ insert, append, remove...）
#     ├── Mapping（__getitem__, __len__, __iter__）
#     │   └── MutableMapping（+ __setitem__, __delitem__...）
#     └── Set
#         └── MutableSet
```

## 11. 参数化泛型与 `TypeVar`

```python
from typing import TypeVar

# =============================================
# TypeVar：定义参数化泛型
# 让函数参数类型和返回类型之间建立关联
# =============================================

T = TypeVar('T')           # 无界 TypeVar：任意类型

# ---- 基本用法 ----
def first(seq: list[T]) -> T:   # 返回类型与输入列表元素类型相同
    return seq[0]

# Mypy 能推断返回类型：
result = first([1, 2, 3])       # 推断为 int
name = first(['Alice', 'Bob'])  # 推断为 str

# ---- 有界 TypeVar（bound）----
from typing import SupportsLessThan   # 示例，实际要自己定义

class SupportsLessThan(Protocol):
    def __lt__(self, other: Any) -> bool: ...

LT = TypeVar('LT', bound=SupportsLessThan)  # 只接受支持 < 的类型

def max_value(seq: list[LT]) -> LT:
    result = seq[0]
    for item in seq[1:]:
        if result < item:   # 安全！LT 保证支持 <
            result = item
    return result

print(max_value([3, 1, 4, 1, 5, 9]))   # 9
print(max_value(['cherry', 'apple', 'banana']))  # 'cherry'
# max_value([[1], [2]])   ← Mypy 警告：list 不支持 <

# ---- 受限 TypeVar（constraints）----
# AnyStr 是内置的受限 TypeVar：只接受 str 或 bytes
from typing import AnyStr

def concat(a: AnyStr, b: AnyStr) -> AnyStr:
    return a + b

concat('hello', ' world')  # str ✅
concat(b'hello', b' world') # bytes ✅
# concat('hello', b' world') # ❌ Mypy 报错：类型不一致
```

## 12. 静态鸭子类型（Protocol）

```python
from typing import Protocol, runtime_checkable

# =============================================
# Protocol：定义"结构子类型"（静态鸭子类型）
# 不需要继承，只要实现了指定的方法就满足
# =============================================

class Drawable(Protocol):
    def draw(self) -> None:
        ...

# 任何实现了 draw 方法的类都满足 Drawable 协议
class Circle:
    def draw(self) -> None:
        print('画圆')

class Square:
    def draw(self) -> None:
        print('画方形')

def render(shape: Drawable) -> None:
    shape.draw()

render(Circle())   # ✅（不需要继承 Drawable）
render(Square())   # ✅
render(42)         # ❌ Mypy 警告：int 没有 draw 方法

# ---- SupportsLessThan：常用于排序的协议 ----
from typing import Any

class SupportsLessThan(Protocol):
    def __lt__(self, other: Any) -> bool: ...

# ---- @runtime_checkable：允许运行时 isinstance 检查 ----
@runtime_checkable
class Sized(Protocol):
    def __len__(self) -> int: ...

print(isinstance([], Sized))      # True（list 有 __len__）
print(isinstance(42, Sized))      # False
```

## 13. Callable 类型

```python
from typing import Callable
from collections.abc import Callable as AbcCallable  # 推荐

# =============================================
# Callable[[参数类型, ...], 返回类型]
# 用于标注可调用对象（函数、方法、类）
# =============================================

# 注解接受一个 int 参数、返回 str 的函数
def apply(func: Callable[[int], str], value: int) -> str:
    return func(value)

apply(str, 42)             # ✅（str 接受 int，返回 str）
apply(lambda x: x * 2, 5)  # ❌ Mypy 警告：返回 int 不是 str

# 多个参数
def binary_op(func: Callable[[int, int], int], a: int, b: int) -> int:
    return func(a, b)

from operator import add
binary_op(add, 3, 4)       # ✅ 7

# 无参数 Callable
def call_later(callback: Callable[[], None]) -> None:
    callback()

# 可变参数的 Callable（用 ...）
def call_any(func: Callable[..., int]) -> int:
    return func()

# 在数据类中使用 Callable 类型
from dataclasses import dataclass, field
from typing import Optional, Callable

@dataclass
class Order:
    customer: str
    items: list[str]
    # promotion 可以是 None 或接受 Order、返回 float 的函数
    promotion: Optional[Callable[['Order'], float]] = None

    def due(self) -> float:
        if self.promotion is None:
            return 100.0
        return 100.0 - self.promotion(self)
```

## 14. 常见误区与陷阱

### ❌ 误区一：以为类型提示在运行时有效

```python
from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float

# ⚠️ 类型提示在运行时完全被忽略！
p = Point('hello', None)   # 不报错！
print(p)   # Point(x='hello', y=None)

# ✅ 需要运行时验证时，使用 __post_init__：
@dataclass
class SafePoint:
    x: float
    y: float

    def __post_init__(self):
        if not isinstance(self.x, (int, float)):
            raise TypeError(f'x 必须是数字，得到 {type(self.x)}')
```

### ❌ 误区二：混淆 Optional 的含义

```python
# ❌ Optional 不是"可选参数"
def f(x: Optional[int]) -> None:  # x 是 int 或 None，但不能省略！
    pass

# f()   → TypeError（没有给 x 传值）

# ✅ 可选参数要用默认值
def g(x: Optional[int] = None) -> None:  # Optional + 默认值
    pass

g()      # ✅ 可以省略 x
g(42)    # ✅ 传入 int
g(None)  # ✅ 传入 None
```

### ❌ 误区三：把函数参数默认值写成类型

```python
# ❌ 错误：把 str 类作为默认值
def f(x=str):    # 这是把 str 类（不是空字符串）作为 x 的默认值
    pass

# ✅ 正确写法：
def f(x: str = ''):    # 注解 str，默认值是空字符串
    pass
```

### ❌ 误区四：使用过时的泛型写法

```python
# ❌ Python 3.9+ 不推荐（但仍支持）
from typing import List, Dict, Tuple

def f(x: List[int]) -> Dict[str, Tuple[int, ...]]:
    pass

# ✅ Python 3.9+ 推荐：直接使用内置类型
def f(x: list[int]) -> dict[str, tuple[int, ...]]:
    pass

# 两种写法均有效，但新代码应使用后者
```

### ❌ 误区五：以为 100% 注解覆盖率是目标

```python
# ⚠️ 过度追求注解覆盖率会引入噪声
# 有些灵活的 Python API 很难完整注解

# 例如：接受任意类型的函数
def process(*args, **kwargs):   # Any 类型，不注解也可以
    pass

# 渐进式类型的核心：可以只为最关键、最容易出错的部分添加类型提示
# 不必追求 100% 覆盖
```
