## 1. 知识框架总览

```
第23章 属性描述符
│
├── 描述符协议
│   ├── __get__(self, instance, owner)   # 读属性
│   ├── __set__(self, instance, value)   # 写属性
│   ├── __delete__(self, instance)       # 删属性
│   └── __set_name__(self, owner, name) # Python 3.6+ 自动注入属性名
│
├── 六大术语
│   ├── 描述符类 (Descriptor Class)     # 实现了描述符协议的类
│   ├── 托管类 (Managed Class)          # 把描述符实例作为类属性的类
│   ├── 描述符实例 (Descriptor Instance) # 描述符类的实例，作为类属性
│   ├── 托管实例 (Managed Instance)     # 托管类创建的实例
│   ├── 托管属性 (Managed Attribute)    # 用户看到的属性名
│   └── 储存属性 (Storage Attribute)    # 实际存储值的地方（实例 __dict__ 中）
│
├── 三个演化版本
│   ├── v3：手动传入 storage_name（有重复问题）
│   ├── v4：__set_name__ 自动获取属性名（推荐）
│   └── v5：抽象基类 + 模板方法（生产级，Django ORM 同款结构）
│
├── 两大类型
│   ├── 覆盖型（有 __set__）
│   │   ├── 完整型（有 __get__ + __set__）→ 最高优先级
│   │   └── 无 __get__ 型（只有 __set__）→ 读时返回描述符对象
│   └── 非覆盖型（无 __set__）→ 可被实例属性遮盖
│
└── 重要推论
    └── 方法 = 非覆盖型描述符（函数实现了 __get__）
```

## 2. 描述符协议：四个核心方法

### 2.1 `__get__`：读属性时触发

```python
def __get__(self, instance, owner):
    """读取托管属性时被调用。

    Args:
        self:     描述符实例（如 LineItem 的类属性 weight，是 Quantity 的实例）
        instance: 托管实例（如某个 LineItem 对象）；通过类访问时为 None
        owner:    托管类（如 LineItem 类本身）

    Returns:
        通过实例访问时，返回储存的属性值。
        通过类访问时，返回描述符实例本身（约定俗成）。
    """
    if instance is None:
        # 通过类访问：如 LineItem.weight
        # 返回描述符本身，支持内省和元编程
        return self
    # 通过实例访问：如 raisins.weight
    # 从托管实例的 __dict__ 中读取（绕过描述符，避免递归）
    return instance.__dict__[self.storage_name]
```

### 2.2 `__set__`：写属性时触发

```python
def __set__(self, instance, value):
    """为托管属性赋值时被调用。

    Args:
        self:     描述符实例
        instance: 托管实例（要写入的目标对象）
        value:    用户赋的值

    ⚠️ 关键：必须把值存到 instance.__dict__，而非 self.__dict__！
       - 描述符实例是类属性，全局只有一份
       - 若存到 self.__dict__，所有托管实例共享同一个值
    """
    if value > 0:
        # ✅ 正确：存到托管实例的 __dict__（每个实例独立）
        instance.__dict__[self.storage_name] = value
    else:
        raise ValueError(f'{self.storage_name} must be > 0')

    # ❌ 错误写法：self.__dict__[self.storage_name] = value
    # 这会把值存到描述符实例里，所有 LineItem 实例共享同一个重量！
```

### 2.3 `__set_name__`：class 主体读取后自动调用（Python 3.6+）

```python
def __set_name__(self, owner, name):
    """class 主体执行完毕后，由 type.__new__ 自动调用。

    Args:
        self:  描述符实例
        owner: 托管类（如 LineItem）
        name:  描述符被赋给的类属性名（如 'weight' 或 'price'）

    效果：
        自动把属性名注入描述符，无需手动传入。
        解决了 weight = Quantity('weight') 重复写属性名的问题。
    """
    self.storage_name = name  # 保存，供 __get__ / __set__ 使用

# 调用时序说明：
# 1. Python 读取 class LineItem 主体
# 2. 执行 weight = Quantity()，创建描述符实例（此时 storage_name 未设置）
# 3. 整个 class 主体读取完毕后，type.__new__ 检查每个类属性
# 4. 发现 Quantity 有 __set_name__，自动调用：
#    LineItem.weight.__set_name__(LineItem, 'weight')
#    LineItem.price.__set_name__(LineItem, 'price')
# 5. 之后用户调用 LineItem(...) 时，__init__ 才运行
```

### 2.4 `__delete__`：`del obj.attr` 时触发（较少使用）

```python
def __delete__(self, instance):
    """del 操作时被调用。

    Args:
        self:     描述符实例
        instance: 托管实例

    实践中很少需要实现此方法。
    """
    del instance.__dict__[self.storage_name]
```

## 3. 六大术语精解

以 `LineItem` + `Quantity` 为例：

```
┌─────────────────────────────────────────────────────┐
│            Quantity（描述符类）                      │
│   - 实现了 __get__、__set__、__set_name__           │
│   - 生产了两个描述符实例：                           │
│     • LineItem.weight（管理 weight 属性）            │
│     • LineItem.price（管理 price 属性）              │
└─────────────────────────────────────────────────────┘
                        │ 描述符实例作为类属性
                        ▼
┌─────────────────────────────────────────────────────┐
│                LineItem（托管类）                    │
│   weight = Quantity()  ← 描述符实例（类属性）        │
│   price  = Quantity()  ← 描述符实例（类属性）        │
└─────────────────────────────────────────────────────┘
                        │ 创建
                        ▼
┌─────────────────────────────────────────────────────┐
│         raisins = LineItem(...)（托管实例）           │
│   __dict__: {                                        │
│     'description': 'Golden raisins',                 │
│     'weight': 10,    ← 储存属性（实际存值的地方）    │
│     'price': 6.95,   ← 储存属性                     │
│   }                                                  │
│   raisins.weight  → 托管属性（公开接口，被描述符管理）│
└─────────────────────────────────────────────────────┘
```

| 术语 | 对应示例 | 一句话说清楚 |
|------|----------|------------|
| **描述符类** | `Quantity` | 实现了描述符协议方法的类 |
| **托管类** | `LineItem` | 把描述符实例声明为类属性的类 |
| **描述符实例** | `LineItem.weight`（Quantity 对象）| 描述符类创建的实例，作为类属性存在 |
| **托管实例** | `raisins`（某个 LineItem 对象）| 托管类创建的实例 |
| **托管属性** | `raisins.weight`（公开接口）| 用户通过点号访问的属性名，由描述符拦截 |
| **储存属性** | `raisins.__dict__['weight']`（实际存值）| 真正存储数据的地方，在托管实例的 `__dict__` 中 |

> **⚠️ 最重要的规则**：描述符实例是**类属性**，全局只有一份。
> 值必须存到**托管实例**的 `__dict__` 里，不能存到描述符实例里。

## 4. LineItem v3：最简描述符（手动命名）

```python
# ============================================================
# 描述符类：Quantity（验证值必须 > 0）
# ============================================================

class Quantity:
    """验证数值必须大于零的描述符。

    v3 版本：storage_name 由外部手动传入，存在重复书写问题。
    """

    def __init__(self, storage_name):
        """初始化描述符，指定储存属性的名称。

        Args:
            storage_name: 在托管实例 __dict__ 中存储值的键名
                         （如 'weight' 或 'price'）
        """
        self.storage_name = storage_name

    def __set__(self, instance, value):
        """拦截属性赋值，进行验证后写入。

        Args:
            instance: 托管实例（如 raisins 这个 LineItem 对象）
            value:    用户赋的值（如 10 或 -1）
        """
        if value > 0:
            # ✅ 写到托管实例的 __dict__（每个实例独立存储）
            instance.__dict__[self.storage_name] = value
        else:
            raise ValueError(f'{self.storage_name} must be > 0')

    def __get__(self, instance, owner):
        """拦截属性读取。

        Args:
            instance: 托管实例；通过类访问时为 None
            owner:    托管类（LineItem）
        """
        if instance is None:
            # 通过类访问（如 LineItem.weight）→ 返回描述符本身
            # 支持内省：LineItem.weight 能返回 Quantity 对象
            return self
        # 通过实例访问 → 从托管实例 __dict__ 中读取
        return instance.__dict__[self.storage_name]
        # ⚠️ 为什么需要 __get__？
        # 当 storage_name != 托管属性名时（如 House.rooms 存为 number_of_rooms）
        # 必须用 __get__ 才能正确返回值


# ============================================================
# 托管类：LineItem（使用两个 Quantity 描述符）
# ============================================================

class LineItem:
    # ⚠️ 缺点：storage_name 写了两遍（'weight' 重复出现）
    # 若复制粘贴时忘记修改，会产生静默 bug
    weight = Quantity('weight')   # 第一个描述符实例
    price  = Quantity('price')    # 第二个描述符实例

    def __init__(self, description, weight, price):
        self.description = description
        self.weight = weight   # 触发 Quantity.__set__，验证 > 0
        self.price  = price    # 触发 Quantity.__set__，验证 > 0

    def subtotal(self):
        return self.weight * self.price   # 触发 Quantity.__get__


# ============================================================
# 测试
# ============================================================

# 正常创建
raisins = LineItem('Golden raisins', 10, 6.95)
print(raisins.weight)          # 10  → 触发 __get__
print(raisins.price)           # 6.95

# 通过类访问，返回描述符对象本身
print(LineItem.weight)         # <Quantity object>（instance=None，__get__ 返回 self）
print(type(LineItem.weight))   # <class 'Quantity'>

# 查看实际存储位置
print(vars(raisins))
# {'description': 'Golden raisins', 'weight': 10, 'price': 6.95}
# 值存在托管实例的 __dict__ 里

# 验证失败
try:
    LineItem('Truffle', 100, 0)    # price = 0，不合法
except ValueError as e:
    print(e)  # price must be > 0

try:
    raisins.weight = -5
except ValueError as e:
    print(e)  # weight must be > 0
```

## 5. LineItem v4：`__set_name__` 自动命名

> **改进**：Python 3.6 引入 `__set_name__`，class 主体读取后自动把属性名注入描述符。
> 再也不需要重复写属性名。

```python
# ============================================================
# 描述符类：Quantity v4（使用 __set_name__）
# ============================================================

class Quantity:
    """验证数值必须大于零的描述符（v4）。

    改进：不再需要 __init__ 手动传入 storage_name。
    Python 在解析 class 主体后自动调用 __set_name__。
    """

    def __set_name__(self, owner, name):
        """class 主体读取完毕后，由 type.__new__ 自动调用。

        Args:
            owner: 托管类（如 LineItem）
            name:  描述符被赋给的属性名（如 'weight' 或 'price'）
        """
        # 无需用户传入，自动记录属性名
        self.storage_name = name
        # 调用时序：
        # class LineItem: weight = Quantity()  ← 创建描述符实例
        #   执行完 class 主体后，Python 自动调用：
        #   LineItem.weight.__set_name__(LineItem, 'weight')
        #   LineItem.price.__set_name__(LineItem, 'price')

    def __set__(self, instance, value):
        """拦截赋值并验证。"""
        if value > 0:
            instance.__dict__[self.storage_name] = value
        else:
            raise ValueError(f'{self.storage_name} must be > 0')

    # ✅ v4 可以省略 __get__！
    # 原因：storage_name 与托管属性名完全相同（'weight' == 'weight'）
    # 读取 raisins.weight 时：
    #   1. 先找数据描述符（Quantity 有 __set__，是覆盖型）
    #   2. Quantity 没有 __get__，跳过
    #   3. 找实例 __dict__['weight'] → 直接返回 10
    # 速度更快，不经过额外的方法调用


# ============================================================
# 托管类：LineItem v4（干净清爽）
# ============================================================

class LineItem:
    # ✅ 不再重复写属性名！
    weight = Quantity()   # __set_name__ 自动注入 'weight'
    price  = Quantity()   # __set_name__ 自动注入 'price'

    def __init__(self, description, weight, price):
        self.description = description
        self.weight = weight
        self.price  = price

    def subtotal(self):
        return self.weight * self.price


# ============================================================
# 实际项目中的典型用法：描述符放在独立模块中
# ============================================================

# model_v4.py（描述符模块）
# class Quantity: ...

# bulkfood.py（业务模块）
# import model_v4 as model
# class LineItem:
#     weight = model.Quantity()  # 像 Django ORM 一样清晰
#     price  = model.Quantity()
#     ...

# ============================================================
# 测试
# ============================================================

item = LineItem('Organic nuts', 5, 15.0)
print(item.weight)  # 5
print(item.price)   # 15.0

# v4 中 __set_name__ 的效果验证
# 访问 LineItem.weight 时，没有 __get__，直接返回描述符对象
print(LineItem.weight)          # <Quantity object>
print(LineItem.weight.storage_name)  # 'weight'（自动注入的）
print(LineItem.price.storage_name)   # 'price'（自动注入的）
```

## 6. LineItem v5：抽象基类 + 模板方法（生产级）

> **改进**：把通用逻辑（`__set_name__`、`__set__`）抽象到基类，子类只需实现 `validate` 方法。
> 这是**模板方法模式**（Template Method Pattern），也是 Django ORM 字段类的标准结构。

```python
import abc


# ============================================================
# 抽象描述符基类：Validated（模板方法模式）
# ============================================================

class Validated(abc.ABC):
    """所有验证型描述符的抽象基类。

    使用模板方法模式：
    - __set_name__ 和 __set__ 在基类中实现（共享逻辑）
    - validate 是抽象方法，子类必须实现（具体验证逻辑）
    """

    def __set_name__(self, owner, name):
        """自动获取属性名（所有子类共享此逻辑）。"""
        self.storage_name = name

    def __set__(self, instance, value):
        """拦截赋值，委托验证，存储结果。

        这是"自委托"（Self-Delegation）模式：
        __set__ 把职责委托给 self.validate（子类实现）。
        """
        # 步骤1：调用子类实现的 validate 方法进行验证
        value = self.validate(self.storage_name, value)
        # 步骤2：将验证（可能被清洗）后的值存入托管实例
        instance.__dict__[self.storage_name] = value
        # 注意：validate 不仅验证，还可以清洗/转换数据
        # 如 NonBlank 会返回 strip() 后的值

    @abc.abstractmethod
    def validate(self, name, value):
        """子类必须实现的验证逻辑。

        Args:
            name:  属性名（用于错误消息）
            value: 待验证的值

        Returns:
            验证通过后的值（可以是清洗/转换后的版本）

        Raises:
            ValueError: 验证失败时抛出
        """
        # 这是抽象方法，无需实现（子类必须覆盖）


# ============================================================
# 具体描述符：Quantity（继承 Validated，只实现验证逻辑）
# ============================================================

class Quantity(Validated):
    """验证数值必须大于零的描述符。"""

    def validate(self, name, value):
        """验证值 > 0。

        Args:
            name:  属性名（如 'weight'）
            value: 待验证的数值

        Returns:
            验证通过的值（原样返回）

        Raises:
            ValueError: 值 <= 0 时抛出
        """
        if value <= 0:
            raise ValueError(f'{name} must be > 0')
        return value  # ✅ 必须返回值！基类 __set__ 用它来存储


# ============================================================
# 具体描述符：NonBlank（继承 Validated，验证字符串非空）
# ============================================================

class NonBlank(Validated):
    """验证字符串不能为空或纯空白字符的描述符。"""

    def validate(self, name, value):
        """验证字符串非空，同时去除首尾空白。

        Args:
            name:  属性名（如 'description'）
            value: 待验证的字符串

        Returns:
            去除首尾空白后的字符串

        Raises:
            ValueError: 去除空白后为空字符串时抛出
        """
        value = value.strip()   # 先去除首尾空白（数据清洗）
        if not value:
            raise ValueError(f'{name} cannot be blank')
        return value            # ✅ 返回清洗后的值（有副作用：自动 strip）


# ============================================================
# 托管类：LineItem v5（三个描述符，逻辑最清晰）
# ============================================================

class LineItem:
    """订单商品，使用描述符自动验证各字段。

    类似 Django 模型的字段声明方式。
    """
    description = NonBlank()   # 描述不能为空
    weight      = Quantity()   # 重量必须 > 0
    price       = Quantity()   # 价格必须 > 0

    def __init__(self, description, weight, price):
        self.description = description  # 触发 NonBlank.__set__ → validate → strip
        self.weight      = weight       # 触发 Quantity.__set__  → validate
        self.price       = price        # 触发 Quantity.__set__  → validate

    def subtotal(self):
        return self.weight * self.price


# ============================================================
# 测试
# ============================================================

# 正常创建
item = LineItem('  Organic apple  ', 2, 5.0)
print(item.description)  # 'Organic apple'（自动 strip！）
print(item.weight)       # 2
print(item.price)        # 5.0

# 验证失败：空描述
try:
    LineItem('   ', 1, 1)   # 纯空白字符
except ValueError as e:
    print(e)  # description cannot be blank

# 验证失败：价格为零
try:
    LineItem('Apple', 1, 0)
except ValueError as e:
    print(e)  # price must be > 0

# 查看类结构
print(type(LineItem.description))  # <class 'NonBlank'>（描述符实例）
print(type(LineItem.weight))       # <class 'Quantity'>（描述符实例）

# 展示继承体系
print(NonBlank.__mro__)
# (<class 'NonBlank'>, <class 'Validated'>, <class 'abc.ABC'>, <class 'object'>)
```

## 7. 覆盖型描述符 vs 非覆盖型描述符

> **⚠️ 重点**：根据是否实现 `__set__`，描述符分为两大类，行为差异显著。

### 7.1 实验台代码

```python
# 以下代码用于演示三种描述符的行为差异

def cls_name(obj):
    """辅助函数，返回对象的类名。"""
    return type(obj).__name__


class Overriding:
    """覆盖型描述符（数据描述符）：同时有 __get__ 和 __set__。"""
    def __get__(self, instance, owner):
        print(f'→ Overriding.__get__('
              f'instance={cls_name(instance) if instance else None}, '
              f'owner={owner.__name__})')
        return None  # 示例中不返回实际值

    def __set__(self, instance, value):
        print(f'→ Overriding.__set__(instance={cls_name(instance)}, value={value})')


class OverridingNoGet:
    """无 __get__ 的覆盖型描述符：只有 __set__，没有 __get__。"""
    def __set__(self, instance, value):
        print(f'→ OverridingNoGet.__set__(instance={cls_name(instance)}, value={value})')


class NonOverriding:
    """非覆盖型描述符（非数据描述符）：只有 __get__，没有 __set__。"""
    def __get__(self, instance, owner):
        print(f'→ NonOverriding.__get__('
              f'instance={cls_name(instance) if instance else None})')
        return None


class Managed:
    """托管类，包含三种描述符和一个普通方法。"""
    over        = Overriding()
    over_no_get = OverridingNoGet()
    non_over    = NonOverriding()

    def spam(self):
        print(f'→ Managed.spam({cls_name(self)})')
```

### 7.2 覆盖型描述符（有 `__get__` + `__set__`）

```python
obj = Managed()

# ---- 读取操作 ----
obj.over
# 输出：→ Overriding.__get__(instance=Managed, owner=Managed)
# 无论如何，读取都触发 __get__

Managed.over
# 输出：→ Overriding.__get__(instance=None, owner=Managed)
# 通过类访问，instance=None

# ---- 写入操作 ----
obj.over = 7
# 输出：→ Overriding.__set__(instance=Managed, value=7)
# __set__ 拦截赋值

# ---- 强行绕过：直接写实例 __dict__ ----
obj.__dict__['over'] = 8
print(vars(obj))   # {'over': 8}（__dict__ 里确实有值了）
obj.over           # 输出：→ Overriding.__get__(...)（__get__ 仍然被调用！）
# ⚠️ 即使绕过写了实例属性，读取仍然触发 __get__，实例属性被完全忽略

# 结论：覆盖型描述符的 __get__ 始终胜出，实例 __dict__ 无法遮盖它
```

### 7.3 无 `__get__` 的覆盖型描述符（只有 `__set__`）

```python
obj = Managed()

# ---- 读取操作 ----
obj.over_no_get
# 没有 __get__，直接返回描述符对象本身：<OverridingNoGet object>

Managed.over_no_get
# 同上：<OverridingNoGet object>

# ---- 写入操作 ----
obj.over_no_get = 7
# 输出：→ OverridingNoGet.__set__(instance=Managed, value=7)
# __set__ 仍然拦截赋值（因为有 __set__，是覆盖型）

obj.over_no_get
# 仍然返回描述符对象（__set__ 没有真正存值）

# ---- 绕过写入实例 __dict__ ----
obj.__dict__['over_no_get'] = 9
obj.over_no_get
# 返回 9（实例属性遮盖了描述符——读操作如此）

obj.over_no_get = 7
# 输出：→ OverridingNoGet.__set__（写操作仍经过描述符）

obj.over_no_get
# 仍然返回 9（写操作经过描述符，但读操作从实例 __dict__ 取）

# 结论：写操作由描述符控制；读操作时，若有同名实例属性则遮盖描述符
```

### 7.4 非覆盖型描述符（只有 `__get__`，无 `__set__`）

```python
obj = Managed()

# ---- 读取操作 ----
obj.non_over
# 输出：→ NonOverriding.__get__(instance=Managed)
# 正常触发 __get__

# ---- 写入操作 ----
obj.non_over = 7
# 没有 __set__，直接写入实例 __dict__（描述符完全被绕过）

obj.non_over
# 返回 7（实例属性遮盖了描述符）

# ---- 通过类访问，描述符仍然存在 ----
Managed.non_over
# 输出：→ NonOverriding.__get__(instance=None)（类上的描述符未受影响）

# ---- 删除实例属性，描述符恢复 ----
del obj.non_over
obj.non_over
# 输出：→ NonOverriding.__get__(instance=Managed)（描述符重新生效）

# 结论：非覆盖型描述符可被实例属性临时遮盖，删除实例属性后自动恢复
```

### 7.5 特殊规则：覆盖类中的描述符

```python
# ⚠️ 无论什么类型的描述符，直接对类赋值都能覆盖它！

obj = Managed()

# 覆盖类上的所有描述符
Managed.over        = 1   # Overriding 描述符被整数替换
Managed.over_no_get = 2   # OverridingNoGet 描述符被整数替换
Managed.non_over    = 3   # NonOverriding 描述符被整数替换

print(obj.over, obj.over_no_get, obj.non_over)
# (1, 2, 3) ← 描述符全都不见了

# 要保护类属性级别的描述符，需要在元类上定义描述符（第24章内容）
```

## 8. 方法是描述符（重要推论）

> **⚠️ 核心推论**：普通函数实现了 `__get__`，没有 `__set__`，所以是**非覆盖型描述符**。
> `self` 不是语言内置魔法，而是函数 `__get__` 把实例绑定到第一个参数的结果。

### 8.1 函数作为描述符的完整机制

```python
import collections


class Text(collections.UserString):
    """演示方法作为描述符的工作机制。"""

    def __repr__(self):
        return f'Text({self.data!r})'

    def reverse(self):
        """将字符串反转。"""
        return self[::-1]


word = Text('forward')

# ---- 通过实例访问方法 → 得到"绑定方法" ----
bound = word.reverse
print(bound)         # <bound method Text.reverse of Text('forward')>
print(type(bound))   # <class 'method'>

# ---- 通过类访问方法 → 得到函数本身 ----
func = Text.reverse
print(func)          # <function Text.reverse at 0x...>
print(type(func))    # <class 'function'>

# 注意：同一个 reverse，通过实例和类访问得到不同的对象

# ---- 手动模拟 Python 的绑定过程 ----
# word.reverse 实际上执行的是：
# Text.__dict__['reverse'].__get__(word, Text)
manual_bound = Text.reverse.__get__(word)   # 绑定到 word
print(manual_bound)  # <bound method Text.reverse of Text('forward')>

unbound = Text.reverse.__get__(None, Text)  # instance=None，返回函数本身
print(unbound)       # <function Text.reverse at 0x...>

# ---- 绑定方法的内部结构 ----
print(bound.__self__)              # Text('forward')（绑定的实例）
print(bound.__func__)              # <function Text.reverse>（原始函数）
print(bound.__func__ is Text.reverse)  # True

# ---- 调用绑定方法的完整过程 ----
# word.reverse()
# = bound()
# = bound.__func__(bound.__self__)
# = Text.reverse(word)
# = word 被传入 self 参数
```

### 8.2 为什么方法可以被实例属性遮盖

```python
obj = Managed()

# 函数没有 __set__（非覆盖型描述符），所以可以被实例属性遮盖
obj.spam = 7         # 直接写入实例 __dict__
obj.spam             # → 7（方法被遮盖了）
# obj.spam()         # TypeError: 'int' object is not callable

# ---- 但特殊方法不受影响 ----
# 解释器查找特殊方法时，从类（type(obj)）查找，而非实例
# repr(obj) 实际上执行 type(obj).__repr__(obj)

class MyObj:
    pass

m = MyObj()
m.__repr__ = lambda: 'fake'  # 写入实例 __dict__
repr(m)    # 仍然返回默认的 <__main__.MyObj object at 0x...>
# ✅ 特殊方法不受实例属性遮盖，因为解释器从类上查找
```

### 8.3 `classmethod` 和 `staticmethod` 也是描述符

```python
class Demo:
    @classmethod
    def cls_method(cls):
        """classmethod 是非覆盖型描述符（只有 __get__）。
        __get__ 返回把 cls 绑定为第一个参数的函数。
        """
        return cls

    @staticmethod
    def static_method():
        """staticmethod 也是非覆盖型描述符。
        __get__ 返回原始函数（不绑定任何参数）。
        """
        return 'static'


# classmethod 的 __get__：绑定类
print(Demo.cls_method())   # <class 'Demo'>
print(Demo().cls_method()) # <class 'Demo'>（实例和类调用结果相同）

# staticmethod 的 __get__：返回原始函数
print(Demo.static_method())    # 'static'
print(Demo().static_method())  # 'static'（同上）
```

## 9. 描述符用法建议

### 9.1 保持简单，优先用 `@property`

```python
# ✅ 1-2 个属性验证 → @property 最简洁
class Circle:
    @property
    def radius(self):
        return self._radius

    @radius.setter
    def radius(self, value):
        if value < 0:
            raise ValueError('radius must be >= 0')
        self._radius = value

    # property 的 __set__ 默认抛 AttributeError（只读特性）
    # property 是覆盖型描述符，自动防止实例属性遮盖
```

### 9.2 只读描述符必须同时有 `__set__`

```python
# ❌ 错误：只有 __get__ 的"只读描述符"——实例属性可以遮盖它！
class BadReadOnly:
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return 42

class Obj:
    value = BadReadOnly()

o = Obj()
o.value = 'overwritten'  # 直接写入实例 __dict__，描述符被遮盖
print(o.value)           # 'overwritten'（只读失效了！）


# ✅ 正确：只读描述符必须同时实现 __set__，在其中抛异常
class ReadOnly:
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__.get(f'_{self.storage_name}', 0)

    def __set__(self, instance, value):
        raise AttributeError(f'{self.storage_name} is read-only')

    def __set_name__(self, owner, name):
        self.storage_name = name
```

### 9.3 纯验证描述符可以只有 `__set__`

```python
# 当 storage_name == 托管属性名时，不需要 __get__
# 读取时直接从实例 __dict__ 拿（更快！）

class Positive(Validated):
    """只验证，不拦截读取。"""

    def validate(self, name, value):
        if value <= 0:
            raise ValueError(f'{name} must be positive')
        return value

class LineItem:
    weight = Positive()  # __set_name__ 自动设置 storage_name = 'weight'
    # 读取 item.weight 时：
    # 1. 找覆盖型描述符 → Positive 有 __set__，是覆盖型
    # 2. Positive 无 __get__ → 跳过
    # 3. 找实例 __dict__['weight'] → 直接返回（快！）
```

### 9.4 缓存计算用只有 `__get__` 的非覆盖型描述符

```python
import math


class CachedArea:
    """计算圆面积，首次计算后缓存到实例 __dict__。"""

    def __get__(self, instance, owner):
        if instance is None:
            return self
        # 计算面积
        area = math.pi * instance.radius ** 2
        # 写入实例 __dict__['area']（同名属性）
        instance.__dict__['area'] = area
        # 之后读取 instance.area 直接从 __dict__ 拿，不再触发 __get__
        return area
    # 无 __set__ → 非覆盖型 → 实例 __dict__['area'] 会遮盖描述符
    # 这正是缓存生效的机制！


class Circle:
    area = CachedArea()  # 非覆盖型描述符

    def __init__(self, radius):
        self.radius = radius


c = Circle(5)
print(c.area)    # 78.539...（首次计算，触发 __get__）
print(c.area)    # 78.539...（从实例 __dict__ 直接返回，不再计算）
# functools.cached_property 的实现原理与此完全相同
```

## 10. 完整属性优先级速查

```
obj.attr 的查找顺序（高 → 低）：

①【最高】覆盖型描述符（数据描述符）
   条件：type(obj).__mro__ 中有同名属性，且该属性有 __set__ 或 __delete__
   例子：@property、Quantity、NonBlank
   特点：无论实例 __dict__ 有什么，都会调用 __get__

②【中等】实例 __dict__
   条件：obj.__dict__ 中有该键
   例子：普通实例属性
   特点：直接返回，无方法调用

③【较低】非覆盖型描述符 / 普通类属性
   条件：type(obj).__mro__ 中有同名属性，且没有 __set__
   例子：函数/方法（只有 __get__）、cached_property、普通类变量
   特点：非覆盖型描述符的 __get__ 在此触发；实例属性可以遮盖它

④【最低】__getattr__
   条件：以上全部失败后调用
   例子：FrozenJSON 的 __getattr__
   特点：兜底方法，用于虚拟属性
```

## 11. 常见误区与陷阱

### ❌ 误区一：把值存到描述符实例里（最常见的严重错误）

```python
# ❌ 错误！所有托管实例共享同一个值
class BadDescriptor:
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return self.value  # 从描述符实例读取 ← 全局共享！

    def __set__(self, instance, value):
        self.value = value  # 存到描述符实例 ← 错误！


class Broken:
    x = BadDescriptor()


a = Broken()
b = Broken()
a.x = 10  # 存到 BadDescriptor 实例（Broken.x）
b.x = 20  # 覆盖了上面的值
print(a.x)  # 20（不是 10！因为 a 和 b 共享同一个描述符实例）


# ✅ 正确：值要存到托管实例（instance.__dict__）
class GoodDescriptor:
    def __set_name__(self, owner, name):
        self.storage_name = name

    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__.get(self.storage_name)  # 从托管实例读

    def __set__(self, instance, value):
        instance.__dict__[self.storage_name] = value  # 存到托管实例
```

### ❌ 误区二：在 `__set__` 中用 `setattr(instance, ...)` 导致递归

```python
class InfiniteLoop:
    def __set_name__(self, owner, name):
        self.name = name

    def __set__(self, instance, value):
        # ❌ 错误：setattr(instance, self.name, value) 会触发 __set__
        setattr(instance, self.name, value)  # 无限递归！

    # ✅ 正确：直接写 __dict__
    def __set__correct(self, instance, value):
        instance.__dict__[self.name] = value  # 正确，不触发 __set__
```

### ❌ 误区三：以为非覆盖型描述符"只读"

```python
class ReadOnlyAttempt:
    """试图用非覆盖型描述符实现只读属性——行不通！"""
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return 42


class MyClass:
    value = ReadOnlyAttempt()


obj = MyClass()
print(obj.value)   # 42（正常读取）
obj.value = 'hack' # 直接写入实例 __dict__，不报错！
print(obj.value)   # 'hack'（只读失效！）

# ✅ 要实现只读，必须用覆盖型描述符（有 __set__）
class TrueReadOnly:
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__.get(f'_{self.name}', 42)

    def __set__(self, instance, value):
        raise AttributeError(f'{self.name} is read-only')  # 覆盖型 + 拒绝写入

    def __set_name__(self, owner, name):
        self.name = name
```

### ❌ 误区四：`__set_name__` 的时序误解

```python
class EarlyAccess:
    def __init__(self):
        # ❌ 错误：__init__ 在 __set_name__ 之前执行
        # 此时 storage_name 还不存在
        print(f'storage_name = {self.storage_name}')  # AttributeError!

    def __set_name__(self, owner, name):
        self.storage_name = name  # 这在 __init__ 之后才调用

# 正确时序：
# 1. weight = EarlyAccess()  ← __init__ 执行（storage_name 还不存在）
# 2. class 主体读取完毕
# 3. type.__new__ 调用 __set_name__（现在 storage_name 才有值）
# 4. 用户调用 LineItem(...) 时，__init__ 才再次运行（实例初始化）
```

## 12. 本章知识对比速查表

### 三种描述符类型对比

| 类型 | 有 `__get__` | 有 `__set__` | 赋值行为 | 读取行为 | 典型例子 |
|------|:-----------:|:-----------:|----------|----------|----------|
| **覆盖型（完整）** | ✅ | ✅ | 调用 `__set__`，实例属性无效 | 调用 `__get__`，始终胜出 | `@property`、`Quantity` |
| **覆盖型（无 get）** | ❌ | ✅ | 调用 `__set__` | 返回描述符对象（或实例属性） | 纯验证字段 |
| **非覆盖型** | ✅ | ❌ | 直接写 `__dict__`，遮盖描述符 | 调用 `__get__`（若未被遮盖）| 函数/方法、`cached_property` |

### `__get__` 方法的参数说明

| 访问方式 | `self` | `instance` | `owner` | 推荐返回值 |
|---------|--------|------------|---------|----------|
| 通过实例 `obj.attr` | 描述符实例 | 托管实例 | 托管类 | 储存的属性值 |
| 通过类 `Cls.attr` | 描述符实例 | `None` | 托管类 | 描述符实例本身 |

### 何时选择哪种属性管理方式

| 场景 | 推荐方案 | 理由 |
|------|----------|------|
| 单个属性只读计算 | `@property`（只写 getter）| 最简洁，自动拒绝写入 |
| 单个属性读写验证 | `@property` + `@setter` | 局部清晰，适合 1-2 个属性 |
| 多个属性相同验证 | 描述符类 + `__set_name__` | 代码复用，Django ORM 同款 |
| 多种验证逻辑可扩展 | `Validated` 基类 + 子类 | 模板方法模式，无限扩展 |
| 计算属性缓存 | `@property + @cache` | 万能，适合各种情况 |
| 轻量缓存（无同名实例属性）| `@cached_property` | 简洁但有局限性 |
| 动态属性名 | `__getattr__` / `__getattribute__` | 属性名不固定，无法用描述符 |
