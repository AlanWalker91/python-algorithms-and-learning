## 题目1：53. 最大子数组和

### 1. 核心考点

**动态规划 / Kadane算法**。考查如何在一维数组中找到和最大的连续子数组，是"连续子数组问题"的经典入门题。

### 2. 解题思路（给新手讲）

想象你在走一条路，路上每一步都会捡到或丢掉一些金币（正数或负数）。你想找到一段连续的路，使得这段路上捡到的金币总数最多。

关键直觉：**当你走到某一步时，你只需要做一个决定——是把之前积累的金币带上继续走，还是从当前位置重新开始？**

如果之前积累的总和是负数，那带着它只会拖后腿，不如丢掉重新开始。如果之前积累的总和是正数或零，那带着它至少不亏。

### 3. 从暴力到优化的推导

#### 暴力解法

枚举所有子数组的起点 i 和终点 j，计算每个子数组的和，取最大值。

- 时间复杂度：O(n^3)，三层循环（枚举起点、终点、计算区间和）
- 用前缀和可以优化到 O(n^2)，但仍然不够快

#### 优化思路

观察暴力解法中的重复计算：当我们固定起点 i，从左到右扩展终点 j 时，`sum(i, j)` 和 `sum(i, j-1)` 只差一个元素。

进一步思考：我们真的需要固定起点吗？

定义 `dp[i]` 为"以第 i 个元素结尾的最大子数组和"。那么：
- `dp[i] = max(dp[i-1] + nums[i], nums[i])`

含义：要么把 nums[i] 接到前面的子数组后面，要么从 nums[i] 重新开始。

#### 最优解法

Kadane算法，本质就是上面的DP，但只需要一个变量记录"当前子数组和"：

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 4. 关键思维点

- **"接"还是"断"**：`dp[i-1] + nums[i]` vs `nums[i]`，等价于判断 `dp[i-1]` 是否大于 0
- **局部最优 -> 全局最优**：每一步维护"以当前元素结尾的最大和"，全局答案就是所有 `dp[i]` 的最大值
- **空间优化**：dp 数组可以压缩为一个变量，因为 `dp[i]` 只依赖 `dp[i-1]`

### 5. 代码实现（Python）

```python
def maxSubArray(nums):
    # cur_sum: 以当前元素结尾的最大子数组和
    cur_sum = nums[0]
    # max_sum: 全局最大子数组和
    max_sum = nums[0]

    # 从第二个元素开始遍历
    for i in range(1, len(nums)):
        # 核心决策：接上前面的子数组，还是从当前重新开始
        # 如果 cur_sum > 0，接上它能让和更大；否则从当前重新开始
        cur_sum = max(cur_sum + nums[i], nums[i])
        # 更新全局最大值
        max_sum = max(max_sum, cur_sum)

    return max_sum
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| Kadane算法 | 连续子数组求最值的标准模板 |
| DP状态定义 | "以第i个元素结尾的xxx"是处理连续子数组问题的常用状态定义 |
| 空间压缩 | 当 dp[i] 只依赖 dp[i-1] 时，可以用一个变量代替数组 |

## 题目2：56. 合并区间

### 1. 核心考点

**排序 + 贪心**。考查对区间问题的处理能力，是"区间合并"类问题的基础模板题。

### 2. 解题思路（给新手讲）

想象你有一堆时间段（比如会议安排），有些会议时间有重叠，你需要把重叠的合并成一个大时间段。

关键直觉：**先按开始时间排序**。排好序后，你只需要从左到右扫一遍。对于当前区间，只有两种情况：
1. 和前一个区间有重叠（当前区间的左端点 <= 前一个区间的右端点） -> 合并，右端点取两者较大值
2. 没有重叠 -> 直接作为一个新区间加入结果

为什么排序后只需要和"前一个"比较？因为排序保证了后面的区间左端点 >= 当前区间左端点，所以如果当前区间和前一个不重叠，后面的区间更不可能和前一个重叠。

### 3. 从暴力到优化的推导

#### 暴力解法

对每一对区间判断是否重叠，如果重叠就合并，然后重新检查所有区间，反复迭代直到没有可合并的区间。

- 时间复杂度：O(n^2) 甚至更高

#### 优化思路

暴力法的问题在于没有顺序，所以需要反复扫描。如果按左端点排序，所有可能重叠的区间都会相邻出现，只需要一次扫描。

#### 最优解法

排序 + 一次线性扫描：

- 时间复杂度：O(n log n)，排序主导
- 空间复杂度：O(n)，存储结果（不算结果空间则 O(1)）

### 4. 关键思维点

- **排序是区间问题的第一步**：绝大多数区间问题都需要先排序
- **合并条件**：`intervals[i][0] <= merged[-1][1]`，即当前左端点 <= 上一个右端点
- **更新右端点**：合并时取 `max(merged[-1][1], intervals[i][1])`，不能直接取当前的右端点（因为可能出现包含关系，如 [1,10] 和 [2,3]）

### 5. 代码实现（Python）

```python
def merge(intervals):
    # 按区间左端点排序
    intervals.sort(key=lambda x: x[0])

    # merged 存储合并后的结果
    merged = []

    for interval in intervals:
        # 如果结果为空，或当前区间与上一个区间不重叠
        if not merged or interval[0] > merged[-1][1]:
            # 直接加入结果
            merged.append(interval)
        else:
            # 有重叠，合并：更新右端点为两者的较大值
            merged[-1][1] = max(merged[-1][1], interval[1])

    return merged
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 区间排序 | 区间问题第一步几乎都是排序（按左端点或右端点） |
| 贪心合并 | 排序后从左到右扫描，逐个判断是否合并 |
| 右端点取max | 合并时不能忘记处理包含关系 |

## 题目3：189. 轮转数组

### 1. 核心考点

**数组翻转技巧**。考查如何在 O(1) 额外空间下实现数组的循环移位，核心是"三次翻转"法。

### 2. 解题思路（给新手讲）

把数组想象成一个环，"轮转k位"就是把最后k个元素搬到最前面。

比如 `[1,2,3,4,5,6,7]` 轮转3位 -> `[5,6,7,1,2,3,4]`

有一个神奇的技巧：**通过三次翻转就能完成轮转！**

想象一本书的页面，你想把最后几页移到最前面：
1. 先把整本书倒过来 -> `[7,6,5,4,3,2,1]`
2. 再把前k页翻回来 -> `[5,6,7,4,3,2,1]`
3. 最后把剩余页翻回来 -> `[5,6,7,1,2,3,4]`

为什么这样做有效？第一次整体翻转把最后k个元素"带"到了前面，但它们的内部顺序反了；后两次翻转分别修复前k个和后n-k个元素的内部顺序。

### 3. 从暴力到优化的推导

#### 暴力解法

每次将数组右移一位（最后一个元素放到最前面），重复k次。

- 时间复杂度：O(n * k)
- 空间复杂度：O(1)

或者使用额外数组：直接计算每个元素的新位置。

- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 优化思路

能不能既 O(n) 时间又 O(1) 空间？答案就是三次翻转法。翻转操作是原地的，每次翻转只需要 O(n) 时间。

#### 最优解法

三次翻转：

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 4. 关键思维点

- **k可能大于n**：需要先 `k = k % n`，因为轮转n次等于没动
- **翻转的本质**：翻转操作只改变顺序不改变元素集合，三次翻转巧妙地实现了"循环移位"
- **左移同理**：如果要左移k位，只需改变翻转的分界点（先翻前k个，再翻后n-k个，再整体翻）

### 5. 代码实现（Python）

```python
def rotate(nums, k):
    n = len(nums)
    # k可能大于数组长度，取模
    k = k % n

    # 定义翻转函数：将 nums[left..right] 原地翻转
    def reverse(left, right):
        while left < right:
            nums[left], nums[right] = nums[right], nums[left]
            left += 1
            right -= 1

    # 第一步：整体翻转
    # [1,2,3,4,5,6,7] -> [7,6,5,4,3,2,1]
    reverse(0, n - 1)

    # 第二步：翻转前k个元素
    # [7,6,5,4,3,2,1] -> [5,6,7,4,3,2,1]
    reverse(0, k - 1)

    # 第三步：翻转后n-k个元素
    # [5,6,7,4,3,2,1] -> [5,6,7,1,2,3,4]
    reverse(k, n - 1)
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 三次翻转法 | 数组轮转/循环移位的经典O(1)空间技巧 |
| 取模处理 | k % n 避免多余操作 |
| 原地翻转 | 双指针交换实现O(1)空间的翻转 |

## 题目4：238. 除自身以外数组的乘积

### 1. 核心考点

**前缀积与后缀积**。考查在不使用除法的前提下，如何利用前后缀分解来计算"除自身以外"的乘积。

### 2. 解题思路（给新手讲）

对于位置 i，`answer[i] = 所有左边元素的乘积 * 所有右边元素的乘积`（不包含自身）。

想象你站在一排人中间，你想知道除了你以外所有人的身高乘积。你可以分别算出"你左边所有人的身高乘积"和"你右边所有人的身高乘积"，然后把这两个数相乘就行了。

所以我们需要两样东西：
- **前缀积 prefix[i]**：nums[0] * nums[1] * ... * nums[i-1]（i 左边所有元素的乘积）
- **后缀积 suffix[i]**：nums[i+1] * nums[i+2] * ... * nums[n-1]（i 右边所有元素的乘积）
- **answer[i] = prefix[i] * suffix[i]**

### 3. 从暴力到优化的推导

#### 暴力解法

对每个位置 i，遍历整个数组跳过 i 计算乘积。

- 时间复杂度：O(n^2)
- 空间复杂度：O(1)

#### 优化思路

暴力法中对每个 i 都重新计算了完整乘积，大量重复。

如果能预先算好"每个位置左边的乘积"和"右边的乘积"，就能 O(1) 算出每个位置的答案。

进一步优化：可以先用结果数组存前缀积，然后从右往左一边计算后缀积一边乘到结果数组中，这样不需要额外的后缀积数组。

#### 最优解法

两次遍历（前缀积 + 后缀积），用一个变量滚动计算后缀积：

- 时间复杂度：O(n)
- 空间复杂度：O(1)（结果数组不算额外空间）

### 4. 关键思维点

- **分解思想**：`除自身外的乘积 = 左边的乘积 * 右边的乘积`
- **不用除法**：题目明确要求不用除法。即使能用除法，遇到0也会有问题
- **空间优化**：利用结果数组先存前缀积，再用一个变量从右向左累乘后缀积

### 5. 代码实现（Python）

```python
def productExceptSelf(nums):
    n = len(nums)
    answer = [1] * n

    # 第一遍：从左到右，计算前缀积
    # answer[i] = nums[0] * nums[1] * ... * nums[i-1]
    prefix = 1
    for i in range(n):
        answer[i] = prefix       # 先把左边的乘积存入答案
        prefix *= nums[i]        # 更新前缀积（把当前元素乘进去）

    # 第二遍：从右到左，计算后缀积并乘到答案中
    # answer[i] *= nums[i+1] * nums[i+2] * ... * nums[n-1]
    suffix = 1
    for i in range(n - 1, -1, -1):
        answer[i] *= suffix      # 把右边的乘积乘到答案中
        suffix *= nums[i]        # 更新后缀积（把当前元素乘进去）

    return answer
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 前缀/后缀分解 | 把"除自身外"拆成"左边"和"右边"两部分分别计算 |
| 滚动变量 | 用一个变量代替整个前缀/后缀数组，节省空间 |
| 两次遍历 | 正向一次（前缀），反向一次（后缀），合并结果 |

## 题目5：41. 缺失的第一个正数

### 1. 核心考点

**原地哈希（Index as Hash Key）**。考查如何在 O(n) 时间 O(1) 空间的严格限制下，利用数组本身作为哈希表。

### 2. 解题思路（给新手讲）

如果没有空间限制，我们可以用一个哈希集合记录出现过的正整数，然后从1开始依次查找，第一个不在集合中的就是答案。

但题目要求 O(1) 空间。怎么办？**把数组本身当作哈希表！**

核心思想：长度为 n 的数组，答案一定在 [1, n+1] 范围内（最多放 n 个正整数，最坏情况答案是 n+1）。所以我们可以把每个值为 v 的元素放到索引 v-1 的位置上（即 nums[v-1] = v），形成"每个萝卜回到自己的坑"的局面。

放好之后，从头扫一遍，第一个 `nums[i] != i + 1` 的位置 i，答案就是 `i + 1`。

### 3. 从暴力到优化的推导

#### 暴力解法

从 1 开始逐个检查每个正整数是否在数组中。

- 时间复杂度：O(n^2)（每次查找 O(n)）
- 用哈希集合可以优化到 O(n) 时间，但空间 O(n)

#### 优化思路

哈希集合用了 O(n) 额外空间。能不能不用额外空间？

关键洞察：数组本身有 n 个位置，我们关心的正整数范围也恰好是 [1, n]。所以可以把数组当成一个"大小为 n 的哈希表"，位置 i 存放值 i+1。

#### 最优解法

原地交换，将每个元素放到它应该在的位置：

- 时间复杂度：O(n)，虽然有嵌套循环，但每个元素最多被交换一次
- 空间复杂度：O(1)

### 4. 关键思维点

- **答案范围缩小**：长度为 n 的数组，答案一定在 [1, n+1] 中
- **原地哈希**：`nums[i]` 应该放到 `nums[nums[i]-1]` 的位置
- **交换而非覆盖**：用 swap 而不是直接赋值，确保不丢失任何元素
- **循环条件**：`while 1 <= nums[i] <= n and nums[nums[i]-1] != nums[i]`，既要确保值在合法范围内，也要确保目标位置还没放好（避免死循环）

### 5. 代码实现（Python）

```python
def firstMissingPositive(nums):
    n = len(nums)

    # 第一步：把每个元素放到它应该在的位置
    # 值为 v 的元素应该放在索引 v-1 的位置
    for i in range(n):
        # 不断交换，直到当前位置放对或无法继续
        while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
            # 将 nums[i] 放到它应该在的位置 nums[i]-1
            correct_idx = nums[i] - 1
            nums[i], nums[correct_idx] = nums[correct_idx], nums[i]

    # 第二步：找第一个不在正确位置的元素
    for i in range(n):
        if nums[i] != i + 1:
            return i + 1  # 缺失的最小正整数

    # 如果所有位置都正确，答案是 n+1
    return n + 1
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 原地哈希 | 利用数组下标作为天然的哈希键，把值 v 放到索引 v-1 |
| 答案范围分析 | 长度为 n 的数组，缺失的最小正整数一定 <= n+1 |
| 交换归位 | 用 while 循环不断交换，直到当前元素到达正确位置或不合法 |
| 避免死循环 | 检查 `nums[nums[i]-1] != nums[i]` 防止重复元素导致无限交换 |
