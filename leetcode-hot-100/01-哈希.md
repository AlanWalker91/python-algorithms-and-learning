## 题目1：1. 两数之和 (Two Sum)

> 给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出和为目标值 `target` 的那两个整数，并返回它们的数组下标。你可以假设每种输入只会对应一个答案，并且不能使用同一个元素两次。

### 1. 核心考点

- **哈希表的查找能力**：利用哈希表将"查找某个值是否存在"的操作从 O(n) 降到 O(1)。
- **"互补配对"思想**：当我们需要找两个数满足某种关系时（如 a + b = target），可以固定一个数，去查找另一个数是否存在。
- 本题属于经典的 **"两数配对查找"** 问题。

### 2. 解题思路（给新手讲）

想象你在一个派对上，主持人说："谁能找到另一个人，让你们两人的号码牌加起来等于 100？"

**笨办法**：你走到每一个人面前，把你的号码牌和对方的加一加，看看是不是 100。如果派对上有 n 个人，你最多要问 n-1 个人。而且每个人都要这样做，所以总共要问 n*(n-1)/2 次。

**聪明办法**：你在门口放一块白板。每个人进来时，先看白板上有没有自己需要的"搭档号码"（即 100 减去自己的号码）。如果有，直接配对成功；如果没有，就把自己的号码写在白板上，留给后面的人查找。

这块白板就是**哈希表**。它让"查找搭档"这件事变得极快，一眼就能看到。

### 3. 从暴力到优化的推导

#### 暴力解法

对数组中的每一个元素 `nums[i]`，遍历它后面的所有元素 `nums[j]`，检查 `nums[i] + nums[j]` 是否等于 `target`。

```
对于每个 i（从 0 到 n-1）：
    对于每个 j（从 i+1 到 n-1）：
        如果 nums[i] + nums[j] == target：
            返回 [i, j]
```

- **时间复杂度**：O(n^2)，两层嵌套循环。
- **空间复杂度**：O(1)，没有使用额外空间。

问题显而易见：当数组很大时（比如 10 万个元素），需要近 50 亿次运算，太慢了。

#### 优化思路

暴力解法慢在哪里？慢在 **内层循环的查找**。对于每一个 `nums[i]`，我们都要线性扫描剩余元素去找 `target - nums[i]`。

关键洞察：**如果能把"查找某个值是否在数组中"这件事变成 O(1)，整体就变成 O(n) 了。**

什么数据结构能做到 O(1) 查找？ -- **哈希表**。

具体做法：
1. 创建一个哈希表，用来存储已经遍历过的元素及其下标。
2. 遍历数组，对于当前元素 `nums[i]`，计算它的"搭档" `complement = target - nums[i]`。
3. 在哈希表中查找 `complement`：
   - 找到了 --> 返回结果。
   - 没找到 --> 把当前元素存入哈希表，继续遍历。

为什么只需要一次遍历？因为当我们遍历到后面的那个数时，前面的那个数已经在哈希表里了。配对一定会在遍历到两个数中较后出现的那个时被发现。

#### 最优解法

使用哈希表的一次遍历法：

- **时间复杂度**：O(n)，只遍历一次数组，每次哈希表查找 O(1)。
- **空间复杂度**：O(n)，哈希表最多存储 n 个元素。

### 4. 关键思维点

- **为什么用哈希表而不是排序？** 排序会破坏原始下标信息，而题目要求返回下标。虽然也可以排序后用双指针，但需要额外记录原始下标，不如哈希表直接。
- **为什么边遍历边插入，而不是先全部插入再查找？** 边遍历边插入可以避免"自己和自己配对"的问题（因为查找时当前元素还没有被插入哈希表），而且一次遍历即可完成。
- **核心 insight**：将"双重循环找配对"转化为"单次遍历 + 哈希表反向查找"。本质上是用空间换时间。

### 5. 代码实现（Python）

```python
def twoSum(nums, target):
    # 创建哈希表：键是数值，值是该数值的下标
    hashmap = {}

    # 遍历数组中的每一个元素
    for i, num in enumerate(nums):
        # 计算当前元素需要的"搭档"
        complement = target - num

        # 在哈希表中查找搭档是否已经出现过
        if complement in hashmap:
            # 找到了！返回搭档的下标和当前下标
            return [hashmap[complement], i]

        # 没找到搭档，把当前元素存入哈希表，留给后面的元素查找
        hashmap[num] = i

    # 题目保证一定有解，理论上不会走到这里
    return []
```

### 6. 套路总结

| 识别信号 | 解题模板 |
|---------|---------|
| 在数组中找两个数满足某种关系 | 哈希表存已遍历元素，查找互补值 |
| 需要 O(1) 查找某个值是否存在 | 哈希表 |
| 需要快速建立"值 -> 下标"的映射 | 哈希表 |

**通用模板**：
```python
def find_pair(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num  # 计算互补值
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
```

## 题目2：49. 字母异位词分组 (Group Anagrams)

> 给你一个字符串数组，请你将字母异位词组合在一起。可以按任意顺序返回结果列表。字母异位词是由重新排列源单词的所有字母得到的一个新单词，所有源单词中的字母通常恰好只用一次。

### 1. 核心考点

- **哈希表的分组能力**：利用哈希表将具有相同特征的元素归为一组。
- **设计哈希键**：如何为"字母异位词"设计一个统一的标识（键），使得所有互为异位词的字符串映射到同一个键。
- 本题属于经典的 **"按特征分组"** 问题。

### 2. 解题思路（给新手讲）

想象你是一个图书管理员，桌上有一堆打乱了字母的卡片：`"eat"`, `"tea"`, `"tan"`, `"ate"`, `"nat"`, `"bat"`。

你的任务是把"本质上用了相同字母"的卡片放到同一个抽屉里。

怎么判断两张卡片是不是"同一类"呢？

**最直观的方法**：把每张卡片上的字母按字母表顺序重新排列。`"eat"` 排完变成 `"aet"`，`"tea"` 排完也变成 `"aet"`，`"ate"` 排完还是 `"aet"`。它们排完都一样，说明它们是同一类！

这个"排列后的结果"就是它们的**身份证号**。你在每个抽屉上贴上身份证号，拿到一张新卡片就排序一下，然后放到对应的抽屉里。

这里的"抽屉"就是**哈希表**，"身份证号"就是**哈希键**。

### 3. 从暴力到优化的推导

#### 暴力解法

对于每对字符串，判断它们是否是字母异位词。判断方法：比较两个字符串排序后是否相同。然后用某种方式把它们分组。

```
对于每对字符串 (s1, s2)：
    如果 sorted(s1) == sorted(s2)：
        它们是同一组
```

- **时间复杂度**：O(n^2 * k*log(k))，其中 n 是字符串个数，k 是字符串最大长度。需要比较每对字符串，每次比较需要排序。
- **空间复杂度**：O(n*k)。

#### 优化思路

暴力解法的问题在于：做了大量重复的"两两比较"。其实我们不需要两两比较，只需要给每个字符串生成一个**唯一标识**，然后按标识分组即可。

**方法一：排序作为键**

把每个字符串排序后的结果作为哈希表的键。互为异位词的字符串排序后结果相同，自然落入同一组。

- 每个字符串排序：O(k*log(k))
- 遍历所有字符串：O(n)
- 总时间：O(n * k*log(k))

**方法二：字符计数作为键**

统计每个字符串中每个字符出现的次数，将计数结果转化为一个元组作为键。例如 `"aab"` 的计数是 `(2,1,0,0,...,0)`（a 出现 2 次，b 出现 1 次，其余 0 次）。

- 每个字符串计数：O(k)
- 遍历所有字符串：O(n)
- 总时间：O(n * k)

方法二在理论上更优，因为避免了排序的 O(k*log(k))，但在实际中，当字符串较短时，两种方法差别不大，排序法代码更简洁。

#### 最优解法

使用哈希表 + 排序键（代码简洁）或字符计数键（理论最优）。

- **时间复杂度**：O(n * k*log(k))（排序法）或 O(n * k)（计数法）。
- **空间复杂度**：O(n * k)，存储所有字符串。

### 4. 关键思维点

- **核心 insight**：分组问题的本质是"如何定义同一类"。能定义出一个统一的标识，就能用哈希表一步到位地分组。
- **为什么排序可以作为标识？** 因为字母异位词包含完全相同的字符，只是顺序不同。排序抹平了顺序差异，保留了字符组成。
- **为什么字符计数也可以？** 同理，异位词的字符频率分布完全相同。
- **选择哈希键的原则**：键必须满足"同组元素映射到相同键，不同组元素映射到不同键"。

### 5. 代码实现（Python）

**方法一：排序作为键（推荐，代码简洁）**

```python
from collections import defaultdict

def groupAnagrams(strs):
    # 创建哈希表：键是排序后的字符串，值是原始字符串的列表
    groups = defaultdict(list)

    for s in strs:
        # 将字符串排序后作为键
        # 例如 "eat" -> "aet", "tea" -> "aet"，它们会归入同一组
        key = ''.join(sorted(s))

        # 把原始字符串加入对应的分组
        groups[key].append(s)

    # 返回所有分组
    return list(groups.values())
```

**方法二：字符计数作为键（理论最优）**

```python
from collections import defaultdict

def groupAnagrams(strs):
    # 创建哈希表：键是字符计数元组，值是原始字符串的列表
    groups = defaultdict(list)

    for s in strs:
        # 统计每个字符的出现次数，生成长度为26的计数数组
        count = [0] * 26
        for c in s:
            count[ord(c) - ord('a')] += 1

        # 将计数数组转为元组作为哈希键（列表不可哈希，元组可以）
        key = tuple(count)

        groups[key].append(s)

    return list(groups.values())
```

### 6. 套路总结

| 识别信号 | 解题模板 |
|---------|---------|
| 把元素按某种等价关系分组 | 哈希表，键 = 等价类的标识 |
| 判断两个对象是否"本质相同" | 找到一个标准化表示（排序/计数/规范形式） |
| 字符串相关的分组/匹配 | 排序 or 字符频率计数 |

**通用模板**：
```python
from collections import defaultdict

def group_by_feature(items):
    groups = defaultdict(list)
    for item in items:
        key = compute_key(item)  # 计算该元素的分组标识
        groups[key].append(item)
    return list(groups.values())
```

**设计哈希键的常见手段**：
- 排序后的结果（字母异位词、同构字符串）
- 字符/元素频率统计
- 数学性质（如余数、质因数分解）

## 题目3：128. 最长连续序列 (Longest Consecutive Sequence)

> 给定一个未排序的整数数组 `nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。要求算法时间复杂度为 O(n)。

### 1. 核心考点

- **哈希集合的 O(1) 查找能力**：利用集合快速判断某个数是否存在。
- **"起点识别"思想**：只从连续序列的起点开始计数，避免重复计算。
- 本题属于经典的 **"序列查找 + 去重"** 问题，要求在 O(n) 时间内完成，排序（O(n*log(n))）不满足要求。

### 2. 解题思路（给新手讲）

想象你有一堆散落在桌上的数字卡片：`100, 4, 200, 1, 3, 2`。

你想找到最长的"能连成一串"的数字序列。比如 `1, 2, 3, 4` 就是一个长度为 4 的连续序列。

**笨办法**：把所有卡片排好序，然后一个一个看相邻的是否连续。但排序要 O(n*log(n))，题目要求 O(n)。

**聪明办法**：

第一步：把所有卡片扔进一个"神奇口袋"（哈希集合），这样你想知道某个数字是否存在，只需要摸一下口袋就知道了。

第二步：现在从口袋里一张一张拿卡片。拿到一张卡片（比如数字 3），你先问自己："2 在不在口袋里？"如果在，说明 3 不是序列的起点（因为前面还有 2），直接跳过它。如果 2 不在口袋里，说明 3 就是某个连续序列的起点！

第三步：既然确定了起点，就从这个起点开始往后数：4 在不在？在！5 在不在？不在。那从 3 开始的连续序列长度是 2（3 和 4）。

关键在于：**只从起点开始计数**。这样每个数字最多被访问两次（一次判断是否是起点，一次在某个序列中被数到），总时间就是 O(n)。

### 3. 从暴力到优化的推导

#### 暴力解法

对数组中的每一个数字 x，从 x 开始不断检查 x+1, x+2, ... 是否在数组中，记录最长的连续序列。

```
最大长度 = 0
对于数组中的每个数 x：
    当前长度 = 1
    当前值 = x
    while (当前值 + 1) 在数组中：
        当前值 += 1
        当前长度 += 1
    最大长度 = max(最大长度, 当前长度)
```

如果"在数组中"的查找用线性扫描：
- **时间复杂度**：O(n^3)，外层 O(n)，内层 while 最多 O(n)，每次查找 O(n)。
- **空间复杂度**：O(1)。

#### 优化思路

**第一步优化：用哈希集合加速查找**

把数组中的所有数字放入哈希集合，使"是否存在"的查询从 O(n) 变为 O(1)。

- 时间复杂度降为 O(n^2)：外层 O(n)，内层 while 最多 O(n)，每次查找 O(1)。

**第二步优化：只从起点开始计数**

上面 O(n^2) 的原因是：很多数字不是序列起点，却还是从它开始往后数了。比如 `[1,2,3,4]`，从 2 开始数会数到 3,4；从 3 开始数会数到 4。大量重复。

关键观察：**如果 x-1 存在于集合中，那么 x 一定不是某个连续序列的起点**（因为 x-1 才是更前面的起点或者更前面的某个数）。

所以我们只需要在 `x-1` 不在集合中时才开始计数。这样，每个连续序列只会从它的真正起点被遍历一次。

**复杂度分析**：虽然有嵌套循环，但每个元素最多被访问两次（一次在外层循环判断是否是起点，一次在内层 while 中被计数），所以总时间是 O(n)。

#### 最优解法

使用哈希集合 + 起点识别：

- **时间复杂度**：O(n)，每个元素最多被处理两次。
- **空间复杂度**：O(n)，存储哈希集合。

### 4. 关键思维点

- **为什么不能排序？** 题目明确要求 O(n) 时间复杂度，排序是 O(n*log(n))，不满足要求。
- **为什么用集合而不是字典？** 这道题只需要判断"某个数是否存在"，不需要存储下标或其他信息，集合足够且更简洁。
- **核心 insight -- "起点识别"**：判断 `x-1` 是否在集合中，如果不在则 x 是起点。这一步将时间从 O(n^2) 降到 O(n)，是整道题最精妙的地方。
- **为什么是 O(n) 而不是 O(n^2)？** 虽然看起来有嵌套循环，但每个数字最多只会被内层 while 循环访问一次（只有起点才会触发 while，且 while 中访问的数字不会再成为起点）。所有 while 循环的迭代次数总和不超过 n。

### 5. 代码实现（Python）

```python
def longestConsecutive(nums):
    # 将所有数字放入哈希集合，去重 + O(1)查找
    num_set = set(nums)

    max_length = 0  # 记录最长连续序列的长度

    for num in num_set:
        # 关键判断：只从序列的起点开始计数
        # 如果 num-1 存在，说明 num 不是起点，跳过
        if num - 1 not in num_set:
            # num 是某个连续序列的起点
            current_num = num
            current_length = 1

            # 从起点开始，不断向后查找连续的数字
            while current_num + 1 in num_set:
                current_num += 1
                current_length += 1

            # 更新最大长度
            max_length = max(max_length, current_length)

    return max_length
```

### 6. 套路总结

| 识别信号 | 解题模板 |
|---------|---------|
| 在无序数组中找连续/相邻关系 | 哈希集合 + 起点识别 |
| 要求 O(n) 时间，不能排序 | 哈希表/集合 |
| 判断某个值是否存在（无需下标） | 哈希集合（set） |

**通用模板**：
```python
def find_longest_consecutive(nums):
    num_set = set(nums)
    max_length = 0

    for num in num_set:
        # 只从起点开始（前驱不存在）
        if num - 1 not in num_set:
            length = 1
            while num + length in num_set:
                length += 1
            max_length = max(max_length, length)

    return max_length
```

**"起点识别"思想的应用场景**：
- 任何需要在无序数据中寻找连续区间/序列的问题
- 需要避免重复遍历的场景
- 核心逻辑：找到每个区间的起始点，只从起点出发探索

## 哈希专题总结

### 何时想到用哈希？

1. **需要快速查找**：当你发现解题瓶颈在于"查找某个元素是否存在"或"查找某个元素的位置"时，哈希表可以将查找从 O(n) 优化到 O(1)。
2. **需要建立映射**：需要将某个值映射到另一个值（值->下标、值->计数、值->分组）时。
3. **需要分组归类**：把具有相同特征的元素归到一起。
4. **需要去重**：快速判断是否已经见过某个元素。

### 哈希表 vs 哈希集合的选择

| 场景 | 选择 | 原因 |
|------|------|------|
| 需要存储额外信息（如下标） | `dict`（哈希表） | 键值对，值可以存附加信息 |
| 只需要判断存在性 | `set`（哈希集合） | 更简洁，语义更清晰 |
| 需要分组 | `defaultdict(list)` | 自动初始化，方便追加 |
| 需要计数 | `Counter` 或 `defaultdict(int)` | 自动计数 |

### 三道题的递进关系

```
题目1（两数之和）  --> 哈希表的基础用法：存值查值，O(1)配对查找
       |
       v
题目2（字母异位词） --> 哈希表的进阶用法：设计键，按特征分组
       |
       v
```
