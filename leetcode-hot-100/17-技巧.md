# 技巧

这一类题目没有统一的算法框架，但每道题都有一个巧妙的核心思路。掌握这些"技巧"的关键不是死记硬背，而是理解"为什么这个方法能行"。

这类题的特点：
1. **看似简单的问题，但加上约束条件后需要巧妙的算法**
2. **常见约束**：O(1) 额外空间、O(n) 时间、不能修改原数组等
3. **解法往往出人意料**：位运算、数学性质、特殊指针技巧

## 题目1：136. 只出现一次的数字

### 1. 核心考点
- **位运算 -- 异或（XOR）的性质**
- **O(1) 空间的约束下如何消除成对出现的元素**

### 2. 解题思路（给新手讲）

给你一个非空整数数组，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现一次的元素。要求 O(n) 时间、O(1) 额外空间。

如果没有空间限制，用哈希表计数就行。但要求 O(1) 空间，就需要用到**异或运算**。

异或（XOR，符号 `^`）的三个关键性质：
1. **任何数和自己异或等于 0**：`a ^ a = 0`
2. **任何数和 0 异或等于自己**：`a ^ 0 = a`
3. **异或满足交换律和结合律**：顺序无所谓

所以把数组所有元素异或在一起：成对的元素互相抵消变成 0，最后剩下的就是那个只出现一次的数。

比如 `[4, 1, 2, 1, 2]`：
`4 ^ 1 ^ 2 ^ 1 ^ 2 = 4 ^ (1 ^ 1) ^ (2 ^ 2) = 4 ^ 0 ^ 0 = 4`

### 3. 从暴力到优化的推导

#### 暴力解法
用哈希表统计每个数出现的次数，然后找出现 1 次的。

```python
def singleNumber(nums: list[int]) -> int:
    count = {}
    for num in nums:
        count[num] = count.get(num, 0) + 1  # 计数

    for num, c in count.items():
        if c == 1:        # 找只出现一次的
            return num
```

时间 O(n)，空间 O(n)。不满足 O(1) 空间的要求。

#### 优化思路
哈希表用了 O(n) 空间。有没有办法不用额外空间？

排序后相邻比较？O(n log n) 时间，也不够好。

异或的性质完美解决：成对的数异或后消失，剩下的就是答案。

#### 最优解法
把所有数异或在一起，结果就是答案。

### 4. 关键思维点
- **异或是"成对消除"的利器**：看到"其他元素出现两次/偶数次"，立刻想到异或
- **异或的本质**：在二进制层面，每一位独立地做"模 2 加法"，出现偶数次的位都变成 0
- **为什么排序不够好**？排序是 O(n log n)，异或是 O(n)
- **拓展**：如果有两个数只出现一次呢？可以用异或 + 分组（LeetCode 260）

### 5. 代码实现（Python）

```python
def singleNumber(nums: list[int]) -> int:
    result = 0  # 初始值为 0（异或的"单位元"）

    for num in nums:
        result ^= num  # 逐个异或，成对的会抵消

    return result  # 最后剩下的就是只出现一次的数
```

**一行版本**：

```python
from functools import reduce
from operator import xor

def singleNumber(nums: list[int]) -> int:
    return reduce(xor, nums)  # 把所有元素用异或累积
```

### 6. 套路总结
- 看到"出现两次/偶数次的元素" + "O(1) 空间" --> 异或
- 异或三大性质：`a^a=0`、`a^0=a`、交换律结合律
- 异或是位运算中最常用的"消除"工具
- 这道题是位运算入门的经典题，务必熟练掌握

## 题目2：169. 多数元素

### 1. 核心考点
- **Boyer-Moore 投票算法**：O(n) 时间 O(1) 空间找多数元素
- **多数元素的数学性质**：出现次数超过 n/2，意味着它比其他所有元素加起来都多

### 2. 解题思路（给新手讲）

给你一个数组，找出其中出现次数**超过 n/2 次**的元素（保证存在）。

如果没有空间限制，用哈希表计数就行。但最优解法是**Boyer-Moore 投票算法**。

形象理解：想象一场"大逃杀"。每个不同的元素代表一个阵营，规则是"不同阵营的两个人同归于尽"。因为多数元素的数量超过一半，所以最后活下来的一定是多数元素。

算法过程：
1. 维护一个"候选人" `candidate` 和他的"票数" `count`
2. 遍历数组：
   - 如果 `count == 0`，换一个新候选人
   - 如果当前元素 == 候选人，票数 +1
   - 如果当前元素 != 候选人，票数 -1（"同归于尽"）
3. 最后的候选人就是多数元素

### 3. 从暴力到优化的推导

#### 暴力解法
用哈希表统计每个元素出现的次数。

```python
def majorityElement(nums: list[int]) -> int:
    count = {}
    n = len(nums)

    for num in nums:
        count[num] = count.get(num, 0) + 1
        if count[num] > n // 2:  # 超过一半就是答案
            return num
```

时间 O(n)，空间 O(n)。

#### 优化思路
排序后中间位置的元素一定是多数元素（因为它超过一半）。但排序是 O(n log n)。

投票算法可以做到 O(n) 时间 O(1) 空间。

#### 最优解法
Boyer-Moore 投票算法：利用"超过一半"这个性质，通过"抵消"来筛选。

### 4. 关键思维点
- **为什么投票算法是对的**？多数元素出现次数 > n/2，其他所有元素出现次数之和 < n/2。一对一抵消后，多数元素一定有剩余
- **count 降到 0 时换候选人**：说明前面的候选人已经被抵消光了，但多数元素在剩余部分中仍然是多数（这是个很精妙的不变量）
- **如果不保证存在多数元素**，需要在最后做一次验证（再遍历一遍数组确认候选人的出现次数）
- **排序解法也值得了解**：`sorted(nums)[n // 2]`，简单但时间是 O(n log n)

### 5. 代码实现（Python）

```python
def majorityElement(nums: list[int]) -> int:
    candidate = None  # 当前候选人
    count = 0         # 候选人的票数

    for num in nums:
        if count == 0:
            candidate = num  # 票数为 0，换新候选人
        if num == candidate:
            count += 1   # 相同，票数 +1
        else:
            count -= 1   # 不同，票数 -1（抵消）

    return candidate  # 最后的候选人就是多数元素
```

### 6. 套路总结
- 看到"出现次数超过 n/2" + "O(1) 空间" --> Boyer-Moore 投票
- 核心思想：不同元素互相抵消，多数元素一定有剩余
- 投票算法的变体可以找出出现次数超过 n/3 的元素（LeetCode 229），维护两个候选人

## 题目3：75. 颜色分类

### 1. 核心考点
- **三路划分 / 荷兰国旗问题**
- **双指针（实际是三指针）**
- **原地排序，只遍历一次**

### 2. 解题思路（给新手讲）

给你一个数组，里面只有 0、1、2 三种值，要求**原地**排序，使得 0 在前、1 在中、2 在后。要求**一趟扫描**完成。

这就是著名的**荷兰国旗问题**（Dutch National Flag Problem），由 Dijkstra 提出。

思路：用三个指针把数组分成四个区域：
```
[0, low)     -- 全是 0
[low, mid)   -- 全是 1
[mid, high]  -- 未处理
(high, n-1]  -- 全是 2
```

`mid` 指针扫描未处理区域，根据 `nums[mid]` 的值：
- 如果是 0：和 `low` 位置交换，`low` 和 `mid` 都右移
- 如果是 1：`mid` 直接右移（1 本来就在中间）
- 如果是 2：和 `high` 位置交换，`high` 左移（注意 `mid` 不动，因为换过来的值还没检查过）

### 3. 从暴力到优化的推导

#### 暴力解法
直接调用排序函数，或者两趟扫描（第一趟统计 0、1、2 的个数，第二趟覆盖写入）。

```python
def sortColors(nums: list[int]) -> None:
    # 两趟扫描法
    count0 = nums.count(0)  # 统计 0 的个数
    count1 = nums.count(1)  # 统计 1 的个数
    count2 = nums.count(2)  # 统计 2 的个数

    idx = 0
    for _ in range(count0):  # 先写 0
        nums[idx] = 0
        idx += 1
    for _ in range(count1):  # 再写 1
        nums[idx] = 1
        idx += 1
    for _ in range(count2):  # 最后写 2
        nums[idx] = 2
        idx += 1
```

这需要两趟扫描，能不能一趟搞定？

#### 优化思路
一趟扫描就搞定 = 在遍历的过程中，同时把 0 甩到左边、把 2 甩到右边、1 留在中间。需要三个指针来维护三个区域的边界。

#### 最优解法
荷兰国旗算法：三个指针 `low`、`mid`、`high` 协作完成。

### 4. 关键思维点
- **为什么遇到 2 交换后 `mid` 不前进**？因为从 `high` 换过来的值可能是 0、1 或 2，需要再检查一次。而遇到 0 交换后 `mid` 可以前进，因为从 `low` 换过来的一定是 1（`low` 到 `mid` 之间全是 1）
- **循环条件是 `mid <= high`**：`mid` 超过 `high` 时，所有元素都处理完了
- **不变量维护**：在任意时刻，`[0, low)` 全是 0，`[low, mid)` 全是 1，`(high, n-1]` 全是 2

### 5. 代码实现（Python）

```python
def sortColors(nums: list[int]) -> None:
    low = 0            # low 左边全是 0
    mid = 0            # mid 是当前正在检查的位置
    high = len(nums) - 1  # high 右边全是 2

    while mid <= high:
        if nums[mid] == 0:
            # 当前是 0，应该放到左边
            nums[low], nums[mid] = nums[mid], nums[low]
            low += 1   # 0 区域右扩
            mid += 1   # 换过来的一定是 1（因为 low <= mid），继续前进
        elif nums[mid] == 1:
            # 当前是 1，本来就应该在中间，直接跳过
            mid += 1
        else:  # nums[mid] == 2
            # 当前是 2，应该放到右边
            nums[mid], nums[high] = nums[high], nums[mid]
            high -= 1  # 2 区域左扩
            # 注意：mid 不动！因为换过来的值还没检查过
```

### 6. 套路总结
- **三路划分**是一种通用技巧，不只用于 0/1/2 排序，快排中的 partition 也用类似思想
- 核心：用指针维护区域边界，把元素"甩"到正确的区域
- 关键细节：和右边交换后不能前进（换过来的值未知），和左边交换后可以前进（换过来的值已知）
- 这个算法是快速排序优化版（三路快排）的核心组件

## 题目4：31. 下一个排列

### 1. 核心考点
- **排列的字典序**
- **贪心 + 逆序查找**
- **对数组的精细操作**

### 2. 解题思路（给新手讲）

给你一个整数数组，找出这个数组在所有排列中**字典序恰好比它大的下一个排列**。如果已经是最大排列（完全降序），则返回最小排列（完全升序）。

比如 `[1, 2, 3]` 的下一个排列是 `[1, 3, 2]`。

怎么找"刚好大一点"的排列？类似于数字加 1 的进位思想：

1. **从右往左，找到第一个"下降"的位置**：即 `nums[i] < nums[i+1]` 的位置 i。这个位置右边是降序排列的（已经是那部分的最大排列了）

2. **在右边部分找到"刚好比 `nums[i]` 大"的数**：从右往左找第一个比 `nums[i]` 大的数 `nums[j]`

3. **交换 `nums[i]` 和 `nums[j]`**

4. **把 i 右边的部分反转**（从降序变成升序），让右边部分变成最小排列

### 3. 从暴力到优化的推导

#### 暴力解法
生成所有排列，排序后找到当前排列的下一个。时间 O(n!)，完全不可行。

```python
from itertools import permutations

def nextPermutation(nums: list[int]) -> None:
    all_perms = sorted(set(permutations(nums)))  # 所有排列排序
    idx = all_perms.index(tuple(nums))            # 找到当前位置
    next_perm = all_perms[(idx + 1) % len(all_perms)]  # 下一个
    nums[:] = list(next_perm)                     # 原地修改
```

#### 优化思路
不需要枚举所有排列。找"下一个排列"有固定的规律：
- 从右往左看，如果一直是升序（如 `5, 4, 3`），说明这部分已经是最大排列，需要往左边找一个"可以进位"的位置
- 找到那个位置后，用右边最小的比它大的数替换它，然后把右边排成最小序列

这就像十进制加 1：从个位开始，如果是 9 就进位，找到第一个不是 9 的位置加 1，后面的位全变成 0。

#### 最优解法
四步走：找下降点 -> 找替换点 -> 交换 -> 反转后缀。O(n) 时间，O(1) 空间。

### 4. 关键思维点
- **为什么从右往左找"下降点"**？右边如果是降序，说明这部分已经是最大排列了，无法再增大。需要在更高位做改变
- **为什么交换后要反转**？交换后右边部分仍然是降序（因为我们选的是"刚好比 `nums[i]` 大"的数，交换不会改变降序性质），反转变成升序就是最小排列
- **边界情况**：如果整个数组就是降序的（已经是最大排列），找不到下降点，直接反转整个数组变成最小排列
- **这个算法就是 C++ STL 中 `next_permutation` 的实现**

### 5. 代码实现（Python）

```python
def nextPermutation(nums: list[int]) -> None:
    n = len(nums)

    # 第一步：从右往左，找第一个 nums[i] < nums[i+1] 的位置
    # 即找到第一个"下降"的位置
    i = n - 2
    while i >= 0 and nums[i] >= nums[i + 1]:
        i -= 1
    # 循环结束后，i 右边（i+1 到 n-1）是降序的
    # 如果 i == -1，说明整个数组是降序的

    if i >= 0:
        # 第二步：从右往左，找第一个比 nums[i] 大的数 nums[j]
        j = n - 1
        while nums[j] <= nums[i]:
            j -= 1
        # nums[j] 是右边部分中最小的比 nums[i] 大的数

        # 第三步：交换 nums[i] 和 nums[j]
        nums[i], nums[j] = nums[j], nums[i]

    # 第四步：反转 i+1 到末尾的部分（从降序变成升序）
    # 如果 i == -1，就是反转整个数组
    left, right = i + 1, n - 1
    while left < right:
        nums[left], nums[right] = nums[right], nums[left]
        left += 1
        right -= 1
```

**详细示例**：以 `[1, 5, 8, 4, 7, 6, 5, 3, 1]` 为例

```
原数组:    [1, 5, 8, 4, 7, 6, 5, 3, 1]
                        ^
第一步: 找到 i=4 (nums[4]=7 > nums[5]=6, 继续)
        找到 i=3 (nums[3]=4 < nums[4]=7, 停！) -> i = 3, nums[i] = 4

第二步: 从右往左找比 4 大的第一个数
        nums[8]=1, nums[7]=3, nums[6]=5 > 4, 停！ -> j = 6

第三步: 交换 nums[3] 和 nums[6]
        [1, 5, 8, 5, 7, 6, 4, 3, 1]

第四步: 反转 i+1=4 到末尾
        [1, 5, 8, 5, 1, 3, 4, 6, 7]
        这就是下一个排列！
```

### 6. 套路总结
- **下一个排列**是一道固定套路题，核心是"从右找下降点 + 交换 + 反转"
- 思路类比十进制加法的进位
- 四步流程要记牢：找下降点 -> 找替换点 -> 交换 -> 反转后缀
- 理解"为什么交换后右边仍然是降序"是关键

## 题目5：287. 寻找重复数

### 1. 核心考点
- **Floyd 判圈算法（快慢指针找环入口）**
- **抽屉原理**
- **将数组问题转化为链表问题**

### 2. 解题思路（给新手讲）

给你一个长度为 n+1 的数组，里面的数字都在 1 到 n 之间，所以至少有一个数字是重复的。找出那个重复的数字。

约束：不能修改数组，只能用 O(1) 额外空间。

**核心转化**：把数组看成一个"隐式链表"。

怎么理解？数组的下标是 0 到 n，值是 1 到 n。对于每个位置 `i`，`nums[i]` 可以看作"下一个位置"（即从 i 跳到 nums[i]）。

- 因为值的范围是 1 到 n，从位置 0 出发永远不会回到 0
- 因为有重复数字，必然有两个不同的位置指向同一个位置 --> 形成了**环**
- **环的入口就是重复的数字**

这样就把问题转化成了"链表找环入口"，用 Floyd 快慢指针即可。

### 3. 从暴力到优化的推导

#### 暴力解法

方法一：排序后找相邻重复（但会修改数组）。

```python
def findDuplicate(nums: list[int]) -> int:
    nums.sort()
    for i in range(1, len(nums)):
        if nums[i] == nums[i - 1]:
            return nums[i]
```

方法二：哈希集合（但用了 O(n) 空间）。

```python
def findDuplicate(nums: list[int]) -> int:
    seen = set()
    for num in nums:
        if num in seen:
            return num
        seen.add(num)
```

两种方法都违反了约束条件。

#### 优化思路
不能排序（修改数组），不能用哈希表（O(n) 空间）。

二分法可以做到 O(n log n) 时间 O(1) 空间（在值域上二分），但有更优的 O(n) 解法。

关键洞察：把数组看作链表，有重复数字 = 链表有环。用 Floyd 判圈算法（龟兔赛跑）找环入口。

#### 最优解法
Floyd 判圈算法，两个阶段：

**阶段一：快慢指针找到相遇点**
- 慢指针每次走一步：`slow = nums[slow]`
- 快指针每次走两步：`fast = nums[nums[fast]]`
- 因为有环，快指针一定会追上慢指针

**阶段二：找到环入口**
- 让一个指针从起点出发，另一个从相遇点出发
- 都每次走一步，它们相遇的位置就是环入口 = 重复数字

### 4. 关键思维点
- **为什么 `nums[i]` 构成链表**？每个位置 i 指向 `nums[i]`，这就是链表的"next 指针"
- **为什么有环**？抽屉原理：n+1 个数放在 1~n 的范围内，必有重复。重复的数字 x 被至少两个位置指向，形成了"多对一"的汇合点
- **为什么环入口就是重复数字**？假设重复数字是 x，那么至少有两个不同的位置 a 和 b 满足 `nums[a] = nums[b] = x`，这意味着从两条路径都可以走到 x，x 就是环的入口
- **Floyd 算法的数学证明**：设起点到环入口距离为 a，环入口到相遇点距离为 b，环长为 c。快指针走 2(a+b)，慢指针走 a+b，差值 a+b 是 c 的倍数。从起点走 a 步 = 从相遇点走 c-b 步 = 环入口

### 5. 代码实现（Python）

```python
def findDuplicate(nums: list[int]) -> int:
    # 阶段一：快慢指针找到环中的相遇点
    slow = nums[0]           # 慢指针，每次走一步
    fast = nums[nums[0]]     # 快指针，每次走两步

    while slow != fast:
        slow = nums[slow]           # 走一步
        fast = nums[nums[fast]]     # 走两步

    # 此时 slow 和 fast 在环内的某个点相遇

    # 阶段二：找环入口（即重复数字）
    # 一个指针从起点出发，另一个从相遇点出发，都每次走一步
    finder = 0  # 从起点出发

    while finder != slow:
        finder = nums[finder]  # 从起点走一步
        slow = nums[slow]      # 从相遇点走一步

    # 两指针相遇的位置就是环入口 = 重复数字
    return finder
```

**二分法解法（也满足约束，但时间 O(n log n)）**：

```python
def findDuplicate(nums: list[int]) -> int:
    # 在值域 [1, n] 上二分
    # 如果 <= mid 的数字个数 > mid，说明重复数在 [1, mid] 中
    lo, hi = 1, len(nums) - 1

    while lo < hi:
        mid = (lo + hi) // 2
        # 统计数组中 <= mid 的元素个数
        count = sum(1 for num in nums if num <= mid)

        if count > mid:
            hi = mid      # 重复数在左半部分
        else:
            lo = mid + 1  # 重复数在右半部分

    return lo
```

### 6. 套路总结
- **数组映射成链表**是一种强大的转化技巧：下标 -> 值 = 节点 -> next
- 有重复 = 有环，重复值 = 环入口
- Floyd 判圈法是找链表环入口的标准方法（LeetCode 142 的直接应用）
- 二分法在值域上二分（而非下标上），利用抽屉原理

## 技巧题总结对比

| 题目 | 核心技巧 | 关键约束 | 为什么暴力不行 |
|------|----------|----------|----------------|
| 只出现一次的数字 | 异或 | O(1) 空间 | 哈希表需要 O(n) 空间 |
| 多数元素 | Boyer-Moore 投票 | O(1) 空间 | 哈希表/排序不够优 |
| 颜色分类 | 三路划分（荷兰国旗） | 一趟扫描、原地 | 两趟扫描可以但不够优 |
| 下一个排列 | 贪心 + 逆序查找 | O(1) 空间、原地 | 枚举排列 O(n!) |
| 寻找重复数 | Floyd 判圈 | 不修改数组、O(1) 空间 | 排序/哈希都违反约束 |

**通用思维**：
1. 先想暴力解法
2. 看约束条件排除哪些做法
3. 利用数据的特殊性质（异或抵消、超过一半、值域有限、数组映射链表）
4. 这些技巧往往不是"推导"出来的，而是"知道就知道"的经典方法，重点在于理解原理
