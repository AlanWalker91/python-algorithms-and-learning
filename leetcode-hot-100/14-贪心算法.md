# 贪心算法

贪心算法的核心思想是：**每一步都做出当前看起来最优的选择，从而希望最终结果也是全局最优的。** 贪心不回头、不反悔，做了选择就一路走到底。它像一个"目光短浅但运气很好"的策略——之所以能成功，是因为问题本身具有**贪心选择性质**（局部最优能推导出全局最优）。

贪心算法没有固定的代码模板，关键在于**发现贪心策略**并**证明其正确性**。

## 题目1：121. 买卖股票的最佳时机

### 1. 核心考点

- **贪心思想**：维护历史最低价，在每一天计算"如果今天卖出能赚多少"
- **一次遍历求最大差值**：在数组中找到 `max(prices[j] - prices[i])`，其中 `j > i`
- 这道题是贪心算法的入门经典题，也可以用动态规划来理解

### 2. 解题思路（给新手讲）

想象你穿越了，知道了未来每天的股价，但规则是：**只能买卖一次**（先买后卖），问你最多能赚多少钱。

比如股价是 `[7, 1, 5, 3, 6, 4]`：
- 第2天买入（价格1），第5天卖出（价格6），利润 = 6 - 1 = 5

**贪心的想法特别自然**：
- 你从左到右看每一天的股价
- 你随时记住"到目前为止最便宜的那天是多少钱"（因为你肯定想在最便宜的时候买入）
- 然后算"如果今天卖掉，能赚多少"
- 所有天里取最大利润就行了

就像逛超市：你一直记着看到过的最低价，每看到一个新价格就想"现在卖能赚多少"，最后取最大的那个。

### 3. 从暴力到优化的推导

#### 暴力解法

双重循环，枚举所有的买入日和卖出日的组合：

```python
def maxProfit(prices):
    n = len(prices)
    max_profit = 0
    for i in range(n):                          # 枚举买入日
        for j in range(i + 1, n):               # 枚举卖出日（必须在买入之后）
            profit = prices[j] - prices[i]      # 计算利润
            max_profit = max(max_profit, profit) # 更新最大利润
    return max_profit
```

时间复杂度 O(n^2)，当 n 很大时会超时。

#### 优化思路

暴力解法中，对于每一个卖出日 `j`，我们都去遍历它前面的所有买入日 `i` 找最小值。但其实我们不需要每次重新找——**只需要一边遍历，一边维护前面的最小值**。

关键洞察：**当你站在第 j 天考虑卖出时，最优的买入日一定是前面价格最低的那一天。**

所以我们只需要一个变量 `min_price` 记录历史最低价，遍历到每一天时：
1. 更新 `min_price`（看看今天是不是更便宜）
2. 计算今天卖出的利润（`prices[j] - min_price`）
3. 更新最大利润

#### 最优解法

一次遍历，O(n) 时间，O(1) 空间：

```python
def maxProfit(prices):
    min_price = float('inf')    # 初始化为正无穷，任何价格都比它小
    max_profit = 0              # 初始利润为0（最差情况就是不买不卖）
    for price in prices:
        min_price = min(min_price, price)           # 更新历史最低价
        max_profit = max(max_profit, price - min_price)  # 更新最大利润
    return max_profit
```

### 4. 关键思维点

- **为什么初始化 `min_price = inf`？** 这样第一天的价格一定会更新它，不需要特殊处理第一天
- **为什么初始化 `max_profit = 0`？** 如果股价一直在跌，不买不卖利润为0，这就是最优解（题目允许不交易）
- **贪心在哪里？** 每一天都"贪心地"假设自己在历史最低点买入了，然后计算当天卖出的利润。这个局部最优决策能推出全局最优
- **为什么先更新 `min_price` 再算利润？** 顺序其实无所谓，因为 `price - price = 0`，不影响 `max_profit`（它初始就是0）

### 5. 代码实现（Python）

```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        min_price = float('inf')    # 记录遍历过程中的最低股价
        max_profit = 0              # 记录最大利润，初始为0（不交易）

        for price in prices:        # 从第1天到最后一天，逐天遍历
            # 贪心第一步：更新历史最低买入价
            min_price = min(min_price, price)
            # 贪心第二步：假设今天卖出，看看利润是否创新高
            max_profit = max(max_profit, price - min_price)

        return max_profit           # 返回整个过程中的最大利润
```

### 6. 套路总结

**识别特征**：求"最大差值"类问题，且有前后顺序约束（先买后卖）。

**通用模板**：维护一个"前缀最值"（本题是前缀最小值），然后遍历时用当前值和前缀最值做计算。

**变体延伸**：
- 如果允许多次买卖 → LeetCode 122（累加所有上涨段）
- 如果限制最多 k 次交易 → 需要动态规划
- 如果有冷冻期或手续费 → 状态机 DP

## 题目2：55. 跳跃游戏

### 1. 核心考点

- **贪心思想**：维护"当前能到达的最远位置"，逐步扩展可达范围
- **可达性分析**：不需要求具体路径，只需要判断终点是否可达
- 理解贪心中"最远覆盖范围"这一经典模型

### 2. 解题思路（给新手讲）

想象你站在一排石头上，每块石头上写了一个数字，代表你站在这块石头上**最多能往前跳几步**。你从第一块石头出发，问：你能不能跳到最后一块石头？

比如 `[2, 3, 1, 1, 4]`：
- 站在第0块（值为2），你最远能跳到第2块
- 站在第1块（值为3），你最远能跳到第4块 —— 已经到终点了！
- 答案：能到达，返回 True

比如 `[3, 2, 1, 0, 4]`：
- 站在第0块（值为3），最远到第3块
- 站在第1块（值为2），最远到第3块
- 站在第2块（值为1），最远到第3块
- 站在第3块（值为0），跳不了了，被"困住"了！
- 答案：不能到达，返回 False

**贪心策略**：我们不关心具体跳到哪里，只关心"**我最远能到哪里**"。遍历每个位置，不断更新能到达的最远位置。如果某个位置超出了最远可达范围，说明到不了了。

就像你在探索迷雾地图：你从左往右走，每走一步就"解锁"一片新区域。只要你当前站的地方在已解锁的区域内，你就能继续探索。如果某个位置还没被解锁，你就走不到那里。

### 3. 从暴力到优化的推导

#### 暴力解法

用 BFS/DFS 搜索所有可能的跳法，看能不能到达终点：

```python
def canJump(nums):
    n = len(nums)
    visited = [False] * n
    visited[0] = True
    queue = [0]                         # BFS 队列
    while queue:
        pos = queue.pop(0)
        if pos == n - 1:                # 到达终点
            return True
        for step in range(1, nums[pos] + 1):  # 尝试跳 1~nums[pos] 步
            next_pos = pos + step
            if next_pos < n and not visited[next_pos]:
                visited[next_pos] = True
                queue.append(next_pos)
    return False
```

时间复杂度最坏 O(n^2)（每个位置都可能跳到后面很多位置），空间复杂度 O(n)。

#### 优化思路

其实我们不需要真的模拟跳跃过程。**关键洞察**是：

> 如果我能到达位置 i，那么从位置 i 出发，我能到达 `i + nums[i]` 以内的所有位置。

所以我们只需要维护一个变量 `max_reach`，表示"当前能到达的最远位置"：
- 遍历每个位置 i，如果 `i <= max_reach`（说明 i 是可达的），就更新 `max_reach = max(max_reach, i + nums[i])`
- 如果 `max_reach >= n - 1`，直接返回 True
- 如果 `i > max_reach`，说明 i 不可达，返回 False

#### 最优解法

一次遍历，O(n) 时间，O(1) 空间。

### 4. 关键思维点

- **为什么不需要模拟跳跃？** 因为我们只关心"能不能到达终点"，不关心具体怎么跳的。最远可达范围已经包含了所有可能的跳法信息
- **贪心在哪里？** 每到一个可达位置，我们都"贪心地"把可达范围扩到最远。不需要回头，不需要考虑"要不要少跳一点"
- **为什么这个贪心是正确的？** 因为跳得远一定不比跳得近差——多跳几步不会有惩罚，所以最远覆盖一定是最优策略
- **遍历的终止条件**：要么 `max_reach >= n - 1`（成功），要么遍历到某个位置 `i > max_reach`（失败）

### 5. 代码实现（Python）

```python
class Solution:
    def canJump(self, nums: List[int]) -> bool:
        max_reach = 0                       # 当前能到达的最远位置

        for i in range(len(nums)):          # 遍历每个位置
            if i > max_reach:               # 如果当前位置不可达
                return False                # 直接返回 False（后面更到不了）
            # 贪心：从位置 i 能跳到的最远位置是 i + nums[i]
            max_reach = max(max_reach, i + nums[i])
            if max_reach >= len(nums) - 1:  # 如果已经能到达终点
                return True                 # 提前返回 True

        return True                         # 遍历完毕，说明每个位置都可达
```

### 6. 套路总结

**识别特征**：判断"能否到达"某个目标，每一步有多种选择，选择范围与当前位置有关。

**通用模板——"最远可达范围"模型**：
```python
max_reach = 0
for i in range(n):
    if i > max_reach:       # 当前位置不可达，直接失败
        return False
    max_reach = max(max_reach, i + nums[i])  # 扩展最远可达范围
return True
```

**核心思想**：不需要枚举所有路径，只需要维护"最远能到哪"。这个"覆盖范围"模型在贪心中非常常用。

## 题目3：45. 跳跃游戏 II

### 1. 核心考点

- **贪心思想**：在每一跳的覆盖范围内，选择能跳到最远的位置作为下一跳的起点
- **BFS 层次遍历的思想**：把"同一跳能到达的所有位置"看作一层
- 从"能不能到"升级到"最少跳几次"，是跳跃游戏的进阶版

### 2. 解题思路（给新手讲）

还是上一题的石头阵，但这次**题目保证一定能到达终点**，问你：**最少跳几次？**

比如 `[2, 3, 1, 1, 4]`：
- 第一跳：从位置0出发，最远到位置2。但我们想想这一跳的范围内（位置1和2），谁的"潜力"最大？
  - 位置1：值为3，如果跳到这里，下一跳最远能到 1+3=4
  - 位置2：值为1，如果跳到这里，下一跳最远能到 2+1=3
  - 位置1更有潜力！
- 第二跳：从位置1出发，一步就能跳到位置4（终点）
- 答案：2次

**贪心策略**：不要急着跳，先看看这一步能到的范围内，谁能让下一步跳得最远，然后再跳。

**生活类比**：你要从城市A到城市Z，每次只能坐一趟公交。你不是随便上一辆车，而是看看当前站能坐的所有公交车，**选那辆能把你送到最远的车**。这样总的换乘次数就最少。

更准确地说，我们可以用 **BFS 的分层思想** 来理解：
- 第0跳：只在位置0（起点）
- 第1跳：能从位置0到达的所有位置
- 第2跳：从第1跳的所有位置出发能到达的所有新位置
- ...
- 当某一层包含终点时，层数就是最少跳跃次数

### 3. 从暴力到优化的推导

#### 暴力解法

BFS，把每个位置当作图的节点，从起点做 BFS 找到终点的最短距离：

```python
def jump(nums):
    n = len(nums)
    if n <= 1:
        return 0
    visited = [False] * n
    visited[0] = True
    queue = [(0, 0)]                            # (位置, 跳数)
    while queue:
        pos, steps = queue.pop(0)
        for step in range(1, nums[pos] + 1):    # 尝试跳 1~nums[pos] 步
            next_pos = pos + step
            if next_pos >= n - 1:               # 到达终点
                return steps + 1
            if not visited[next_pos]:
                visited[next_pos] = True
                queue.append((next_pos, steps + 1))
    return -1
```

时间复杂度 O(n^2)，空间复杂度 O(n)。

#### 优化思路

BFS 的核心是"分层"，但其实我们不需要真的维护一个队列。观察发现：

> 每一"层"（每一跳能到的范围）是一个**连续的区间**。

因为位置是连续的、从左到右排列的，所以第 k 跳能到的位置一定是一个区间 `[left, right]`。我们只需要两个指针来表示当前层的边界：

- `end`：当前这一跳的最远边界
- `farthest`：在当前这一跳范围内，下一跳能到的最远位置

当遍历到 `end` 时，说明当前这一跳结束了，跳数+1，然后把 `end` 更新为 `farthest`。

#### 最优解法

一次遍历，O(n) 时间，O(1) 空间。

### 4. 关键思维点

- **为什么这是贪心？** 每一跳都选择"让下一跳能到最远"的方案，即"跳跃范围内的最大潜力"
- **`end` 变量的含义**：当前这一跳能到达的最远位置（类似 BFS 的层边界）
- **`farthest` 变量的含义**：遍历当前层时，不断更新"下一层能到的最远位置"
- **什么时候 `jumps++`？** 当 `i == end` 时，说明当前层已经遍历完了，必须再跳一次
- **为什么不遍历最后一个位置？** 遍历到 `n-2` 即可。因为如果在 `n-2` 位置时 `i == end`，说明还需要一跳才能到终点；如果遍历到 `n-1`，可能会多算一跳
- **和上一题的关系**：上一题是判断"能不能到"，本题在此基础上加了层次计数

### 5. 代码实现（Python）

```python
class Solution:
    def jump(self, nums: List[int]) -> int:
        n = len(nums)
        jumps = 0           # 跳跃次数
        end = 0             # 当前这一跳能到达的最远边界
        farthest = 0        # 在当前跳范围内，下一跳能到的最远位置

        # 注意：只遍历到 n-2，不需要遍历最后一个位置
        for i in range(n - 1):
            # 更新从位置 i 出发能到达的最远位置
            farthest = max(farthest, i + nums[i])

            # 当遍历到当前跳的边界时，必须再跳一次
            if i == end:
                jumps += 1          # 跳跃次数 +1
                end = farthest      # 更新边界为下一跳的最远位置

                # 优化：如果已经能到达终点，提前退出
                if end >= n - 1:
                    break

        return jumps
```

### 6. 套路总结

**识别特征**：在"覆盖范围"模型上求**最少步数/次数**，本质上是贪心版的 BFS。

**通用模板——"分层贪心跳跃"模型**：
```python
jumps = 0
end = 0             # 当前层的边界
farthest = 0        # 下一层的最远边界
for i in range(n - 1):
    farthest = max(farthest, i + nums[i])
    if i == end:
        jumps += 1
        end = farthest
return jumps
```

**本质**：每一跳选择能到最远的方案，类似 BFS 的层次遍历，但用贪心避免了显式建队列。

## 题目4：763. 划分字母区间

### 1. 核心考点

- **贪心思想**：每个片段必须包含其中字母的所有出现，贪心地选择最小的合法片段
- **区间合并的变体**：把每个字母的出现范围看作一个区间，问题转化为合并重叠区间
- **双指针/滑动窗口思想**：维护当前片段的右边界

### 2. 解题思路（给新手讲）

给你一个字符串，要求把它切成尽可能多的片段，使得**每个字母只出现在一个片段中**。

比如 `"ababcbacadefegdehijhklij"`：
- 字母 a 出现在位置 0,2,4,7,8 → 所以包含 a 的片段至少要覆盖位置 0~8
- 字母 b 出现在位置 1,3,5 → 包含 b 的片段至少覆盖 1~5
- 因为 a 和 b 的范围有重叠（都在 0~8 内），所以它们必须在同一个片段里
- 以此类推，最终切出 `"ababcbaca"`, `"defegde"`, `"hijhklij"` 三个片段

**生活类比**：想象你在整理一堆文件，每份文件属于不同的项目。规则是：同一个项目的文件必须放在同一个文件夹里。你要把文件按顺序分成尽可能多的文件夹，但不能打乱原始顺序。

**贪心策略**：
1. 先预处理：记录每个字母最后一次出现的位置
2. 从左到右遍历字符串，维护当前片段的右边界
3. 遇到一个字母，当前片段的右边界至少要延伸到这个字母最后出现的位置
4. 当遍历到当前片段的右边界时，说明这个片段可以"关闭"了——它内部的所有字母都不会在后面出现

### 3. 从暴力到优化的推导

#### 暴力解法

暴力尝试所有可能的切割方案，检查每种方案是否合法（每个字母只在一个片段中）：

```python
def partitionLabels(s):
    n = len(s)
    # 尝试找到第一个最短的合法前缀片段
    for end in range(n):
        # 检查 s[0:end+1] 是否是一个合法片段
        # 即其中所有字母都不在 s[end+1:] 中出现
        prefix_chars = set(s[:end + 1])
        suffix_chars = set(s[end + 1:])
        if not prefix_chars & suffix_chars:     # 没有交集
            # 找到一个合法切割点，递归处理后面的部分
            rest = partitionLabels(s[end + 1:]) if end + 1 < n else []
            return [end + 1] + rest
    return [n]
```

这种方法每次切割都需要检查集合交集，而且是递归的，时间复杂度较高。

#### 优化思路

**关键洞察**：要判断一个片段是否可以"关闭"，我们只需要知道**片段内每个字母最后一次出现的位置**。如果当前遍历位置已经到达了片段内所有字母的最远位置，就说明可以切割了。

所以分两步走：
1. **预处理**：一次遍历记录每个字母的最后出现位置（一个长度26的数组或字典）
2. **贪心遍历**：从左到右扫描字符串，用一个变量 `end` 维护当前片段的右边界。每遇到一个字母，就把 `end` 更新为 `max(end, 该字母的最后出现位置)`。当 `i == end` 时，片段关闭。

#### 最优解法

两次遍历，O(n) 时间，O(1) 空间（字母表大小固定为26）。

### 4. 关键思维点

- **为什么先记录最后出现位置？** 因为只有知道一个字母"最远在哪"，才能判断当前片段是否已经"收纳"了这个字母的所有出现
- **`end` 变量的含义**：当前片段至少要延伸到的位置。每遇到一个新字母，这个边界可能要右移
- **什么时候可以切割？** 当 `i == end` 时，说明从 `start` 到 `i` 的所有字母都不会在 `i` 之后出现，可以安全切割
- **贪心在哪里？** 每次尽可能早地切割（一旦 `i == end` 就立刻切），这样片段数最多
- **为什么贪心是正确的？** 一旦 `i == end`，当前片段已经是包含所有必要字母的最小片段。如果不在这里切，合并到后面只会让片段更少，不会更多
- **和区间合并的关系**：如果把每个字母的 `[首次出现, 最后出现]` 看作一个区间，本题等价于合并所有重叠区间

### 5. 代码实现（Python）

```python
class Solution:
    def partitionLabels(self, s: str) -> List[int]:
        # 第一步：记录每个字母最后一次出现的位置
        last = {}
        for i, ch in enumerate(s):
            last[ch] = i            # 后面的会覆盖前面的，最终记录的就是最后出现位置

        # 第二步：贪心遍历，找到所有切割点
        result = []
        start = 0       # 当前片段的起始位置
        end = 0         # 当前片段的右边界（至少要到这里）

        for i, ch in enumerate(s):
            # 当前字母的最后出现位置可能扩展片段的右边界
            end = max(end, last[ch])

            # 当遍历位置到达片段右边界时，可以切割
            if i == end:
                result.append(end - start + 1)  # 记录片段长度
                start = end + 1                  # 下一个片段的起始位置

        return result
```

### 6. 套路总结

**识别特征**：需要将序列"分段/分组"，约束条件是某些元素必须在同一段中。

**通用模板——"区间合并贪心"模型**：
```python
# 第一步：确定每个元素的"影响范围"（区间右端点）
last = {}
for i, ch in enumerate(s):
    last[ch] = i

# 第二步：贪心扩展和切割
start = end = 0
result = []
for i, ch in enumerate(s):
    end = max(end, last[ch])    # 扩展当前区间
    if i == end:                # 到达区间右端点，可以切割
        result.append(end - start + 1)
        start = end + 1
```

**本质**：每个字母定义了一个必须被包含的区间，当所有区间合并完成（`i == end`），就得到了一个最小合法片段。

# 贪心算法专题总结

## 一、什么是贪心算法？

贪心算法是一种**在每一步选择中都采取当前最优的选择**的策略，期望通过局部最优选择达到全局最优。

**三个关键特征**：
1. **贪心选择性质**：全局最优解可以通过一系列局部最优选择来达到
2. **最优子结构**：问题的最优解包含子问题的最优解
3. **无后效性**：当前的选择不会影响之前的选择（做了决定不反悔）

## 二、贪心算法的常见模型

| 模型 | 典型题目 | 核心策略 |
|------|----------|----------|
| **前缀最值** | 121. 买卖股票 | 维护前缀最小值/最大值，用当前值与之比较 |
| **最远覆盖** | 55. 跳跃游戏 | 维护能到达的最远位置，判断可达性 |
| **分层跳跃** | 45. 跳跃游戏 II | 类似 BFS 分层，每层选最远，统计层数 |
| **区间贪心** | 763. 划分字母区间 | 预处理区间端点，贪心合并或切割 |

## 三、贪心 vs 动态规划

这是面试和学习中最常被问到的问题，两者有相似之处但本质不同：

| 对比维度 | 贪心算法 | 动态规划 |
|----------|----------|----------|
| **决策方式** | 每步做当前最优选择，不回头 | 考虑所有子问题，综合做最优选择 |
| **是否回退** | 不回退，一条路走到底 | 通过状态转移"回顾"之前的结果 |
| **正确性保证** | 需要证明贪心选择性质（局部最优=全局最优） | 只要状态转移方程正确就一定正确 |
| **时间复杂度** | 通常更低（O(n) 或 O(n log n)） | 通常更高（O(n^2) 或 O(n*k)） |
| **空间复杂度** | 通常 O(1) | 通常 O(n) 或 O(n^2) |
| **适用范围** | 窄，需要问题具有贪心选择性质 | 广，几乎所有最优化问题都能用 |
| **思维难度** | 难在"发现贪心策略并证明正确" | 难在"定义状态和推导转移方程" |

**一句话总结**：
> 贪心是"目光短浅但运气好"——每步只看眼前最优，恰好能得到全局最优。动态规划是"深谋远虑"——通过记录所有子问题的结果来做全局最优决策。

**什么时候用贪心，什么时候用 DP？**

- 如果你能证明"当前最优选择不会影响后续选择的最优性"，用贪心
- 如果当前选择会影响后续的可选方案（即有"后效性"），必须用 DP
- 不确定时，先试贪心看看能不能举出反例；如果能举出反例，就用 DP

**以买卖股票为例**：
- 只买卖一次（121题）→ 贪心就够了（维护最低价即可）
- 多次买卖但有冷冻期 → 必须用 DP（今天的决策影响明天能否操作）
- 限制 k 次交易 → 必须用 DP（需要记录"还剩几次交易机会"这个状态）

## 四、贪心算法的解题步骤

1. **分析问题**：看看能否拆分成一系列子选择
2. **设计贪心策略**：想出一个"每步怎么选"的规则
3. **验证正确性**：
   - **举反例法**：试试能不能找到一个贪心策略失败的例子
   - **交换论证法**：假设最优解和贪心解不同，证明"换成贪心的选择不会更差"
4. **编码实现**：贪心代码通常很简洁，一次遍历搞定

## 五、做题口诀

> **看到"最大/最小/最多/最少"，先想贪心再想 DP。**
> **贪心的关键不是写代码，而是"想清楚贪什么"。**
> **如果贪心能举反例，老老实实写 DP。**
