## 题目1：160. 相交链表

### 1. 核心考点

- 双指针的巧妙应用
- 链表遍历与路径长度差的消除

### 2. 解题思路（给新手讲）

想象两个人从不同起点出发，走向同一个终点。路程不一样长，怎么让他们同时到达交点？

```
A:      a1 -> a2 ------\
                         -> c1 -> c2 -> c3
B: b1 -> b2 -> b3 -----/
```

诀窍：**让两个人都走一遍对方的路**。A走完自己的路后去走B的路，B走完自己的路后去走A的路。这样两人走的总路程相同，一定会在交点相遇！

- A走的路程：`a + c + b`
- B走的路程：`b + c + a`
- 两者相等，所以一定同时到达交点。

### 3. 从暴力到优化的推导

#### 暴力解法

对A的每个节点，遍历B看是否存在相同节点。时间复杂度 O(m*n)，空间 O(1)。

```python
def getIntersectionNode(headA, headB):
    pa = headA
    while pa:
        pb = headB
        while pb:
            if pa is pb:
                return pa
            pb = pb.next
        pa = pa.next
    return None
```

#### 优化思路

暴力法太慢，能不能用哈希表记录一条链表的所有节点？时间 O(m+n)，空间 O(m)。

```python
def getIntersectionNode(headA, headB):
    seen = set()
    pa = headA
    while pa:
        seen.add(pa)
        pa = pa.next
    pb = headB
    while pb:
        if pb in seen:
            return pb
        pb = pb.next
    return None
```

#### 最优解法

双指针法，时间 O(m+n)，空间 O(1)。

```python
def getIntersectionNode(headA, headB):
    # pa、pb 分别从两个链表头出发
    pa, pb = headA, headB
    # 当两个指针相遇时停止（包括都走到 None 的情况）
    while pa is not pb:
        # pa 走完 A 链表就去走 B 链表
        pa = pa.next if pa else headB
        # pb 走完 B 链表就去走 A 链表
        pb = pb.next if pb else headA
    # 相遇点就是交点（或者都是 None，表示不相交）
    return pa
```

### 4. 关键思维点

- **消除长度差**是核心思想：两个指针各走一遍对方的路，总路程相等。
- 如果不相交，两个指针最终都会走到 `None`，此时 `pa is pb` 为 `True`（都是 None），循环结束。
- 比较的是 **节点对象本身**（`is`），不是节点的值。

### 5. 代码实现（Python）

```python
class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None

def getIntersectionNode(headA: ListNode, headB: ListNode) -> ListNode:
    pa, pb = headA, headB          # 双指针分别指向两个链表头
    while pa is not pb:            # 两指针未相遇就继续走
        pa = pa.next if pa else headB  # A走完跳到B的头
        pb = pb.next if pb else headA  # B走完跳到A的头
    return pa                      # 返回相遇节点（交点或None）
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 双指针消除差距 | 两个指针交替走对方的路，自动对齐 |
| 适用场景 | 两条链表找交点、对齐起点 |
| 时间/空间 | O(m+n) / O(1) |

## 题目2：206. 反转链表

### 1. 核心考点

- 链表指针翻转的基本功
- 迭代 vs 递归两种写法

### 2. 解题思路（给新手讲）

原来链表是：`1 -> 2 -> 3 -> 4 -> 5 -> None`

反转后变成：`None <- 1 <- 2 <- 3 <- 4 <- 5`

就是把每个箭头反过来。关键问题：**改了 next 指针后，怎么找到下一个节点？** 所以需要提前保存。

画图理解迭代过程：

```
初始:   prev=None,  curr=1 -> 2 -> 3
第1步:  None <- 1   curr=2 -> 3       (prev=1, curr=2)
第2步:  None <- 1 <- 2   curr=3       (prev=2, curr=3)
第3步:  None <- 1 <- 2 <- 3           (prev=3, curr=None)
返回 prev=3，即新头节点
```

### 3. 从暴力到优化的推导

#### 暴力解法

把所有值存到数组，反转数组，重建链表。时间 O(n)，空间 O(n)。

```python
def reverseList(head):
    vals = []
    curr = head
    while curr:
        vals.append(curr.val)
        curr = curr.next
    curr = head
    for val in reversed(vals):
        curr.val = val
        curr = curr.next
    return head
```

#### 优化思路

不需要额外空间，直接原地改指针方向。

#### 最优解法

迭代法，时间 O(n)，空间 O(1)。

```python
def reverseList(head):
    prev, curr = None, head
    while curr:
        nxt = curr.next    # 先保存下一个节点
        curr.next = prev   # 反转指针
        prev = curr        # prev 前进
        curr = nxt         # curr 前进
    return prev
```

### 4. 关键思维点

- **三个指针**：`prev`、`curr`、`nxt`，缺一不可。
- 操作顺序很重要：先保存 `nxt`，再改 `curr.next`，否则链就断了。
- 递归写法的核心：信任递归会帮你反转后面的部分，你只需要处理当前节点。

### 5. 代码实现（Python）

```python
# 迭代法（推荐）
def reverseList(head: ListNode) -> ListNode:
    prev = None                # prev 初始化为 None（新链表的尾部）
    curr = head                # curr 从头开始
    while curr:                # 遍历整条链表
        nxt = curr.next        # 保存下一个节点（防止断链）
        curr.next = prev       # 当前节点指向前一个节点（反转）
        prev = curr            # prev 前进一步
        curr = nxt             # curr 前进一步
    return prev                # prev 就是新的头节点

# 递归法
def reverseList(head: ListNode) -> ListNode:
    if not head or not head.next:   # 空链表或只有一个节点
        return head
    new_head = reverseList(head.next)  # 递归反转后面的部分
    head.next.next = head              # 让下一个节点指向自己
    head.next = None                   # 断开原来的指向
    return new_head                    # 返回新的头节点
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 三指针迭代反转 | prev/curr/nxt 是反转链表的标准模板 |
| 操作顺序 | 保存next -> 反转指针 -> 移动prev -> 移动curr |
| 递归反转 | 信任递归 + 处理当前节点的指向 |

## 题目3：234. 回文链表

### 1. 核心考点

- 快慢指针找中点
- 反转链表（复用206题）
- 链表的原地操作

### 2. 解题思路（给新手讲）

回文就是正着读反着读一样，比如 `1 -> 2 -> 2 -> 1`。

数组判断回文很简单（双指针从两头往中间），但链表只能单向遍历。怎么办？

**核心思路：把后半段反转，然后和前半段逐一比较。**

```
原链表:     1 -> 2 -> 2 -> 1
找中点:     1 -> 2 | 2 -> 1
反转后半:   1 -> 2 | 1 -> 2
逐一比较:   1==1, 2==2  => 回文!
```

### 3. 从暴力到优化的推导

#### 暴力解法

把链表值全存到数组里，用数组判断回文。时间 O(n)，空间 O(n)。

```python
def isPalindrome(head):
    vals = []
    curr = head
    while curr:
        vals.append(curr.val)
        curr = curr.next
    return vals == vals[::-1]
```

#### 优化思路

能不能不用额外空间？可以！找到链表中点，反转后半段，然后比较。

#### 最优解法

快慢指针 + 反转后半段，时间 O(n)，空间 O(1)。

```python
def isPalindrome(head):
    # 1. 快慢指针找中点
    slow, fast = head, head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    # slow 现在在中点（偶数时在后半段开头，奇数时在正中间）

    # 2. 反转后半段
    prev = None
    curr = slow
    while curr:
        nxt = curr.next
        curr.next = prev
        prev = curr
        curr = nxt
    # prev 现在是反转后的后半段头节点

    # 3. 比较前半段和反转后的后半段
    left, right = head, prev
    while right:  # 后半段更短或等长
        if left.val != right.val:
            return False
        left = left.next
        right = right.next
    return True
```

### 4. 关键思维点

- **快慢指针找中点**是链表的高频技巧：快指针走两步，慢指针走一步，快指针到末尾时慢指针刚好在中间。
- 这道题组合了两个子问题：找中点 + 反转链表。
- 奇数长度链表中间节点无所谓，比较时后半段短一些就行。

### 5. 代码实现（Python）

```python
def isPalindrome(head: ListNode) -> bool:
    # 第一步：快慢指针找中点
    slow, fast = head, head
    while fast and fast.next:          # 快指针走两步
        slow = slow.next               # 慢指针走一步
        fast = fast.next.next
    # 此时 slow 指向中间位置

    # 第二步：反转后半段链表
    prev = None
    curr = slow
    while curr:
        nxt = curr.next                # 保存下一个
        curr.next = prev               # 反转指针
        prev = curr                    # 前移
        curr = nxt
    # prev 是反转后的后半段头节点

    # 第三步：前后两段逐一比较
    left, right = head, prev
    while right:                       # 以较短的后半段为准
        if left.val != right.val:      # 值不同则不是回文
            return False
        left = left.next
        right = right.next
    return True                        # 全部匹配，是回文
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 快慢指针找中点 | fast走2步，slow走1步，fast到底时slow在中间 |
| 反转后半段比较 | 链表回文判断的经典做法 |
| 子问题组合 | 找中点 + 反转链表，两个基础技巧的组合 |

## 题目4：141. 环形链表

### 1. 核心考点

- 快慢指针判环
- Floyd 判圈算法

### 2. 解题思路（给新手讲）

想象两个人在操场跑步，一个跑得快一个跑得慢。如果操场是环形的，快的人迟早会追上（套圈）慢的人。如果是直线跑道，快的人会先跑到终点。

```
有环:   1 -> 2 -> 3 -> 4
                  ^         |
                  |         v
                  7 <- 6 <- 5

无环:   1 -> 2 -> 3 -> 4 -> None
```

- 快指针每次走2步，慢指针每次走1步。
- 如果有环，快指针一定会追上慢指针（两者相遇）。
- 如果无环，快指针会先到达 `None`。

### 3. 从暴力到优化的推导

#### 暴力解法

用哈希表记录走过的节点，如果遇到重复的就有环。时间 O(n)，空间 O(n)。

```python
def hasCycle(head):
    seen = set()
    curr = head
    while curr:
        if curr in seen:
            return True
        seen.add(curr)
        curr = curr.next
    return False
```

#### 优化思路

哈希表需要额外空间，能不能 O(1) 空间？用快慢指针！

#### 最优解法

快慢指针法，时间 O(n)，空间 O(1)。

```python
def hasCycle(head):
    slow, fast = head, head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            return True
    return False
```

### 4. 关键思维点

- 快指针比慢指针每轮多走1步，所以在环中一定会追上（不会跳过）。
- 循环条件是 `fast and fast.next`，保证快指针能安全走两步。
- 注意：先移动再判断，不要一开始就判断（两个指针初始都在 head）。

### 5. 代码实现（Python）

```python
def hasCycle(head: ListNode) -> bool:
    slow, fast = head, head        # 两个指针都从头出发
    while fast and fast.next:      # 快指针能走两步才继续
        slow = slow.next           # 慢指针走一步
        fast = fast.next.next      # 快指针走两步
        if slow is fast:           # 相遇了，说明有环
            return True
    return False                   # 快指针到了终点，无环
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 快慢指针判环 | 快走2步慢走1步，相遇则有环 |
| Floyd判圈 | 经典算法，O(1)空间判断环 |
| 循环条件 | `while fast and fast.next` 保证安全 |

## 题目5：142. 环形链表 II

### 1. 核心考点

- Floyd 判圈算法的进阶
- 数学推导找环入口

### 2. 解题思路（给新手讲）

141题只需要判断有没有环，这题要找到环的入口节点。

```
       a        b
head -----> 入口 -----> 相遇点
              ^              |
              |     c        |
              +--------------+
```

**数学推导**（简化版）：
- 设 head 到入口距离为 `a`，入口到相遇点距离为 `b`，相遇点再走回入口距离为 `c`。
- 慢指针走了 `a + b` 步。
- 快指针走了 `a + b + n(b+c)` 步（多绕了 n 圈）。
- 快指针速度是慢指针2倍：`2(a+b) = a + b + n(b+c)`。
- 化简得 `a = c + (n-1)(b+c)`。
- 含义：**从 head 走 a 步 = 从相遇点走 c 步 + 绕几圈**。
- 所以：**一个指针从 head 出发，一个从相遇点出发，都走一步，一定在入口相遇！**

### 3. 从暴力到优化的推导

#### 暴力解法

用哈希表，第一个重复访问的就是入口。时间 O(n)，空间 O(n)。

```python
def detectCycle(head):
    seen = set()
    curr = head
    while curr:
        if curr in seen:
            return curr
        seen.add(curr)
        curr = curr.next
    return None
```

#### 优化思路

快慢指针找相遇点，然后利用数学关系找入口。

#### 最优解法

Floyd 算法完整版，时间 O(n)，空间 O(1)。

```python
def detectCycle(head):
    slow, fast = head, head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:  # 找到相遇点
            # 一个从head走，一个从相遇点走
            ptr = head
            while ptr is not slow:
                ptr = ptr.next
                slow = slow.next
            return ptr  # 相遇处就是环入口
    return None
```

### 4. 关键思维点

- 第一阶段：快慢指针找相遇点（和141题一样）。
- 第二阶段：一个从 head，一个从相遇点，同速前进，相遇处就是入口。
- 数学推导是关键：`a = c + (n-1)(b+c)`，理解这个等式就理解了整个算法。

### 5. 代码实现（Python）

```python
def detectCycle(head: ListNode) -> ListNode:
    slow, fast = head, head
    # 第一阶段：快慢指针找相遇点
    while fast and fast.next:
        slow = slow.next               # 慢指针走一步
        fast = fast.next.next          # 快指针走两步
        if slow is fast:               # 相遇了
            # 第二阶段：找入口
            ptr = head                 # ptr 从头出发
            while ptr is not slow:     # 两个指针同速前进
                ptr = ptr.next
                slow = slow.next
            return ptr                 # 相遇处就是环入口
    return None                        # 无环
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| Floyd完整版 | 第一阶段找相遇点，第二阶段找入口 |
| 数学推导 | a = c + (n-1)(b+c) |
| 两阶段指针 | 相遇后，head和相遇点同速走，交汇即入口 |

## 题目6：21. 合并两个有序链表

### 1. 核心考点

- 链表的基本操作
- 虚拟头节点（dummy node）的使用
- 归并排序的基础

### 2. 解题思路（给新手讲）

就像合并两副已经排好序的扑克牌：每次比较两副牌顶部的牌，小的先放到新的牌堆里。

```
list1:  1 -> 2 -> 4
list2:  1 -> 3 -> 4

合并过程：
比较 1,1 -> 取 1(list1)
比较 2,1 -> 取 1(list2)
比较 2,3 -> 取 2
比较 4,3 -> 取 3
比较 4,4 -> 取 4(list1)
剩下 4(list2) 直接接上

结果:  1 -> 1 -> 2 -> 3 -> 4 -> 4
```

**虚拟头节点**：创建一个 dummy 节点作为新链表的起点，避免处理"头节点是谁"的边界问题。

### 3. 从暴力到优化的推导

#### 暴力解法

把两个链表的值全放到数组里，排序，重建链表。时间 O((m+n)log(m+n))，空间 O(m+n)。

#### 优化思路

既然两个链表已经有序，直接利用这个性质，像拉拉链一样合并。

#### 最优解法

双指针归并，时间 O(m+n)，空间 O(1)。

```python
def mergeTwoLists(list1, list2):
    dummy = ListNode(0)     # 虚拟头节点
    curr = dummy
    while list1 and list2:
        if list1.val <= list2.val:
            curr.next = list1
            list1 = list1.next
        else:
            curr.next = list2
            list2 = list2.next
        curr = curr.next
    curr.next = list1 if list1 else list2  # 剩余部分直接接上
    return dummy.next
```

### 4. 关键思维点

- **dummy 节点**是链表题的常用技巧，省去判断头节点的麻烦。最后返回 `dummy.next`。
- 循环结束后，一定有一个链表还有剩余，直接接上即可（已经有序的）。
- 这个合并逻辑是归并排序的核心步骤。

### 5. 代码实现（Python）

```python
def mergeTwoLists(list1: ListNode, list2: ListNode) -> ListNode:
    dummy = ListNode(0)            # 创建虚拟头节点
    curr = dummy                   # curr 用来构建新链表
    while list1 and list2:         # 两个链表都还有节点
        if list1.val <= list2.val: # 取较小的那个
            curr.next = list1      # 接到新链表后面
            list1 = list1.next     # list1 前进
        else:
            curr.next = list2      # 接到新链表后面
            list2 = list2.next     # list2 前进
        curr = curr.next           # 新链表指针前进
    # 把剩余部分接上（只会有一个链表有剩余）
    curr.next = list1 if list1 else list2
    return dummy.next              # dummy.next 就是合并后的头节点
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| dummy 虚拟头节点 | 避免处理头节点的特殊情况 |
| 双指针归并 | 每次取较小的接到结果链表后面 |
| 剩余直接接上 | 循环结束后把非空的那条接上 |

## 题目7：2. 两数相加

### 1. 核心考点

- 链表遍历与构建
- 进位处理
- 模拟加法运算

### 2. 解题思路（给新手讲）

两个数用链表**逆序**表示，比如 `342` 表示为 `2 -> 4 -> 3`。

直接从头开始加就相当于从个位开始加，非常方便！就像小学竖式加法：

```
  2 -> 4 -> 3     (342)
+ 5 -> 6 -> 4     (465)
= 7 -> 0 -> 8     (807)

竖式：
    3 4 2
+   4 6 5
---------
    8 0 7
```

关键：**别忘了进位**！比如 `5 + 7 = 12`，本位写 `2`，进位 `1`。

### 3. 从暴力到优化的推导

#### 暴力解法

把两个链表转成数字，相加，再转回链表。但数字可能非常大，超出整数范围（虽然 Python 没有整数溢出问题，但思路不够通用）。

#### 优化思路

直接模拟竖式加法，逐位相加。

#### 最优解法

模拟加法，时间 O(max(m,n))，空间 O(max(m,n))。

```python
def addTwoNumbers(l1, l2):
    dummy = ListNode(0)
    curr = dummy
    carry = 0
    while l1 or l2 or carry:
        val = carry
        if l1:
            val += l1.val
            l1 = l1.next
        if l2:
            val += l2.val
            l2 = l2.next
        carry = val // 10
        curr.next = ListNode(val % 10)
        curr = curr.next
    return dummy.next
```

### 4. 关键思维点

- 循环条件是 `l1 or l2 or carry`，三个条件缺一不可。特别是最后的 `carry`——比如 `999 + 1 = 1000`，最后还有一个进位。
- 链表已经是逆序的，直接从头遍历就是从低位到高位。
- 用 `//` 取商得进位，用 `%` 取余得本位数字。

### 5. 代码实现（Python）

```python
def addTwoNumbers(l1: ListNode, l2: ListNode) -> ListNode:
    dummy = ListNode(0)                # 虚拟头节点
    curr = dummy                       # 结果链表的指针
    carry = 0                          # 进位，初始为 0
    while l1 or l2 or carry:           # 还有数字没加完或者还有进位
        val = carry                    # 先加上进位
        if l1:                         # l1 还有节点
            val += l1.val              # 加上 l1 的值
            l1 = l1.next               # l1 前进
        if l2:                         # l2 还有节点
            val += l2.val              # 加上 l2 的值
            l2 = l2.next               # l2 前进
        carry = val // 10              # 计算新的进位
        curr.next = ListNode(val % 10) # 本位数字作为新节点
        curr = curr.next               # 结果链表前进
    return dummy.next                  # 返回结果链表头
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 模拟竖式加法 | 逐位相加 + 进位 |
| 循环条件三合一 | `l1 or l2 or carry`，别忘了最后的进位 |
| dummy + 逐位构建 | 标准的链表构建模板 |

## 题目8：19. 删除链表的倒数第N个结点

### 1. 核心考点

- 快慢指针（间隔 N 步）
- 虚拟头节点处理边界

### 2. 解题思路（给新手讲）

倒数第 N 个，直觉上需要知道链表长度。但有个巧妙的方法：**让快指针先走 N 步，然后快慢一起走，快指针到底时慢指针就在倒数第 N 个位置！**

```
删除倒数第2个节点：
链表:  1 -> 2 -> 3 -> 4 -> 5

快指针先走2步:
slow              fast
  |                |
dummy -> 1 -> 2 -> 3 -> 4 -> 5

然后一起走，直到fast到末尾:
              slow              fast
                |                |
dummy -> 1 -> 2 -> 3 -> 4 -> 5 -> None

slow.next 就是要删的节点(4)，跳过它即可
```

### 3. 从暴力到优化的推导

#### 暴力解法

先遍历一遍算长度 L，然后删第 `L - N + 1` 个节点。需要两次遍历。

```python
def removeNthFromEnd(head, n):
    # 第一遍：算长度
    length = 0
    curr = head
    while curr:
        length += 1
        curr = curr.next
    # 第二遍：找到并删除
    dummy = ListNode(0)
    dummy.next = head
    curr = dummy
    for _ in range(length - n):
        curr = curr.next
    curr.next = curr.next.next
    return dummy.next
```

#### 优化思路

用快慢指针，一次遍历就够了。

#### 最优解法

快慢指针，时间 O(n)，空间 O(1)。

```python
def removeNthFromEnd(head, n):
    dummy = ListNode(0)
    dummy.next = head
    fast = slow = dummy
    # 快指针先走 n+1 步
    for _ in range(n + 1):
        fast = fast.next
    # 快慢一起走
    while fast:
        fast = fast.next
        slow = slow.next
    # 删除 slow 的下一个节点
    slow.next = slow.next.next
    return dummy.next
```

### 4. 关键思维点

- 快指针先走 `n+1` 步（不是 n 步），这样 slow 停在要删节点的**前一个**，方便删除。
- **一定要用 dummy 节点**！如果删的是头节点（比如链表只有一个节点），没有 dummy 会很麻烦。
- 快慢指针间隔固定为 n，利用这个间隔把"倒数"转化为"正数"。

### 5. 代码实现（Python）

```python
def removeNthFromEnd(head: ListNode, n: int) -> ListNode:
    dummy = ListNode(0)            # 虚拟头节点
    dummy.next = head
    fast = slow = dummy            # 都从 dummy 出发
    for _ in range(n + 1):         # 快指针先走 n+1 步
        fast = fast.next
    while fast:                    # 快慢一起走，直到快指针到末尾
        fast = fast.next
        slow = slow.next
    slow.next = slow.next.next     # 跳过（删除）目标节点
    return dummy.next              # 返回真正的头节点
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 快慢指针间隔N | 快先走N步，然后同步走，快到底时慢在倒数第N |
| dummy节点 | 处理删除头节点的边界情况 |
| 先走N+1步 | 让slow停在要删节点的前面，方便操作 |

## 题目9：24. 两两交换链表中的节点

### 1. 核心考点

- 链表指针操作的细节
- 虚拟头节点
- 迭代 vs 递归

### 2. 解题思路（给新手讲）

把相邻的两个节点交换位置：`1 -> 2 -> 3 -> 4` 变成 `2 -> 1 -> 4 -> 3`。

画图是关键！每次处理一对节点：

```
交换前:  prev -> node1 -> node2 -> next_pair
交换后:  prev -> node2 -> node1 -> next_pair

具体指针操作（3步）：
1. prev.next = node2         (prev 指向 node2)
2. node1.next = node2.next   (node1 指向下一对的开头)
3. node2.next = node1        (node2 指向 node1)

然后 prev 移到 node1（因为交换后 node1 在后面）
```

### 3. 从暴力到优化的推导

#### 暴力解法

把链表的值存到数组，两两交换数组的值，再写回链表。但这不是真正的节点交换。

#### 优化思路

直接操作指针，原地完成交换。

#### 最优解法

迭代法，时间 O(n)，空间 O(1)。

```python
def swapPairs(head):
    dummy = ListNode(0)
    dummy.next = head
    prev = dummy
    while prev.next and prev.next.next:
        node1 = prev.next
        node2 = prev.next.next
        # 三步交换
        prev.next = node2
        node1.next = node2.next
        node2.next = node1
        # prev 移到交换后的第二个节点
        prev = node1
    return dummy.next
```

### 4. 关键思维点

- 每次交换需要3步指针操作，顺序不能错。
- `prev` 指向每一对的前一个节点，所以循环条件是 `prev.next and prev.next.next`（确保后面还有一对可以交换）。
- 交换完后，`prev` 移到 `node1`（交换后 node1 在后面，变成了下一对的 prev）。

### 5. 代码实现（Python）

```python
def swapPairs(head: ListNode) -> ListNode:
    dummy = ListNode(0)                # 虚拟头节点
    dummy.next = head
    prev = dummy                       # prev 指向每对节点之前
    while prev.next and prev.next.next:  # 后面还有一对节点
        node1 = prev.next              # 第一个节点
        node2 = prev.next.next         # 第二个节点
        # 三步交换指针
        prev.next = node2              # prev 指向 node2
        node1.next = node2.next        # node1 指向下一对
        node2.next = node1             # node2 指向 node1
        prev = node1                   # prev 前移到 node1（现在在后面）
    return dummy.next                  # 返回新头节点

# 递归法（更简洁）
def swapPairs(head: ListNode) -> ListNode:
    if not head or not head.next:      # 不足两个节点，无需交换
        return head
    node1 = head                       # 第一个节点
    node2 = head.next                  # 第二个节点
    node1.next = swapPairs(node2.next) # node1 指向后面交换好的结果
    node2.next = node1                 # node2 指向 node1
    return node2                       # node2 变成新的头
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 画图明确指针操作 | 交换前后的指向画出来，操作就清晰了 |
| 三步交换法 | prev->node2, node1->next_pair, node2->node1 |
| prev移动到node1 | 交换后node1在后面，变成下一对的prev |

## 题目10：25. K个一组翻转链表

### 1. 核心考点

- 链表分组操作
- 反转链表的灵活运用（指定区间反转）
- 复杂指针操作

### 2. 解题思路（给新手讲）

这是24题（两两交换）的升级版，每 K 个节点一组进行反转。不够 K 个的不翻转。

```
K=3, 链表: 1 -> 2 -> 3 -> 4 -> 5

第一组 [1,2,3] 反转:  3 -> 2 -> 1
第二组 [4,5] 不够K个，保持原样:  4 -> 5

结果: 3 -> 2 -> 1 -> 4 -> 5
```

思路分三步：
1. **检查**剩余节点是否够 K 个。
2. **反转**这 K 个节点。
3. **连接**反转后的组与前后部分。

### 3. 从暴力到优化的推导

#### 暴力解法

用数组存所有值，分组反转数组内的值，重建链表。时间 O(n)，空间 O(n)。

#### 优化思路

直接在链表上操作指针，每次反转一组 K 个节点。

#### 最优解法

分组反转，时间 O(n)，空间 O(1)。

```python
def reverseKGroup(head, k):
    dummy = ListNode(0)
    dummy.next = head
    prev_group_end = dummy

    while True:
        # 检查是否还有 k 个节点
        kth = prev_group_end
        for _ in range(k):
            kth = kth.next
            if not kth:
                return dummy.next  # 不够 k 个，直接返回

        next_group_start = kth.next  # 保存下一组的开头

        # 反转当前组的 k 个节点
        prev, curr = next_group_start, prev_group_end.next
        for _ in range(k):
            nxt = curr.next
            curr.next = prev
            prev = curr
            curr = nxt

        # 连接：prev_group_end 指向反转后的头（kth），更新 prev_group_end
        tmp = prev_group_end.next  # 反转前的头 = 反转后的尾
        prev_group_end.next = kth  # 连接到反转后的头
        prev_group_end = tmp       # 更新为反转后的尾
```

### 4. 关键思维点

- 先**探测**够不够 K 个，不够就不反转，直接返回。
- 反转一组时，`prev` 初始化为 `next_group_start`（下一组的开头），这样反转后自动和后面连上。
- 反转后，原来的组头变成了组尾，原来的组尾（kth）变成了组头。
- `prev_group_end` 是关键变量，始终指向上一组的末尾。

### 5. 代码实现（Python）

```python
def reverseKGroup(head: ListNode, k: int) -> ListNode:
    dummy = ListNode(0)
    dummy.next = head
    prev_group_end = dummy             # 上一组的末尾节点

    while True:
        # 第一步：检查剩余节点是否 >= k 个
        kth = prev_group_end
        for _ in range(k):
            kth = kth.next
            if not kth:                # 不够 k 个
                return dummy.next      # 直接返回结果

        next_group_start = kth.next    # 下一组的起始节点

        # 第二步：反转当前这 k 个节点
        prev = next_group_start        # prev 初始指向下一组开头
        curr = prev_group_end.next     # curr 从当前组的第一个开始
        for _ in range(k):             # 反转 k 个节点
            nxt = curr.next            # 保存下一个
            curr.next = prev           # 反转指针
            prev = curr                # 前移
            curr = nxt                 # 前移

        # 第三步：连接
        tmp = prev_group_end.next      # 当前组反转前的头（现在是尾）
        prev_group_end.next = kth      # 上一组末尾连接到当前组反转后的头
        prev_group_end = tmp           # 更新 prev_group_end 为当前组的尾
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 先探测再反转 | 先检查够不够K个，不够就不动 |
| 分组反转链表 | 24题的推广，每K个一反转 |
| prev初始化技巧 | prev初始化为下一组开头，反转后自动连接 |
| prev_group_end | 始终追踪上一组末尾，用于连接 |

## 题目11：138. 随机链表的复制

### 1. 核心考点

- 深拷贝的概念
- 哈希表映射（原节点 -> 新节点）
- 交织链表法（进阶）

### 2. 解题思路（给新手讲）

每个节点有两个指针：`next` 和 `random`。深拷贝要求创建全新的节点，且 `random` 指针也要正确地指向新链表中对应的节点。

难点在于：复制时，`random` 指向的节点可能还没被创建。

**哈希表法**：先创建所有新节点，用哈希表记录"原节点 -> 新节点"的映射，再设置 `next` 和 `random`。

```
原链表:  A(random->C) -> B(random->A) -> C(random->B)
哈希表:  {A: A', B: B', C: C'}
新链表:  A'(random->C') -> B'(random->A') -> C'(random->B')
```

### 3. 从暴力到优化的推导

#### 暴力解法

暴力法和哈希表法思路一致，这里哈希表法就是标准解法。

#### 优化思路

哈希表用 O(n) 空间。可以用交织链表法优化到 O(1) 空间。

交织法三步：
1. 在每个原节点后面插入一个拷贝节点：`A -> A' -> B -> B' -> C -> C'`
2. 设置拷贝节点的 `random`：`A'.random = A.random.next`
3. 拆分出拷贝链表

#### 最优解法

**哈希表法**（代码简洁，推荐掌握）：

```python
def copyRandomList(head):
    if not head:
        return None
    old_to_new = {}
    # 第一遍：创建所有新节点
    curr = head
    while curr:
        old_to_new[curr] = Node(curr.val)
        curr = curr.next
    # 第二遍：设置 next 和 random
    curr = head
    while curr:
        old_to_new[curr].next = old_to_new.get(curr.next)
        old_to_new[curr].random = old_to_new.get(curr.random)
        curr = curr.next
    return old_to_new[head]
```

### 4. 关键思维点

- 深拷贝的核心问题：**对象引用**。`random` 指向的不是值，而是特定的节点对象。
- 哈希表建立"旧节点到新节点"的映射，这样就能找到 `random` 对应的新节点。
- `old_to_new.get(curr.next)` 用 `get` 方法处理 `None` 的情况。

### 5. 代码实现（Python）

```python
class Node:
    def __init__(self, x, next=None, random=None):
        self.val = int(x)
        self.next = next
        self.random = random

def copyRandomList(head: Node) -> Node:
    if not head:
        return None

    # 哈希表：原节点 -> 新节点
    old_to_new = {}

    # 第一遍遍历：创建所有新节点（只复制值）
    curr = head
    while curr:
        old_to_new[curr] = Node(curr.val)  # 创建对应的新节点
        curr = curr.next

    # 第二遍遍历：连接 next 和 random 指针
    curr = head
    while curr:
        new_node = old_to_new[curr]
        new_node.next = old_to_new.get(curr.next)      # 设置 next
        new_node.random = old_to_new.get(curr.random)   # 设置 random
        curr = curr.next

    return old_to_new[head]  # 返回新链表的头节点
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 哈希表映射 | 原节点 -> 新节点，解决random指针的映射问题 |
| 两遍遍历 | 第一遍创建节点，第二遍连接指针 |
| 深拷贝模板 | 先建映射，再设引用 |

## 题目12：148. 排序链表

### 1. 核心考点

- 链表的归并排序
- 快慢指针找中点
- 合并两个有序链表（复用21题）

### 2. 解题思路（给新手讲）

要求 O(n log n) 时间复杂度排序链表，最适合的就是**归并排序**。

数组的归并排序需要额外空间来合并，但链表合并只需要改指针，天然适合归并排序！

```
归并排序三步：
1. 找中点，分成两半（快慢指针）
2. 递归排序左半和右半
3. 合并两个有序链表（21题）

例子: 4 -> 2 -> 1 -> 3
分割: [4, 2] 和 [1, 3]
递归: [2, 4] 和 [1, 3]
合并: 1 -> 2 -> 3 -> 4
```

### 3. 从暴力到优化的推导

#### 暴力解法

把链表的值存到数组，数组排序，写回链表。时间 O(n log n)，空间 O(n)。

```python
def sortList(head):
    vals = []
    curr = head
    while curr:
        vals.append(curr.val)
        curr = curr.next
    vals.sort()
    curr = head
    for val in vals:
        curr.val = val
        curr = curr.next
    return head
```

#### 优化思路

直接在链表上做归并排序，不需要额外数组。

#### 最优解法

自顶向下归并排序，时间 O(n log n)，空间 O(log n)（递归栈）。

```python
def sortList(head):
    if not head or not head.next:
        return head
    # 快慢指针找中点
    slow, fast = head, head.next
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    mid = slow.next
    slow.next = None  # 断开
    # 递归排序两半
    left = sortList(head)
    right = sortList(mid)
    # 合并
    return merge(left, right)

def merge(l1, l2):
    dummy = ListNode(0)
    curr = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            curr.next = l1
            l1 = l1.next
        else:
            curr.next = l2
            l2 = l2.next
        curr = curr.next
    curr.next = l1 if l1 else l2
    return dummy.next
```

### 4. 关键思维点

- 找中点时，`fast` 初始化为 `head.next`（而不是 `head`），这样对于偶数个节点，`slow` 停在前半段的最后一个节点，方便断开。
- **断开链表**是关键步骤：`slow.next = None`，否则递归会死循环。
- `merge` 函数就是21题的合并两个有序链表，直接复用。

### 5. 代码实现（Python）

```python
def sortList(head: ListNode) -> ListNode:
    # 递归终止：空链表或只有一个节点
    if not head or not head.next:
        return head

    # 第一步：快慢指针找中点
    slow, fast = head, head.next   # fast 从 head.next 开始
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    mid = slow.next                # mid 是后半段的头
    slow.next = None               # 断开前后两段

    # 第二步：递归排序左右两半
    left = sortList(head)          # 排序前半段
    right = sortList(mid)          # 排序后半段

    # 第三步：合并两个有序链表
    return merge(left, right)

def merge(l1: ListNode, l2: ListNode) -> ListNode:
    dummy = ListNode(0)            # 虚拟头节点
    curr = dummy
    while l1 and l2:               # 两个链表都还有节点
        if l1.val <= l2.val:       # 取较小的
            curr.next = l1
            l1 = l1.next
        else:
            curr.next = l2
            l2 = l2.next
        curr = curr.next
    curr.next = l1 if l1 else l2   # 接上剩余部分
    return dummy.next
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 链表归并排序 | 找中点 + 递归 + 合并，三步走 |
| fast从head.next开始 | 确保偶数节点时slow停在前半段末尾 |
| 断开链表 | slow.next = None，否则递归死循环 |
| 复用合并逻辑 | merge 就是21题的解法 |

## 题目13：23. 合并K个升序链表

### 1. 核心考点

- 优先队列（最小堆）
- 分治法
- 21题（合并两个有序链表）的推广

### 2. 解题思路（给新手讲）

有 K 个已排序的链表，要合并成一个大的有序链表。

**方法一：优先队列**

想象你有 K 碗面条，每碗面条从细到粗排好了。你要把它们合成一碗，从细到粗排列。怎么做？每次从每碗里看最上面那根（最细的），把 K 碗中最细的那根挑出来放到新碗里。

用最小堆（优先队列）维护 K 个链表当前的头节点，每次取最小的。

**方法二：分治法**

两两合并，类似归并排序的合并过程。

```
K=4:  [l1, l2, l3, l4]
第一轮:  merge(l1,l2), merge(l3,l4)  -> [m1, m2]
第二轮:  merge(m1, m2)               -> [result]
```

### 3. 从暴力到优化的推导

#### 暴力解法

把所有节点的值放到一个数组，排序，重建链表。时间 O(N log N)，空间 O(N)，其中 N 是所有节点总数。

#### 优化思路

利用每个链表已经有序的性质，用堆或分治来优化。

#### 最优解法

**优先队列法**，时间 O(N log k)，空间 O(k)。

```python
import heapq

def mergeKLists(lists):
    dummy = ListNode(0)
    curr = dummy
    heap = []
    # 把每个链表的头节点入堆
    for i, lst in enumerate(lists):
        if lst:
            heapq.heappush(heap, (lst.val, i, lst))
    while heap:
        val, i, node = heapq.heappop(heap)
        curr.next = node
        curr = curr.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    return dummy.next
```

**分治法**，时间 O(N log k)，空间 O(log k)。

```python
def mergeKLists(lists):
    if not lists:
        return None
    return divide(lists, 0, len(lists) - 1)

def divide(lists, lo, hi):
    if lo == hi:
        return lists[lo]
    mid = (lo + hi) // 2
    left = divide(lists, lo, mid)
    right = divide(lists, mid + 1, hi)
    return merge(left, right)
```

### 4. 关键思维点

- 优先队列法中，堆中最多 K 个元素，每次操作 O(log k)。
- 堆元素用三元组 `(val, i, node)`，其中 `i` 是链表编号，用来避免节点值相同时比较 `ListNode` 对象（Python 不知道怎么比较两个 ListNode）。
- 分治法本质就是重复调用"合并两个有序链表"。

### 5. 代码实现（Python）

```python
import heapq

def mergeKLists(lists: list) -> ListNode:
    dummy = ListNode(0)                # 虚拟头节点
    curr = dummy
    heap = []

    # 初始化：每个链表的头节点入堆
    for i, lst in enumerate(lists):
        if lst:
            # (节点值, 链表编号, 节点对象)
            heapq.heappush(heap, (lst.val, i, lst))

    while heap:
        val, i, node = heapq.heappop(heap)  # 取出最小的
        curr.next = node                     # 接到结果链表
        curr = curr.next
        if node.next:                        # 该链表还有下一个节点
            heapq.heappush(heap, (node.next.val, i, node.next))

    return dummy.next

# 分治法
def mergeKLists(lists: list) -> ListNode:
    if not lists:
        return None
    return divide(lists, 0, len(lists) - 1)

def divide(lists, lo, hi):
    if lo == hi:                       # 只有一个链表
        return lists[lo]
    mid = (lo + hi) // 2
    left = divide(lists, lo, mid)      # 递归处理左半
    right = divide(lists, mid + 1, hi) # 递归处理右半
    return merge(left, right)          # 合并两个有序链表

            l1 = l1.next

            l2 = l2.next
```

| 套路 | 说明 |
|------|------|
| 最小堆维护K个头节点 | 每次取最小的，O(N log k) |
| 堆元素三元组 | (val, index, node) 避免比较节点对象 |
| 分治两两合并 | 重复利用21题的合并逻辑 |

## 题目14：146. LRU缓存

### 1. 核心考点

- 哈希表 + 双向链表的组合数据结构
- LRU（Least Recently Used）淘汰策略
- `get` 和 `put` 操作都要 O(1)

### 2. 解题思路（给新手讲）

LRU 缓存就像一个固定大小的书架：
- 每次**取**一本书（get），把它放到最前面。
- 每次**放**一本新书（put），放到最前面。如果书架满了，把最久没用的那本（最后面的）扔掉。

问题来了：怎么在 O(1) 时间内完成？

- **查找**用哈希表：O(1) 找到节点。
- **排序**（维护使用顺序）用双向链表：O(1) 删除任意节点并移到头部。

```
最近使用 <----> 最久未使用
   HEAD <-> A <-> B <-> C <-> TAIL
   |                           |
   最新                       最旧

get(B): 把B移到最前面
   HEAD <-> B <-> A <-> C <-> TAIL

put(D), 容量满了: 删除TAIL前面的C，插入D到HEAD后面
   HEAD <-> D <-> B <-> A <-> TAIL
```

### 3. 从暴力到优化的推导

#### 暴力解法

用一个列表 + 字典。每次 get/put 时遍历列表找到元素并移到最前面。get O(n)，put O(n)。

#### 优化思路

用双向链表代替列表，配合哈希表直接定位节点，删除和插入都是 O(1)。

#### 最优解法

哈希表 + 双向链表，get O(1)，put O(1)。

```python
class DLinkedNode:
    def __init__(self, key=0, val=0):
        self.key = key
        self.val = val
        self.prev = None
        self.next = None

class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = {}  # key -> DLinkedNode
        # 哨兵节点（dummy head 和 dummy tail）
        self.head = DLinkedNode()
        self.tail = DLinkedNode()
        self.head.next = self.tail
        self.tail.prev = self.head
```

### 4. 关键思维点

- 双向链表需要**两个哨兵节点**（dummy head 和 dummy tail），避免处理边界。
- 哈希表的 key 是缓存的 key，value 是双向链表的节点。
- 节点中要存 key（不仅仅是 val），因为淘汰节点时需要同步删除哈希表中的条目。
- 四个核心操作：
  - `_add_to_head(node)`：把节点插到头部（最近使用）。
  - `_remove_node(node)`：从链表中删除节点。
  - `_move_to_head(node)`：移到头部 = 删除 + 插入头部。
  - `_pop_tail()`：删除尾部节点（最久未使用）。

### 5. 代码实现（Python）

```python
class DLinkedNode:
    """双向链表节点"""
    def __init__(self, key=0, val=0):
        self.key = key             # 存 key 是为了淘汰时从哈希表删除
        self.val = val
        self.prev = None
        self.next = None

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity   # 缓存容量
        self.cache = {}            # 哈希表: key -> 链表节点
        self.size = 0              # 当前缓存大小
        # 初始化双向链表的哨兵节点
        self.head = DLinkedNode()  # 虚拟头（最近使用端）
        self.tail = DLinkedNode()  # 虚拟尾（最久未使用端）
        self.head.next = self.tail
        self.tail.prev = self.head

    def _add_to_head(self, node: DLinkedNode):
        """把节点添加到双向链表头部（head之后）"""
        node.prev = self.head          # node 的 prev 指向 head
        node.next = self.head.next     # node 的 next 指向原来的第一个节点
        self.head.next.prev = node     # 原来第一个节点的 prev 指向 node
        self.head.next = node          # head 的 next 指向 node

    def _remove_node(self, node: DLinkedNode):
        """从双向链表中删除一个节点"""
        node.prev.next = node.next     # 前一个节点跳过 node
        node.next.prev = node.prev     # 后一个节点跳过 node

    def _move_to_head(self, node: DLinkedNode):
        """把节点移到头部（标记为最近使用）"""
        self._remove_node(node)        # 先从原位置删除
        self._add_to_head(node)        # 再插入到头部

    def _pop_tail(self) -> DLinkedNode:
        """弹出尾部节点（最久未使用的）"""
        node = self.tail.prev          # 尾部哨兵的前一个才是真正的尾节点
        self._remove_node(node)        # 从链表中删除
        return node                    # 返回被删的节点（需要其 key 来更新哈希表）

    def get(self, key: int) -> int:
        if key not in self.cache:      # 缓存中没有
            return -1
        node = self.cache[key]         # 通过哈希表 O(1) 找到节点
        self._move_to_head(node)       # 移到头部（标记为最近使用）
        return node.val                # 返回值

    def put(self, key: int, value: int):
        if key in self.cache:          # key 已存在：更新值，移到头部
            node = self.cache[key]
            node.val = value
            self._move_to_head(node)
        else:                          # key 不存在：新建节点
            node = DLinkedNode(key, value)
            self.cache[key] = node     # 加入哈希表
            self._add_to_head(node)    # 加入链表头部
            self.size += 1
            if self.size > self.capacity:   # 超容量：淘汰最久未使用的
                tail = self._pop_tail()     # 弹出尾部节点
                del self.cache[tail.key]    # 从哈希表中删除
                self.size -= 1
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 哈希表 + 双向链表 | 哈希表O(1)查找，双向链表O(1)增删 |
| 两个哨兵节点 | head 和 tail 避免边界判断 |
| 节点存key | 淘汰时需要从哈希表同步删除 |
| 四个核心方法 | add_to_head, remove, move_to_head, pop_tail |

### 做链表题的通用流程

1. **画图**：画出链表的初始状态、操作过程、最终状态。
2. **想清楚指针操作的顺序**：先保存要断开的指针，再修改。
3. **考虑边界**：空链表、只有一个节点、要操作的是头节点或尾节点。
4. **用 dummy 节点**：只要涉及增删操作，dummy 节点几乎总是好用的。
5. **写代码前模拟一遍**：拿一个小例子（3-4个节点）手动模拟整个过程。
