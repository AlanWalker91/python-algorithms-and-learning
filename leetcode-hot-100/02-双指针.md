# 分类名：双指针

> 双指针是一种通过维护两个指针（索引）在数组或链表上协同移动来降低时间复杂度的技巧。根据指针的运动方向，可分为**同向双指针（快慢指针）**和**相向双指针（对撞指针）**两大类。

---

## 题目1：283. 移动零 (Move Zeroes)

> 给定一个数组 `nums`，编写一个函数将所有 `0` 移动到数组的末尾，同时保持非零元素的相对顺序。必须在原数组上操作，不能拷贝额外的数组。

### 1. 核心考点

本题核心考查**同向双指针（快慢指针）**的应用。属于"数组原地操作 / 数组分区"类问题。关键在于理解如何用一个慢指针维护"已处理区域的边界"，用一个快指针扫描整个数组。

### 2. 解题思路（给新手讲）

想象你在整理一排书架上的书。书架上有些位置是空的（零），有些位置有书（非零）。你的任务是把所有的书都推到书架左边，空位全部留在右边，并且书的先后顺序不能变。

你可以这样做：
- 左手（慢指针）指着书架最左边，表示"下一本书应该放在这里"。
- 右手（快指针）从左到右一本一本地检查。
- 每当右手发现一本书（非零元素），就把它搬到左手指着的位置，然后左手往右移一格。
- 右手扫完整个书架后，左手右边的所有位置都填上空（零）。

这样一趟扫描就搞定了！

### 3. 从暴力到优化的推导

#### 暴力解法

最直接的想法：每次找到一个 `0`，就把它后面的所有元素往前移一位，然后在末尾放一个 `0`。

```python
def moveZeroes(nums):
    n = len(nums)
    for i in range(n):
        if nums[i] == 0:
            # 找到后面第一个非零元素，交换
            for j in range(i + 1, n):
                if nums[j] != 0:
                    nums[i], nums[j] = nums[j], nums[i]
                    break
```

**时间复杂度**：O(n^2)，最坏情况下每个零都要扫描后面所有元素。
**空间复杂度**：O(1)。

#### 优化思路

暴力解法的问题在于：每找到一个零，都要重新扫描后面的部分，做了大量重复工作。

观察发现：我们不需要"找零然后搬非零"，可以换个角度——**直接把所有非零元素按顺序放到前面，剩下的位置全填零**。

这就像是只关注"有书的位置"，按顺序一本本摆好，最后空位自然就在右边了。

用两个指针可以一趟扫描完成：
- `slow`：指向下一个要放非零元素的位置。
- `fast`：从头到尾扫描每一个元素。

当 `fast` 遇到非零元素时，把它放到 `slow` 的位置，`slow` 右移。这样 `slow` 左边全是按顺序排好的非零元素。

#### 最优解法

使用同向双指针，一趟扫描，遇到非零就和 `slow` 位置交换（交换比覆盖+填零更简洁）。

**时间复杂度**：O(n)，只扫描一遍。
**空间复杂度**：O(1)，原地操作。

### 4. 关键思维点

- **为什么用两个指针？** 一个负责"扫描"，一个负责"安置"，各司其职，互不干扰。
- **slow 指针的含义**：`slow` 左边的元素都是已经处理好的非零元素，`slow` 指向的是"下一个空位"。
- **为什么用交换而不是直接赋值？** 交换操作天然地把零"换到后面"，不需要额外的填零步骤。
- **为什么 slow 不会超过 fast？** 因为 `slow` 只在 `fast` 发现非零元素时才移动，而 `fast` 每一步都移动，所以 `slow <= fast` 始终成立。
- **相对顺序为什么不会乱？** 因为 `fast` 是从左到右扫描的，非零元素被"收集"到前面的顺序就是它们原始的顺序。

### 5. 代码实现（Python）

```python
def moveZeroes(nums: list[int]) -> None:
    """
    同向双指针法（快慢指针）
    slow 维护非零区域的右边界
    fast 负责扫描整个数组
    """
    slow = 0  # slow 指向下一个非零元素应该放置的位置

    for fast in range(len(nums)):
        if nums[fast] != 0:
            # 发现非零元素，和 slow 位置交换
            # 交换后 slow 位置就是非零元素，slow 右移
            nums[slow], nums[fast] = nums[fast], nums[slow]
            slow += 1

    # 结束后，slow 左边全是非零元素（保持原顺序），右边全是零
```

**执行过程示例**：`nums = [0, 1, 0, 3, 12]`

| 步骤 | fast | nums[fast] | 操作 | slow | 数组状态 |
|------|------|-----------|------|------|---------|
| 0 | 0 | 0 | 跳过 | 0 | [0, 1, 0, 3, 12] |
| 1 | 1 | 1 | 交换[0]和[1] | 1 | [1, 0, 0, 3, 12] |
| 2 | 2 | 0 | 跳过 | 1 | [1, 0, 0, 3, 12] |
| 3 | 3 | 3 | 交换[1]和[3] | 2 | [1, 3, 0, 0, 12] |
| 4 | 4 | 12 | 交换[2]和[4] | 3 | [1, 3, 12, 0, 0] |

### 6. 套路总结

**识别方法**：当题目要求"原地操作数组"、"把满足某条件的元素移到一边"时，考虑同向双指针。

**解题模板**：

```python
# 同向双指针模板：数组分区
def partition(nums, condition):
    slow = 0
    for fast in range(len(nums)):
        if condition(nums[fast]):  # 满足条件的元素放到前面
            nums[slow], nums[fast] = nums[fast], nums[slow]
            slow += 1
```

## 题目2：11. 盛最多水的容器 (Container With Most Water)

> 给定一个长度为 `n` 的整数数组 `height`，其中 `height[i]` 表示第 `i` 条线的高度。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。返回容器可以储存的最大水量。

### 1. 核心考点

本题核心考查**相向双指针（对撞指针）**的应用。属于"贪心 + 双指针"类问题。关键在于理解为什么移动较短的那条边是安全的（不会错过最优解）。

### 2. 解题思路（给新手讲）

想象你站在一条河的两岸，两岸各有一排高矮不一的栅栏柱子。你要选两根柱子，在它们之间拉一块布接水，接到的水量 = 两根柱子之间的距离 x 较矮那根柱子的高度。

怎么找到能接最多水的两根柱子？

最聪明的方法：先从最远的两根柱子开始（距离最大），然后逐步缩小范围。

- 两根柱子之间水量 = 宽度 x 最小高度。
- 如果我们移动较**高**的柱子，宽度变小了，而高度最多和之前一样（受限于矮的那根），所以水量只会变少或不变——没意义！
- 如果我们移动较**矮**的柱子，宽度虽然变小了，但高度有可能变大，水量就**有可能**增加！

所以每次都移动矮的那根，就能保证不错过最优解。

### 3. 从暴力到优化的推导

#### 暴力解法

枚举所有的 (i, j) 组合，计算每一对的水量，取最大值。

```python
def maxArea(height):
    n = len(height)
    max_water = 0
    for i in range(n):
        for j in range(i + 1, n):
            water = (j - i) * min(height[i], height[j])
            max_water = max(max_water, water)
    return max_water
```

**时间复杂度**：O(n^2)，需要枚举所有组合。
**空间复杂度**：O(1)。

#### 优化思路

暴力解法枚举了 n*(n-1)/2 种组合，能不能减少枚举量？

核心观察：**面积 = 宽度 x 高度（取较矮边）**。

如果我们从最大宽度开始（左右两端），每次缩小宽度时，要想面积增大，必须让高度增大。而高度由较矮的边决定，所以移动较矮的边才有可能让高度增加。

这个贪心策略的正确性证明：假设最优解是 (i*, j*)。当我们的指针 left 和 right 还没有收缩到 (i*, j*) 时，每次移动矮边，都不会跳过最优解，因为：
- 如果 `height[left] < height[right]`，那么 left 和 left 右边任何柱子的组合，水量都不超过 `(right - left) * height[left]`（因为高度受限于 left）。所以 left 可以安全地右移。

#### 最优解法

使用对撞双指针，从两端向中间收缩，每次移动较矮的那根柱子。

**时间复杂度**：O(n)，每个元素最多被访问一次。
**空间复杂度**：O(1)。

### 4. 关键思维点

- **为什么从两端开始？** 两端开始保证了初始宽度最大，之后的缩小过程是有目的的搜索。
- **为什么移动矮的那边？** 这是整道题最核心的 insight。移动高的那边，宽度减小，高度不可能超过矮边，面积必然减小。移动矮的那边，虽然宽度减小，但高度有机会增大，面积可能增大。
- **会不会错过最优解？** 不会。每次移动矮边时，被"跳过"的所有组合的面积都不超过当前面积（因为宽度更小，高度受矮边限制不会更大）。
- **两边相等时移动哪边？** 都可以，因为无论移动哪边，另一边和当前这边的最优组合已经计算过了。

### 5. 代码实现（Python）

```python
def maxArea(height: list[int]) -> int:
    """
    对撞双指针法
    从两端开始，每次移动较矮的一边
    """
    left = 0                # 左指针，从最左端开始
    right = len(height) - 1  # 右指针，从最右端开始
    max_water = 0            # 记录最大水量

    while left < right:
        # 计算当前两根柱子围成的水量
        # 宽度 = right - left，高度 = 较矮的那根
        w = right - left
        h = min(height[left], height[right])
        max_water = max(max_water, w * h)

        # 移动较矮的那一边（贪心策略）
        if height[left] < height[right]:
            left += 1     # 左边矮，移动左指针
        else:
            right -= 1    # 右边矮（或相等），移动右指针

    return max_water
```

**执行过程示例**：`height = [1, 8, 6, 2, 5, 4, 8, 3, 7]`

| 步骤 | left | right | h[left] | h[right] | 面积 | max_water | 移动 |
|------|------|-------|---------|----------|------|-----------|------|
| 1 | 0 | 8 | 1 | 7 | 8 | 8 | left++ |
| 2 | 1 | 8 | 8 | 7 | 49 | 49 | right-- |
| 3 | 1 | 7 | 8 | 3 | 18 | 49 | right-- |
| 4 | 1 | 6 | 8 | 8 | 40 | 49 | right-- |
| 5 | 1 | 5 | 8 | 4 | 16 | 49 | right-- |
| 6 | 1 | 4 | 8 | 5 | 15 | 49 | right-- |
| 7 | 1 | 3 | 8 | 2 | 4 | 49 | right-- |
| 8 | 1 | 2 | 8 | 6 | 6 | 49 | right-- |

最终结果：49。

### 6. 套路总结

**识别方法**：当题目涉及"在数组两端选取元素进行某种计算并求最优"时，考虑对撞双指针。特别是当"缩小搜索范围"有明确的贪心依据时。

**解题模板**：

```python

def two_pointer_shrink(arr):
    left, right = 0, len(arr) - 1
    result = 0
    while left < right:
        # 计算当前结果
        current = calculate(arr, left, right)
        result = max(result, current)
        # 根据贪心策略决定移动哪一边
        if should_move_left(arr, left, right):
            left += 1
        else:
            right -= 1

```

## 题目3：15. 三数之和 (3Sum)

> 给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。返回所有和为 `0` 且不重复的三元组。

### 1. 核心考点

本题核心考查**排序 + 固定一个数 + 对撞双指针**的组合技巧。属于"多数之和"类问题的经典代表。难点在于**去重处理**——如何避免输出重复的三元组。

### 2. 解题思路（给新手讲）

想象你有一堆标了数字的卡片，要从中选出三张卡片，使得数字之和为零。

笨办法是所有三张的组合都试一遍，但太慢了。

聪明的做法：
1. **先排序**：把卡片从小到大排好。排序之后，相同的数字就挨在一起了，方便跳过重复。
2. **固定一张卡片**：从左到右依次选一张卡片作为"第一张"，问题就变成了"在剩下的卡片中找两张，使它们的和等于 -第一张的值"。
3. **对撞双指针找另外两张**：因为数组已排序，可以用左右指针高效地找到这两张卡片。

比如排序后是 `[-4, -1, -1, 0, 1, 2]`：
- 固定第一张 `-4`，在后面找和为 `4` 的两个数 → 找不到。
- 固定第一张 `-1`，在后面找和为 `1` 的两个数 → `(-1, 2)` 和 `(0, 1)` 都行！
- 第二个 `-1` 跳过（和前一个相同，避免重复）。
- 固定 `0`，找和为 `0` 的两个数 → 找不到。

### 3. 从暴力到优化的推导

#### 暴力解法

三重循环枚举所有三元组，用集合去重。

```python
def threeSum(nums):
    n = len(nums)
    result = set()
    for i in range(n):
        for j in range(i + 1, n):
            for k in range(j + 1, n):
                if nums[i] + nums[j] + nums[k] == 0:
                    result.add(tuple(sorted([nums[i], nums[j], nums[k]])))
    return [list(t) for t in result]
```

**时间复杂度**：O(n^3)，三重循环。
**空间复杂度**：O(n)，用于存储去重的集合。

#### 优化思路

**第一步优化——降维**：三数之和可以拆解为"固定一个数 + 两数之和"。两数之和在有序数组上可以用双指针 O(n) 解决，所以总复杂度降为 O(n^2)。

**第二步优化——去重**：
- 先排序，这样相同的元素相邻。
- 固定第一个数时，如果和前一个相同，跳过（避免重复三元组）。
- 双指针找到一组解后，左右指针都要跳过相同的值。

**为什么排序后双指针能工作？**
- 数组有序后，左指针 `left` 指向较小值，右指针 `right` 指向较大值。
- 三数之和太大 → 减小 `right`（让大的变小）。
- 三数之和太小 → 增大 `left`（让小的变大）。
- 刚好等于零 → 记录答案，两个指针同时移动并跳过重复值。

#### 最优解法

排序 O(n log n) + 固定一个数遍历 O(n) x 双指针 O(n) = O(n^2)。

**时间复杂度**：O(n^2)。
**空间复杂度**：O(1)（不算输出结果的空间；排序可能需要 O(log n)）。

### 4. 关键思维点

- **为什么要先排序？** 两个原因：(1) 使双指针的移动方向有明确依据（太大就减小右端，太小就增大左端）；(2) 相同元素相邻，方便去重。
- **去重的关键**：固定数和前一个相同时跳过；找到解后，`left` 和 `right` 都要跳过相邻的重复值。注意是和"前一个"比较，而不是和"后一个"比较，否则会漏掉 `[-1, -1, 2]` 这样的合法解。
- **提前终止**：如果固定的数 > 0，后面的数都是正数，三数之和不可能为零，可以直接 break。
- **为什么不用哈希表？** 虽然两数之和可以用哈希表 O(1) 查找，但在三数之和中去重逻辑非常复杂。排序 + 双指针在去重方面更加自然、简洁。

### 5. 代码实现（Python）

```python
def threeSum(nums: list[int]) -> list[list[int]]:
    """
    排序 + 固定一个数 + 对撞双指针
    """
    nums.sort()             # 第一步：排序，为双指针和去重做准备
    result = []
    n = len(nums)

    for i in range(n - 2):  # 固定第一个数，留至少两个位置给 left 和 right
        # 剪枝：如果当前数 > 0，后面的数都更大，三数之和不可能为 0
        if nums[i] > 0:
            break

        # 去重：跳过和前一个相同的数（注意是 i > 0，不是 i >= 0）
        if i > 0 and nums[i] == nums[i - 1]:
            continue

        # 在 i 的右边用对撞双指针找两个数，使三数之和为 0
        target = -nums[i]    # 需要找的两数之和
        left = i + 1         # 左指针
        right = n - 1        # 右指针

        while left < right:
            two_sum = nums[left] + nums[right]

            if two_sum < target:
                left += 1     # 和太小，左指针右移，增大和
            elif two_sum > target:
                right -= 1    # 和太大，右指针左移，减小和
            else:
                # 找到一组解
                result.append([nums[i], nums[left], nums[right]])

                # 左指针去重：跳过相同的值
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                # 右指针去重：跳过相同的值
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1

                # 两个指针同时移动，继续寻找下一组解
                left += 1
                right -= 1

    return result
```

**执行过程示例**：`nums = [-1, 0, 1, 2, -1, -4]`

排序后：`[-4, -1, -1, 0, 1, 2]`

| i | nums[i] | target | left | right | 过程 | 结果 |
|---|---------|--------|------|-------|------|------|
| 0 | -4 | 4 | 1 | 5 | -1+2=1<4, 左移... 找不到 | - |
| 1 | -1 | 1 | 2 | 5 | -1+2=1=target | [-1,-1,2] |
| 1 | -1 | 1 | 3 | 4 | 0+1=1=target | [-1,0,1] |
| 2 | -1 | 跳过（和i=1相同） | - | - | - | - |
| 3 | 0 | 0 | 4 | 5 | 1+2=3>0 | - |

最终结果：`[[-1, -1, 2], [-1, 0, 1]]`

### 6. 套路总结

**识别方法**：凡是"N 数之和"问题，核心套路都是**排序 + 固定 N-2 个数 + 双指针找最后两个数**。

**解题模板**：

```python
# N数之和通用框架（以三数之和为例）
def nSum(nums, target):
    nums.sort()
    result = []
    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i - 1]:  # 去重
            continue
        # 转化为两数之和
        left, right = i + 1, len(nums) - 1
        while left < right:
            s = nums[i] + nums[left] + nums[right]
            if s < target:
                left += 1
            elif s > target:
                right -= 1
            else:
                result.append([nums[i], nums[left], nums[right]])
                # 去重 + 移动
                while left < right and nums[left] == nums[left + 1]: left += 1
                while left < right and nums[right] == nums[right - 1]: right -= 1
                left += 1
                right -= 1
    return result
```

## 题目4：42. 接雨水 (Trapping Rain Water)

> 给定 `n` 个非负整数表示每个宽度为 `1` 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

### 1. 核心考点

本题核心考查**相向双指针**的高级应用。属于"模拟 / 贪心"类问题。除了双指针，还可以用动态规划、单调栈等方法解决，但双指针是空间最优的解法。关键 insight 是：每个位置能接的水量取决于它左右两侧最高柱子中较矮的那个。

### 2. 解题思路（给新手讲）

想象你用积木搭了一排高低不同的墙壁，然后往里面灌水。水会填满低洼处，直到溢出。

关键问题是：每个位置上方能存多少水？

答案是：**这个位置上方的水面高度 - 这个位置柱子的高度**。

而水面高度由什么决定？就像水桶效应——由这个位置左边最高的墙和右边最高的墙中**较矮的那个**决定。水面不可能高过较矮的墙，否则就从矮的那边溢出去了。

所以公式是：`位置 i 的接水量 = min(左边最高, 右边最高) - height[i]`

用双指针怎么做呢？从两端往中间走：
- 维护 `left_max`（左边已扫过区域的最高柱子）和 `right_max`（右边已扫过区域的最高柱子）。
- 哪边的 max 更小，就处理哪边的指针。因为较小的那个 max 就是"短板"，它决定了那个位置的水面高度，不管另一边实际有多高都没关系。

### 3. 从暴力到优化的推导

#### 暴力解法

对每个位置，向左扫描找最高柱子，向右扫描找最高柱子，然后计算该位置的接水量。

```python
def trap(height):
    n = len(height)
    water = 0
    for i in range(1, n - 1):  # 第一个和最后一个位置不可能接水
        left_max = max(height[:i + 1])     # 左边（含自身）最高
        right_max = max(height[i:])        # 右边（含自身）最高
        water += min(left_max, right_max) - height[i]
    return water
```

**时间复杂度**：O(n^2)，每个位置都要扫描左右两边。
**空间复杂度**：O(1)。

#### 优化思路

**方法一：预处理（动态规划思路）**

暴力法重复计算了左右最大值。可以预先算好两个数组：
- `left_max[i]`：从左到 i 的最大高度。
- `right_max[i]`：从右到 i 的最大高度。

这样每个位置的接水量直接查表计算，时间 O(n)，但空间也变成 O(n)。

```python
def trap(height):
    n = len(height)
    if n == 0: return 0
    left_max = [0] * n
    right_max = [0] * n
    left_max[0] = height[0]
    for i in range(1, n):
        left_max[i] = max(left_max[i - 1], height[i])
    right_max[n - 1] = height[n - 1]
    for i in range(n - 2, -1, -1):
        right_max[i] = max(right_max[i + 1], height[i])
    water = 0
    for i in range(n):
        water += min(left_max[i], right_max[i]) - height[i]
    return water
```

**方法二：对撞双指针（终极优化）**

能不能不用额外数组，把空间降到 O(1)？可以！

核心观察：我们不需要同时知道某个位置的 `left_max` 和 `right_max` 的精确值。我们只需要知道**较小的那个**。

- 维护 `left_max` 和 `right_max`。
- 如果 `left_max < right_max`，那么对于 `left` 位置，它的水面由 `left_max` 决定（因为右边有一根更高的柱子兜底，水不会从右边漏掉）。处理 `left`，然后 `left` 右移。
- 反之，处理 `right`，然后 `right` 左移。

#### 最优解法

对撞双指针，维护左右最大值，每次处理较小一侧。

**时间复杂度**：O(n)。
**空间复杂度**：O(1)。

### 4. 关键思维点

- **每个位置的接水量公式**：`min(left_max, right_max) - height[i]`，这是整道题的根基。
- **为什么处理 left_max 较小的那一侧？** 因为 `left_max < right_max` 意味着左边的最高柱子是"短板"。对于 `left` 位置来说，水面高度就是 `left_max`，不管右边的最大值到底是多少（反正它更大，水不会从右边溢出）。这就是双指针能代替两个数组的关键。
- **left_max 和 right_max 的更新时机**：进入某一侧处理之前，先更新该侧的 max。这保证了 max 值包含当前位置。
- **为什么第一个和最后一个位置不接水？** 因为没有另一边的墙壁挡住水。
- **和"盛最多水的容器"的区别**：盛水容器只看两根线之间的水（一个矩形），接雨水要计算每个位置的水量之和（一个不规则形状）。

### 5. 代码实现（Python）

```python
def trap(height: list[int]) -> int:
    """
    对撞双指针法
    核心：哪边的 max 更小，就处理哪边
    """
    if not height:
        return 0

    left = 0                  # 左指针
    right = len(height) - 1   # 右指针
    left_max = 0              # 左边已扫过区域的最高柱子
    right_max = 0             # 右边已扫过区域的最高柱子
    water = 0                 # 累计接水量

    while left < right:
        # 更新两侧的最大值
        left_max = max(left_max, height[left])
        right_max = max(right_max, height[right])

        if left_max < right_max:
            # 左边的 max 更小，左边是"短板"
            # left 位置的水面高度 = left_max
            # 该位置接水量 = left_max - height[left]
            water += left_max - height[left]
            left += 1         # 处理完，左指针右移
        else:
            # 右边的 max 更小（或相等），右边是"短板"
            # right 位置的水面高度 = right_max
            # 该位置接水量 = right_max - height[right]
            water += right_max - height[right]
            right -= 1        # 处理完，右指针左移

    return water
```

### 6. 套路总结

**识别方法**：当题目中每个位置的结果取决于"左右两侧的某个极值"时，可以考虑对撞双指针 + 维护左右极值。

**解题模板**：

```python
# 对撞双指针 + 左右极值模板
def two_pointer_with_extremes(arr):
    left, right = 0, len(arr) - 1
    left_max = right_max = 0
    result = 0
    while left < right:
        left_max = max(left_max, arr[left])
        right_max = max(right_max, arr[right])
        if left_max < right_max:
            result += left_max - arr[left]  # 处理左侧
            left += 1
        else:
            result += right_max - arr[right]  # 处理右侧
            right -= 1
    return result
```

**接雨水的三种解法对比**：

| 方法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|-----------|-----------|------|
| 暴力法 | O(n^2) | O(1) | 简单直观，但慢 |
| 预处理数组（DP） | O(n) | O(n) | 好理解，但需要额外空间 |
| 对撞双指针 | O(n) | O(1) | 时间空间双优，但需要理解"短板"思想 |

## 双指针专题总结

### 两大类型

| 类型 | 特点 | 代表题目 |
|------|------|---------|
| 同向双指针（快慢指针） | 两个指针同方向移动，一快一慢 | 283. 移动零 |
| 相向双指针（对撞指针） | 两个指针从两端向中间移动 | 11. 盛最多水的容器、15. 三数之和、42. 接雨水 |

### 什么时候想到双指针？

1. **数组原地操作**：需要在不用额外空间的情况下移除/移动/去重元素 → 同向双指针。
2. **有序数组上找组合**：在排序后的数组上寻找满足某种条件的元素对/组 → 对撞双指针。
3. **两端向中间缩小搜索范围**：有明确的贪心策略支持跳过部分候选 → 对撞双指针。
4. **每个位置依赖左右两侧信息**：可以用对撞双指针 + 维护左右状态来避免额外空间。

### 双指针的核心思想

双指针的本质是**用 O(1) 的额外空间换取减少一层循环**，将 O(n^2) 优化到 O(n)。它之所以能工作，是因为指针的移动方向有明确的单调性保证——每次移动都能排除一部分不可能的候选解。
