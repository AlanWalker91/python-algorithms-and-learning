## 题目1：46. 全排列

### 1. 核心考点

- **回溯框架**：理解"选择 -> 递归 -> 撤销"的核心模式
- **排列问题**：每个元素都要用到，区别于子集/组合问题
- **visited标记**：用标记数组避免同一元素重复使用

### 2. 解题思路（给新手讲）

给你一个不重复的数组，比如 [1,2,3]，请列出所有可能的排列。

想象你有三个空位 `_ _ _`，第一个位置可以放1、2或3，假设放了1，第二个位置只能从剩下的2、3中选，以此类推。每填完三个位置就得到一个排列。

回溯就是模拟这个过程：
1. 做选择：从还没用过的数字中选一个
2. 递归到下一层：继续填下一个位置
3. 撤销选择：退回来，试另一个数字

### 3. 从暴力到优化的推导

#### 暴力解法

多重循环嵌套，n 个数字就写 n 层循环。但 n 不确定，无法写出固定层数的循环。

#### 优化思路

用递归代替嵌套循环，递归的深度就是"层数"。每层递归就是在做一次选择，回溯就是撤销选择后换下一个试。

#### 最优解法

回溯法，时间 O(N! * N)，空间 O(N)。全排列总数就是 N!，没法更优了。

### 4. 关键思维点

- **回溯三步曲**：做选择 -> 递归 -> 撤销选择
- **排列 vs 子集**：排列关注顺序（[1,2]和[2,1]不同），需要 visited 标记
- **递归出口**：路径长度等于数组长度时收集结果
- **收集结果时要拷贝**：`result.append(path[:])` 而不是 `result.append(path)`

### 5. 代码实现（Python）

```python
class Solution:
    def permute(self, nums: list[int]) -> list[list[int]]:
        result = []     # 存放所有排列结果
        path = []       # 当前正在构建的排列
        used = [False] * len(nums)  # 标记哪些数字已经被用过

        def backtrack():
            # 递归出口：排列长度等于数组长度，找到一个完整排列
            if len(path) == len(nums):
                result.append(path[:])  # 注意要拷贝！path 后续会被修改
                return

            # 遍历所有数字，尝试每一种选择
            for i in range(len(nums)):
                if used[i]:        # 已经用过的数字跳过
                    continue

                # 做选择
                path.append(nums[i])   # 把这个数字加入当前排列
                used[i] = True         # 标记为已使用

                # 递归到下一层
                backtrack()

                # 撤销选择（回溯）
                path.pop()             # 把这个数字从排列中移除
                used[i] = False        # 标记为未使用

        backtrack()
        return result
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 题型 | 排列问题（顺序有关）|
| 核心套路 | 回溯三步曲：选择 -> 递归 -> 撤销 |
| 去重方式 | used 数组标记已使用元素 |
| 结果收集 | path[:] 拷贝当前路径 |
| 时间复杂度 | O(N! * N) |
| 空间复杂度 | O(N) 递归栈深度 |

## 题目2：78. 子集

### 1. 核心考点

- **回溯法求子集**：理解子集问题与排列问题的区别
- **start索引控制**：通过起始索引避免重复子集
- **每个节点都是答案**：不像排列只在叶子节点收集结果

### 2. 解题思路（给新手讲）

给你 [1,2,3]，请返回所有子集：[], [1], [2], [3], [1,2], [1,3], [2,3], [1,2,3]。

对于每个元素，只有两种状态：选或不选。可以看作一棵决策树：
- 到达每一个节点都可以收集一个子集（包括空集）
- 用 start 索引保证只向后选，避免 [1,2] 和 [2,1] 重复

与排列的关键区别：子集不关心顺序，[1,2] 和 [2,1] 是同一个子集，所以用 start 索引而不是 used 数组。

### 3. 从暴力到优化的推导

#### 暴力解法

二进制枚举：n 个元素有 2^n 个子集，用 0 到 2^n-1 的二进制数表示每个元素选或不选。可行但不如回溯直观。

#### 优化思路

回溯法天然地枚举了所有子集，思路更清晰，且可以方便地添加剪枝条件。

#### 最优解法

回溯法，时间 O(N * 2^N)，空间 O(N)。子集总数为 2^N，没法更优。

### 4. 关键思维点

- **start索引**：排列用 used 数组，子集/组合用 start 索引
- **每个节点都收集结果**：不是只在叶子节点收集
- **不需要递归出口**：for 循环到数组末尾自然结束

### 5. 代码实现（Python）

```python
class Solution:
    def subsets(self, nums: list[int]) -> list[list[int]]:
        result = []   # 存放所有子集
        path = []     # 当前正在构建的子集

        def backtrack(start):
            # 每到一个节点就收集当前路径作为一个子集
            result.append(path[:])  # 拷贝当前路径

            # 从 start 开始遍历，保证不回头选（避免重复子集）
            for i in range(start, len(nums)):
                path.append(nums[i])     # 做选择：选择 nums[i]
                backtrack(i + 1)         # 递归：下一层从 i+1 开始
                path.pop()              # 撤销选择

        backtrack(0)  # 从索引0开始
        return result
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 题型 | 子集问题（顺序无关）|
| 核心套路 | 回溯 + start 索引避免重复 |
| 与排列的区别 | 排列用 used 数组，子集用 start 索引 |
| 收集时机 | 每个节点都收集，不只是叶子 |
| 时间复杂度 | O(N * 2^N) |
| 空间复杂度 | O(N) |

## 题目3：17. 电话号码的字母组合

### 1. 核心考点

- **映射关系**：数字到字母的映射（电话键盘）
- **多叉树回溯**：每个数字对应多个字母，形成多叉树
- **组合生成**：逐位数字选择一个字母，拼出所有可能

### 2. 解题思路（给新手讲）

手机九宫格键盘：2->abc, 3->def, ..., 9->wxyz。给你一串数字如 "23"，求所有可能的字母组合。

想象成树：第一层是数字2对应的 a/b/c 三个分支，第二层每个分支下面又有数字3对应的 d/e/f。所有从根到叶子的路径就是所有组合：ad, ae, af, bd, be, bf, cd, ce, cf。

### 3. 从暴力到优化的推导

#### 暴力解法

嵌套循环，有几个数字就嵌套几层。但数字个数不固定，循环层数无法确定。

#### 优化思路

递归代替嵌套循环：每层递归处理一个数字，在该数字对应的所有字母中选择一个，递归处理下一个数字。

#### 最优解法

回溯法，时间 O(4^N * N)，N 为输入数字个数（最坏每个数字对应4个字母）。

### 4. 关键思维点

- **用索引控制层级**：第 index 层递归处理第 index 个数字
- **不需要 start 或 used**：因为每层选择的范围不同（不同数字对应不同字母集）
- **多叉树的分支数不固定**：有的数字对应3个字母，有的对应4个

### 5. 代码实现（Python）

```python
class Solution:
    def letterCombinations(self, digits: str) -> list[str]:
        # 特判：空输入返回空列表
        if not digits:
            return []

        # 数字到字母的映射表（电话键盘）
        phone_map = {
            '2': 'abc', '3': 'def', '4': 'ghi', '5': 'jkl',
            '6': 'mno', '7': 'pqrs', '8': 'tuv', '9': 'wxyz'
        }

        result = []   # 存放所有组合结果
        path = []     # 当前正在构建的组合

        def backtrack(index):
            """index 表示当前处理到第几个数字"""
            # 递归出口：所有数字都处理完了，收集结果
            if index == len(digits):
                result.append(''.join(path))  # 把字符列表拼成字符串
                return

            digit = digits[index]              # 当前数字
            letters = phone_map[digit]         # 当前数字对应的字母集

            for letter in letters:             # 遍历每个可选字母
                path.append(letter)            # 做选择
                backtrack(index + 1)           # 递归处理下一个数字
                path.pop()                     # 撤销选择

        backtrack(0)
        return result
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 题型 | 组合问题（不同层选择范围不同）|
| 核心套路 | 映射表 + 回溯 |
| 与排列/子集的区别 | 每层的选择集合不同，由输入数字决定 |
| 递归参数 | index 表示当前处理到第几个数字 |
| 时间复杂度 | O(4^N * N) |
| 空间复杂度 | O(N) |

## 题目4：39. 组合总和

### 1. 核心考点

- **回溯求组合**：在候选数组中找出所有和为 target 的组合
- **元素可重复使用**：同一个数字可以选多次
- **剪枝优化**：提前终止不可能成功的分支

### 2. 解题思路（给新手讲）

给你一组候选数字 [2,3,6,7] 和目标值 target=7。找出所有加起来等于7的组合，每个数字可以无限次重复使用。

这就像凑零钱：你有面额为 2、3、6、7 的硬币（每种无限个），问有几种方式凑出7元。

关键规则：
- 元素可以重复使用 -> 递归时不跳到 i+1，而是从 i 开始
- 用 start 索引避免重复组合（[2,3,2] 和 [2,2,3] 只算一种）
- 当剩余目标值变为0时，找到一个有效组合
- 当剩余目标值变为负数时，剪枝

### 3. 从暴力到优化的推导

#### 暴力解法

不加剪枝的回溯，盲目尝试所有分支直到 target < 0 才回头。

#### 优化思路

先排序，当 candidates[i] > remaining 时，后面的数字更大，也不可能满足，直接 break。这叫排序剪枝。

#### 最优解法

回溯 + 排序剪枝，避免大量无效搜索。

### 4. 关键思维点

- **可重复 vs 不可重复**：可重复则递归传 i（不跳过当前元素），不可重复则传 i+1
- **start 索引避免重复组合**：只往后选不往前选
- **排序剪枝**：排序后如果当前数字已经大于剩余值，直接跳过后面所有数字

### 5. 代码实现（Python）

```python
class Solution:
    def combinationSum(self, candidates: list[int], target: int) -> list[list[int]]:
        result = []
        path = []
        candidates.sort()  # 排序，为了后面的剪枝优化

        def backtrack(start, remaining):
            """
            start: 当前可选的起始索引
            remaining: 还需要凑的值
            """
            # 递归出口：剩余值为0，找到一个有效组合
            if remaining == 0:
                result.append(path[:])
                return

            # 从 start 开始遍历候选数字
            for i in range(start, len(candidates)):
                # 剪枝：如果当前数字已经大于剩余值，后面的更大，全部跳过
                if candidates[i] > remaining:
                    break

                path.append(candidates[i])         # 做选择
                # 递归：注意传 i 而不是 i+1，因为每个数字可以重复使用
                backtrack(i, remaining - candidates[i])
                path.pop()                          # 撤销选择

        backtrack(0, target)
        return result
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 题型 | 组合总和（元素可重复）|
| 核心套路 | 回溯 + start 索引 + 排序剪枝 |
| 可重复的关键 | 递归传 i 而非 i+1 |
| 剪枝条件 | candidates[i] > remaining 时 break |
| 时间复杂度 | O(N^(T/M))，T为target，M为最小候选值 |
| 空间复杂度 | O(T/M) 递归深度 |
| 对比题 | 40.组合总和II（每个数字只能用一次）|

## 题目5：22. 括号生成

### 1. 核心考点

- **回溯生成有效括号**：在生成过程中用规则剪枝
- **合法性判断**：任何时刻左括号数量 >= 右括号数量
- **对称递归**：选择"加左括号"或"加右括号"

### 2. 解题思路（给新手讲）

给你 n 对括号，请生成所有有效的括号组合。比如 n=3 时有 ["((()))","(()())","(())()","()(())","()()()"]。

关键规则很简单：
- 左括号最多用 n 个
- 右括号最多用 n 个
- 任何时候，已用的右括号数不能超过已用的左括号数（否则就是 ")(" 这种非法情况）

把它想象成一棵二叉决策树：每一步要么加 `(`，要么加 `)`。但不是所有路径都走，只有满足上面规则的才继续。

### 3. 从暴力到优化的推导

#### 暴力解法

生成所有 2n 长度的括号字符串（2^(2n)种），然后逐个验证合法性。大量无效字符串被白白生成。

#### 优化思路

在生成过程中就保证合法性——"合法的才继续生成"。通过跟踪左右括号的使用数量，在不合法的时刻直接剪枝，避免走死路。

#### 最优解法

带剪枝的回溯，时间复杂度 O(4^n / sqrt(n))（第n个卡特兰数）。

### 4. 关键思维点

- **两个条件控制选择**：left < n 可以加左括号；right < left 可以加右括号
- **不需要 visited 或 start**：因为每一步的选择只有两个（加左或加右），靠计数控制
- **结果在叶子节点收集**：当字符串长度达到 2n 时

### 5. 代码实现（Python）

```python
class Solution:
    def generateParenthesis(self, n: int) -> list[str]:
        result = []   # 存放所有有效括号组合
        path = []     # 当前正在构建的括号序列

        def backtrack(left, right):
            """
            left: 已使用的左括号数量
            right: 已使用的右括号数量
            """
            # 递归出口：左右括号都用完了，得到一个有效组合
            if left == n and right == n:
                result.append(''.join(path))
                return

            # 选择1：加左括号（条件：左括号还没用完）
            if left < n:
                path.append('(')           # 做选择
                backtrack(left + 1, right)  # 递归
                path.pop()                 # 撤销选择

            # 选择2：加右括号（条件：右括号数量 < 左括号数量，保证合法）
            if right < left:
                path.append(')')           # 做选择
                backtrack(left, right + 1)  # 递归
                path.pop()                 # 撤销选择

        backtrack(0, 0)
        return result
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 题型 | 括号生成（合法性约束的回溯）|
| 核心套路 | 回溯 + 左右括号计数剪枝 |
| 两个剪枝条件 | left < n 可加左；right < left 可加右 |
| 时间复杂度 | O(4^n / sqrt(n)) |
| 空间复杂度 | O(n) |
| 特殊之处 | 不需要 for 循环，每步只有两个选择 |

## 题目6：79. 单词搜索

### 1. 核心考点

- **二维网格上的回溯**：在网格中搜索路径
- **四方向DFS**：上下左右探索
- **原地标记**：避免额外 visited 数组

### 2. 解题思路（给新手讲）

给你一个字母网格和一个单词，问能不能在网格中找到一条路径，依次经过单词的每个字母。路径可以上下左右走，但同一个格子不能重复使用。

做法：遍历网格找到单词首字母的位置作为起点，然后从起点出发做 DFS：
- 当前格子的字母是否匹配单词的当前字符？
- 匹配的话向四个方向递归搜索下一个字符
- 搜索过程中标记已访问，回溯时取消标记

### 3. 从暴力到优化的推导

#### 暴力解法

尝试网格中所有可能的路径，检查是否拼出目标单词。路径数量是指数级的。

#### 优化思路

回溯法：逐字符匹配，不匹配立即回头。加上原地标记避免重复访问。这样大量不匹配的路径在早期就被剪掉了。

#### 最优解法

回溯 + 原地标记，时间 O(M*N*3^L)，L为单词长度（除了来路，每步最多3个方向）。

### 4. 关键思维点

- **原地标记技巧**：把当前格子临时改为特殊字符（如 `'#'`），回溯时恢复
- **匹配索引**：用 index 跟踪当前匹配到单词的第几个字符
- **提前返回**：匹配完单词最后一个字符就返回 True，不用走完整个网格

### 5. 代码实现（Python）

```python
class Solution:
    def exist(self, board: list[list[str]], word: str) -> bool:
        rows, cols = len(board), len(board[0])

        def backtrack(r, c, index):
            """
            r, c: 当前位置
            index: 当前匹配到 word 的第几个字符
            """
            # 递归出口：所有字符都匹配成功
            if index == len(word):
                return True

            # 越界、字符不匹配，直接返回 False
            if (r < 0 or r >= rows or c < 0 or c >= cols
                    or board[r][c] != word[index]):
                return False

            # 原地标记：把当前格子临时替换，防止重复访问
            temp = board[r][c]
            board[r][c] = '#'

            # 向四个方向递归搜索下一个字符
            found = (backtrack(r + 1, c, index + 1) or   # 下
                     backtrack(r - 1, c, index + 1) or   # 上
                     backtrack(r, c + 1, index + 1) or   # 右
                     backtrack(r, c - 1, index + 1))     # 左

            # 回溯：恢复当前格子的原始值
            board[r][c] = temp

            return found

        # 遍历网格，找到单词首字母的位置作为搜索起点
        for r in range(rows):
            for c in range(cols):
                if board[r][c] == word[0]:      # 首字母匹配
                    if backtrack(r, c, 0):       # 从这里开始搜索
                        return True

        return False  # 所有起点都试过了，没找到
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 题型 | 二维网格路径搜索 |
| 核心套路 | DFS回溯 + 原地标记 |
| 原地标记 | 临时改为 '#'，回溯时恢复 |
| 短路优化 | 用 or 连接四方向，找到就立即返回 |
| 时间复杂度 | O(M * N * 3^L) |
| 空间复杂度 | O(L) 递归栈深度 |

## 题目7：131. 分割回文串

### 1. 核心考点

- **回溯枚举分割位置**：在字符串的每个位置决定是否切一刀
- **回文判断**：判断子串是否是回文
- **动态规划预处理（可选优化）**：提前计算所有子串是否为回文

### 2. 解题思路（给新手讲）

给你字符串 "aab"，把它分割成若干段，要求每一段都是回文串。返回所有可能的分割方案。

"aab" 的答案是 [["a","a","b"], ["aa","b"]]。

想象在字符串的字符之间"切"：a|a|b 或 aa|b。回溯法就是在每个可能的位置决定"切到哪里"：
- 从当前位置 start 开始，尝试截取 start 到 i 的子串
- 如果这段子串是回文，就"切下来"放入路径，然后从 i+1 继续切剩下的
- 如果不是回文就跳过

### 3. 从暴力到优化的推导

#### 暴力解法

枚举所有可能的分割方式，然后逐个检查每段是否为回文。分割方式有 2^(n-1) 种（每个间隙切或不切）。

#### 优化思路

在回溯过程中边切边判断回文，如果当前截取的子串不是回文就直接跳过，大量剪枝。还可以用 DP 预处理所有子串是否为回文，避免重复计算。

#### 最优解法

回溯 + 即时回文判断（或 DP 预处理），时间 O(N * 2^N)。

### 4. 关键思维点

- **分割问题 = 子集问题的变体**：在每个间隙决定切不切
- **start 索引**：表示当前要从哪个位置开始切
- **剪枝关键**：只有回文子串才继续递归
- **递归出口**：start 到达字符串末尾，说明整个字符串被成功分割

### 5. 代码实现（Python）

```python
class Solution:
    def partition(self, s: str) -> list[list[str]]:
        result = []   # 存放所有分割方案
        path = []     # 当前的分割方案

        def is_palindrome(sub: str) -> bool:
            """判断字符串是否是回文"""
            return sub == sub[::-1]

        def backtrack(start):
            """
            start: 当前切割的起始位置
            """
            # 递归出口：切到了字符串末尾，整个字符串被成功分割
            if start == len(s):
                result.append(path[:])
                return

            # 尝试所有可能的切割位置：从 start 到末尾
            for end in range(start + 1, len(s) + 1):
                substring = s[start:end]       # 截取子串 s[start:end]

                if is_palindrome(substring):   # 只有回文子串才继续
                    path.append(substring)     # 做选择：把这段回文加入方案
                    backtrack(end)             # 递归：从 end 继续切剩下的部分
                    path.pop()                 # 撤销选择

        backtrack(0)
        return result
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 题型 | 字符串分割（分割型回溯）|
| 核心套路 | 回溯 + 回文判断剪枝 |
| 与子集的关系 | 分割位置的选择等价于子集问题 |
| 剪枝 | 非回文子串直接跳过 |
| 时间复杂度 | O(N * 2^N) |
| 空间复杂度 | O(N) |
| 优化方向 | 用 DP 预处理 is_palindrome 表 |

## 题目8：51. N皇后

### 1. 核心考点

- **经典回溯问题**：逐行放置皇后
- **冲突检测**：列冲突、主对角线冲突、副对角线冲突
- **对角线编码技巧**：用 `row-col` 和 `row+col` 唯一标识对角线

### 2. 解题思路（给新手讲）

在 N x N 的棋盘上放 N 个皇后，要求任意两个皇后不能在同一行、同一列、同一对角线上。

逐行放置：因为每行恰好放一个皇后，所以按行递归。每到一行，尝试在每一列放一个皇后，检查是否与已放置的皇后冲突。不冲突就放下去递归到下一行，冲突就跳过。

怎么快速判断冲突？维护三个集合：
- `cols`：记录哪些列已被占用
- `diag1`：记录哪些主对角线已被占用（主对角线上 row - col 相同）
- `diag2`：记录哪些副对角线已被占用（副对角线上 row + col 相同）

### 3. 从暴力到优化的推导

#### 暴力解法

在 N^2 个格子中选 N 个放皇后，然后检查合法性。组合数量为 C(N^2, N)，天文数字。

#### 优化思路

逐行放置大幅减少搜索空间（每行只选一列），加上列和对角线的集合判断，O(1) 检测冲突。

#### 最优解法

逐行回溯 + 集合冲突检测，时间 O(N!)。

### 4. 关键思维点

- **逐行放置**：保证每行恰好一个皇后，不用检查行冲突
- **对角线编码**：主对角线 row-col 相等，副对角线 row+col 相等，这是固定的数学性质
- **集合判断冲突**：O(1) 时间判断某个位置是否安全
- **结果构建**：用一维数组 queens[row] = col 记录每行皇后放在哪列

### 5. 代码实现（Python）

```python
class Solution:
    def solveNQueens(self, n: int) -> list[list[str]]:
        result = []
        queens = [-1] * n  # queens[i] = j 表示第i行的皇后放在第j列

        # 三个集合用于快速冲突检测
        cols = set()       # 已被占用的列
        diag1 = set()      # 已被占用的主对角线（row - col）
        diag2 = set()      # 已被占用的副对角线（row + col）

        def backtrack(row):
            """逐行放置皇后，当前处理第 row 行"""
            # 递归出口：所有行都放完了，收集结果
            if row == n:
                # 把 queens 数组转换成棋盘字符串格式
                board = []
                for i in range(n):
                    line = '.' * queens[i] + 'Q' + '.' * (n - queens[i] - 1)
                    board.append(line)
                result.append(board)
                return

            # 尝试在第 row 行的每一列放置皇后
            for col in range(n):
                # 冲突检测：列、主对角线、副对角线任一被占用则跳过
                if col in cols or (row - col) in diag1 or (row + col) in diag2:
                    continue

                # 做选择：放置皇后
                queens[row] = col
                cols.add(col)
                diag1.add(row - col)
                diag2.add(row + col)

                # 递归到下一行
                backtrack(row + 1)

                # 撤销选择：移除皇后
                queens[row] = -1
                cols.remove(col)
                diag1.remove(row - col)
                diag2.remove(row + col)

        backtrack(0)
        return result
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 题型 | 约束满足（棋盘放置）|
| 核心套路 | 逐行回溯 + 集合冲突检测 |
| 冲突检测 | cols + diag1(row-col) + diag2(row+col) |
| 为何逐行 | 每行恰好一个皇后，自动排除行冲突 |
| 时间复杂度 | O(N!) |
| 空间复杂度 | O(N) |
| 结果格式 | queens 数组 -> 棋盘字符串 |

## 回溯专题总结

### 回溯问题三大类型对比

| 类型 | 代表题 | 关键区别 | 去重手段 |
|------|--------|----------|----------|
| 排列 | 46.全排列 | 顺序有关，[1,2] != [2,1] | used 数组 |
| 子集/组合 | 78.子集, 39.组合总和 | 顺序无关，[1,2] == [2,1] | start 索引 |
| 分割 | 131.分割回文串 | 在字符串间隙切割 | start 索引 |
| 棋盘 | 51.N皇后 | 二维约束 | 逐行 + 冲突集合 |
| 网格搜索 | 79.单词搜索 | 二维路径搜索 | 原地标记 |

### 回溯模板

```python
def backtrack(路径, 选择列表):
    if 满足结束条件:
        收集结果
        return

    for 选择 in 选择列表:
        if 不合法:
            continue       # 剪枝
        做选择
        backtrack(路径, 选择列表)
        撤销选择
```

### 核心口诀

- **回溯 = 穷举 + 剪枝**
- **做选择 -> 递归 -> 撤销选择**（三步曲）
- **排列用 used，组合用 start**
- **结果拷贝 path[:]，不要直接加 path**
