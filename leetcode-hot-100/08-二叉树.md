## 题目1：94. 二叉树的中序遍历

### 1. 核心考点

- 二叉树的遍历方式（前序、中序、后序）
- 递归与迭代两种实现方式
- 栈模拟递归的思路

### 2. 解题思路（给新手讲）

中序遍历的顺序是：**左子树 -> 根节点 -> 右子树**。

想象你站在一棵树前面，你要按照"先看完左边所有的，再看当前的，最后看右边所有的"这个顺序来记录每个节点的值。

对于二叉搜索树来说，中序遍历的结果恰好是升序排列的，这是一个非常重要的性质。

### 3. 从暴力到优化的推导

#### 暴力解法

最直观的方式就是递归。递归天然地符合树的结构：处理左子树、处理当前节点、处理右子树。

```python
class Solution:
    def inorderTraversal(self, root: Optional[TreeNode]) -> List[int]:
        result = []

        def dfs(node):
            if not node:          # 递归终止条件：空节点直接返回
                return
            dfs(node.left)        # 先递归遍历左子树
            result.append(node.val)  # 再记录当前节点
            dfs(node.right)       # 最后递归遍历右子树

        dfs(root)
        return result
```

- 时间复杂度：O(n)，每个节点访问一次
- 空间复杂度：O(n)，递归栈的深度最坏为 n

#### 优化思路

递归本质上是用系统栈来保存状态。我们可以用一个显式的栈来模拟这个过程，从而实现迭代版本。

核心思路：不断把左孩子压入栈，直到没有左孩子了，弹出栈顶（就是"最左"的节点），记录它的值，然后转向它的右孩子。

#### 最优解法

```python
class Solution:
    def inorderTraversal(self, root: Optional[TreeNode]) -> List[int]:
        result = []
        stack = []
        curr = root

        while curr or stack:        # 当前节点不为空 或 栈不为空时继续
            while curr:             # 一路向左，把所有左孩子压栈
                stack.append(curr)
                curr = curr.left
            curr = stack.pop()      # 弹出栈顶，即当前最左的未处理节点
            result.append(curr.val) # 记录该节点的值（中序位置）
            curr = curr.right       # 转向右子树

        return result
```

- 时间复杂度：O(n)
- 空间复杂度：O(n)

### 4. 关键思维点

- 递归写法虽然简单，但面试中经常会追问迭代写法，务必掌握
- 迭代的关键在于"一路向左压栈"，弹出时再处理右子树
- 中序遍历 + BST = 有序数组，这个性质在后面的题目中会反复用到

### 5. 代码实现（Python）

```python
class Solution:
    def inorderTraversal(self, root: Optional[TreeNode]) -> List[int]:
        """迭代法实现中序遍历"""
        result = []       # 存储遍历结果
        stack = []         # 显式栈，模拟递归
        curr = root        # 当前正在处理的节点

        while curr or stack:
            # 第一步：一路向左走到底，沿途节点全部压栈
            while curr:
                stack.append(curr)
                curr = curr.left

            # 第二步：弹出栈顶节点（最左边未处理的节点）
            curr = stack.pop()

            # 第三步：记录该节点值（中序位置：左-根-右，此时左已处理完）
            result.append(curr.val)

            # 第四步：转向右子树，下一轮循环会继续处理右子树的左链
            curr = curr.right

        return result
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 递归三要素 | 终止条件 + 递归调用 + 当前层处理 |
| 迭代模板 | 一路向左压栈 -> 弹出处理 -> 转右子树 |
| 中序 = 有序 | 对 BST 做中序遍历得到有序数组 |

## 题目2：104. 二叉树的最大深度

### 1. 核心考点

- 递归思维：将大问题拆解为子问题
- 分治法的经典应用
- BFS 层序遍历求深度

### 2. 解题思路（给新手讲）

树的最大深度就是从根节点到最远叶子节点的路径上的节点数。

你可以这样想：一棵树的最大深度 = max(左子树深度, 右子树深度) + 1。这就是典型的递归思维——把大问题变成两个小问题。

就像量一棵真实的树的高度：你分别量左边最高的枝和右边最高的枝，取较高的那个，再加上树干本身（根节点）的高度。

### 3. 从暴力到优化的推导

#### 暴力解法

暴力方式可以用 BFS（广度优先搜索），一层一层地遍历，数一共有多少层。

```python
from collections import deque

class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        if not root:
            return 0
        queue = deque([root])
        depth = 0
        while queue:
            depth += 1                        # 每处理完一层，深度 +1
            for _ in range(len(queue)):       # 遍历当前层的所有节点
                node = queue.popleft()
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)
        return depth
```

- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 优化思路

递归写法更加简洁，天然符合树的结构。递归的核心思想是：空节点深度为 0，非空节点深度 = max(左, 右) + 1。

#### 最优解法

```python
class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        if not root:              # 空节点深度为 0
            return 0
        left = self.maxDepth(root.left)    # 左子树深度
        right = self.maxDepth(root.right)  # 右子树深度
        return max(left, right) + 1        # 当前树深度 = 较深的子树 + 1
```

- 时间复杂度：O(n)
- 空间复杂度：O(h)，h 为树的高度，最坏 O(n)

### 4. 关键思维点

- 树的问题首先想递归，递归首先想"当前节点要做什么"
- 分治思维：问题拆解为左右子树的子问题 + 合并结果
- 这是最经典的"自底向上"递归模型，很多题目都是它的变种

### 5. 代码实现（Python）

```python
class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        # 递归终止条件：空节点的深度是 0
        if not root:
            return 0

        # 分别计算左右子树的最大深度
        left_depth = self.maxDepth(root.left)
        right_depth = self.maxDepth(root.right)

        # 当前树的最大深度 = 左右子树中较大的那个 + 1（算上当前根节点）
        return max(left_depth, right_depth) + 1
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 自底向上递归 | 先递归到底，再把子结果合并上来 |
| 分治公式 | 树的某性质 = f(左子树性质, 右子树性质) |
| 空节点是 base case | 几乎所有树的递归，空节点都是终止条件 |

## 题目3：226. 翻转二叉树

### 1. 核心考点

- 递归操作二叉树
- 理解"翻转"的含义：交换每个节点的左右子树
- 前序/后序遍历都可以，但中序遍历需要注意

### 2. 解题思路（给新手讲）

翻转二叉树就是把每个节点的左孩子和右孩子交换。想象你拿着一面镜子对着这棵树，镜子里看到的就是翻转后的样子。

关键洞察：你不需要一次性想清楚整棵树怎么翻转，你只需要对每个节点做一件事——交换它的左右孩子。递归会帮你搞定所有节点。

### 3. 从暴力到优化的推导

#### 暴力解法

可以用 BFS 层序遍历，逐个节点交换左右子树。

```python
from collections import deque

class Solution:
    def invertTree(self, root: Optional[TreeNode]) -> Optional[TreeNode]:
        if not root:
            return None
        queue = deque([root])
        while queue:
            node = queue.popleft()
            node.left, node.right = node.right, node.left  # 交换左右子树
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        return root
```

#### 优化思路

递归写法更加简洁：对当前节点交换左右子树，然后递归翻转左右子树。

#### 最优解法

```python
class Solution:
    def invertTree(self, root: Optional[TreeNode]) -> Optional[TreeNode]:
        if not root:              # 空节点不需要翻转
            return None
        # 交换当前节点的左右子树
        root.left, root.right = root.right, root.left
        # 递归翻转左右子树
        self.invertTree(root.left)
        self.invertTree(root.right)
        return root
```

- 时间复杂度：O(n)
- 空间复杂度：O(h)

### 4. 关键思维点

- 交换左右孩子可以在递归前（前序）或递归后（后序）做，但不能在中间（中序）做，因为中序交换后左子树变成了原来的右子树，会导致右子树被翻转两次
- 这道题是递归操作树结构的入门题，务必理解"每个节点做同样的事"这个思想

### 5. 代码实现（Python）

```python
class Solution:
    def invertTree(self, root: Optional[TreeNode]) -> Optional[TreeNode]:
        # 递归终止条件：空节点直接返回
        if not root:
            return None

        # 交换当前节点的左右子树（核心操作，就这一行）
        root.left, root.right = root.right, root.left

        # 递归地翻转左子树（此时左子树是原来的右子树）
        self.invertTree(root.left)

        # 递归地翻转右子树（此时右子树是原来的左子树）
        self.invertTree(root.right)

        # 返回当前根节点
        return root
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 递归修改树 | 对每个节点做相同操作 + 递归处理子树 |
| 前序 vs 后序 | 修改结构时前序后序都可以，中序要小心 |
| Python 交换 | `a, b = b, a` 一行搞定 |

## 题目4：101. 对称二叉树

### 1. 核心考点

- 双指针递归（同时递归两棵子树）
- 对称的定义：左子树的左 = 右子树的右，左子树的右 = 右子树的左
- 递归函数的参数设计

### 2. 解题思路（给新手讲）

判断一棵树是否对称，就是判断它的左子树和右子树是否互为镜像。

怎么判断两棵树互为镜像？你可以同时遍历两棵树，一棵往左走的时候另一棵往右走，一棵往右走的时候另一棵往左走，每一步都检查两个节点的值是否相等。

就像照镜子：左边的人举左手，镜子里的人举右手——对应的是"反着来"的。

### 3. 从暴力到优化的推导

#### 暴力解法

可以先翻转右子树，然后判断左右子树是否相同。但这样需要额外修改树。

```python
class Solution:
    def isSymmetric(self, root: Optional[TreeNode]) -> bool:
        if not root:
            return True
        # 将中序遍历结果 + 层级信息对比（不推荐，仅作思路参考）
        # 这个方法有很多边界问题，不如直接双指针递归
```

#### 优化思路

直接定义一个辅助函数 `isMirror(left, right)`，同时比较两个节点：
1. 两个都为空 -> 对称
2. 一个为空一个不为空 -> 不对称
3. 两个值不相等 -> 不对称
4. 递归比较 left.left 与 right.right，以及 left.right 与 right.left

#### 最优解法

```python
class Solution:
    def isSymmetric(self, root: Optional[TreeNode]) -> bool:
        if not root:
            return True
        return self.isMirror(root.left, root.right)

    def isMirror(self, left, right):
        if not left and not right:    # 都为空，对称
            return True
        if not left or not right:     # 一个为空一个不为空，不对称
            return False
        # 值相等，且 left 的左 = right 的右，left 的右 = right 的左
        return (left.val == right.val
                and self.isMirror(left.left, right.right)
                and self.isMirror(left.right, right.left))
```

- 时间复杂度：O(n)
- 空间复杂度：O(h)

### 4. 关键思维点

- 对称 != 左右子树相同，对称是**镜像**关系
- 递归函数需要两个参数（两个节点），这是"双指针递归"的典型应用
- 类似题目：判断两棵树是否相同（100 题），只是递归方向不同

### 5. 代码实现（Python）

```python
class Solution:
    def isSymmetric(self, root: Optional[TreeNode]) -> bool:
        # 空树是对称的
        if not root:
            return True
        # 判断左右子树是否互为镜像
        return self.isMirror(root.left, root.right)

    def isMirror(self, left: Optional[TreeNode], right: Optional[TreeNode]) -> bool:
        # 两个都为空：对称
        if not left and not right:
            return True
        # 其中一个为空：不对称
        if not left or not right:
            return False
        # 当前节点值必须相等
        # 且 left 的左子树要和 right 的右子树镜像对称
        # 且 left 的右子树要和 right 的左子树镜像对称
        return (left.val == right.val
                and self.isMirror(left.left, right.right)
                and self.isMirror(left.right, right.left))
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 双指针递归 | 同时递归两个节点，比较它们的关系 |
| 镜像比较 | left.left vs right.right, left.right vs right.left |
| 三段式判断 | 都空/一空/都不空，这是双节点递归的标准模板 |

## 题目5：543. 二叉树的直径

### 1. 核心考点

- 树的直径：任意两个节点之间的最长路径
- 路径不一定经过根节点
- 在求深度的过程中"顺便"更新全局最大值

### 2. 解题思路（给新手讲）

二叉树的直径是任意两个节点之间最长路径的**边数**。

关键洞察：对于任何一个节点，经过它的最长路径 = 它的左子树深度 + 右子树深度。我们遍历所有节点，找到这个值的最大值就是答案。

打个比方：把每个节点想象成一个山谷的最低点，从它出发往左能爬多高（左子树深度），往右能爬多高（右子树深度），两边加起来就是经过这个山谷的最长路线。

### 3. 从暴力到优化的推导

#### 暴力解法

对每个节点都计算一次左右子树的深度，然后取最大的左+右。但这样会有大量重复计算。

```python
class Solution:
    def diameterOfBinaryTree(self, root: Optional[TreeNode]) -> int:
        self.ans = 0

        def depth(node):
            if not node:
                return 0
            return max(depth(node.left), depth(node.right)) + 1

        # 对每个节点都单独算一次左右深度（大量重复计算）
        def traverse(node):
            if not node:
                return
            left_d = depth(node.left)
            right_d = depth(node.right)
            self.ans = max(self.ans, left_d + right_d)
            traverse(node.left)
            traverse(node.right)

        traverse(root)
        return self.ans
```

- 时间复杂度：O(n^2)，每个节点都重新算深度

#### 优化思路

核心优化：在计算深度的递归过程中，**顺便**更新直径的最大值。这样只需要一次 DFS 就够了。

#### 最优解法

```python
class Solution:
    def diameterOfBinaryTree(self, root: Optional[TreeNode]) -> int:
        self.ans = 0

        def depth(node):
            if not node:
                return 0
            left = depth(node.left)     # 左子树深度
            right = depth(node.right)   # 右子树深度
            self.ans = max(self.ans, left + right)  # 更新直径
            return max(left, right) + 1  # 返回当前节点的深度

        depth(root)
        return self.ans
```

- 时间复杂度：O(n)
- 空间复杂度：O(h)

### 4. 关键思维点

- 直径 = 某个节点的左子树深度 + 右子树深度 的最大值
- "在求 A 的过程中顺便求 B"是树题中非常常见的技巧
- 注意直径是**边数**，不是节点数，所以深度用边数衡量时，叶子节点深度为 0

### 5. 代码实现（Python）

```python
class Solution:
    def diameterOfBinaryTree(self, root: Optional[TreeNode]) -> int:
        self.ans = 0  # 用实例变量记录全局最大直径

        def depth(node):
            """计算节点的深度，同时更新全局直径"""
            if not node:
                return 0

            # 递归计算左右子树的深度
            left = depth(node.left)
            right = depth(node.right)

            # 经过当前节点的最长路径 = 左深度 + 右深度
            # 用它来更新全局最大直径
            self.ans = max(self.ans, left + right)

            # 返回当前节点的深度（选左右中较大的 + 1）
            return max(left, right) + 1

        depth(root)
        return self.ans
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 深度 + 全局变量 | 在求深度时顺便更新全局最优解 |
| 路径 = 左深度 + 右深度 | 经过某节点的最长路径公式 |
| 一次 DFS 多个目的 | 避免重复计算，是树题优化的核心思路 |

## 题目6：102. 二叉树的层序遍历

### 1. 核心考点

- BFS（广度优先搜索）
- 队列的使用
- 如何区分"层"：记录当前层的节点数

### 2. 解题思路（给新手讲）

层序遍历就是一层一层地遍历树，同一层的节点从左到右排列。

实现方式是用**队列**（先进先出）：先把根节点放入队列，然后每次取出当前层的所有节点，把它们的子节点放入队列。

关键技巧：在处理每一层之前，先记录队列的长度 `size`，然后只处理 `size` 个节点，这样就能精确区分每一层。

### 3. 从暴力到优化的推导

#### 暴力解法

可以用 DFS + 记录深度的方式实现，但不够直观。

```python
class Solution:
    def levelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        result = []

        def dfs(node, level):
            if not node:
                return
            if level == len(result):      # 遇到新的一层，新建列表
                result.append([])
            result[level].append(node.val)  # 加入对应层
            dfs(node.left, level + 1)
            dfs(node.right, level + 1)

        dfs(root, 0)
        return result
```

#### 优化思路

BFS 是层序遍历最自然的实现方式，用队列逐层处理。

#### 最优解法

```python
from collections import deque

class Solution:
    def levelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root:
            return []
        result = []
        queue = deque([root])

        while queue:
            level_size = len(queue)   # 当前层有多少个节点
            level = []
            for _ in range(level_size):  # 只处理当前层的节点
                node = queue.popleft()
                level.append(node.val)
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)
            result.append(level)

        return result
```

- 时间复杂度：O(n)
- 空间复杂度：O(n)

### 4. 关键思维点

- BFS 用队列，DFS 用栈（或递归），这是最基本的区分
- `len(queue)` 技巧是层序遍历的核心，它让你能区分不同层的节点
- 层序遍历是很多题目的基础（右视图、锯齿遍历等等）

### 5. 代码实现（Python）

```python
from collections import deque

class Solution:
    def levelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        # 特判：空树返回空列表
        if not root:
            return []

        result = []                  # 最终结果：二维列表
        queue = deque([root])        # 队列初始化，放入根节点

        while queue:
            level_size = len(queue)  # 记录当前层的节点数量（关键！）
            level = []               # 当前层的节点值列表

            for _ in range(level_size):
                node = queue.popleft()       # 取出队首节点
                level.append(node.val)       # 记录节点值

                # 将下一层的节点（左右孩子）加入队列
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)

            result.append(level)     # 当前层处理完毕，加入结果

        return result
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| BFS 模板 | 队列 + while 循环 + len(queue) 区分层 |
| 层级处理 | for _ in range(level_size) 只处理当前层 |
| 变体技巧 | 改变遍历方向、只取最后一个等可以衍生很多题目 |

## 题目7：108. 将有序数组转换为二叉搜索树

### 1. 核心考点

- 二叉搜索树 (BST) 的性质
- 分治法：每次取中间元素作为根
- 递归构建树

### 2. 解题思路（给新手讲）

给你一个升序排列的数组，要求转换成一棵高度平衡的二叉搜索树。

关键洞察：BST 的中序遍历是有序的，所以有序数组的**中间元素**就是根节点！中间元素左边的构成左子树，右边的构成右子树。然后对左右两部分递归地做同样的事情。

就像找一本书的"中心页"作为目录的根，左半部分是上册，右半部分是下册，每册再找中心页......

### 3. 从暴力到优化的推导

#### 暴力解法

没有明显的暴力解法，因为直接用分治就是最自然的思路。如果不用分治，构建的 BST 可能不平衡。

#### 优化思路

分治法天然保证平衡：每次取中间元素，左右两部分大小差最多为 1。

#### 最优解法

```python
class Solution:
    def sortedArrayToBST(self, nums: List[int]) -> Optional[TreeNode]:
        if not nums:
            return None
        mid = len(nums) // 2               # 取中间位置
        root = TreeNode(nums[mid])          # 中间元素作为根
        root.left = self.sortedArrayToBST(nums[:mid])      # 左半部分构建左子树
        root.right = self.sortedArrayToBST(nums[mid + 1:])  # 右半部分构建右子树
        return root
```

- 时间复杂度：O(n)
- 空间复杂度：O(log n)，递归栈深度

### 4. 关键思维点

- 有序数组 + BST = 取中间作为根，这是一个固定搭配
- 分治法的三步走：分（取中间）、治（递归构建左右子树）、合（连接到根节点）
- 选 `len // 2` 保证左右子树大小尽量相等，从而保证高度平衡

### 5. 代码实现（Python）

```python
class Solution:
    def sortedArrayToBST(self, nums: List[int]) -> Optional[TreeNode]:
        # 用索引避免数组切片的开销
        def build(left, right):
            # 递归终止条件：区间为空
            if left > right:
                return None

            # 取中间元素作为当前子树的根节点
            mid = (left + right) // 2
            root = TreeNode(nums[mid])

            # 递归构建左子树：用 [left, mid-1] 的元素
            root.left = build(left, mid - 1)

            # 递归构建右子树：用 [mid+1, right] 的元素
            root.right = build(mid + 1, right)

            return root

        return build(0, len(nums) - 1)
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 有序数组构建 BST | 取中间元素作为根，递归构建 |
| 分治三步走 | 分（取中点）、治（递归）、合（连接） |
| 索引代替切片 | `build(left, right)` 避免 `nums[:mid]` 的额外开销 |

## 题目8：98. 验证二叉搜索树

### 1. 核心考点

- BST 的定义：左子树所有节点 < 根 < 右子树所有节点
- 中序遍历 BST 得到严格递增序列
- 递归时传递上下界

### 2. 解题思路（给新手讲）

验证一棵树是否是合法的二叉搜索树。

常见错误：只检查"左孩子 < 根 < 右孩子"是不够的！BST 要求的是左子树的**所有**节点都小于根，右子树的**所有**节点都大于根。

正确思路有两种：
1. **上下界法**：每个节点的值必须在一个范围 `(lower, upper)` 内，递归时不断缩小范围
2. **中序遍历法**：BST 的中序遍历必须是严格递增的

### 3. 从暴力到优化的推导

#### 暴力解法

中序遍历整棵树，得到一个数组，然后检查数组是否严格递增。

```python
class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
        vals = []

        def inorder(node):
            if not node:
                return
            inorder(node.left)
            vals.append(node.val)
            inorder(node.right)

        inorder(root)
        # 检查是否严格递增
        for i in range(1, len(vals)):
            if vals[i] <= vals[i - 1]:
                return False
        return True
```

- 时间复杂度：O(n)
- 空间复杂度：O(n)，存储所有节点值

#### 优化思路

不需要存储所有节点值，只需要在中序遍历时记录前一个节点的值，每次比较即可。

或者用更优雅的上下界递归法：

#### 最优解法

```python
class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
        def validate(node, lower=float('-inf'), upper=float('inf')):
            if not node:
                return True
            if node.val <= lower or node.val >= upper:
                return False
            # 左子树的所有值必须 < node.val
            # 右子树的所有值必须 > node.val
            return (validate(node.left, lower, node.val)
                    and validate(node.right, node.val, upper))

        return validate(root)
```

- 时间复杂度：O(n)
- 空间复杂度：O(h)

### 4. 关键思维点

- BST 的约束是全局的（所有左子树节点 < 根），不是局部的（只看直接孩子）
- 上下界法的精髓：往左走时更新上界（不能超过父节点），往右走时更新下界（不能小于父节点）
- 中序遍历法利用了 BST 的核心性质，更容易理解

### 5. 代码实现（Python）

```python
class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
        def validate(node, lower=float('-inf'), upper=float('inf')):
            # 空节点是合法的
            if not node:
                return True

            # 当前节点值必须严格在 (lower, upper) 范围内
            if node.val <= lower or node.val >= upper:
                return False

            # 验证左子树：左子树所有值必须 < node.val（更新上界）
            if not validate(node.left, lower, node.val):
                return False

            # 验证右子树：右子树所有值必须 > node.val（更新下界）
            if not validate(node.right, node.val, upper):
                return False

            return True

        return validate(root)
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 上下界递归 | 传递合法范围 (lower, upper)，不断缩小 |
| 中序遍历验证 | BST 中序必须严格递增 |
| 易错点 | 只比较父子是不够的，必须比较整个子树 |

## 题目9：230. 二叉搜索树中第K小的元素

### 1. 核心考点

- BST 中序遍历 = 有序序列
- 中序遍历的迭代实现
- 提前终止：找到第 K 个就停下

### 2. 解题思路（给新手讲）

在 BST 中找第 K 小的元素。

因为 BST 的中序遍历是有序的（从小到大），所以第 K 小的元素就是中序遍历的第 K 个元素。

你不需要遍历完整棵树——遍历到第 K 个就可以提前返回了。

### 3. 从暴力到优化的推导

#### 暴力解法

中序遍历整棵树存入数组，然后返回第 K 个。

```python
class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        vals = []

        def inorder(node):
            if not node:
                return
            inorder(node.left)
            vals.append(node.val)
            inorder(node.right)

        inorder(root)
        return vals[k - 1]  # 第 K 小的（0-indexed 是 k-1）
```

- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 优化思路

不需要遍历完所有节点，遍历到第 K 个就停下。用迭代中序遍历，每弹出一个节点就 K 减 1，当 K 为 0 时返回。

#### 最优解法

```python
class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        stack = []
        curr = root

        while curr or stack:
            while curr:
                stack.append(curr)
                curr = curr.left
            curr = stack.pop()
            k -= 1
            if k == 0:
                return curr.val
            curr = curr.right
```

- 时间复杂度：O(H + k)，H 是树高
- 空间复杂度：O(H)

### 4. 关键思维点

- BST + 第 K 小 = 中序遍历 + 计数，这是固定搭配
- 迭代版中序遍历可以随时停下来，比递归更灵活
- 如果频繁查询第 K 小，可以在每个节点存储左子树大小来加速

### 5. 代码实现（Python）

```python
class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        stack = []    # 显式栈
        curr = root   # 当前节点

        while curr or stack:
            # 一路向左，把所有左孩子压栈
            while curr:
                stack.append(curr)
                curr = curr.left

            # 弹出栈顶（当前最小的未处理节点）
            curr = stack.pop()

            # 计数：每弹出一个节点，k 减 1
            k -= 1

            # 如果已经是第 K 个，直接返回
            if k == 0:
                return curr.val

            # 转向右子树
            curr = curr.right
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| BST 中序 = 有序 | BST 的核心性质，反复使用 |
| 迭代可提前终止 | 比递归更适合"找到就停"的场景 |
| 第 K 小/大 | 中序遍历 + 计数器 |

## 题目10：199. 二叉树的右视图

### 1. 核心考点

- BFS 层序遍历
- 每层的最后一个节点就是右视图看到的
- 也可以用 DFS（优先访问右子树）

### 2. 解题思路（给新手讲）

想象你站在二叉树的右侧，能看到的节点就是每一层最右边的那个节点。

用层序遍历（BFS），每一层遍历完后，最后一个节点就是右视图能看到的节点。

### 3. 从暴力到优化的推导

#### 暴力解法

用 DFS，先遍历右子树再遍历左子树，每到达一个新的深度时记录第一个遇到的节点。

```python
class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> List[int]:
        result = []

        def dfs(node, depth):
            if not node:
                return
            if depth == len(result):     # 第一次到达这个深度
                result.append(node.val)  # 因为先遍历右子树，所以第一个就是最右的
            dfs(node.right, depth + 1)   # 先右
            dfs(node.left, depth + 1)    # 后左

        dfs(root, 0)
        return result
```

#### 优化思路

BFS 更直观：每层取最后一个。

#### 最优解法

```python
from collections import deque

class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> List[int]:
        if not root:
            return []
        result = []
        queue = deque([root])

        while queue:
            level_size = len(queue)
            for i in range(level_size):
                node = queue.popleft()
                if i == level_size - 1:      # 当前层的最后一个节点
                    result.append(node.val)
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)

        return result
```

- 时间复杂度：O(n)
- 空间复杂度：O(n)

### 4. 关键思维点

- BFS 取每层最后一个 = 右视图，取每层第一个 = 左视图
- DFS 解法的核心：先访问右子树，第一次到达某深度时记录的就是最右节点
- `depth == len(result)` 这个判断技巧很精妙：result 的长度正好等于已经记录了多少层

### 5. 代码实现（Python）

```python
from collections import deque

class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> List[int]:
        # 特判空树
        if not root:
            return []

        result = []
        queue = deque([root])

        while queue:
            level_size = len(queue)      # 当前层的节点数

            for i in range(level_size):
                node = queue.popleft()

                # 当前层的最后一个节点 -> 右视图能看到的
                if i == level_size - 1:
                    result.append(node.val)

                # 将下一层的节点加入队列
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)

        return result
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| BFS 层序遍历变体 | 每层取最后一个 = 右视图 |
| DFS 优先右子树 | 先右后左 + 首次到达某深度时记录 |
| 层序遍历的应用 | 左视图、右视图、锯齿遍历都是层序遍历的变体 |

## 题目11：114. 二叉树展开为链表

### 1. 核心考点

- 将二叉树原地展开为链表（只有右孩子）
- 展开顺序是前序遍历的顺序
- 如何在遍历的同时修改树的结构

### 2. 解题思路（给新手讲）

把一棵二叉树"拉平"成一个只有右子节点的链表，节点的顺序按照前序遍历（根-左-右）。

难点在于：你需要原地修改，不能创建新的节点。

最直观的思路：对于每个节点，把它的左子树插入到它和右子树之间。具体来说，找到左子树的最右节点，把原来的右子树挂到那个最右节点的右边，然后把左子树移到右边。

### 3. 从暴力到优化的推导

#### 暴力解法

先前序遍历存入数组，然后重建链表。

```python
class Solution:
    def flatten(self, root: Optional[TreeNode]) -> None:
        if not root:
            return
        nodes = []

        def preorder(node):
            if not node:
                return
            nodes.append(node)
            preorder(node.left)
            preorder(node.right)

        preorder(root)
        for i in range(len(nodes) - 1):
            nodes[i].left = None
            nodes[i].right = nodes[i + 1]
```

- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 优化思路

原地修改，不用额外空间。对于每个节点，如果它有左子树：
1. 找到左子树的最右节点
2. 把当前节点的右子树挂到最右节点的右边
3. 把左子树移到右边，左边置空

#### 最优解法

```python
class Solution:
    def flatten(self, root: Optional[TreeNode]) -> None:
        curr = root
        while curr:
            if curr.left:
                # 找到左子树的最右节点
                predecessor = curr.left
                while predecessor.right:
                    predecessor = predecessor.right
                # 把右子树挂到左子树的最右节点后面
                predecessor.right = curr.right
                # 左子树移到右边
                curr.right = curr.left
                curr.left = None
            curr = curr.right
```

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 4. 关键思维点

- 前序遍历的展开顺序：根 -> 左子树全部 -> 右子树全部
- "左子树的最右节点"就是前序遍历中左子树最后被访问的节点，它的下一个正好是右子树
- 这种"找前驱节点"的技巧也叫 Morris 遍历的思想

### 5. 代码实现（Python）

```python
class Solution:
    def flatten(self, root: Optional[TreeNode]) -> None:
        """原地将二叉树展开为链表（前序遍历顺序）"""
        curr = root

        while curr:
            if curr.left:
                # 第一步：找到左子树的最右节点（前驱节点）
                predecessor = curr.left
                while predecessor.right:
                    predecessor = predecessor.right

                # 第二步：把当前节点的右子树接到前驱节点的右边
                predecessor.right = curr.right

                # 第三步：把左子树移到右边
                curr.right = curr.left

                # 第四步：左子树置空
                curr.left = None

            # 移到下一个节点（此时一定是往右走）
            curr = curr.right
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 原地修改树结构 | 找前驱节点 + 重新连接 |
| 前序展开 | 左子树插入到根和右子树之间 |
| O(1) 空间 | 通过寻找前驱节点避免使用栈 |

## 题目12：105. 从前序与中序遍历序列构造二叉树

### 1. 核心考点

- 前序遍历的第一个元素是根节点
- 中序遍历中，根节点左边是左子树，右边是右子树
- 分治法递归构建

### 2. 解题思路（给新手讲）

给你前序遍历和中序遍历的结果，让你还原出原来的二叉树。

核心逻辑：
1. 前序遍历的第一个元素一定是当前树的**根节点**
2. 在中序遍历中找到这个根节点的位置，它左边的所有元素构成**左子树**，右边的所有元素构成**右子树**
3. 根据左子树的元素个数，在前序遍历中划分出左子树和右子树的前序序列
4. 对左右子树递归执行上述过程

打个比方：前序遍历像是"报到顺序"（领导先报到，然后左部门，再右部门），中序遍历像是"座位安排"（左部门在左，领导在中间，右部门在右）。结合两个信息就能还原整个组织结构。

### 3. 从暴力到优化的推导

#### 暴力解法

按上述逻辑递归，但每次在中序遍历中用 `index()` 查找根节点位置，时间为 O(n)。

```python
class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
        if not preorder:
            return None
        root_val = preorder[0]                    # 前序第一个是根
        root = TreeNode(root_val)
        mid = inorder.index(root_val)             # 在中序中找根的位置，O(n)
        root.left = self.buildTree(preorder[1:mid+1], inorder[:mid])
        root.right = self.buildTree(preorder[mid+1:], inorder[mid+1:])
        return root
```

- 时间复杂度：O(n^2)，每层递归都做一次 `index()` 和数组切片

#### 优化思路

1. 用哈希表预存中序遍历中每个值的索引，查找从 O(n) 变为 O(1)
2. 用索引区间代替数组切片，避免额外空间

#### 最优解法

```python
class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
        # 预处理：中序遍历值 -> 索引 的映射
        inorder_map = {val: idx for idx, val in enumerate(inorder)}
        self.pre_idx = 0  # 前序遍历的当前指针

        def build(in_left, in_right):
            if in_left > in_right:
                return None

            root_val = preorder[self.pre_idx]  # 前序的当前元素就是根
            self.pre_idx += 1
            root = TreeNode(root_val)

            mid = inorder_map[root_val]  # O(1) 找到根在中序中的位置
            root.left = build(in_left, mid - 1)    # 构建左子树
            root.right = build(mid + 1, in_right)  # 构建右子树
            return root

        return build(0, len(inorder) - 1)
```

- 时间复杂度：O(n)
- 空间复杂度：O(n)

### 4. 关键思维点

- 前序确定根，中序确定左右子树的范围——这是固定搭配
- 哈希表优化查找是常见技巧
- `self.pre_idx` 按前序顺序递增：先处理根，再处理左子树的根，再处理右子树的根
- 注意：必须先构建左子树再构建右子树，因为前序遍历的顺序是 根-左-右

### 5. 代码实现（Python）

```python
class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
        # 建立中序遍历 值->索引 的哈希映射，方便快速查找根的位置
        inorder_map = {val: idx for idx, val in enumerate(inorder)}

        # 前序遍历的全局指针，按顺序取出每个根节点
        self.pre_idx = 0

        def build(in_left, in_right):
            """在 inorder[in_left..in_right] 范围内构建子树"""
            # 递归终止条件：区间为空
            if in_left > in_right:
                return None

            # 前序遍历的当前元素就是这棵子树的根
            root_val = preorder[self.pre_idx]
            self.pre_idx += 1

            # 创建根节点
            root = TreeNode(root_val)

            # 在中序遍历中找到根的位置
            mid = inorder_map[root_val]

            # 先构建左子树（前序遍历：根-左-右，所以左子树先于右子树）
            root.left = build(in_left, mid - 1)

            # 再构建右子树
            root.right = build(mid + 1, in_right)

            return root

        return build(0, len(inorder) - 1)
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 前序 + 中序还原树 | 前序定根，中序分左右 |
| 哈希表加速查找 | 预处理索引映射，O(1) 定位 |
| 全局指针 + 索引区间 | 避免数组切片，提高效率 |

## 题目13：437. 路径总和 III

### 1. 核心考点

- 前缀和在树上的应用
- 哈希表记录前缀和的出现次数
- DFS + 回溯

### 2. 解题思路（给新手讲）

找出二叉树中路径和等于目标值的路径数量。路径不需要从根节点开始，也不需要到叶子节点结束，但必须是**从上到下**的。

暴力思路：对每个节点都尝试作为路径的起点，往下遍历累加求和。但这样太慢了。

优化思路：联想一维数组中的前缀和问题。如果 `prefixSum[j] - prefixSum[i] = target`，说明从 i 到 j 的区间和等于 target。在树上也是一样的！从根到当前节点的路径和减去从根到某个祖先节点的路径和，就是中间那段路径的和。

### 3. 从暴力到优化的推导

#### 暴力解法

对每个节点作为起点，向下DFS寻找和为 target 的路径。

```python
class Solution:
    def pathSum(self, root: Optional[TreeNode], targetSum: int) -> int:
        def dfs(node, remain):
            """以 node 为起点，寻找和为 remain 的路径"""
            if not node:
                return 0
            count = 0
            if node.val == remain:   # 找到一条
                count += 1
            count += dfs(node.left, remain - node.val)
            count += dfs(node.right, remain - node.val)
            return count

        if not root:
            return 0
        # 对每个节点都作为起点尝试
        return (dfs(root, targetSum)
                + self.pathSum(root.left, targetSum)
                + self.pathSum(root.right, targetSum))
```

- 时间复杂度：O(n^2)，最坏情况
- 空间复杂度：O(n)

#### 优化思路

使用前缀和 + 哈希表：
1. 维护从根到当前节点的路径前缀和 `curr_sum`
2. 用哈希表 `prefix_count` 记录路径上每个前缀和出现的次数
3. 如果 `curr_sum - target` 在哈希表中存在，说明有一段路径的和恰好等于 target

#### 最优解法

```python
from collections import defaultdict

class Solution:
    def pathSum(self, root: Optional[TreeNode], targetSum: int) -> int:
        prefix_count = defaultdict(int)
        prefix_count[0] = 1   # 空路径前缀和为 0，出现 1 次
        self.count = 0

        def dfs(node, curr_sum):
            if not node:
                return
            curr_sum += node.val
            # 查找：是否存在前缀和 = curr_sum - target
            self.count += prefix_count[curr_sum - targetSum]
            # 记录当前前缀和
            prefix_count[curr_sum] += 1
            # 递归左右子树
            dfs(node.left, curr_sum)
            dfs(node.right, curr_sum)
            # 回溯：离开当前节点时恢复状态
            prefix_count[curr_sum] -= 1

        dfs(root, 0)
        return self.count
```

- 时间复杂度：O(n)
- 空间复杂度：O(n)

### 4. 关键思维点

- 前缀和思想从一维数组推广到树——路径就是从根到当前节点的"数组"
- **回溯**是关键：离开一个节点时必须把它的前缀和从哈希表中减掉，因为这个前缀和只对当前路径有效，不能影响其他分支
- `prefix_count[0] = 1` 的含义：空前缀（从根节点开始的路径）出现了 1 次

### 5. 代码实现（Python）

```python
from collections import defaultdict

class Solution:
    def pathSum(self, root: Optional[TreeNode], targetSum: int) -> int:
        # 哈希表：前缀和 -> 出现次数
        prefix_count = defaultdict(int)
        # 空路径的前缀和为 0，初始化出现 1 次
        prefix_count[0] = 1
        self.count = 0

        def dfs(node, curr_sum):
            if not node:
                return

            # 更新从根到当前节点的前缀和
            curr_sum += node.val

            # 核心：如果 curr_sum - targetSum 出现过
            # 说明从那个位置到当前节点的路径和 = targetSum
            self.count += prefix_count[curr_sum - targetSum]

            # 把当前前缀和加入哈希表
            prefix_count[curr_sum] += 1

            # 递归处理左右子树
            dfs(node.left, curr_sum)
            dfs(node.right, curr_sum)

            # 回溯：离开当前节点，撤销当前前缀和的记录
            # 因为这个前缀和不应该影响其他分支的计算
            prefix_count[curr_sum] -= 1

        dfs(root, 0)
        return self.count
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 树上前缀和 | 把一维前缀和推广到树的路径上 |
| 哈希表 + 回溯 | 记录前缀和次数，离开节点时恢复 |
| prefix[0] = 1 | 空路径初始化，处理从根开始的路径 |

## 题目14：236. 二叉树的最近公共祖先

### 1. 核心考点

- 最近公共祖先 (LCA) 的定义
- 递归的后序遍历思想
- 自底向上的信息传递

### 2. 解题思路（给新手讲）

给定二叉树的两个节点 p 和 q，找到它们的最近公共祖先（LCA）。

最近公共祖先的定义：一个节点是 p 和 q 的公共祖先，如果 p 和 q 分别在它的左右子树中（或者它本身就是 p 或 q 之一）。

递归思路：对于每个节点，问三个问题：
1. 左子树中是否包含 p 或 q？
2. 右子树中是否包含 p 或 q？
3. 当前节点自身是否是 p 或 q？

如果以上三个条件中有两个为真，那么当前节点就是 LCA。

### 3. 从暴力到优化的推导

#### 暴力解法

从根开始，找到到 p 和 q 的路径，然后比较两条路径找到最后一个相同的节点。

```python
class Solution:
    def lowestCommonAncestor(self, root, p, q):
        def find_path(node, target, path):
            if not node:
                return False
            path.append(node)
            if node == target:
                return True
            if find_path(node.left, target, path) or find_path(node.right, target, path):
                return True
            path.pop()  # 回溯
            return False

        path_p, path_q = [], []
        find_path(root, p, path_p)
        find_path(root, q, path_q)

        lca = None
        for a, b in zip(path_p, path_q):
            if a == b:
                lca = a
            else:
                break
        return lca
```

- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 优化思路

用一次递归解决：递归函数返回"在这棵子树中找到了 p/q 吗"。

#### 最优解法

```python
class Solution:
    def lowestCommonAncestor(self, root: 'TreeNode', p: 'TreeNode', q: 'TreeNode') -> 'TreeNode':
        # 终止条件：空节点，或者找到了 p 或 q
        if not root or root == p or root == q:
            return root

        # 在左子树中找
        left = self.lowestCommonAncestor(root.left, p, q)
        # 在右子树中找
        right = self.lowestCommonAncestor(root.right, p, q)

        # 如果左右都找到了，说明 p 和 q 分别在左右子树中，当前节点就是 LCA
        if left and right:
            return root
        # 否则返回不为空的那个
        return left if left else right
```

- 时间复杂度：O(n)
- 空间复杂度：O(h)

### 4. 关键思维点

- 递归函数的返回值含义：在以 root 为根的子树中，p 和 q 的最近公共祖先（如果只找到其中一个，就返回那一个）
- `if left and right: return root` —— 这是核心判断：左右都有结果，说明 p 和 q 分居两侧
- `return left if left else right` —— p 和 q 都在同一侧，或者只找到了一个
- 这是一道"自底向上"传递信息的经典题

### 5. 代码实现（Python）

```python
class Solution:
    def lowestCommonAncestor(self, root: 'TreeNode', p: 'TreeNode', q: 'TreeNode') -> 'TreeNode':
        # 递归终止条件：
        # 1. root 为空 -> 没找到，返回 None
        # 2. root 是 p 或 q -> 找到了目标节点，返回它自己
        if not root or root == p or root == q:
            return root

        # 后序遍历：先递归处理左右子树
        left = self.lowestCommonAncestor(root.left, p, q)
        right = self.lowestCommonAncestor(root.right, p, q)

        # 情况1：p 和 q 分别在左右子树中 -> 当前节点就是 LCA
        if left and right:
            return root

        # 情况2：p 和 q 都在左子树中（或只找到一个在左边）
        # 情况3：p 和 q 都在右子树中（或只找到一个在右边）
        return left if left else right
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 后序遍历 + 自底向上 | 先递归到底，再把结果往上传 |
| 分类讨论 | 两边都有 -> 当前是LCA；只有一边 -> 往上传 |
| LCA 经典递归 | 终止条件包含 root == p/q，是关键 |

## 题目15：124. 二叉树中的最大路径和

### 1. 核心考点

- 路径可以不经过根节点
- 节点值可以为负数，所以不能贪心地全部选取
- 在递归求"单侧最大贡献值"的同时更新全局最大路径和

### 2. 解题思路（给新手讲）

找出二叉树中路径和最大的路径。路径可以从任意节点开始，到任意节点结束（但必须连续），不一定经过根节点。

这道题和"二叉树的直径"非常类似！不同之处在于：
- 直径求的是边数，这里求的是节点值之和
- 节点值可能是负数，所以选择路径时需要考虑"要不要带上某个子树"

核心思想：对每个节点，计算它的**单侧最大贡献值**（从该节点出发往下走的最大路径和），同时更新全局的最大路径和（左贡献 + 当前值 + 右贡献）。

### 3. 从暴力到优化的推导

#### 暴力解法

枚举所有可能的路径起终点，计算路径和。但树的路径组合太多，不可行。

#### 优化思路

递归的思路：对于每个节点 node：
1. 计算左子树的最大贡献值 `left_gain`（如果为负就取 0，即不选）
2. 计算右子树的最大贡献值 `right_gain`（如果为负就取 0）
3. 经过当前节点的最大路径和 = `left_gain + node.val + right_gain`，用它更新全局最大值
4. 返回给父节点的贡献值 = `node.val + max(left_gain, right_gain)`（只能选一侧，因为路径不能分叉）

#### 最优解法

```python
class Solution:
    def maxPathSum(self, root: Optional[TreeNode]) -> int:
        self.max_sum = float('-inf')

        def max_gain(node):
            if not node:
                return 0
            # 左右子树的最大贡献值（负数则不选，取0）
            left = max(max_gain(node.left), 0)
            right = max(max_gain(node.right), 0)
            # 经过当前节点的最大路径和
            path_sum = node.val + left + right
            self.max_sum = max(self.max_sum, path_sum)
            # 返回给父节点：只能选一侧
            return node.val + max(left, right)

        max_gain(root)
        return self.max_sum
```

- 时间复杂度：O(n)
- 空间复杂度：O(h)

### 4. 关键思维点

- **返回值 vs 更新值** 要分清楚：
  - 更新全局最大值时可以同时选左右两侧（形成"拱形"路径）
  - 返回给父节点时只能选一侧（路径不能分叉）
- `max(gain, 0)`：如果子树贡献为负数，就不选（等价于路径不延伸到那边）
- 这道题的递归结构和"二叉树的直径"几乎一模一样，区别在于用节点值代替了边数

### 5. 代码实现（Python）

```python
class Solution:
    def maxPathSum(self, root: Optional[TreeNode]) -> int:
        # 全局最大路径和，初始化为负无穷（因为节点值可能全是负数）
        self.max_sum = float('-inf')

        def max_gain(node):
            """计算以 node 为起点向下的最大贡献值"""
            if not node:
                return 0

            # 递归计算左右子树的最大贡献值
            # 如果贡献值为负数，就不选（取 0），相当于不走那条路
            left_gain = max(max_gain(node.left), 0)
            right_gain = max(max_gain(node.right), 0)

            # 经过当前节点的最大路径和（可以同时选左右两侧）
            # 这是一条"拱形"路径：左子树 -> 当前节点 -> 右子树
            current_path_sum = node.val + left_gain + right_gain

            # 用当前路径和更新全局最大值
            self.max_sum = max(self.max_sum, current_path_sum)

            # 返回给父节点的贡献值：只能选左右中较大的一侧
            # 因为路径不能分叉——父节点只能从一侧"接入"当前节点
            return node.val + max(left_gain, right_gain)

        max_gain(root)
        return self.max_sum
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 贡献值递归 | 返回单侧最大贡献，更新双侧路径和 |
| max(gain, 0) | 负数贡献不要，等于"剪枝" |
| 与直径题对比 | 结构几乎一样，直径用边数，此题用节点值之和 |
| 返回值 vs 更新值 | 返回给父节点只能选一侧，更新全局可以选两侧 |

def dfs(node):
