# 多维动态规划

多维动态规划的核心特点是：**状态需要用两个或多个维度来描述**。与一维 DP 只需要 `dp[i]` 不同，多维 DP 通常需要 `dp[i][j]` 来表示状态，因为问题本身涉及两个变化的量（比如两个字符串的位置、网格的行列坐标等）。

掌握多维 DP 的关键：
1. **明确状态定义** -- `dp[i][j]` 到底代表什么
2. **找出状态转移** -- `dp[i][j]` 能从哪些子状态推导出来
3. **确定边界条件** -- 哪些 `dp[i][j]` 可以直接确定值
4. **确定遍历方向** -- 保证计算 `dp[i][j]` 时，所依赖的状态已经算过

## 题目1：62. 不同路径

### 1. 核心考点
- **二维网格 DP**：状态定义在二维坐标上
- **路径计数问题**：求方案总数，典型的 DP 应用场景
- **组合数学**（可选的数学解法）

### 2. 解题思路（给新手讲）

想象你站在一个 m x n 的网格的左上角，你的目标是走到右下角。每一步只能**向右**或者**向下**走。问一共有多少条不同的路径。

关键观察：你到达某个格子 `(i, j)` 只有两种方式 --
- 从上方 `(i-1, j)` 走下来
- 从左边 `(i, j-1)` 走过来

所以到达 `(i, j)` 的路径数 = 到达 `(i-1, j)` 的路径数 + 到达 `(i, j-1)` 的路径数。

这就是经典的**加法原理**：两种互斥的到达方式，总数相加。

### 3. 从暴力到优化的推导

#### 暴力解法
用递归枚举所有路径。从 `(0, 0)` 出发，每一步选择向右或向下，直到到达 `(m-1, n-1)`。

```python
def uniquePaths(m: int, n: int) -> int:
    def dfs(i, j):
        if i == m - 1 and j == n - 1:  # 到达终点
            return 1
        if i >= m or j >= n:  # 越界
            return 0
        return dfs(i + 1, j) + dfs(i, j + 1)  # 向下 + 向右
    return dfs(0, 0)
```

时间复杂度：O(2^(m+n))，大量重复计算。

#### 优化思路
递归过程中，`dfs(i, j)` 会被多次调用（比如 `(2,3)` 可以从 `(1,3)` 和 `(2,2)` 都递归到），这就是**重叠子问题**。加上记忆化或者改成自底向上的 DP 即可。

#### 最优解法

**状态定义**：`dp[i][j]` = 从起点 `(0, 0)` 到达位置 `(i, j)` 的路径数

**转移方程**：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`

**边界条件**：
- 第一行所有格子只能从左边走过来，路径数都是 1
- 第一列所有格子只能从上面走下来，路径数都是 1

### 4. 关键思维点
- **为什么是加法不是乘法**？因为"从上面来"和"从左边来"是两种**互斥**的到达方式，用加法原理
- **边界为什么是1**？第一行只能一直向右走，只有一条路；第一列只能一直向下走，也只有一条路
- **空间优化**：因为每一行只依赖上一行，可以用一维数组滚动更新，空间从 O(mn) 降到 O(n)

### 5. 代码实现（Python）

```python
def uniquePaths(m: int, n: int) -> int:
    # 创建 m x n 的 DP 表，初始化全为 1
    # 实际上只需要初始化第一行和第一列为 1
    dp = [[1] * n for _ in range(m)]

    # 从 (1,1) 开始填表，因为第一行和第一列已经确定
    for i in range(1, m):          # 遍历每一行（从第2行开始）
        for j in range(1, n):      # 遍历每一列（从第2列开始）
            # 到达 (i,j) 的路径数 = 从上面来 + 从左边来
            dp[i][j] = dp[i - 1][j] + dp[i][j - 1]

    # 右下角就是答案
    return dp[m - 1][n - 1]
```

**空间优化版本**：

```python
def uniquePaths(m: int, n: int) -> int:
    # 只用一维数组，滚动更新
    dp = [1] * n  # 初始化为第一行的值，全是 1

    for i in range(1, m):          # 从第2行开始
        for j in range(1, n):      # 从第2列开始
            # dp[j] 更新前存的是上一行的值（即 dp[i-1][j]）
            # dp[j-1] 已经更新过了，存的是当前行左边的值（即 dp[i][j-1]）
            dp[j] += dp[j - 1]

    return dp[n - 1]
```

### 6. 套路总结
- **网格路径问题** = 二维 DP 的经典入门题
- 状态定义跟着坐标走：`dp[i][j]` 表示到达 `(i, j)` 的某种属性
- 只能向右/向下走 --> 转移只来自上方和左方
- 求路径数 --> 加法（把来源的数加起来）
- 求最值 --> 取 min/max（见下一题）

## 题目2：64. 最小路径和

### 1. 核心考点
- **二维网格 DP**：在网格上求最优值
- **状态转移取 min**：与上一题结构相同，但从"求路径数"变为"求最小和"
- **DP 的万能适配性**：同一个网格模型，改一下转移就能解决不同问题

### 2. 解题思路（给新手讲）

给你一个 m x n 的网格，每个格子里有一个非负整数。从左上角走到右下角，每步只能向右或向下，找一条路径使得路径上的数字之和最小。

和上一题的唯一区别：上一题求"有多少条路径"，这一题求"哪条路径的数字和最小"。

到达 `(i, j)` 只能从上面或左边来，所以：
- 到达 `(i, j)` 的最小路径和 = min(从上面来的最小路径和, 从左边来的最小路径和) + 当前格子的值

### 3. 从暴力到优化的推导

#### 暴力解法
递归枚举所有路径，取路径和的最小值。

```python
def minPathSum(grid: list[list[int]]) -> int:
    m, n = len(grid), len(grid[0])

    def dfs(i, j):
        if i == 0 and j == 0:        # 起点
            return grid[0][0]
        if i < 0 or j < 0:           # 越界，返回无穷大
            return float('inf')
        # 从上面或左边来，取较小的那个，加上当前格子
        return min(dfs(i - 1, j), dfs(i, j - 1)) + grid[i][j]

    return dfs(m - 1, n - 1)
```

时间复杂度：O(2^(m+n))，同样大量重复计算。

#### 优化思路
加记忆化，或者直接用自底向上的 DP。本题甚至可以**原地修改** grid 数组来做，不需要额外空间。

#### 最优解法

**状态定义**：`dp[i][j]` = 从起点 `(0, 0)` 到达 `(i, j)` 的最小路径和

**转移方程**：`dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]`

**边界条件**：
- `dp[0][0] = grid[0][0]`（起点）
- 第一行：`dp[0][j] = dp[0][j-1] + grid[0][j]`（只能从左边来）
- 第一列：`dp[i][0] = dp[i-1][0] + grid[i][0]`（只能从上面来）

### 4. 关键思维点
- **和"不同路径"的对比**：结构完全一样，只是把"加法计数"换成了"取 min + 累加权值"
- **求路径数用加法，求最短路径用 min**：这是 DP 问题的核心区别
- **原地修改**：可以直接修改 grid 数组，`grid[i][j]` 更新后就代表 `dp[i][j]`，空间 O(1)
- **边界处理要小心**：第一行和第一列只有一个方向可以到达，不能取 min

### 5. 代码实现（Python）

```python
def minPathSum(grid: list[list[int]]) -> int:
    m, n = len(grid), len(grid[0])

    # 创建 DP 表
    dp = [[0] * n for _ in range(m)]
    dp[0][0] = grid[0][0]  # 起点的最小路径和就是自己

    # 初始化第一行：只能从左边来，不断累加
    for j in range(1, n):
        dp[0][j] = dp[0][j - 1] + grid[0][j]

    # 初始化第一列：只能从上面来，不断累加
    for i in range(1, m):
        dp[i][0] = dp[i - 1][0] + grid[i][0]

    # 填表：每个格子取上方和左方的较小值，加上当前值
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j]

    return dp[m - 1][n - 1]  # 右下角就是答案
```

**原地修改版本（空间 O(1)）**：

```python
def minPathSum(grid: list[list[int]]) -> int:
    m, n = len(grid), len(grid[0])

    # 直接在 grid 上修改，grid[i][j] 变成"到达 (i,j) 的最小路径和"
    for i in range(m):
        for j in range(n):
            if i == 0 and j == 0:
                continue  # 起点不动
            elif i == 0:
                grid[i][j] += grid[i][j - 1]  # 第一行只能从左边来
            elif j == 0:
                grid[i][j] += grid[i - 1][j]  # 第一列只能从上面来
            else:
                grid[i][j] += min(grid[i - 1][j], grid[i][j - 1])

    return grid[m - 1][n - 1]
```

### 6. 套路总结
- 网格最短路径 = 网格路径数的"变体"，把加法换成 min
- 遇到网格 DP 题，先想"到达 `(i,j)` 能从哪里来"
- 求方案数 --> 把来源加起来
- 求最值 --> 把来源取 min/max，再加上当前代价
- 原地修改是常见的空间优化技巧

## 题目3：5. 最长回文子串

### 1. 核心考点
- **区间 DP**：状态定义在子串的两个端点上
- **回文的递归结构**：一个回文串去掉两端后仍然是回文
- **中心扩展法**（另一种常用解法）

### 2. 解题思路（给新手讲）

给你一个字符串，找出其中最长的回文子串（连续的一段）。

什么是回文？正着读和反着读一样，比如 "aba"、"abba"。

**DP 思路**：判断 `s[i..j]` 是否是回文，可以分解为：
- `s[i]` 和 `s[j]` 是否相等？
- 去掉两端后，`s[i+1..j-1]` 是否是回文？

如果两个条件都满足，那么 `s[i..j]` 就是回文。这就是回文的**递归结构**。

### 3. 从暴力到优化的推导

#### 暴力解法
枚举所有子串，逐个检查是否是回文。

```python
def longestPalindrome(s: str) -> str:
    n = len(s)
    result = ""

    for i in range(n):              # 枚举起点
        for j in range(i, n):       # 枚举终点
            sub = s[i:j + 1]
            if sub == sub[::-1] and len(sub) > len(result):
                result = sub        # 更新最长回文

    return result
```

时间复杂度：O(n^3)，枚举子串 O(n^2)，检查回文 O(n)。

#### 优化思路
暴力解法中，检查每个子串是否回文需要 O(n)。如果我们能利用已有结果，把检查降到 O(1) 就好了。

关键发现：`s[i..j]` 是否回文，取决于 `s[i+1..j-1]` 是否回文（更短的子串）。如果我们先算短子串，再算长子串，就能复用结果。

#### 最优解法

**状态定义**：`dp[i][j]` = `s[i..j]` 是否是回文串（True/False）

**转移方程**：
```
dp[i][j] = (s[i] == s[j]) and dp[i+1][j-1]
```

**边界条件**：
- 单个字符一定是回文：`dp[i][i] = True`
- 两个相邻字符：`dp[i][i+1] = (s[i] == s[i+1])`

**遍历顺序**：因为 `dp[i][j]` 依赖 `dp[i+1][j-1]`（更短的子串），所以需要**先算短子串，再算长子串**。可以按子串长度从小到大遍历。

### 4. 关键思维点
- **状态是布尔值**：`dp[i][j]` 存的是 True/False，不是数值。最终答案需要在所有 True 中找最长的
- **遍历顺序很关键**：必须保证算 `dp[i][j]` 时，`dp[i+1][j-1]` 已经算过。按长度从短到长遍历可以保证这一点
- **为什么 `j - i < 3` 时不需要检查内部**？因为去掉两端后剩 0 或 1 个字符，一定是回文
- **中心扩展法更优**：DP 是 O(n^2) 时间 + O(n^2) 空间，中心扩展法是 O(n^2) 时间 + O(1) 空间

### 5. 代码实现（Python）

**DP 解法**：

```python
def longestPalindrome(s: str) -> str:
    n = len(s)
    if n < 2:
        return s

    # dp[i][j] 表示 s[i..j] 是否是回文
    dp = [[False] * n for _ in range(n)]

    start = 0   # 记录最长回文子串的起始位置
    max_len = 1  # 记录最长回文子串的长度

    # 边界：每个单字符都是回文
    for i in range(n):
        dp[i][i] = True

    # 按子串长度从短到长遍历
    for length in range(2, n + 1):       # 子串长度从 2 到 n
        for i in range(n - length + 1):  # 起点 i
            j = i + length - 1           # 终点 j

            if s[i] != s[j]:
                dp[i][j] = False  # 两端不相等，不是回文
            else:
                if length <= 3:
                    # 长度为 2 或 3，两端相等就是回文
                    # 因为去掉两端后剩 0 或 1 个字符
                    dp[i][j] = True
                else:
                    # 两端相等，取决于内部是否回文
                    dp[i][j] = dp[i + 1][j - 1]

            # 如果是回文且更长，更新答案
            if dp[i][j] and length > max_len:
                start = i
                max_len = length

    return s[start:start + max_len]
```

**中心扩展法（推荐）**：

```python
def longestPalindrome(s: str) -> str:
    n = len(s)
    if n < 2:
        return s

    start = 0    # 最长回文子串的起始位置
    max_len = 1  # 最长回文子串的长度

    def expand(left, right):
        """从中心向两边扩展，找到最长的回文"""
        while left >= 0 and right < n and s[left] == s[right]:
            left -= 1   # 左边界往左扩
            right += 1  # 右边界往右扩
        # 循环结束时，left 和 right 各多走了一步
        # 回文串是 s[left+1 .. right-1]，长度是 right - left - 1
        return left + 1, right - left - 1

    for i in range(n):
        # 以 s[i] 为中心的奇数长度回文（如 "aba"）
        start1, len1 = expand(i, i)
        # 以 s[i] 和 s[i+1] 为中心的偶数长度回文（如 "abba"）
        start2, len2 = expand(i, i + 1)

        # 取更长的那个
        if len1 > max_len:
            start = start1
            max_len = len1
        if len2 > max_len:
            start = start2
            max_len = len2

    return s[start:start + max_len]
```

### 6. 套路总结
- **区间 DP** 的典型模式：状态是 `dp[i][j]`，表示子串/子区间 `[i, j]` 的某种属性
- 遍历顺序：**长度从小到大**，保证短子串先算
- 回文问题的本质：**两端相等 + 内部也是回文**
- 中心扩展法是回文子串问题的实用技巧，空间更优

## 题目4：1143. 最长公共子序列

### 1. 核心考点
- **双序列 DP**：两个字符串各一个维度，状态定义在两个位置上
- **子序列 vs 子串**：子序列可以不连续，子串必须连续
- **经典状态转移**：匹配/不匹配两种情况

### 2. 解题思路（给新手讲）

给两个字符串 `text1` 和 `text2`，找它们的最长公共子序列（LCS）的长度。

子序列是从字符串中**删掉一些字符（可以不删）**后剩下的字符按原顺序排列的结果。比如 "ace" 是 "abcde" 的子序列。

思路：比较两个字符串的每一个位置，对齐着来看。

想象两个指针分别指向 `text1` 和 `text2` 的某个位置：
- 如果当前字符相同，这个字符一定属于 LCS，两个指针都前进
- 如果不同，要么跳过 `text1` 的当前字符，要么跳过 `text2` 的当前字符，取更优的那个

### 3. 从暴力到优化的推导

#### 暴力解法
枚举 `text1` 的所有子序列（2^m 个），检查每个是否也是 `text2` 的子序列。

```python
def longestCommonSubsequence(text1: str, text2: str) -> int:
    def dfs(i, j):
        """返回 text1[i:] 和 text2[j:] 的 LCS 长度"""
        if i == len(text1) or j == len(text2):
            return 0  # 任一字符串用完，LCS 长度为 0

        if text1[i] == text2[j]:
            # 当前字符相同，属于 LCS，两边都前进
            return 1 + dfs(i + 1, j + 1)
        else:
            # 不同，尝试跳过 text1[i] 或跳过 text2[j]
            return max(dfs(i + 1, j), dfs(i, j + 1))

    return dfs(0, 0)
```

时间复杂度：O(2^(m+n))，指数级。

#### 优化思路
`dfs(i, j)` 会被大量重复调用。比如 `dfs(3, 4)` 可能从多条路径到达。加记忆化或改成 DP 表即可。

#### 最优解法

**状态定义**：`dp[i][j]` = `text1[0..i-1]` 和 `text2[0..j-1]` 的 LCS 长度

> 注意：这里用 `dp[i][j]` 表示前 i 个字符和前 j 个字符的 LCS，这样 `dp[0][j]` 和 `dp[i][0]` 自然就是 0（空串和任何串的 LCS 为 0），边界处理更方便。

**转移方程**：
```
如果 text1[i-1] == text2[j-1]：
    dp[i][j] = dp[i-1][j-1] + 1     -- 当前字符匹配，LCS 长度 +1
否则：
    dp[i][j] = max(dp[i-1][j], dp[i][j-1])  -- 跳过其中一个字符
```

**边界条件**：`dp[0][j] = dp[i][0] = 0`（空串的 LCS 为 0）

### 4. 关键思维点
- **状态定义用"前 i 个字符"而不是"下标 i"**：这样 `dp[0][...]` 天然代表空串，边界更清晰
- **匹配时为什么是 `dp[i-1][j-1] + 1`**？因为这个字符确定属于 LCS，所以在"两边都少一个字符"的基础上 +1
- **不匹配时为什么取 max**？因为不匹配时，当前两个字符至少有一个不属于 LCS，所以要尝试两种"跳过"的可能
- **这道题是很多 DP 题的基础**：编辑距离、最短编辑路径等都是 LCS 的变体

### 5. 代码实现（Python）

```python
def longestCommonSubsequence(text1: str, text2: str) -> int:
    m, n = len(text1), len(text2)

    # dp[i][j] = text1 前 i 个字符和 text2 前 j 个字符的 LCS 长度
    # 多开一行一列，dp[0][j] = dp[i][0] = 0 表示空串
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):        # 遍历 text1 的每个字符
        for j in range(1, n + 1):    # 遍历 text2 的每个字符
            if text1[i - 1] == text2[j - 1]:
                # 当前字符相同：这个字符加入 LCS
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                # 当前字符不同：跳过 text1[i-1] 或 text2[j-1]，取较大值
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]  # 答案：两个完整字符串的 LCS 长度
```

**空间优化版本**：

```python
def longestCommonSubsequence(text1: str, text2: str) -> int:
    m, n = len(text1), len(text2)

    # 只用一维数组
    dp = [0] * (n + 1)

    for i in range(1, m + 1):
        prev = 0  # 保存 dp[i-1][j-1]，即左上角的值
        for j in range(1, n + 1):
            temp = dp[j]  # 先保存，等下要用作下一轮的 prev
            if text1[i - 1] == text2[j - 1]:
                dp[j] = prev + 1   # 左上角 + 1
            else:
                dp[j] = max(dp[j], dp[j - 1])  # max(上方, 左方)
            prev = temp  # 更新 prev 为当前列的旧值

    return dp[n]
```

### 6. 套路总结
- **双序列 DP** 的经典模式：两个字符串，两个维度
- 状态定义用"前 i 个 / 前 j 个"，边界自然为 0
- 两个字符相同 --> 来自左上角（`dp[i-1][j-1]`）+ 1
- 两个字符不同 --> 来自上方或左方的较大/较小值
- LCS 是很多字符串 DP 问题的基础模板

## 题目5：72. 编辑距离

### 1. 核心考点
- **双序列 DP 的经典难题**：两个字符串之间的转换
- **三种操作的统一处理**：插入、删除、替换
- **状态转移的多方向来源**：`dp[i][j]` 可以从三个方向转移过来

### 2. 解题思路（给新手讲）

给两个字符串 `word1` 和 `word2`，可以对 `word1` 进行三种操作：插入一个字符、删除一个字符、替换一个字符。问最少需要多少次操作才能把 `word1` 变成 `word2`。

这是经典的**编辑距离**问题（也叫 Levenshtein 距离），在拼写检查、DNA 序列比对等领域广泛应用。

关键洞察：从后往前看两个字符串的最后一个字符
- 如果 `word1[i]` == `word2[j]`：不需要操作，问题缩小为 `word1[0..i-1]` 变成 `word2[0..j-1]`
- 如果不同，有三种选择：
  - **替换**：把 `word1[i]` 替换成 `word2[j]`，然后解决 `word1[0..i-1]` -> `word2[0..j-1]`
  - **删除**：删掉 `word1[i]`，然后解决 `word1[0..i-1]` -> `word2[0..j]`
  - **插入**：在 `word1` 末尾插入 `word2[j]`，然后解决 `word1[0..i]` -> `word2[0..j-1]`

取三种操作中的最小值。

### 3. 从暴力到优化的推导

#### 暴力解法
递归尝试所有操作组合。

```python
def minDistance(word1: str, word2: str) -> int:
    def dfs(i, j):
        """将 word1[0..i-1] 变成 word2[0..j-1] 的最少操作数"""
        if i == 0:
            return j  # word1 为空，需要插入 j 个字符
        if j == 0:
            return i  # word2 为空，需要删除 i 个字符

        if word1[i - 1] == word2[j - 1]:
            return dfs(i - 1, j - 1)  # 字符相同，不需要操作

        return 1 + min(
            dfs(i - 1, j - 1),  # 替换
            dfs(i - 1, j),      # 删除
            dfs(i, j - 1)       # 插入
        )

    return dfs(len(word1), len(word2))
```

时间复杂度：O(3^(m+n))，极慢。

#### 优化思路
重叠子问题非常严重，同一个 `(i, j)` 会被多次计算。用 DP 表消除重复。

#### 最优解法

**状态定义**：`dp[i][j]` = 将 `word1` 的前 i 个字符变成 `word2` 的前 j 个字符所需的最少操作数

**转移方程**：
```
如果 word1[i-1] == word2[j-1]：
    dp[i][j] = dp[i-1][j-1]          -- 字符相同，不需要额外操作

否则：
    dp[i][j] = 1 + min(
        dp[i-1][j-1],   -- 替换：把 word1[i-1] 换成 word2[j-1]
        dp[i-1][j],      -- 删除：删掉 word1[i-1]
        dp[i][j-1]       -- 插入：在 word1 中插入 word2[j-1]
    )
```

**边界条件**：
- `dp[0][j] = j`：空串变成长度为 j 的串，需要插入 j 次
- `dp[i][0] = i`：长度为 i 的串变成空串，需要删除 i 次

### 4. 关键思维点

- **三种操作对应三个方向**：
  - 替换 --> 来自左上角 `dp[i-1][j-1]` + 1
  - 删除 --> 来自上方 `dp[i-1][j]` + 1（word1 少了一个字符）
  - 插入 --> 来自左方 `dp[i][j-1]` + 1（word2 少了一个字符）

- **为什么"插入"对应 `dp[i][j-1]`**？插入一个字符后，word2 的最后一个字符被匹配了，问题变成 word1 前 i 个字符变成 word2 前 j-1 个字符

- **和 LCS 的关系**：LCS 的转移也是看两个字符是否相同，分"匹配"和"不匹配"两种情况。编辑距离是 LCS 的"加强版"，不匹配时多了一个替换操作（左上角来源）

- **边界很直观**：空串变成目标串，只能一个个插入；源串变成空串，只能一个个删除

### 5. 代码实现（Python）

```python
def minDistance(word1: str, word2: str) -> int:
    m, n = len(word1), len(word2)

    # dp[i][j] = word1 前 i 个字符变成 word2 前 j 个字符的最少操作数
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    # 边界：空串到长度为 j 的串，需要 j 次插入
    for j in range(n + 1):
        dp[0][j] = j

    # 边界：长度为 i 的串到空串，需要 i 次删除
    for i in range(m + 1):
        dp[i][0] = i

    # 填表
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i - 1] == word2[j - 1]:
                # 字符相同，不需要额外操作
                dp[i][j] = dp[i - 1][j - 1]
            else:
                # 字符不同，三种操作取最小
                dp[i][j] = 1 + min(
                    dp[i - 1][j - 1],  # 替换
                    dp[i - 1][j],      # 删除 word1[i-1]
                    dp[i][j - 1]       # 插入 word2[j-1]
                )

    return dp[m][n]
```

**空间优化版本**：

```python
def minDistance(word1: str, word2: str) -> int:
    m, n = len(word1), len(word2)

    # 只用一维数组 + 一个变量保存左上角
    dp = list(range(n + 1))  # dp[j] = j，初始化为第 0 行

    for i in range(1, m + 1):
        prev = dp[0]   # 保存左上角的值 dp[i-1][j-1]
        dp[0] = i       # dp[i][0] = i（边界条件）

        for j in range(1, n + 1):
            temp = dp[j]  # 保存当前值，下一轮用作 prev

            if word1[i - 1] == word2[j - 1]:
                dp[j] = prev  # 左上角
            else:
                dp[j] = 1 + min(
                    prev,       # 左上角（替换）
                    dp[j],      # 上方（删除），还没更新，存的是上一行的值
                    dp[j - 1]   # 左方（插入），已更新为当前行的值
                )

            prev = temp  # 更新 prev

    return dp[n]
```

### 6. 套路总结
- 编辑距离是**双序列 DP 的集大成者**，融合了三种操作
- 状态定义用"前 i 个 / 前 j 个"，边界清晰
- 三种操作 <-> 三个方向：左上（替换）、上（删除）、左（插入），每个 +1
- 字符相同时"免费"，直接取左上角，不用 +1
- 是 LCS 的升级版，LCS 只有"匹配/不匹配"两种情况，编辑距离在"不匹配"时多了一个方向

## 多维 DP 总结对比

| 题目 | 状态定义 | 转移方向 | 操作 |
|------|----------|----------|------|
| 不同路径 | `dp[i][j]` = 到达 (i,j) 的路径数 | 上 + 左 | 加法 |
| 最小路径和 | `dp[i][j]` = 到达 (i,j) 的最小和 | min(上, 左) + 权值 | 取 min |
| 最长回文子串 | `dp[i][j]` = s[i..j] 是否回文 | 左下角（更短子串） | 布尔判断 |
| 最长公共子序列 | `dp[i][j]` = 前 i 和前 j 的 LCS | 匹配: 左上+1; 不匹配: max(上,左) | 取 max |
| 编辑距离 | `dp[i][j]` = 前 i 变成前 j 的最少操作 | min(左上, 上, 左) + 1 | 取 min |

**通用思路**：
1. 明确"两个维度分别代表什么"
2. 画出 DP 表，手动填几个格子找规律
3. 确定遍历方向（保证依赖的状态已算过）
4. 思考空间优化（滚动数组、原地修改）
