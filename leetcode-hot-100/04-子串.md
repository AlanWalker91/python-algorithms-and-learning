# 子串

子串问题的核心是处理「连续」的子数组或子字符串。常用技巧包括前缀和、单调队列、滑动窗口等，关键在于识别问题特征并选择合适的数据结构。

---

## 题目1：560. 和为K的子数组

### 1. 核心考点

**前缀和 + 哈希表**。本题考查的是如何高效地统计和为 K 的连续子数组个数。这不是一道典型的滑动窗口题（因为数组元素可以为负数，窗口没有单调性），而是利用前缀和将「区间和」转化为「两个前缀和之差」，再用哈希表快速查找。

### 2. 解题思路（给新手讲）

假设你在记账，每天记录收入或支出（可正可负）。你想知道有多少段「连续的天数」，总收支恰好等于 K。

一个笨办法是把每种连续天数的组合都加一遍，但这太慢了。

聪明的办法是：你先算出从第 1 天到每一天的「累计总额」（前缀和）。比如：
- 到第 3 天累计 10 元
- 到第 7 天累计 17 元
- 那么第 4 天到第 7 天这段时间的总收支 = 17 - 10 = 7 元

所以问题变成了：在所有的「累计总额」中，有多少对 (i, j) 满足 `prefix[j] - prefix[i] = K`？

等价于：对于每个 `prefix[j]`，有多少个之前的 `prefix[i]` 等于 `prefix[j] - K`？

这就可以用哈希表边走边查了！就像你拿着一个本子，每算出一个累计总额就记下来，同时往回翻本子看有没有正好差 K 的记录。

### 3. 从暴力到优化的推导

#### 暴力解法

枚举所有子数组的起点和终点，逐个计算子数组和：

```python
def subarraySum(nums: list, k: int) -> int:
    n = len(nums)
    count = 0
    for i in range(n):        # 枚举起点
        total = 0
        for j in range(i, n): # 枚举终点
            total += nums[j]
            if total == k:
                count += 1
    return count
```

时间复杂度：O(n^2)，空间复杂度：O(1)。

#### 优化思路

先定义前缀和：`prefix[j] = nums[0] + nums[1] + ... + nums[j-1]`。

那么子数组 `nums[i..j-1]` 的和 = `prefix[j] - prefix[i]`。

我们要找满足 `prefix[j] - prefix[i] = k` 的 (i, j) 对数，即 `prefix[i] = prefix[j] - k`。

遍历到位置 j 时，只要知道之前有多少个前缀和等于 `prefix[j] - k`，就能得到以 j 结尾的满足条件的子数组个数。

用哈希表存储「每个前缀和出现的次数」，就能在 O(1) 时间内查到。

**为什么不能用滑动窗口？** 因为数组元素可以为负数。窗口扩大时和可能减小，窗口缩小时和可能增大，没有单调性，无法决定何时移动左指针。

#### 最优解法

一次遍历，维护前缀和及其出现次数的哈希表。

时间复杂度：O(n)，空间复杂度：O(n)。

### 4. 关键思维点

1. **前缀和的转化**：子数组和 = 两个前缀和之差，这是处理「区间和」问题的基本功
2. **哈希表的角色**：存储历史前缀和的出现次数，实现 O(1) 查找
3. **初始化 `{0: 1}`**：前缀和为 0 出现 1 次，对应从数组开头到当前位置的子数组和恰好为 k 的情况
4. **与 Two Sum 的联系**：本质上是前缀和数组上的 Two Sum 问题（找两数之差等于 k）
5. **不能用滑动窗口**：元素可以为负数，窗口和没有单调性

### 5. 代码实现（Python）

```python
def subarraySum(nums: list, k: int) -> int:
    # 哈希表：前缀和 -> 出现次数
    # 初始化 {0: 1}，表示前缀和为 0 出现了 1 次
    # （这处理了从下标 0 开始的子数组和恰好为 k 的情况）
    prefix_count = {0: 1}
    # 当前前缀和
    prefix_sum = 0
    # 结果计数
    count = 0

    for num in nums:
        # 更新前缀和
        prefix_sum += num

        # 查找：之前有多少个前缀和等于 prefix_sum - k
        # 如果存在，说明从那些位置到当前位置的子数组和为 k
        if prefix_sum - k in prefix_count:
            count += prefix_count[prefix_sum - k]

        # 将当前前缀和记入哈希表
        prefix_count[prefix_sum] = prefix_count.get(prefix_sum, 0) + 1

    return count
```

### 6. 套路总结

**识别信号**：题目涉及「连续子数组的和」，且数组元素可以为负数。

**前缀和 + 哈希表模板**：

```python
def prefix_sum_pattern(nums, target):
    prefix_count = {0: 1}  # 前缀和 -> 出现次数
    prefix_sum = 0
    ans = 0

    for num in nums:
        prefix_sum += num
        # 查找是否存在满足条件的历史前缀和
        if prefix_sum - target in prefix_count:
            ans += prefix_count[prefix_sum - target]
        # 记录当前前缀和
        prefix_count[prefix_sum] = prefix_count.get(prefix_sum, 0) + 1

    return ans
```

**适用场景**：
- 和为 K 的子数组个数（本题）
- 和能被 K 整除的子数组（LeetCode 974）
- 0 和 1 个数相等的最长子数组（LeetCode 525，把 0 变成 -1）

## 题目2：239. 滑动窗口最大值

### 1. 核心考点

**单调队列（单调递减双端队列）**。本题考查的是如何在 O(1) 时间内获取固定大小窗口中的最大值。这是单调队列最经典的应用场景。

### 2. 解题思路（给新手讲）

想象一个窗户（宽度为 k），在一排高低不同的柱子前从左往右滑动。每滑到一个位置，你要报告窗户里最高的柱子有多高。

最笨的方法：每滑一步，把窗户里所有柱子都看一遍找最高的，这要 O(k) 时间。

聪明的方法：你维护一个"候选名单"。这个名单有一个规则 -- **如果新来的柱子比名单末尾的柱子还高，那名单末尾那些矮柱子永远不可能成为最大值了**（因为新柱子比它们高，还比它们晚离开窗口），直接淘汰它们。

这个候选名单就是「单调递减队列」：
- 队列从头到尾，元素递减
- 队头永远是当前窗口的最大值
- 新元素进来时，从队尾淘汰所有比它小的元素
- 窗口滑动时，如果队头元素已经不在窗口内，从队头移除

就像一个擂台赛：新来的选手会把所有比自己弱的选手都淘汰掉，但不会挑战比自己强的选手。

### 3. 从暴力到优化的推导

#### 暴力解法

对每个窗口位置，扫描窗口内的所有元素取最大值：

```python
def maxSlidingWindow(nums: list, k: int) -> list:
    n = len(nums)
    ans = []
    for i in range(n - k + 1):
        ans.append(max(nums[i:i+k]))  # 每个窗口取 max
    return ans
```

时间复杂度：O(n * k)，当 k 很大时效率低下。

#### 优化思路

暴力解法的问题：每个窗口都重新扫描 k 个元素找最大值，大量重复计算。

思考：窗口滑动一步，只有一个元素进来、一个元素出去。能否维护某种数据结构，让"查询最大值"和"更新"都很快？

- **堆（优先队列）**：可以 O(1) 取最大值，但删除非堆顶元素是 O(k) 的，需要延迟删除，复杂度 O(n log n)
- **单调队列**：每个元素最多入队一次、出队一次，总复杂度 O(n)

单调队列的关键insight：如果一个元素 `nums[i]` 比它后面进入窗口的元素 `nums[j]`（j > i）小，那么只要 `nums[j]` 在窗口内，`nums[i]` 就永远不可能成为最大值。所以 `nums[i]` 可以安全淘汰。

#### 最优解法

使用双端队列（deque），维护一个递减序列的下标队列。

时间复杂度：O(n)，每个元素最多入队出队各一次。空间复杂度：O(k)。

### 4. 关键思维点

1. **存下标而非存值**：队列中存储元素的下标，便于判断队头元素是否已滑出窗口
2. **单调性的维护**：新元素入队前，从队尾弹出所有比它小的元素（它们永远不可能成为答案了）
3. **淘汰的合理性**：如果 `nums[j] >= nums[i]` 且 `j > i`，那么 `nums[i]` 在 `nums[j]` 存在期间永远不会是最大值，因为 `nums[j]` 更大且更晚离开窗口
4. **队列大小不固定**：队列中的元素个数不一定等于 k，可能只有 1 个（当前窗口中某个元素特别大）

### 5. 代码实现（Python）

```python
from collections import deque

def maxSlidingWindow(nums: list, k: int) -> list:
    n = len(nums)
    # 双端队列，存储元素下标，保持对应值单调递减
    dq = deque()
    ans = []

    for i in range(n):
        # --- 步骤1：移除队头过期元素 ---
        # 如果队头下标已经不在窗口 [i-k+1, i] 内，弹出
        if dq and dq[0] < i - k + 1:
            dq.popleft()

        # --- 步骤2：维护单调递减性 ---
        # 从队尾弹出所有比 nums[i] 小的元素
        # 因为这些元素在 nums[i] 存在期间不可能成为最大值
        while dq and nums[dq[-1]] <= nums[i]:
            dq.pop()

        # --- 步骤3：当前元素下标入队 ---
        dq.append(i)

        # --- 步骤4：记录答案 ---
        # 当窗口大小达到 k 时（i >= k-1），队头就是窗口最大值
        if i >= k - 1:
            ans.append(nums[dq[0]])

    return ans
```

### 6. 套路总结

**识别信号**：题目涉及「固定窗口大小」+「求窗口内的最大/最小值」。

**单调队列模板**：

```python
from collections import deque

def monotonic_queue_pattern(nums, k):
    dq = deque()  # 存下标，维护单调递减（求最大值）或单调递增（求最小值）
    ans = []

    for i in range(len(nums)):
        # 1. 移除过期元素（队头不在窗口内）
        if dq and dq[0] < i - k + 1:
            dq.popleft()

        # 2. 维护单调性（从队尾弹出不满足单调性的元素）
        while dq and nums[dq[-1]] <= nums[i]:  # 求最小值时改为 >=
            dq.pop()

        # 3. 入队
        dq.append(i)

        # 4. 窗口形成后记录答案
        if i >= k - 1:
            ans.append(nums[dq[0]])

    return ans
```

**单调队列 vs 单调栈**：
- **单调队列**：两头都能操作，适合滑动窗口问题，关注区间最值
- **单调栈**：只操作栈顶，适合找「下一个更大/更小元素」

## 题目3：76. 最小覆盖子串

### 1. 核心考点

**变长滑动窗口 + 字符频次匹配**。本题是滑动窗口的高难度应用，考查如何用双指针动态调整窗口大小，在满足「包含目标所有字符」的前提下找到最短的子串。

### 2. 解题思路（给新手讲）

想象你在一条很长的彩色珠子项链上找一段最短的区间，使得这段区间内包含了所有你需要的颜色（每种颜色可能需要多颗）。

你的策略是：
1. **先扩大范围**：右手不断往右拉，把新的珠子纳入视野，直到你集齐了所有需要的颜色
2. **再缩小范围**：集齐之后，试着把左手往右收，丢掉一些多余的珠子，看看能不能在保持集齐的前提下缩短区间
3. **记录最短的那段**：每次集齐状态下，记录当前区间长度，留下最短的

右手负责"寻找可行解"，左手负责"优化可行解"。这就是滑动窗口求最小值的经典模式。

### 3. 从暴力到优化的推导

#### 暴力解法

枚举所有子串，检查每个子串是否包含 t 的所有字符：

```python
from collections import Counter

def minWindow(s: str, t: str) -> str:
    t_count = Counter(t)
    min_len = float('inf')
    result = ""

    for i in range(len(s)):
        for j in range(i + len(t), len(s) + 1):
            sub = s[i:j]
            sub_count = Counter(sub)
            # 检查 sub 是否包含 t 的所有字符
            if all(sub_count.get(c, 0) >= t_count[c] for c in t_count):
                if j - i < min_len:
                    min_len = j - i
                    result = sub
                break  # 找到最短的合法子串就跳出内层
    return result
```

时间复杂度：O(n^2 * m)，非常慢。

#### 优化思路

观察暴力解法中的冗余：
1. 内层循环每次都重新统计字符频次 -- 可以用增量更新
2. 找到一个合法子串后，起点直接跳到 `i+1` -- 但其实左边界可以更精确地收缩

滑动窗口的思路：
- 右指针扩展，直到窗口包含 t 的所有字符（找到可行解）
- 左指针收缩，在保持可行的前提下尽量缩短窗口（优化可行解）
- 记录过程中最短的可行窗口

如何高效判断「窗口是否包含 t 的所有字符」？
- 维护两个频次表：`need`（t 中每个字符需要的个数）和 `window`（当前窗口中的个数）
- 用一个计数器 `formed` 记录"已满足的字符种类数"
- 当 `formed == 需要的种类数` 时，窗口合法

#### 最优解法

双指针滑动窗口，右指针扩展找到可行解，左指针收缩优化可行解。

时间复杂度：O(n + m)，其中 n 是 s 的长度，m 是 t 的长度。空间复杂度：O(字符集大小)。

### 4. 关键思维点

1. **求最小用外层收缩**：窗口合法时收缩左边界（用 while），不合法时扩展右边界。和求最大值时相反（求最大值是不合法时收缩，合法时扩展）
2. **formed 计数器**：避免每次都比较两个频次表，只在字符频次达标/不达标的"边界时刻"更新
3. **t 中可能有重复字符**：比如 t = "AABC"，窗口中 A 必须出现至少 2 次
4. **空字符串的处理**：如果 s 比 t 短，或者 s 中根本凑不齐 t 的所有字符，返回空串
5. **和 438 题的对比**：438 是定长窗口找异位词，76 是变长窗口找最小覆盖，核心的频次匹配逻辑相似

### 5. 代码实现（Python）

```python
from collections import Counter

def minWindow(s: str, t: str) -> str:
    if not s or not t or len(s) < len(t):
        return ""

    # need: t 中每个字符需要的个数
    need = Counter(t)
    # 需要满足的字符种类数
    required = len(need)

    # 窗口内各字符的频次
    window = {}
    # 已满足的字符种类数
    formed = 0

    # 记录最优答案：(窗口长度, 左边界, 右边界)
    ans = (float('inf'), 0, 0)

    # 左指针
    left = 0

    for right in range(len(s)):
        # --- 扩展：右边界字符进入窗口 ---
        ch = s[right]
        window[ch] = window.get(ch, 0) + 1

        # 如果该字符在 t 中，且窗口中的数量刚好达到 t 中要求的数量
        if ch in need and window[ch] == need[ch]:
            formed += 1

        # --- 收缩：当窗口已经包含 t 的所有字符时，尝试缩小窗口 ---
        while formed == required:
            # 更新最优答案
            if right - left + 1 < ans[0]:
                ans = (right - left + 1, left, right)

            # 尝试移除左边界字符
            left_ch = s[left]
            window[left_ch] -= 1
            # 如果移除后该字符不再满足要求
            if left_ch in need and window[left_ch] < need[left_ch]:
                formed -= 1

            # 左指针右移
            left += 1

    # 返回结果
    return "" if ans[0] == float('inf') else s[ans[1]:ans[2] + 1]
```

### 6. 套路总结

**识别信号**：题目要求「包含/覆盖某些条件的最短子串/最小窗口」。

**求最小值的滑动窗口模板**：

```python
def min_window_pattern(s, condition):
    left = 0
    ans = float('inf')  # 或记录最小区间

    for right in range(len(s)):
        # 1. 扩展：将 s[right] 加入窗口

        # 2. 收缩：当窗口满足条件时，尝试缩小
        while 窗口满足条件:
            ans = min(ans, right - left + 1)  # 更新答案
            # 将 s[left] 移出窗口
            left += 1

    return ans
```

**求最小 vs 求最大的区别**：

| 场景 | 何时更新答案 | 何时收缩窗口 |
|------|------------|------------|
| 求最大值（如第3题） | 窗口合法时更新 | 窗口不合法时收缩（while 不合法） |
| 求最小值（如本题） | 收缩前更新 | 窗口合法时收缩（while 合法） |

**滑动窗口三大经典**：
1. **无重复最长子串**（第3题）：变长窗口求最大，set 维护状态
2. **字母异位词**（第438题）：定长窗口，频次比较
3. **最小覆盖子串**（本题）：变长窗口求最小，频次匹配 + formed 计数器
