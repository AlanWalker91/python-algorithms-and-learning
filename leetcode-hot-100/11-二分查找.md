# 二分查找

二分查找是一种在**有序**数据中高效搜索的算法，核心思想是每次排除一半的搜索空间，将时间复杂度从 O(n) 降低到 O(log n)。

---

## 题目1：35. 搜索插入位置

### 1. 核心考点

- 标准二分查找模板
- 理解二分查找结束时左右指针的含义
- 找到"第一个大于等于 target 的位置"

### 2. 解题思路（给新手讲）

给你一个**排好序**的数组和一个目标值 target：
- 如果 target 在数组中，返回它的下标
- 如果 target 不在数组中，返回它**应该被插入的位置**

本质上就是在找：**第一个 >= target 的元素的位置**。

举个例子：`nums = [1, 3, 5, 6]`，`target = 5`，那 5 在下标 2，返回 2。
如果 `target = 2`，2 应该插在 1 和 3 之间，返回下标 1。

### 3. 从暴力到优化的推导

#### 暴力解法

从头到尾扫一遍，找到第一个 >= target 的位置：

```python
def searchInsert(nums, target):
    for i in range(len(nums)):           # 逐个检查每个元素
        if nums[i] >= target:            # 找到第一个 >= target 的位置
            return i                     # 直接返回
    return len(nums)                     # 如果都比 target 小，插到末尾
```

时间复杂度 O(n)，对于有序数组太浪费了。

#### 优化思路

数组是**有序**的，天然适合二分查找：
- 每次看中间元素，如果中间元素 >= target，答案在左半边（包含 mid）
- 如果中间元素 < target，答案在右半边
- 每次排除一半，O(log n) 就能找到

#### 最优解法

```python
def searchInsert(nums, target):
    left, right = 0, len(nums) - 1       # 初始化搜索范围 [left, right]
    while left <= right:                  # 当搜索范围非空时继续
        mid = left + (right - left) // 2  # 计算中间位置（防溢出写法）
        if nums[mid] >= target:           # 中间值 >= target
            right = mid - 1              # 答案可能是 mid 或更左，缩小右边界
        else:                            # 中间值 < target
            left = mid + 1              # 答案一定在 mid 右边，缩小左边界
    return left                          # 循环结束时 left 就是插入位置
```

时间复杂度 O(log n)，空间复杂度 O(1)。

### 4. 关键思维点

1. **为什么返回 left？** 循环结束时 `left = right + 1`，left 指向的就是"第一个 >= target 的位置"。如果所有元素都 < target，left 会等于 `len(nums)`，正好是末尾插入位置。
2. **`left + (right - left) // 2` vs `(left + right) // 2`**：前者防止整数溢出，Python 中虽然不会溢出，但这是好习惯。
3. **二分查找的本质**：不是"查找某个值"，而是"在有序空间中，利用单调性每次排除一半"。

### 5. 代码实现（Python）

```python
class Solution:
    def searchInsert(self, nums: list[int], target: int) -> int:
        left, right = 0, len(nums) - 1   # 搜索区间 [left, right]

        while left <= right:              # 区间非空就继续搜索
            mid = left + (right - left) // 2  # 取中间位置

            if nums[mid] == target:       # 正好找到目标值
                return mid                # 直接返回下标
            elif nums[mid] < target:      # 中间值太小
                left = mid + 1            # 去右半边找
            else:                         # 中间值太大
                right = mid - 1           # 去左半边找

        return left                       # 没找到时，left 就是插入位置
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 适用场景 | 有序数组中查找目标位置 |
| 核心模板 | `while left <= right` + 收缩边界 |
| 返回值含义 | `left` = 第一个 >= target 的位置 |
| 时间复杂度 | O(log n) |
| 易错点 | 边界条件：`left <= right` 还是 `left < right`，取决于搜索区间定义 |

## 题目2：74. 搜索二维矩阵

### 1. 核心考点

- 将二维矩阵"展平"为一维有序数组的思维
- 二分查找中一维下标与二维坐标的转换
- 利用矩阵的有序性质

### 2. 解题思路（给新手讲）

给你一个矩阵，每行从左到右递增，且每行的第一个元素大于上一行的最后一个元素。判断矩阵中是否存在目标值。

关键观察：如果把这个矩阵一行行拼起来，就是一个**完全有序的一维数组**！

比如：
```
[1,  3,  5,  7]
[10, 11, 16, 20]
[23, 30, 34, 60]
```
拼起来就是 `[1, 3, 5, 7, 10, 11, 16, 20, 23, 30, 34, 60]`。

我们不需要真的拼起来，只需要用数学公式把一维下标转换为二维坐标就行。

### 3. 从暴力到优化的推导

#### 暴力解法

```python
def searchMatrix(matrix, target):
    for row in matrix:                    # 遍历每一行
        for val in row:                   # 遍历每个元素
            if val == target:             # 找到目标值
                return True
    return False                          # 遍历完没找到
```

时间复杂度 O(m * n)，完全没利用有序性。

#### 优化思路

把矩阵看作一维有序数组，长度为 `m * n`：
- 一维下标 `idx` 对应的二维坐标是 `(idx // n, idx % n)`
- 其中 `n` 是列数
- 然后就是标准的一维二分查找

#### 最优解法

```python
def searchMatrix(matrix, target):
    m, n = len(matrix), len(matrix[0])    # m 行 n 列
    left, right = 0, m * n - 1           # 把矩阵看作长度为 m*n 的一维数组

    while left <= right:                  # 标准二分查找
        mid = left + (right - left) // 2  # 一维中间下标
        row = mid // n                   # 转换为行号
        col = mid % n                    # 转换为列号
        val = matrix[row][col]           # 取出对应的值

        if val == target:                # 找到目标
            return True
        elif val < target:               # 目标在右边
            left = mid + 1
        else:                            # 目标在左边
            right = mid - 1

    return False                         # 没找到
```

时间复杂度 O(log(m * n))，空间复杂度 O(1)。

### 4. 关键思维点

1. **一维与二维的转换公式**：`row = idx // n, col = idx % n`。这是处理二维问题的通用技巧。
2. **为什么能展平？** 因为题目保证了"每行第一个元素 > 上一行最后一个元素"，所以整个矩阵拼起来是严格递增的。
3. **另一种思路**：先二分找到 target 可能在哪一行（比较每行首元素），再在那一行内二分查找。两次二分，复杂度也是 O(log m + log n) = O(log(mn))。

### 5. 代码实现（Python）

```python
class Solution:
    def searchMatrix(self, matrix: list[list[int]], target: int) -> bool:
        m, n = len(matrix), len(matrix[0])    # 获取矩阵的行数和列数
        left, right = 0, m * n - 1           # 虚拟一维数组的首尾下标

        while left <= right:
            mid = left + (right - left) // 2  # 一维中间下标
            # 将一维下标转换为二维坐标
            val = matrix[mid // n][mid % n]   # mid // n 是行号，mid % n 是列号

            if val == target:
                return True                   # 找到了
            elif val < target:
                left = mid + 1                # 目标在右半部分
            else:
                right = mid - 1               # 目标在左半部分

        return False                          # 搜索结束，未找到
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 适用场景 | 行列均有序且行间递增的矩阵搜索 |
| 核心技巧 | 二维展平为一维 + 标准二分 |
| 坐标转换 | `row = idx // 列数, col = idx % 列数` |
| 时间复杂度 | O(log(m*n)) |
| 易错点 | 注意是除以**列数** n，不是行数 m |

## 题目3：34. 在排序数组中查找元素的第一个和最后一个位置

### 1. 核心考点

- 二分查找的两个变体：查找左边界、查找右边界
- 理解"找第一个等于 target 的"和"找最后一个等于 target 的"区别
- 二分查找中如何处理重复元素

### 2. 解题思路（给新手讲）

给你一个有序数组（可能有重复元素），要你找到 target 第一次出现和最后一次出现的位置。

比如 `nums = [5, 7, 7, 8, 8, 10]`，`target = 8`，返回 `[3, 4]`。

普通二分找到 8 就停了，但这里可能有多个 8。我们需要两次二分：
1. 第一次找**左边界**：第一个等于 target 的位置
2. 第二次找**右边界**：最后一个等于 target 的位置

### 3. 从暴力到优化的推导

#### 暴力解法

```python
def searchRange(nums, target):
    first, last = -1, -1
    for i in range(len(nums)):            # 从左往右扫
        if nums[i] == target:
            if first == -1:               # 记录第一次出现的位置
                first = i
            last = i                      # 不断更新最后一次出现的位置
    return [first, last]
```

时间复杂度 O(n)，没利用有序性。

#### 优化思路

分两次二分：
- **找左边界**：当 `nums[mid] == target` 时，不急着返回，继续往左找（`right = mid - 1`）
- **找右边界**：当 `nums[mid] == target` 时，不急着返回，继续往右找（`left = mid + 1`）

#### 最优解法

```python
def searchRange(nums, target):
    def find_left(nums, target):
        """找到 target 第一次出现的位置"""
        left, right = 0, len(nums) - 1
        result = -1                       # 默认没找到
        while left <= right:
            mid = left + (right - left) // 2
            if nums[mid] == target:
                result = mid              # 记录当前位置
                right = mid - 1           # 继续往左找，看有没有更早出现的
            elif nums[mid] < target:
                left = mid + 1
            else:
                right = mid - 1
        return result

    def find_right(nums, target):
        """找到 target 最后一次出现的位置"""
        left, right = 0, len(nums) - 1
        result = -1
        while left <= right:
            mid = left + (right - left) // 2
            if nums[mid] == target:
                result = mid              # 记录当前位置
                left = mid + 1            # 继续往右找，看有没有更晚出现的
            elif nums[mid] < target:
                left = mid + 1
            else:
                right = mid - 1
        return result

    return [find_left(nums, target), find_right(nums, target)]
```

时间复杂度 O(log n)，空间复杂度 O(1)。

### 4. 关键思维点

1. **找到了不要停**：普通二分找到 target 就返回了，但找边界时要继续收缩，用一个变量 `result` 记录最新找到的位置。
2. **左边界 vs 右边界的区别**：仅在 `nums[mid] == target` 时不同——找左边界往左收缩 `right = mid - 1`，找右边界往右收缩 `left = mid + 1`。
3. **另一种理解**：左边界 = "第一个 >= target 的位置"，右边界 = "最后一个 <= target 的位置"。

### 5. 代码实现（Python）

```python
class Solution:
    def searchRange(self, nums: list[int], target: int) -> list[int]:
        def binary_search(find_first: bool) -> int:
            """
            find_first=True 时找左边界，find_first=False 时找右边界
            """
            left, right = 0, len(nums) - 1
            result = -1                       # 初始化为 -1 表示没找到

            while left <= right:
                mid = left + (right - left) // 2

                if nums[mid] == target:
                    result = mid              # 找到了先记录
                    if find_first:
                        right = mid - 1       # 找左边界：继续往左
                    else:
                        left = mid + 1        # 找右边界：继续往右
                elif nums[mid] < target:
                    left = mid + 1            # 目标在右边
                else:
                    right = mid - 1           # 目标在左边

            return result

        return [binary_search(True), binary_search(False)]
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 适用场景 | 有序数组中有重复元素，找第一个/最后一个位置 |
| 核心技巧 | 找到 target 不停，继续收缩边界 |
| 左边界 | `nums[mid] == target` 时 `right = mid - 1` |
| 右边界 | `nums[mid] == target` 时 `left = mid + 1` |
| 时间复杂度 | O(log n)，两次二分 |
| 易错点 | 别忘了用 result 变量记录，不能直接 return mid |

## 题目4：33. 搜索旋转排序数组

### 1. 核心考点

- 在"部分有序"的数组中使用二分查找
- 判断哪一半是有序的
- 在有序的那一半中判断 target 是否在范围内

### 2. 解题思路（给新手讲）

一个原本有序的数组被"旋转"了，比如 `[0,1,2,4,5,6,7]` 变成 `[4,5,6,7,0,1,2]`。要在其中找 target。

旋转之后，数组被分成了**两段有序的部分**。虽然整体不有序了，但二分查找每次看中间点时，**至少有一半是有序的**。

我们可以利用有序的那一半来判断 target 在哪边：
- 如果左半边有序，检查 target 是否落在左半边的范围内
- 如果右半边有序，检查 target 是否落在右半边的范围内

### 3. 从暴力到优化的推导

#### 暴力解法

```python
def search(nums, target):
    for i in range(len(nums)):            # 直接线性扫描
        if nums[i] == target:
            return i
    return -1
```

时间复杂度 O(n)。

#### 优化思路

虽然整体不有序，但旋转数组有一个重要性质：**以 mid 为界，左右两半至少有一半是有序的**。

判断方法：
- 如果 `nums[left] <= nums[mid]`，说明左半边 `[left, mid]` 是有序的
- 否则右半边 `[mid, right]` 是有序的

然后在有序的那一半中判断 target 是否在其中：
- 在的话，往那边搜索
- 不在的话，往另一边搜索

#### 最优解法

```python
def search(nums, target):
    left, right = 0, len(nums) - 1

    while left <= right:
        mid = left + (right - left) // 2

        if nums[mid] == target:           # 找到目标
            return mid

        # 判断哪一半是有序的
        if nums[left] <= nums[mid]:       # 左半边有序
            # target 在左半边有序区间内吗？
            if nums[left] <= target < nums[mid]:
                right = mid - 1           # 在左半边，缩小右边界
            else:
                left = mid + 1            # 不在左半边，去右半边找
        else:                             # 右半边有序
            # target 在右半边有序区间内吗？
            if nums[mid] < target <= nums[right]:
                left = mid + 1            # 在右半边，缩小左边界
            else:
                right = mid - 1           # 不在右半边，去左半边找

    return -1
```

时间复杂度 O(log n)，空间复杂度 O(1)。

### 4. 关键思维点

1. **核心判断：`nums[left] <= nums[mid]`**。注意这里是 `<=` 不是 `<`，因为当 `left == mid` 时（只有两个元素），左半边只有一个元素，也算有序。
2. **在有序半边中用范围判断**：比如左半边有序时，检查 `nums[left] <= target < nums[mid]`。注意左边取 `<=`，右边取 `<`（因为 mid 已经检查过不等于 target）。
3. **为什么不能先找旋转点再二分？** 可以，但那需要两次二分。这种方法一次二分就搞定，更优雅。

### 5. 代码实现（Python）

```python
class Solution:
    def search(self, nums: list[int], target: int) -> int:
        left, right = 0, len(nums) - 1

        while left <= right:
            mid = left + (right - left) // 2

            if nums[mid] == target:           # 命中目标
                return mid

            # 步骤1：判断哪半边有序
            if nums[left] <= nums[mid]:       # 左半边 [left..mid] 有序
                # 步骤2：target 在有序的左半边吗？
                if nums[left] <= target < nums[mid]:
                    right = mid - 1           # 是，去左半边
                else:
                    left = mid + 1            # 否，去右半边
            else:                             # 右半边 [mid..right] 有序
                # 步骤2：target 在有序的右半边吗？
                if nums[mid] < target <= nums[right]:
                    left = mid + 1            # 是，去右半边
                else:
                    right = mid - 1           # 否，去左半边

        return -1                             # 未找到
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 适用场景 | 旋转排序数组（无重复元素）中搜索 |
| 核心技巧 | 先判断哪半边有序，再判断 target 在不在有序那半边 |
| 判断有序 | `nums[left] <= nums[mid]` 则左半有序 |
| 时间复杂度 | O(log n) |
| 易错点 | 边界判断的 `<=` 和 `<` 要搞清楚 |

## 题目5：153. 寻找旋转排序数组中的最小值

### 1. 核心考点

- 旋转排序数组的性质
- 二分查找找"转折点"
- 比较 `nums[mid]` 与 `nums[right]` 的关系

### 2. 解题思路（给新手讲）

一个有序数组被旋转了若干次（比如 `[1,2,3,4,5]` 变成 `[3,4,5,1,2]`），找最小值。

最小值就在**旋转的"断点"**处。比如 `[3,4,5,1,2]`，从 5 到 1 有一个"跳崖"，1 就是最小值。

如果没有旋转（比如 `[1,2,3,4,5]`），最小值就是第一个元素。

### 3. 从暴力到优化的推导

#### 暴力解法

```python
def findMin(nums):
    return min(nums)                      # 直接用内置函数，O(n)
```

或者：

```python
def findMin(nums):
    for i in range(1, len(nums)):         # 找到第一个比前一个小的元素
        if nums[i] < nums[i - 1]:
            return nums[i]
    return nums[0]                        # 没找到说明没旋转，返回首元素
```

时间复杂度 O(n)。

#### 优化思路

关键观察：比较 `nums[mid]` 和 `nums[right]`：
- 如果 `nums[mid] > nums[right]`：说明最小值在 mid 的右边（mid 处于旋转的"高处"）
- 如果 `nums[mid] <= nums[right]`：说明最小值在 mid 或 mid 的左边（mid 处于"低处"）

为什么不和 `nums[left]` 比？因为和 `nums[right]` 比可以更直接地判断 mid 和最小值的位置关系。

#### 最优解法

```python
def findMin(nums):
    left, right = 0, len(nums) - 1

    while left < right:                   # 注意：这里是 < 不是 <=
        mid = left + (right - left) // 2

        if nums[mid] > nums[right]:       # mid 在"高处"，最小值在右半边
            left = mid + 1               # 最小值一定不是 mid，跳过
        else:                            # mid 在"低处"，最小值是 mid 或在左半边
            right = mid                  # mid 可能就是最小值，不能跳过

    return nums[left]                    # left == right 时就是最小值
```

时间复杂度 O(log n)，空间复杂度 O(1)。

### 4. 关键思维点

1. **为什么用 `while left < right` 而不是 `<=`？** 因为我们最终要找到一个确定的位置（最小值），`left == right` 时就找到了。如果用 `<=` 会死循环。
2. **为什么和 `nums[right]` 比？** 如果和 `nums[left]` 比，当 `nums[mid] > nums[left]` 时，无法判断最小值在哪边（可能根本没旋转）。而和 `nums[right]` 比可以准确判断。
3. **`nums[mid] > nums[right]` 时为什么 `left = mid + 1`？** 因为 mid 比右边大，说明 mid 不可能是最小值，可以安全跳过。
4. **else 时为什么 `right = mid` 而不是 `mid - 1`？** 因为 mid 本身可能就是最小值，不能跳过。

### 5. 代码实现（Python）

```python
class Solution:
    def findMin(self, nums: list[int]) -> int:
        left, right = 0, len(nums) - 1

        while left < right:                   # 搜索区间 [left, right]
            mid = left + (right - left) // 2  # 中间位置

            if nums[mid] > nums[right]:
                # mid 比右端大，说明旋转点（最小值）在 mid 右边
                # 例如 [4, 5, 6, 1, 2]，mid=6 > right=2
                left = mid + 1                # mid 不可能是最小值
            else:
                # mid 比右端小或相等，说明最小值在 mid 或 mid 左边
                # 例如 [6, 1, 2, 3, 4]，mid=2 < right=4
                right = mid                   # mid 可能是最小值，不能排除

        return nums[left]                     # left == right，找到最小值
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 适用场景 | 旋转排序数组找最小值（无重复元素） |
| 核心技巧 | 比较 `nums[mid]` 和 `nums[right]` |
| 循环条件 | `while left < right`（找确定位置） |
| 收缩方式 | `left = mid + 1` 或 `right = mid` |
| 时间复杂度 | O(log n) |
| 易错点 | 不要和 `nums[left]` 比较，会导致判断出错 |

## 题目6：4. 寻找两个正序数组的中位数

### 1. 核心考点

- 将"找中位数"转化为"找第 k 小"问题
- 在两个有序数组上同时二分
- 理解分割线（partition）的思想
- 达到 O(log(m+n)) 的时间复杂度

### 2. 解题思路（给新手讲）

给你两个有序数组 nums1 和 nums2，找它们合并后的中位数，要求时间复杂度 O(log(m+n))。

**中位数的本质**：把所有数分成个数相等的两半，左半边的最大值 <= 右半边的最小值。

核心思路——**在短数组上二分找分割线**：
1. 假设在 nums1 上切一刀，位置为 i（左边 i 个元素）
2. 那么 nums2 上需要切的位置 j = (m+n+1)//2 - i（保证左半总数 = 右半总数）
3. 找到满足条件的分割位置：nums1 左边最大值 <= nums2 右边最小值，且 nums2 左边最大值 <= nums1 右边最小值

### 3. 从暴力到优化的推导

#### 暴力解法

```python
def findMedianSortedArrays(nums1, nums2):
    merged = sorted(nums1 + nums2)        # 合并后排序
    n = len(merged)
    if n % 2 == 1:                        # 奇数个元素
        return merged[n // 2]             # 中间那个
    else:                                 # 偶数个元素
        return (merged[n // 2 - 1] + merged[n // 2]) / 2  # 中间两个的平均
```

时间复杂度 O((m+n) log(m+n))，不满足要求。

#### 优化思路

既然要 O(log(m+n))，必须用二分。核心想法：

**在较短的数组上做二分**，找一个分割位置 i：
- nums1 的前 i 个元素 + nums2 的前 j 个元素 = 总数的左半边
- 其中 j = (m + n + 1) // 2 - i
- 合法条件：nums1[i-1] <= nums2[j] 且 nums2[j-1] <= nums1[i]

为什么在短数组上二分？因为短数组范围小，效率更高，而且避免 j 越界。

#### 最优解法

```python
def findMedianSortedArrays(nums1, nums2):
    # 确保 nums1 是较短的数组
    if len(nums1) > len(nums2):
        nums1, nums2 = nums2, nums1

    m, n = len(nums1), len(nums2)
    half = (m + n + 1) // 2              # 左半边应有的元素个数

    left, right = 0, m                   # 在 nums1 上二分，切割位置 [0, m]

    while left <= right:
        i = left + (right - left) // 2   # nums1 的切割位置（左边有 i 个元素）
        j = half - i                     # nums2 的切割位置（左边有 j 个元素）

        # 处理边界：切割位置在数组边缘时用正负无穷
        nums1_left = float('-inf') if i == 0 else nums1[i - 1]
        nums1_right = float('inf') if i == m else nums1[i]
        nums2_left = float('-inf') if j == 0 else nums2[j - 1]
        nums2_right = float('inf') if j == n else nums2[j]

        if nums1_left <= nums2_right and nums2_left <= nums1_right:
            # 找到合法的分割位置
            if (m + n) % 2 == 1:         # 总数为奇数
                return max(nums1_left, nums2_left)  # 左半边的最大值
            else:                        # 总数为偶数
                return (max(nums1_left, nums2_left) +
                        min(nums1_right, nums2_right)) / 2
        elif nums1_left > nums2_right:   # nums1 左边太大，i 要左移
            right = i - 1
        else:                            # nums2 左边太大，i 要右移
            left = i + 1
```

时间复杂度 O(log(min(m, n)))，空间复杂度 O(1)。

### 4. 关键思维点

1. **为什么在短数组上二分？** 在短数组上二分，i 的范围是 [0, m]，j = half - i 一定在 [0, n] 范围内（因为 m <= n）。如果在长数组上二分，j 可能越界。
2. **边界处理用正负无穷**：当 i=0 时，nums1 左边没有元素，用 `-inf`；当 i=m 时，nums1 右边没有元素，用 `+inf`。这样比较时不会影响结果。
3. **half = (m+n+1)//2 中 +1 的作用**：当总数为奇数时，让左半边多一个元素，这样中位数就是左半边的最大值。
4. **这道题是 Hard 难度**：面试中能写出来的很少，重要的是理解思路。

### 5. 代码实现（Python）

```python
class Solution:
    def findMedianSortedArrays(self, nums1: list[int], nums2: list[int]) -> float:
        # 确保 nums1 是较短的数组，这样二分范围更小
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1

        m, n = len(nums1), len(nums2)
        half = (m + n + 1) // 2           # 左半边总共需要的元素个数

        # 在 nums1 上做二分，i 表示 nums1 中划分到左边的元素个数
        left, right = 0, m

        while left <= right:
            i = left + (right - left) // 2    # nums1 的分割位置
            j = half - i                      # nums2 的分割位置（自动算出）

            # 获取分割线两侧的四个值（边界情况用无穷大/小处理）
            nums1_left_max = float('-inf') if i == 0 else nums1[i - 1]
            nums1_right_min = float('inf') if i == m else nums1[i]
            nums2_left_max = float('-inf') if j == 0 else nums2[j - 1]
            nums2_right_min = float('inf') if j == n else nums2[j]

            # 检查分割是否合法：左边的最大值 <= 右边的最小值
            if nums1_left_max <= nums2_right_min and nums2_left_max <= nums1_right_min:
                # 分割合法，计算中位数
                if (m + n) % 2 == 1:
                    # 总数奇数：中位数 = 左半边最大值
                    return max(nums1_left_max, nums2_left_max)
                else:
                    # 总数偶数：中位数 = (左半边最大 + 右半边最小) / 2
                    left_max = max(nums1_left_max, nums2_left_max)
                    right_min = min(nums1_right_min, nums2_right_min)
                    return (left_max + right_min) / 2

            elif nums1_left_max > nums2_right_min:
                # nums1 左边最大值太大，说明 i 切多了，往左移
                right = i - 1
            else:
                # nums2 左边最大值太大，说明 i 切少了，往右移
                left = i + 1

        return 0.0  # 不会到这里（题目保证有解）
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 适用场景 | 两个有序数组找中位数，要求 O(log) |
| 核心技巧 | 在短数组上二分找分割线，使左右两半满足有序条件 |
| 分割条件 | `nums1[i-1] <= nums2[j]` 且 `nums2[j-1] <= nums1[i]` |
| 关键公式 | `j = (m+n+1)//2 - i` |
| 边界处理 | 用 `float('inf')` 和 `float('-inf')` |
| 时间复杂度 | O(log(min(m,n))) |
| 难度 | Hard，面试高频但难度最大的二分题 |

## 二分查找总结

### 三种常见模板

| 模板 | 循环条件 | 适用场景 | 返回值 |
|------|---------|---------|--------|
| 标准二分 | `left <= right` | 找确定的目标值 | mid 或 -1 |
| 左边界二分 | `left <= right` | 找第一个 >= target 的位置 | left |
| 收缩二分 | `left < right` | 找满足条件的最小/最大位置 | left (=right) |

### 二分查找适用条件

1. **有序性**：数据整体或部分有序
2. **单调性**：存在某个判断条件，一边全满足，一边全不满足
3. **可二分性**：每次可以排除一半的搜索空间

### 常见陷阱

1. **死循环**：`left < right` 时用 `mid = left + (right - left) // 2` 避免死循环
2. **边界溢出**：注意 `left = mid + 1` 和 `right = mid` 的搭配
3. **整数溢出**：用 `left + (right - left) // 2` 而非 `(left + right) // 2`
4. **漏掉答案**：该用 `right = mid` 时不要写成 `right = mid - 1`
