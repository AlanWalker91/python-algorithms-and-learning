# 堆（Heap）

堆是一种特殊的完全二叉树结构，分为最大堆（父节点 >= 子节点）和最小堆（父节点 <= 子节点）。Python 中的 `heapq` 模块提供的是最小堆。堆的核心操作——插入和删除堆顶——时间复杂度都是 O(log n)，这使得它在"动态维护极值"类问题中非常高效。

---

## 题目1：215. 数组中的第K个最大元素

### 1. 核心考点

- **堆（优先队列）的使用**：用固定大小为 K 的最小堆来维护"前 K 大"的元素。
- **快速选择算法（Quick Select）**：基于快排思想的 O(n) 平均时间算法。
- 理解"第 K 大"和"第 K 小"的区别，以及如何用最小堆解决"第 K 大"问题。

### 2. 解题思路（给新手讲）

想象你是一个班主任，班上有很多学生，你想找到成绩排名第 K 的学生。

**最直觉的方法**：把所有成绩从大到小排一遍，直接取第 K 个。这当然可以，但排序要 O(n log n)。

**更聪明的方法**：你手里始终只拿着 K 张成绩单。每来一个新成绩，如果比你手里最低的那个高，就把最低的扔掉，换上新的。最后你手里最低的那个，就是第 K 大的。

这就是"大小为 K 的最小堆"的思路——堆顶永远是这 K 个元素中最小的，也就是整个数组的第 K 大。

### 3. 从暴力到优化的推导

#### 暴力解法

直接排序，取第 K 大。

```python
def findKthLargest(nums, k):
    # 排序整个数组，降序排列
    nums.sort(reverse=True)
    # 直接返回第 k 个（下标 k-1）
    return nums[k - 1]
```

- 时间复杂度：O(n log n)，排序的代价
- 空间复杂度：O(1)（原地排序）
- 问题：我们只需要第 K 大，却对整个数组排了序，做了太多不必要的工作。

#### 优化思路

**思考**：我们不需要完整排序，只需要知道"谁是第 K 大"。

**方法一：最小堆（大小为 K）**

维护一个大小为 K 的最小堆：
1. 先把前 K 个元素放入堆中。
2. 从第 K+1 个元素开始，如果当前元素大于堆顶（堆中最小值），就替换堆顶并调整堆。
3. 遍历完成后，堆顶就是第 K 大的元素。

为什么堆顶是答案？因为堆中始终保存着当前见过的最大的 K 个元素，而堆顶是这 K 个中最小的，即"第 K 大"。

**方法二：快速选择（Quick Select）**

借助快排的 partition 思想：
1. 随机选一个 pivot，把数组分成两半。
2. 如果 pivot 恰好在第 K 大的位置，直接返回。
3. 否则递归处理其中一半。

平均时间复杂度 O(n)，但最坏情况 O(n^2)。

#### 最优解法

```python
import heapq

def findKthLargest(nums, k):
    # 方法一：最小堆，时间 O(n log k)，空间 O(k)
    # 用前 k 个元素建立最小堆
    min_heap = nums[:k]
    heapq.heapify(min_heap)  # O(k) 建堆

    # 遍历剩余元素
    for num in nums[k:]:
        if num > min_heap[0]:          # 当前元素比堆顶大
            heapq.heapreplace(min_heap, num)  # 弹出堆顶，压入新元素，O(log k)

    # 堆顶就是第 k 大的元素
    return min_heap[0]
```

```python
import random

def findKthLargest(nums, k):
    # 方法二：快速选择，平均时间 O(n)
    # 我们要找的是第 k 大，即排序后下标为 len(nums) - k 的元素
    target = len(nums) - k

    def quick_select(left, right):
        # 随机选择 pivot，避免最坏情况
        pivot_idx = random.randint(left, right)
        # 把 pivot 换到末尾
        nums[pivot_idx], nums[right] = nums[right], nums[pivot_idx]
        pivot = nums[right]

        # store_idx 记录"比 pivot 小的元素"应该放的位置
        store_idx = left
        for i in range(left, right):
            if nums[i] < pivot:
                nums[i], nums[store_idx] = nums[store_idx], nums[i]
                store_idx += 1

        # 把 pivot 放到最终位置
        nums[store_idx], nums[right] = nums[right], nums[store_idx]

        # 判断 pivot 的位置和 target 的关系
        if store_idx == target:
            return nums[store_idx]       # 找到了
        elif store_idx < target:
            return quick_select(store_idx + 1, right)  # 在右半部分找
        else:
            return quick_select(left, store_idx - 1)    # 在左半部分找

    return quick_select(0, len(nums) - 1)
```

### 4. 关键思维点

| 思维点 | 说明 |
|--------|------|
| 最小堆找最大 | 大小为 K 的最小堆，堆顶就是"第 K 大"——这是一个反直觉但非常经典的技巧 |
| heapreplace 的效率 | `heapreplace` = 弹出堆顶 + 压入新元素，比分开做 `heappop` + `heappush` 更高效 |
| 快速选择的随机化 | 随机选 pivot 是关键，否则对有序数组退化为 O(n^2) |
| 第 K 大 vs 第 K 小 | 第 K 大 = 排序后下标 n-k；用最小堆找第 K 大，用最大堆找第 K 小 |

### 5. 代码实现（Python）

```python
import heapq

class Solution:
    def findKthLargest(self, nums: list[int], k: int) -> int:
        """
        用大小为 k 的最小堆找第 k 大元素
        时间复杂度: O(n log k)  空间复杂度: O(k)
        """
        # 取前 k 个元素建最小堆
        min_heap = nums[:k]
        heapq.heapify(min_heap)          # 建堆 O(k)

        # 遍历剩余元素，维护堆的大小为 k
        for num in nums[k:]:
            if num > min_heap[0]:        # 只有比堆顶大的才有资格进堆
                heapq.heapreplace(min_heap, num)  # 替换堆顶 O(log k)

        # 堆中存着最大的 k 个元素，堆顶是其中最小的 = 第 k 大
        return min_heap[0]
```

### 6. 套路总结

| 场景 | 方法 |
|------|------|
| 求前 K 大 / 第 K 大 | 大小为 K 的最小堆 |
| 求前 K 小 / 第 K 小 | 大小为 K 的最大堆（Python 中取负数模拟） |
| 只需要一次查询且追求平均 O(n) | 快速选择算法 |
| 数据流场景（动态添加元素） | 始终维护堆 |
| `heapq.nlargest(k, nums)` | Python 内置的简洁写法，底层也是堆实现 |

## 题目2：347. 前K个高频元素

### 1. 核心考点

- **哈希表统计频率** + **堆维护 Top K**：两步组合拳。
- 理解"前 K 高频"本质上就是"频率的第 K 大"问题。
- 也可以用桶排序做到 O(n) 时间。

### 2. 解题思路（给新手讲）

想象你在统计一个班级里，哪些名字出现得最多。

**第一步**：先数一遍每个名字出现了几次（哈希表统计频率）。

**第二步**：从所有名字中找出出现次数最多的 K 个。

第二步本质上就是"找前 K 大"的问题——和上一题 215 一模一样！只不过这里比较的不是元素本身，而是元素的出现频率。

### 3. 从暴力到优化的推导

#### 暴力解法

统计频率后排序，取前 K 个。

```python
from collections import Counter

def topKFrequent(nums, k):
    # 统计每个元素的出现次数
    count = Counter(nums)
    # 按频率降序排序，取前 k 个元素
    sorted_items = sorted(count.keys(), key=lambda x: count[x], reverse=True)
    return sorted_items[:k]
```

- 时间复杂度：O(n log n)，排序占主导
- 空间复杂度：O(n)
- 问题：排序了所有不同的元素，但我们只要前 K 个。

#### 优化思路

**思路一：最小堆（大小为 K）**

和 215 题一样，用大小为 K 的最小堆维护频率最高的 K 个元素：
1. 统计频率。
2. 遍历所有元素，维护大小为 K 的最小堆（按频率排序）。
3. 堆中剩下的就是频率 Top K。

时间复杂度降为 O(n log k)。

**思路二：桶排序（Bucket Sort）**

频率的范围是 [1, n]，可以建 n+1 个桶：
- 第 i 个桶存放出现了 i 次的元素。
- 从后往前遍历桶，收集 K 个元素即可。

时间复杂度 O(n)，这是最优的。

#### 最优解法

```python
from collections import Counter

def topKFrequent(nums, k):
    # 桶排序法，时间 O(n)
    count = Counter(nums)           # 统计频率 O(n)
    n = len(nums)

    # 创建桶：buckets[i] 存放出现了 i 次的元素
    buckets = [[] for _ in range(n + 1)]
    for num, freq in count.items():
        buckets[freq].append(num)

    # 从高频到低频遍历桶，收集结果
    result = []
    for freq in range(n, 0, -1):    # 从频率 n 到 1
        for num in buckets[freq]:
            result.append(num)
            if len(result) == k:    # 收集够 k 个就返回
                return result

    return result
```

### 4. 关键思维点

| 思维点 | 说明 |
|--------|------|
| 两步分离 | 先统计频率（哈希表），再找 Top K（堆或桶排序），各干各的 |
| 最小堆存 Top K | 和 215 题完全一样的思路，只是比较标准换成了频率 |
| 桶排序的巧妙 | 频率范围有限（最多 n），所以可以用桶排序做到 O(n) |
| Counter 是好帮手 | Python 的 Counter 一行搞定频率统计 |

### 5. 代码实现（Python）

```python
import heapq
from collections import Counter

class Solution:
    def topKFrequent(self, nums: list[int], k: int) -> list[int]:
        """
        方法一：最小堆法
        时间复杂度: O(n log k)  空间复杂度: O(n)
        """
        # 第一步：统计每个元素的频率
        count = Counter(nums)                   # {元素: 出现次数}

        # 第二步：用大小为 k 的最小堆维护频率最高的 k 个
        # heapq.nlargest 内部就是用堆实现的
        # 按频率取前 k 大的元素
        return heapq.nlargest(k, count.keys(), key=count.get)

    def topKFrequent_bucket(self, nums: list[int], k: int) -> list[int]:
        """
        方法二：桶排序法
        时间复杂度: O(n)  空间复杂度: O(n)
        """
        # 第一步：统计频率
        count = Counter(nums)

        # 第二步：建桶，下标表示频率
        n = len(nums)
        buckets = [[] for _ in range(n + 1)]    # buckets[i] = 出现 i 次的元素列表
        for num, freq in count.items():
            buckets[freq].append(num)           # 把元素放到对应频率的桶中

        # 第三步：从高频桶往低频桶收集，直到凑够 k 个
        result = []
        for freq in range(n, 0, -1):            # 频率从大到小遍历
            for num in buckets[freq]:           # 取出该频率下的所有元素
                result.append(num)
                if len(result) == k:            # 够了就返回
                    return result

        return result
```

### 6. 套路总结

| 场景 | 方法 |
|------|------|
| 求频率 Top K | 哈希表统计 + 最小堆 / 桶排序 |
| 值域有限时求 Top K | 桶排序 O(n) 最优 |
| Python 快速写法 | `heapq.nlargest(k, count.keys(), key=count.get)` |
| 需要同时返回元素和频率 | 堆中存 `(freq, num)` 元组 |
| 数据流中实时求 Top K | 用堆动态维护 |

## 题目3：295. 数据流的中位数

### 1. 核心考点

- **对顶堆（双堆）**：用一个最大堆和一个最小堆配合，动态维护中位数。
- 理解"数据流"意味着数据是一个一个来的，每次加入新数据后都要能快速返回当前中位数。
- 维护两个堆的平衡关系是关键。

### 2. 解题思路（给新手讲）

想象你在组织一场排队活动。所有人按身高排成一排，你需要随时知道"中间那个人的身高"。

**问题是**：人一个一个来，你不能每来一个人就重新排一次队。

**巧妙的方法**：把队伍分成两半：
- **左半部分**（矮的那一半）：你只关心这半里面最高的人 --> 用**最大堆**
- **右半部分**（高的那一半）：你只关心这半里面最矮的人 --> 用**最小堆**

这样，中位数一定在"左半最大"和"右半最小"之间产生！

**维护规则**：
1. 左半部分的大小要么等于右半部分，要么比右半部分多 1。
2. 左半部分的所有元素 <= 右半部分的所有元素。

### 3. 从暴力到优化的推导

#### 暴力解法

每次加入新元素后，对整个数组排序，取中间值。

```python
class MedianFinder:
    def __init__(self):
        self.data = []              # 存储所有数据

    def addNum(self, num):
        self.data.append(num)       # 直接追加
        self.data.sort()            # 重新排序

    def findMedian(self):
        n = len(self.data)
        if n % 2 == 1:              # 奇数个，取中间
            return self.data[n // 2]
        else:                       # 偶数个，取中间两个的平均
            return (self.data[n // 2 - 1] + self.data[n // 2]) / 2
```

- `addNum` 时间复杂度：O(n log n)，每次都排序
- `findMedian` 时间复杂度：O(1)
- 问题：如果有大量数据不断流入，每次排序太慢了。

#### 优化思路

**用插入排序优化**：维护有序数组，每次用二分查找找到插入位置，O(log n) 找位置 + O(n) 插入（移动元素）。`addNum` 降为 O(n)，但还是不够快。

**终极方案：对顶堆**

核心洞察：中位数只和"中间的一两个元素"有关，我们不需要维护完整的排序，只需要知道：
- 较小那一半中的最大值
- 较大那一半中的最小值

用两个堆完美实现：

```
  最大堆（左半）          最小堆（右半）
    存较小的一半            存较大的一半
    堆顶 = 左半最大         堆顶 = 右半最小
         \                /
          中位数在这里产生
```

**加入新元素的流程**：
1. 先加入最大堆（左半）。
2. 把最大堆的堆顶弹出，加入最小堆（右半）。这保证了左半的元素都 <= 右半。
3. 如果最小堆比最大堆多了，把最小堆堆顶弹出，还给最大堆。这保证了大小平衡。

#### 最优解法

```python
import heapq

class MedianFinder:
    def __init__(self):
        # 最大堆（存较小的一半），Python 没有最大堆，用取负数模拟
        self.max_heap = []  # 存负数，堆顶的负数最小 = 原始值最大
        # 最小堆（存较大的一半）
        self.min_heap = []  # 正常最小堆

    def addNum(self, num):
        # 第一步：先加入最大堆（取负数）
        heapq.heappush(self.max_heap, -num)

        # 第二步：把最大堆的堆顶移到最小堆
        # 保证左半所有元素 <= 右半所有元素
        max_top = -heapq.heappop(self.max_heap)
        heapq.heappush(self.min_heap, max_top)

        # 第三步：平衡大小——最大堆的大小 >= 最小堆的大小
        if len(self.min_heap) > len(self.max_heap):
            min_top = heapq.heappop(self.min_heap)
            heapq.heappush(self.max_heap, -min_top)

    def findMedian(self):
        if len(self.max_heap) > len(self.min_heap):
            # 奇数个元素，中位数是最大堆堆顶
            return -self.max_heap[0]
        else:
            # 偶数个元素，中位数是两个堆顶的平均值
            return (-self.max_heap[0] + self.min_heap[0]) / 2
```

### 4. 关键思维点

| 思维点 | 说明 |
|--------|------|
| 对顶堆的本质 | 把有序序列"从中间劈开"，左半用最大堆，右半用最小堆 |
| Python 模拟最大堆 | Python 的 heapq 只有最小堆，存负数就变成了最大堆 |
| 三步添加法 | 先入左 -> 左顶移到右 -> 右多了就还给左。这个顺序保证了两个性质同时成立 |
| 平衡条件 | 左半大小 = 右半大小（偶数），或左半 = 右半 + 1（奇数） |
| 不需要完整排序 | 我们只关心"中间"，对顶堆让我们 O(1) 拿到中位数 |

### 5. 代码实现（Python）

```python
import heapq

class MedianFinder:
    """
    对顶堆（双堆）维护数据流的中位数
    addNum: O(log n)    findMedian: O(1)
    空间复杂度: O(n)
    """

    def __init__(self):
        # max_heap: 存较小的一半（取负数模拟最大堆）
        # 大小：要么等于 min_heap，要么比 min_heap 多 1
        self.max_heap = []
        # min_heap: 存较大的一半（正常最小堆）
        self.min_heap = []

    def addNum(self, num: int) -> None:
        """添加一个数到数据结构中"""
        # 第一步：新元素先入最大堆（左半）
        heapq.heappush(self.max_heap, -num)

        # 第二步：将最大堆堆顶移到最小堆（右半）
        # 这一步确保左半所有元素 <= 右半所有元素
        top = -heapq.heappop(self.max_heap)
        heapq.heappush(self.min_heap, top)

        # 第三步：如果右半比左半多，把右半堆顶还给左半
        # 这一步确保 len(max_heap) >= len(min_heap)
        if len(self.min_heap) > len(self.max_heap):
            top = heapq.heappop(self.min_heap)
            heapq.heappush(self.max_heap, -top)

    def findMedian(self) -> float:
        """返回当前所有元素的中位数"""
        if len(self.max_heap) > len(self.min_heap):
            # 总数为奇数：中位数 = 最大堆堆顶（左半的最大值）
            return -self.max_heap[0]
        else:
            # 总数为偶数：中位数 = 两个堆顶的平均值
            return (-self.max_heap[0] + self.min_heap[0]) / 2.0


# 使用示例
# finder = MedianFinder()
# finder.addNum(1)    # 数据: [1], 中位数 = 1
# finder.addNum(2)    # 数据: [1, 2], 中位数 = 1.5
# finder.findMedian() # 返回 1.5
# finder.addNum(3)    # 数据: [1, 2, 3], 中位数 = 2
# finder.findMedian() # 返回 2.0
```

### 6. 套路总结

| 场景 | 方法 |
|------|------|
| 数据流求中位数 | 对顶堆（最大堆 + 最小堆） |
| 动态维护第 K 大/小 | 类似对顶堆思路，调整两堆大小比例 |
| Python 最大堆 | 存负数，取出时再取负 |
| 对顶堆的三步添加法 | 入左 -> 左顶移右 -> 右多还左（固定套路，背下来） |
| 滑动窗口中位数 | 对顶堆 + 延迟删除（进阶变体） |
