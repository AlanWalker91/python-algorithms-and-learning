## 题目1：73. 矩阵置零

### 1. 核心考点

**原地标记**。考查如何在不使用额外空间（或仅 O(1) 空间）的情况下，记录哪些行和列需要被置零，核心是"用矩阵的第一行和第一列作为标记数组"。

### 2. 解题思路（给新手讲）

如果某个位置是0，那整行整列都要变成0。

最直观的想法：先扫一遍矩阵，记下哪些行有0、哪些列有0，然后再扫一遍去置零。这需要额外的数组来记录。

但我们可以更聪明：**用矩阵自身的第一行和第一列来做标记！**

比如 `matrix[2][3] == 0`，我们就在 `matrix[2][0]` 和 `matrix[0][3]` 上做标记（置零），表示第2行和第3列需要被置零。

但这样做有一个问题：第一行和第一列本身也可能需要被置零，而它们又被用作标记，会互相干扰。所以需要用两个额外变量分别记录"第一行是否需要置零"和"第一列是否需要置零"。

### 3. 从暴力到优化的推导

#### 暴力解法

用一个同样大小的矩阵作为副本，先在副本上标记，再把原矩阵对应行列置零。

- 时间复杂度：O(m * n)
- 空间复杂度：O(m * n)

#### 优化思路

用两个数组 `row[m]` 和 `col[n]` 记录哪些行和列需要置零，空间优化到 O(m + n)。

再进一步：矩阵的第一行和第一列本身就可以充当这两个数组！只需额外两个布尔变量记录第一行/第一列自身是否需要置零。

#### 最优解法

用第一行第一列作为标记：

- 时间复杂度：O(m * n)
- 空间复杂度：O(1)

### 4. 关键思维点

- **标记与执行分离**：先标记后置零，不能边扫描边修改（否则新置的零会导致连锁反应）
- **借用自身空间**：第一行/列充当标记数组是空间优化的核心技巧
- **处理顺序**：先处理非第一行/列的部分，最后再处理第一行和第一列（否则标记会被提前破坏）
- **两个额外变量**：`first_row_zero` 和 `first_col_zero` 解决第一行/列本身的标记冲突

### 5. 代码实现（Python）

```python
def setZeroes(matrix):
    m, n = len(matrix), len(matrix[0])

    # 先记录第一行和第一列自身是否含有0
    first_row_zero = any(matrix[0][j] == 0 for j in range(n))
    first_col_zero = any(matrix[i][0] == 0 for i in range(m))

    # 用第一行和第一列作为标记数组
    # 扫描除第一行第一列以外的部分
    for i in range(1, m):
        for j in range(1, n):
            if matrix[i][j] == 0:
                matrix[i][0] = 0  # 标记第 i 行需要置零
                matrix[0][j] = 0  # 标记第 j 列需要置零

    # 根据标记，将对应行置零（跳过第一行）
    for i in range(1, m):
        if matrix[i][0] == 0:
            for j in range(1, n):
                matrix[i][j] = 0

    # 根据标记，将对应列置零（跳过第一列）
    for j in range(1, n):
        if matrix[0][j] == 0:
            for i in range(1, m):
                matrix[i][j] = 0

    # 最后处理第一行
    if first_row_zero:
        for j in range(n):
            matrix[0][j] = 0

    # 最后处理第一列
    if first_col_zero:
        for i in range(m):
            matrix[i][0] = 0
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 原地标记 | 利用矩阵自身（第一行/列）作为辅助存储空间 |
| 先标记后执行 | 避免修改操作影响后续判断 |
| 边界特殊处理 | 第一行/列既是标记又是数据，需要额外变量和特殊处理顺序 |

## 题目2：54. 螺旋矩阵

### 1. 核心考点

**模拟 / 边界控制**。考查按照特定顺序遍历二维矩阵的能力，核心是维护四个边界并逐层收缩。

### 2. 解题思路（给新手讲）

想象你在剥洋葱：一层一层地从外往里剥。每一层的路径是一个"口"字形：先从左到右走最上面一行，再从上到下走最右边一列，再从右到左走最下面一行，最后从下到上走最左边一列。

走完一层后，四个边界各往里缩一步，继续剥下一层，直到所有元素都被遍历完。

用四个变量维护当前层的边界：
- `top`：当前最上面一行
- `bottom`：当前最下面一行
- `left`：当前最左边一列
- `right`：当前最右边一列

每走完一条边，就收缩对应的边界。

### 3. 从暴力到优化的推导

#### 暴力解法

用一个布尔矩阵 `visited` 记录每个位置是否被访问过，然后模拟方向转弯（碰到边界或已访问就右转）。

- 时间复杂度：O(m * n)
- 空间复杂度：O(m * n)（visited矩阵）

#### 优化思路

不需要 visited 矩阵，只要维护四个边界就能精确控制遍历范围，每走完一条边就收缩边界。

#### 最优解法

四边界法：

- 时间复杂度：O(m * n)
- 空间复杂度：O(1)（不算结果数组）

### 4. 关键思维点

- **四个边界**：`top, bottom, left, right` 精确控制当前层的范围
- **收缩时机**：每走完一条边就收缩对应边界（走完上边 -> top++，走完右边 -> right-- 等）
- **提前终止**：每次收缩后都要检查 `top <= bottom` 和 `left <= right`，因为矩阵不一定是正方形，可能中途某个方向就走完了
- **单行/单列特殊情况**：当只剩一行或一列时，不需要走回头路

### 5. 代码实现（Python）

```python
def spiralOrder(matrix):
    if not matrix:
        return []

    result = []
    # 初始化四个边界
    top, bottom = 0, len(matrix) - 1
    left, right = 0, len(matrix[0]) - 1

    while top <= bottom and left <= right:
        # 从左到右：遍历上边一行
        for j in range(left, right + 1):
            result.append(matrix[top][j])
        top += 1  # 上边界下移

        # 从上到下：遍历右边一列
        for i in range(top, bottom + 1):
            result.append(matrix[i][right])
        right -= 1  # 右边界左移

        # 从右到左：遍历下边一行（需要检查是否还有行）
        if top <= bottom:
            for j in range(right, left - 1, -1):
                result.append(matrix[bottom][j])
            bottom -= 1  # 下边界上移

        # 从下到上：遍历左边一列（需要检查是否还有列）
        if left <= right:
            for i in range(bottom, top - 1, -1):
                result.append(matrix[i][left])
            left += 1  # 左边界右移

    return result
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 四边界法 | 用 top/bottom/left/right 控制遍历范围，每轮收缩 |
| 分方向遍历 | 右 -> 下 -> 左 -> 上，每走完一边收缩一次 |
| 边界检查 | 走"左"和"上"之前要检查边界是否仍然合法，防止重复遍历 |

## 题目3：48. 旋转图像

### 1. 核心考点

**矩阵变换（转置 + 翻转）**。考查如何将 n x n 矩阵原地顺时针旋转90度，核心是将旋转分解为两个简单操作的组合。

### 2. 解题思路（给新手讲）

顺时针旋转90度，直觉上很复杂：每个元素要跑到一个新位置。但有一个非常优雅的拆解：

**顺时针旋转90度 = 先沿主对角线转置 + 再左右翻转每一行**

举个例子：
```
原矩阵        转置后         左右翻转后（旋转完成）
1 2 3        1 4 7          7 4 1
4 5 6   ->   2 5 8   ->     8 5 2
7 8 9        3 6 9          9 6 3
```

为什么这样等价于顺时针旋转90度？

旋转90度的规则是 `new[j][n-1-i] = old[i][j]`。
转置的规则是 `new[j][i] = old[i][j]`。
左右翻转的规则是 `new[i][n-1-j] = old[i][j]`。

组合起来：先转置把 `(i,j)` 变成 `(j,i)`，再左右翻转把 `(j,i)` 变成 `(j,n-1-i)`，刚好就是旋转90度的结果。

### 3. 从暴力到优化的推导

#### 暴力解法

创建一个新矩阵，按规则 `new[j][n-1-i] = old[i][j]` 逐个填入。

- 时间复杂度：O(n^2)
- 空间复杂度：O(n^2)

#### 优化思路

题目要求原地旋转。观察到旋转时4个位置的元素会形成一个循环：`(i,j) -> (j,n-1-i) -> (n-1-i,n-1-j) -> (n-1-j,i) -> (i,j)`。可以4个一组做循环交换，但实现起来容易出错。

更简洁的方法：将旋转分解为"转置 + 翻转"两步，每步都是原地操作。

#### 最优解法

转置 + 左右翻转：

- 时间复杂度：O(n^2)
- 空间复杂度：O(1)

### 4. 关键思维点

- **分解思想**：复杂的旋转操作可以分解为两个简单操作（转置 + 翻转）
- **转置只遍历上三角**：`for i in range(n): for j in range(i+1, n)`，避免交换两次等于没交换
- **不同旋转方向的分解**：
  - 顺时针90度 = 转置 + 左右翻转
  - 逆时针90度 = 转置 + 上下翻转
  - 旋转180度 = 上下翻转 + 左右翻转

### 5. 代码实现（Python）

```python
def rotate(matrix):
    n = len(matrix)

    # 第一步：沿主对角线转置
    # 交换 matrix[i][j] 和 matrix[j][i]
    for i in range(n):
        for j in range(i + 1, n):  # 只遍历上三角，避免交换回来
            matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]

    # 第二步：左右翻转每一行
    for i in range(n):
        left, right = 0, n - 1
        while left < right:
            matrix[i][left], matrix[i][right] = matrix[i][right], matrix[i][left]
            left += 1
            right -= 1
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 旋转 = 转置 + 翻转 | 将矩阵旋转分解为两步简单操作 |
| 上三角遍历 | 转置时只遍历 j > i 的部分，防止重复交换 |
| 原地操作 | 转置和翻转都只需要交换元素，不需要额外空间 |

## 题目4：240. 搜索二维矩阵 II

### 1. 核心考点

**矩阵搜索 / 利用有序性的削减搜索空间**。考查如何利用"每行递增、每列递增"的特性，从特定角出发逐步缩小搜索范围。核心技巧是"从右上角或左下角开始搜索"。

### 2. 解题思路（给新手讲）

矩阵的每一行从左到右递增，每一列从上到下递增。我们要在里面找一个目标值。

关键直觉：**从右上角出发！**

为什么选右上角？因为站在右上角 `(0, n-1)` 时，你有两个明确的移动方向：
- 当前值比目标大 -> 往左走一列（列减小，值变小）
- 当前值比目标小 -> 往下走一行（行增大，值变大）
- 当前值等于目标 -> 找到了！

每一步都能**排除一整行或一整列**，效率非常高。

类比：想象你在一栋大楼的右上角，每一层（行）的房间号从左到右递增，每个位置（列）的楼层从上到下递增。你要找某个房间：当前房间号太大就往左走，楼层太低就往下走。

从左上角出发就不行——往右和往下都是变大，无法做出明确的二分方向选择。

### 3. 从暴力到优化的推导

#### 暴力解法

遍历整个矩阵，逐个比较。

- 时间复杂度：O(m * n)

#### 优化思路

可以对每一行做二分查找：O(m * log n)。但还没有充分利用"列也有序"的性质。

终极优化：从右上角（或左下角）出发，每步排除一行或一列。

#### 最优解法

右上角搜索法：

- 时间复杂度：O(m + n)，最多走 m + n 步
- 空间复杂度：O(1)

### 4. 关键思维点

- **起点选择**：右上角或左下角是唯一两个可以提供"二分方向"的起点
  - 右上角：左边更小，下面更大
  - 左下角：右边更大，上面更小
  - 左上角和右下角不行——两个方向同增或同减，无法做出选择
- **每步排除一行或一列**：这是 O(m + n) 复杂度的保证
- **类BST结构**：如果把右上角看作根节点，左边是"左子树"（更小），下边是"右子树"（更大），这就是一个BST的搜索过程

### 5. 代码实现（Python）

```python
def searchMatrix(matrix, target):
    if not matrix or not matrix[0]:
        return False

    m, n = len(matrix), len(matrix[0])

    # 从右上角出发
    row, col = 0, n - 1

    while row < m and col >= 0:
        current = matrix[row][col]

        if current == target:
            # 找到目标
            return True
        elif current > target:
            # 当前值太大，往左走（排除当前列）
            col -= 1
        else:
            # 当前值太小，往下走（排除当前行）
            row += 1

    # 超出边界仍未找到
    return False
```

### 6. 套路总结

| 套路 | 说明 |
|------|------|
| 右上角/左下角出发 | 行列均有序的矩阵搜索的标准起点 |
| 每步排除一行或一列 | 利用有序性大幅削减搜索空间，O(m+n) |
| 类BST思维 | 把矩阵看作一棵BST，右上角是根节点 |
