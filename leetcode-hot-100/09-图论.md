## 题目1：200. 岛屿数量

### 1. 核心考点

- **DFS/BFS遍历**：在二维网格上进行深度优先或广度优先搜索
- **连通分量计数**：每一个由 `'1'` 组成的连通区域就是一个岛屿
- **visited标记**：避免重复访问同一个格子

### 2. 解题思路（给新手讲）

想象你站在一张航拍地图上往下看，`'1'` 是陆地，`'0'` 是水。你要数有多少块独立的岛屿。

做法很直觉：从左上角开始一个一个格子扫描，遇到一块陆地 `'1'` 就知道"这里有一个岛"，然后用 DFS 把整个岛走一遍，走过的地方标记为 `'0'`（沉岛法），这样这座岛就不会被再数一次。继续扫描下一个格子，每次遇到新的 `'1'` 就说明发现了新的岛屿，计数 +1。

### 3. 从暴力到优化的推导

#### 暴力解法

暴力的想法是：对每个 `'1'`，都检查它是否属于之前已发现的岛屿。但这需要维护所有岛屿的坐标集合，每次判断归属代价很高。

#### 优化思路

换个角度：不去判断"这个1属于哪个岛"，而是在发现一个新的 `'1'` 时，主动把整座岛"淹掉"——用 DFS/BFS 把与它连通的所有 `'1'` 都变成 `'0'`。这样后续扫描就不会重复计数了。

#### 最优解法

DFS 沉岛法，时间复杂度 O(M*N)，空间复杂度 O(M*N)（最坏递归深度）。

### 4. 关键思维点

- **沉岛技巧**：访问过的陆地直接置 `'0'`，既做了标记又省去了额外的 visited 数组
- **四方向遍历**：上下左右四个方向是二维网格问题的标配
- **遍历触发条件**：只在遇到 `'1'` 时才启动 DFS，启动一次就代表一个岛

### 5. 代码实现（Python）

```python
class Solution:
    def numIslands(self, grid: list[list[str]]) -> int:
        # 特判：空网格直接返回0
        if not grid or not grid[0]:
            return 0

        rows, cols = len(grid), len(grid[0])  # 获取网格的行数和列数
        count = 0  # 岛屿计数器

        def dfs(r, c):
            """从(r,c)出发，把整座岛沉掉"""
            # 越界或遇到水，直接返回
            if r < 0 or r >= rows or c < 0 or c >= cols or grid[r][c] == '0':
                return
            grid[r][c] = '0'  # 沉岛：把当前陆地变成水，防止重复访问
            # 向上下左右四个方向递归
            dfs(r + 1, c)  # 下
            dfs(r - 1, c)  # 上
            dfs(r, c + 1)  # 右
            dfs(r, c - 1)  # 左

        # 遍历整个网格的每一个格子
        for r in range(rows):
            for c in range(cols):
                if grid[r][c] == '1':  # 发现一块新陆地
                    count += 1         # 岛屿数 +1
                    dfs(r, c)          # 把这座岛全部沉掉

        return count
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 题型 | 二维网格连通分量计数 |
| 核心套路 | 遍历 + DFS/BFS 标记 |
| 沉岛法 | 将访问过的 `'1'` 改为 `'0'`，省去 visited 数组 |
| 时间复杂度 | O(M * N) |
| 空间复杂度 | O(M * N)，递归栈最坏情况 |
| 类似题 | 695.岛屿最大面积、130.被围绕的区域 |

## 题目2：994. 腐烂的橘子

### 1. 核心考点

- **多源BFS**：从所有腐烂橘子同时出发，逐层扩散
- **层序遍历计时**：BFS 每扩散一层就是过了1分钟
- **边界判断**：最终需要检查是否还有新鲜橘子残留

### 2. 解题思路（给新手讲）

把网格想象成一个果盘：0是空格，1是新鲜橘子，2是腐烂的橘子。每过1分钟，腐烂橘子会传染上下左右四个方向的新鲜橘子。问最少多少分钟全部腐烂？如果有橘子永远不会腐烂，返回-1。

关键洞察：所有腐烂橘子是**同时**开始扩散的，这就是"多源BFS"——把所有起点在第0分钟就全部放入队列，然后一层一层往外扩。

### 3. 从暴力到优化的推导

#### 暴力解法

模拟法：每一分钟扫描整个网格，把所有与腐烂橘子相邻的新鲜橘子变腐烂，直到没有变化为止。时间复杂度 O((M*N)^2)。

#### 优化思路

不要每分钟都全盘扫描，而是用队列只关注"前线"——当前刚腐烂的橘子。这些橘子只需要检查各自的四个邻居。这就是 BFS 的思路，时间复杂度降为 O(M*N)。

#### 最优解法

多源 BFS，时间 O(M*N)，空间 O(M*N)。

### 4. 关键思维点

- **多源 vs 单源**：多源BFS就是起点有多个，全部同时入队。思路和单源BFS完全一样，只是初始化不同
- **层数 = 时间**：BFS 按层遍历，每处理完一层就是过了1分钟
- **结束检查**：BFS 结束后必须检查有没有剩余的新鲜橘子，有则返回 -1

### 5. 代码实现（Python）

```python
from collections import deque

class Solution:
    def orangesRotting(self, grid: list[list[int]]) -> int:
        rows, cols = len(grid), len(grid[0])
        queue = deque()    # BFS 队列
        fresh = 0          # 新鲜橘子计数

        # 第一步：遍历网格，找出所有腐烂橘子(入队)，统计新鲜橘子数量
        for r in range(rows):
            for c in range(cols):
                if grid[r][c] == 2:
                    queue.append((r, c))  # 腐烂橘子作为BFS起点
                elif grid[r][c] == 1:
                    fresh += 1            # 记录新鲜橘子数量

        # 特判：如果一开始就没有新鲜橘子，直接返回0
        if fresh == 0:
            return 0

        minutes = 0        # 经过的分钟数
        directions = [(1, 0), (-1, 0), (0, 1), (0, -1)]  # 四个方向

        # 第二步：BFS 层序遍历
        while queue:
            size = len(queue)            # 当前层的节点数（当前这一分钟要处理的橘子）
            has_rotted = False           # 这一分钟是否有新橘子被感染

            for _ in range(size):
                r, c = queue.popleft()   # 取出一个腐烂橘子
                for dr, dc in directions:
                    nr, nc = r + dr, c + dc  # 计算邻居坐标
                    # 邻居在界内且是新鲜橘子
                    if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == 1:
                        grid[nr][nc] = 2         # 感染它
                        queue.append((nr, nc))   # 入队，下一轮继续扩散
                        fresh -= 1               # 新鲜橘子减少
                        has_rotted = True         # 标记本轮有感染发生

            if has_rotted:
                minutes += 1  # 只有真正发生感染的轮次才计时

        # 第三步：检查是否还有新鲜橘子
        return minutes if fresh == 0 else -1
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 题型 | 多源BFS + 最短传播时间 |
| 核心套路 | 所有起点同时入队，按层遍历计时 |
| 易错点 | 忘记检查最终是否还有新鲜橘子 |
| 时间复杂度 | O(M * N) |
| 空间复杂度 | O(M * N) |
| 多源BFS模板 | 初始化把所有源点入队 -> 按层BFS -> 每层时间+1 |

## 题目3：207. 课程表

### 1. 核心考点

- **拓扑排序**：判断有向图中是否存在环
- **入度表 + BFS（Kahn算法）**：通过入度为0的节点逐步剥离图
- **图的构建**：邻接表表示有向图

### 2. 解题思路（给新手讲）

你有 n 门课程要修，有些课有先修要求。比如想上课程1必须先上课程0。问：你能不能修完所有课程？

本质就是：把课程关系画成有向图（A -> B 表示学完A才能学B），如果图里有环，就说明存在循环依赖（A要先学B，B又要先学A），不可能修完，返回 False。

怎么检测环？用拓扑排序：先找没有先修课的课程（入度为0），先"修完"它们，然后把它们的后续课程的先修要求去掉。重复这个过程，如果最终所有课程都能修完，就没有环。

### 3. 从暴力到优化的推导

#### 暴力解法

DFS 检测环：对每个节点做 DFS，用三色标记法（白/灰/黑）判断是否有回边。实现相对复杂。

#### 优化思路

BFS 拓扑排序（Kahn算法）更直观：统计每个节点的入度，入度为0的先处理。处理一个节点就把它的后继入度减1，如果后继入度变为0就入队。最后检查处理了多少节点。

#### 最优解法

BFS 拓扑排序，时间 O(V+E)，空间 O(V+E)。V是课程数，E是先修关系数。

### 4. 关键思维点

- **有向图建模**：先修关系天然就是有向边
- **环 = 无法完成**：有向图中有环意味着循环依赖
- **入度为0 = 无前置依赖**：这些节点可以直接处理
- **计数法判环**：如果最终处理的节点数 < 总数，说明有环

### 5. 代码实现（Python）

```python
from collections import deque, defaultdict

class Solution:
    def canFinish(self, numCourses: int, prerequisites: list[list[int]]) -> bool:
        # 第一步：构建邻接表和入度数组
        graph = defaultdict(list)  # 邻接表：graph[a] = [b1, b2...] 表示学完a可以学b1, b2...
        in_degree = [0] * numCourses  # 每门课的入度（有多少先修课）

        for course, prereq in prerequisites:
            graph[prereq].append(course)  # prereq -> course 的有向边
            in_degree[course] += 1         # course 的入度 +1

        # 第二步：把所有入度为0的课程（没有先修要求）入队
        queue = deque()
        for i in range(numCourses):
            if in_degree[i] == 0:
                queue.append(i)

        # 第三步：BFS 逐层剥离
        count = 0  # 记录已经"修完"的课程数
        while queue:
            course = queue.popleft()  # 取出一门可以修的课
            count += 1                 # 修完它，计数 +1

            # 遍历这门课的所有后续课程
            for next_course in graph[course]:
                in_degree[next_course] -= 1   # 后续课程的先修要求减少一个
                if in_degree[next_course] == 0:
                    queue.append(next_course)  # 如果先修全部满足，可以修了

        # 第四步：如果修完的课程数等于总课程数，说明没有环
        return count == numCourses
```

### 6. 套路总结

| 要素 | 内容 |
|------|------|
| 题型 | 有向图判环 / 拓扑排序 |
| 核心套路 | BFS + 入度表（Kahn算法）|
| 建图方式 | 邻接表 + 入度数组 |
| 判环条件 | 处理节点数 < 总节点数 则有环 |
| 时间复杂度 | O(V + E) |
| 空间复杂度 | O(V + E) |
| 类似题 | 210.课程表II（要求输出拓扑序列）|

## 题目4：208. 实现Trie（前缀树）

### 1. 核心考点

- **前缀树数据结构**：理解 Trie 的多叉树结构
- **节点设计**：每个节点包含子节点映射和"是否为单词结尾"的标志
- **三个操作**：insert（插入）、search（精确搜索）、startsWith（前缀查找）

### 2. 解题思路（给新手讲）

前缀树就像一本字典的目录结构。假设要存 "apple" 和 "app"：

```
根
 └─ a
     └─ p
         └─ p (is_end=True，因为 "app" 是一个完整单词)
             └─ l
                 └─ e (is_end=True，因为 "apple" 是一个完整单词)
```

每个节点不存完整单词，只存一个字符（体现在父节点到子节点的边上）。从根到某个标记了 `is_end=True` 的节点的路径就构成一个完整的单词。

- **插入**：沿着单词的每个字符往下走，没有路就创建新节点，最后标记 is_end
- **搜索**：沿着单词的每个字符往下走，走完后检查 is_end 是否为 True
- **前缀查找**：沿着前缀的每个字符往下走，只要能走完就返回 True（不需要检查 is_end）

### 3. 从暴力到优化的推导

#### 暴力解法

用列表存储所有单词。插入 O(1)，搜索 O(N*L)（N个单词，平均长度L），前缀查找也是 O(N*L)。

#### 优化思路

利用单词之间共享前缀的特性。"apple"和"app"共享前缀"app"，没必要存两份。Trie 把公共前缀合并到同一条路径上。

#### 最优解法

Trie 前缀树。插入/搜索/前缀查找的时间复杂度都是 O(L)，L 为单词长度，与已存储的单词总数无关。

### 4. 关键思维点

- **节点设计**：`children` 字典 + `is_end` 布尔值就够了
- **search vs startsWith 的区别**：search 要求 is_end=True，startsWith 不要求
- **公共前缀共享**：这是 Trie 节省空间的核心原理
- **每条边代表一个字符**：节点本身不存字符，字符体现在边上

### 5. 代码实现（Python）

```python
class TrieNode:
    """前缀树的节点"""
    def __init__(self):
        self.children = {}    # 子节点映射：字符 -> TrieNode
        self.is_end = False   # 标记是否是某个单词的结尾


class Trie:
    def __init__(self):
        self.root = TrieNode()  # 根节点不存储任何字符

    def insert(self, word: str) -> None:
        """将单词插入前缀树"""
        node = self.root           # 从根节点出发
        for char in word:          # 逐个字符往下走
            if char not in node.children:
                node.children[char] = TrieNode()  # 没有这条路就创建新节点
            node = node.children[char]             # 走到子节点
        node.is_end = True         # 标记单词结尾

    def search(self, word: str) -> bool:
        """搜索前缀树中是否存在完整的单词"""
        node = self._find_node(word)  # 沿着 word 走到尽头
        # 节点存在且是单词结尾才算找到
        return node is not None and node.is_end

    def startsWith(self, prefix: str) -> bool:
        """判断前缀树中是否有以 prefix 开头的单词"""
        return self._find_node(prefix) is not None  # 只要能走完前缀就行

    def _find_node(self, prefix: str):
        """辅助函数：沿着 prefix 路径走，返回终点节点（找不到返回 None）"""
        node = self.root
        for char in prefix:
            if char not in node.children:
                return None              # 路径断了，找不到
            node = node.children[char]   # 继续往下走
        return node                      # 返回终点节点
```

| 要素 | 内容 |

| 题型 | 数据结构设计（前缀树）|
