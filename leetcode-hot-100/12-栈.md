# 栈

栈是一种**后进先出（LIFO）**的数据结构，就像一摞盘子——只能从顶部放入和取出。栈在算法中有三大经典应用场景：**括号匹配**、**辅助设计数据结构**、**单调栈**。尤其是单调栈，它是解决"下一个更大/更小元素"类问题的利器。

## 题目1：20. 有效的括号

### 1. 核心考点

- 栈的基本操作（压栈、弹栈、判空）
- 括号匹配问题——栈的最经典应用
- 边界条件处理（多余左括号、多余右括号、类型不匹配）

### 2. 解题思路（给新手讲）

想象你在检查一段代码里的括号是否配对。规则很简单：
- 每个左括号 `(`、`[`、`{` 都必须有一个**对应类型**的右括号来关闭它
- 括号必须**按正确顺序**关闭，不能交叉

比如 `([])` 是合法的，但 `([)]` 是非法的。

怎么检查呢？拿一摞盘子来类比：
1. 从左往右扫描字符串
2. 遇到**左括号**，就把它"放到盘子堆顶"（压栈）
3. 遇到**右括号**，就看盘子堆顶的左括号是不是和它配对
   - 配对就把顶部盘子拿走（弹栈）
   - 不配对就说明字符串无效
4. 最后检查盘子堆是否清空了——如果还有剩余的左括号，也无效

### 3. 从暴力到优化的推导

#### 暴力解法

不断地从字符串中删除相邻的配对括号 `()`、`[]`、`{}`，直到无法删除为止。如果最终字符串为空，则有效。

```python
def isValid(s):
    while '()' in s or '[]' in s or '{}' in s:
        s = s.replace('()', '').replace('[]', '').replace('{}', '')
    return s == ''
```

时间复杂度 O(n^2)，每次替换都要扫描整个字符串，最多替换 n/2 次。

#### 优化思路

暴力解法反复扫描字符串太浪费了。仔细想一下，**最后出现的左括号**应该**最先被匹配**——这正好是"后进先出"的特性，天然适合用栈！

用栈就可以**一次遍历**完成所有匹配：
- 左括号入栈，右括号和栈顶匹配
- 任何时候匹配失败就直接返回 False

#### 最优解法

一次遍历 + 栈匹配，时间复杂度 O(n)，空间复杂度 O(n)。

### 4. 关键思维点

- **为什么用栈**：括号匹配要求"最近的左括号最先被匹配"，这就是后进先出
- **用哈希表做映射**：右括号 -> 左括号的映射，避免写一堆 if-else
- **三种无效情况**：(1) 右括号多了（栈为空时遇到右括号）(2) 类型不匹配（栈顶括号和当前右括号不配对）(3) 左括号多了（遍历结束栈不为空）

### 5. 代码实现（Python）

```python
def isValid(s: str) -> bool:
    # 用哈希表存储右括号到左括号的映射
    mapping = {')': '(', ']': '[', '}': '{'}
    stack = []  # 初始化空栈

    for char in s:                          # 逐个字符扫描
        if char in mapping:                 # 当前是右括号
            # 栈为空或栈顶不匹配，直接返回 False
            if not stack or stack[-1] != mapping[char]:
                return False
            stack.pop()                     # 匹配成功，弹出栈顶
        else:
            stack.append(char)              # 当前是左括号，压入栈

    return len(stack) == 0                  # 栈为空说明全部匹配
```

### 6. 套路总结

**括号匹配问题**的通用模板：
- 遇到"开"的符号 -> 入栈
- 遇到"关"的符号 -> 和栈顶匹配
- 最终检查栈是否为空

这个套路可以扩展到：HTML 标签匹配、表达式求值、路径简化等问题。

## 题目2：155. 最小栈

### 1. 核心考点

- 数据结构设计
- 辅助栈（用额外栈维护状态信息）
- 在常数时间内获取最小值

### 2. 解题思路（给新手讲）

设计一个栈，除了正常的 `push`、`pop`、`top` 操作之外，还要能在 **O(1) 时间**内返回栈中的最小值。

难点在哪？普通栈可以 O(1) 压入弹出，但找最小值通常需要 O(n) 遍历。

生活中的类比：假设你在叠盘子，每个盘子上写了一个数字。老板随时可能问你"当前这摞盘子里最小的数字是多少？"你不可能每次都翻遍所有盘子。

解决办法：**再准备一摞辅助盘子**，这摞盘子上记录的是"到当前为止的最小值"。

```
操作过程示例：push(-2), push(0), push(-3)

主栈:     [-2, 0, -3]      <- 存原始数据
辅助栈:   [-2, -2, -3]     <- 存"到当前位置为止的最小值"
                               ↑ 每次 push 时，辅助栈压入 min(新元素, 辅助栈栈顶)

getMin() 直接返回辅助栈栈顶 = -3

pop() 弹出 -3:
主栈:     [-2, 0]
辅助栈:   [-2, -2]
getMin() 返回辅助栈栈顶 = -2  ✓ 正确！
```

### 3. 从暴力到优化的推导

#### 暴力解法

用普通栈存数据，每次调用 `getMin()` 时遍历整个栈找最小值：

```python
class MinStack:
    def __init__(self):
        self.stack = []

    def push(self, val):
        self.stack.append(val)

    def pop(self):
        self.stack.pop()

    def top(self):
        return self.stack[-1]

    def getMin(self):
        return min(self.stack)  # O(n) 遍历找最小值
```

`getMin()` 时间复杂度 O(n)，不满足题目要求。

#### 优化思路

问题在于：弹出一个元素后，最小值可能会变化。我们需要**记住每个历史状态下的最小值**。

关键 insight：栈的元素是"逐个增加、逐个减少"的。当我压入第 k 个元素时，前 k 个元素的最小值是确定的。当我弹出一个元素后，前 k-1 个元素的最小值也应该是已知的。

所以我们可以用一个**辅助栈**，和主栈**同步操作**：
- 主栈压入一个元素时，辅助栈压入"当前最小值"
- 主栈弹出一个元素时，辅助栈也弹出
- 辅助栈的栈顶始终是当前状态下的最小值

#### 最优解法

双栈方案：主栈 + 辅助最小栈。所有操作均为 O(1)。

### 4. 关键思维点

- **为什么需要辅助栈**：因为弹出元素后最小值会变，需要回退到之前的最小值，而栈天然支持"回退"
- **辅助栈和主栈同步**：压入时一起压，弹出时一起弹，保持长度一致
- **辅助栈压入什么**：`min(当前要压入的值, 辅助栈栈顶)`，这样辅助栈栈顶永远是当前状态下的最小值
- **空间换时间**：多用 O(n) 空间，换来 O(1) 的 getMin

### 5. 代码实现（Python）

```python
class MinStack:
    def __init__(self):
        self.stack = []          # 主栈，存储所有元素
        self.min_stack = []      # 辅助栈，存储每个状态下的最小值

    def push(self, val: int) -> None:
        self.stack.append(val)   # 主栈正常压入
        # 辅助栈压入：当前值和辅助栈栈顶的较小者
        # 如果辅助栈为空，直接压入当前值
        cur_min = min(val, self.min_stack[-1] if self.min_stack else val)
        self.min_stack.append(cur_min)

    def pop(self) -> None:
        self.stack.pop()         # 主栈弹出
        self.min_stack.pop()     # 辅助栈同步弹出

    def top(self) -> int:
        return self.stack[-1]    # 返回主栈栈顶

    def getMin(self) -> int:
        return self.min_stack[-1]  # 返回辅助栈栈顶，即当前最小值
```

### 6. 套路总结

**辅助栈设计**的通用思路：
- 当需要在栈操作中**维护某种全局状态**（最小值、最大值、频率等）时，考虑用辅助栈
- 辅助栈和主栈**同步操作**，这样回退时状态也能回退
- 本质是用空间换时间，把 O(n) 的查询优化到 O(1)

类似的设计思想也用于：最大栈、带频率的栈等问题。

## 题目3：394. 字符串解码

### 1. 核心考点

- 栈处理嵌套结构
- 括号嵌套问题的通用栈解法
- 如何用栈保存和恢复"上下文"

### 2. 解题思路（给新手讲）

给你一个编码字符串，比如 `3[a2[c]]`，要你解码成 `accaccacc`。

规则：
- `k[encoded_string]` 表示 `encoded_string` 重复 k 次
- 可以嵌套，比如 `3[a2[c]]` = `3[a + cc]` = `3[acc]` = `accaccacc`

这个问题的难点在于**嵌套**。当你遇到内层的 `2[c]` 时，你需要先处理它，得到 `cc`，然后把结果拼回外层。

用生活中的例子：这就像**俄罗斯套娃**。你必须先打开最里面的娃娃，再一层层往外拼。

怎么做？用栈来模拟这个"进入内层、返回外层"的过程：

```
处理 "3[a2[c]]" 的过程：

字符  操作                           栈状态                  当前串   当前数字
'3'   记录数字                       []                     ""       3
'['   把(当前串,数字)压栈，重置      [(""，3)]              ""       0
'a'   拼接到当前串                   [(""，3)]              "a"      0
'2'   记录数字                       [(""，3)]              "a"      2
'['   把(当前串,数字)压栈，重置      [("",3), ("a",2)]      ""       0
'c'   拼接到当前串                   [("",3), ("a",2)]      "c"      0
']'   弹出("a",2)，当前串="a"+2*"c"  [("",3)]               "acc"    0
']'   弹出("",3)，当前串=""+3*"acc"  []                     "accaccacc"

结果："accaccacc" ✓
```

### 3. 从暴力到优化的推导

#### 暴力解法

从最内层括号开始，反复用正则替换，直到没有括号：

```python
import re
def decodeString(s):
    while '[' in s:
        # 找到最内层的 k[...] 并替换
        s = re.sub(r'(\d+)\[([a-zA-Z]*)\]', lambda m: int(m.group(1)) * m.group(2), s)
    return s
```

时间复杂度取决于嵌套深度，最坏 O(n^2)。每次正则替换都要扫描整个字符串。

#### 优化思路

暴力解法每次只能处理一层，多层嵌套需要反复扫描。

核心观察：遇到 `[` 就像"进入新的一层"，遇到 `]` 就像"从内层返回"。**这种进入/返回的模式天然适合栈**。

我们需要在栈中保存什么？进入内层前的**上下文**：
1. 之前已经拼好的字符串（外层的部分结果）
2. 当前层的重复次数

遇到 `]` 时，从栈中恢复上下文，把内层结果重复后拼接到外层结果上。

#### 最优解法

一次遍历 + 栈保存上下文，时间复杂度 O(n * 总展开长度)，空间复杂度 O(n)。

### 4. 关键思维点

- **栈保存的是"上下文"**：每次遇到 `[`，当前的字符串和数字就是外层的上下文，压入栈中保存
- **遇到 `]` 就恢复上下文**：弹出外层的字符串和数字，把当前结果重复后拼接
- **数字可能多位**：比如 `12[a]`，需要把 `1` 和 `2` 拼成 `12`
- **这种"进入/退出"模式**是栈的经典应用场景，类似于函数调用栈

### 5. 代码实现（Python）

```python
def decodeString(s: str) -> str:
    stack = []              # 栈中保存 (之前的字符串, 重复次数)
    cur_str = ""            # 当前正在构建的字符串
    cur_num = 0             # 当前的重复次数

    for char in s:
        if char.isdigit():
            # 数字可能是多位的，比如 "12" -> 1*10+2=12
            cur_num = cur_num * 10 + int(char)
        elif char == '[':
            # 遇到左括号：保存当前上下文（当前串和重复次数）到栈
            stack.append((cur_str, cur_num))
            # 重置当前状态，准备处理括号内的内容
            cur_str = ""
            cur_num = 0
        elif char == ']':
            # 遇到右括号：从栈中恢复外层上下文
            prev_str, num = stack.pop()
            # 外层字符串 + 当前字符串重复 num 次
            cur_str = prev_str + num * cur_str
        else:
            # 普通字母，直接拼接到当前字符串
            cur_str += char

    return cur_str
```

### 6. 套路总结

**嵌套结构问题**的栈解法模板：
1. 遇到"进入"标记（如 `[`）-> 保存当前上下文到栈，重置状态
2. 遇到"退出"标记（如 `]`）-> 从栈中恢复上下文，合并结果
3. 栈中保存的是"外层的部分结果"

这个模板适用于：嵌套括号计算、HTML/XML 解析、嵌套列表展开等问题。

## 题目4：739. 每日温度

### 1. 核心考点

- **单调栈**（Monotonic Stack）——栈内元素保持单调递减
- "下一个更大元素"问题的经典模型
- 理解单调栈为什么能把 O(n^2) 优化到 O(n)

### 2. 解题思路（给新手讲）

给你一个温度数组 `temperatures`，对于每一天，你要找出**还要等几天才能遇到更高的温度**。如果后面没有更高的温度，就填 0。

比如 `[73, 74, 75, 71, 69, 72, 76, 73]`，输出 `[1, 1, 4, 2, 1, 1, 0, 0]`。

最朴素的想法：对每一天，往后逐天找更高温度。但这太慢了。

**单调栈的直觉**：

想象你排队，每个人举着一块写有温度的牌子。新来一个人（新的一天），如果他的温度比前面排着的人高，那些人就找到答案了——"等了这么多天，终于遇到更高温度了！"

具体来说：我们维护一个**从栈底到栈顶温度递减**的栈（存下标）：
- 新温度来了，如果比栈顶温度高，栈顶那天的答案就找到了
- 不断弹出栈顶，直到栈顶温度 >= 新温度，或栈为空
- 然后把新的一天压入栈

```
示例：temperatures = [73, 74, 75, 71, 69, 72, 76, 73]

用下标表示，栈中存的是下标：

第0天(73): 栈空，直接入栈                 栈: [0]
第1天(74): 74>73，弹出0，ans[0]=1-0=1     栈: [1]
第2天(75): 75>74，弹出1，ans[1]=2-1=1     栈: [2]
第3天(71): 71<75，直接入栈                 栈: [2, 3]
第4天(69): 69<71，直接入栈                 栈: [2, 3, 4]
第5天(72): 72>69，弹出4，ans[4]=5-4=1
           72>71，弹出3，ans[3]=5-3=2
           72<75，入栈                     栈: [2, 5]
第6天(76): 76>72，弹出5，ans[5]=6-5=1
           76>75，弹出2，ans[2]=6-2=4
           栈空，入栈                      栈: [6]
第7天(73): 73<76，直接入栈                 栈: [6, 7]

遍历结束，栈中剩余 [6,7]，它们没找到更高温度，ans 保持 0

最终结果: [1, 1, 4, 2, 1, 1, 0, 0] ✓
```

### 3. 从暴力到优化的推导

#### 暴力解法

对每一天，往后遍历找第一个更高温度：

```python
def dailyTemperatures(temperatures):
    n = len(temperatures)
    ans = [0] * n
    for i in range(n):                      # 对每一天
        for j in range(i + 1, n):           # 往后找
            if temperatures[j] > temperatures[i]:  # 找到更高温度
                ans[i] = j - i              # 记录等待天数
                break                       # 找到第一个就停
    return ans
```

时间复杂度 O(n^2)，最坏情况（递减序列）每天都要扫到末尾。

#### 优化思路

暴力解法的问题：每天都独立地往后找，大量重复比较。

关键观察：如果第 i 天温度比第 i+1 天低，那第 i 天的答案就是 1。如果第 i 天温度比第 i+1 天高，那第 i 天的答案至少要跳过第 i+1 天。

**单调栈的本质**是维护一个"待解决问题"的列表：
- 栈中存的是"还没找到更高温度的那些天"
- 新的一天如果温度更高，就能一次性解决栈中所有温度比它低的天
- 每个元素**最多入栈一次、出栈一次**，所以总时间是 O(n)

#### 最优解法

单调递减栈，时间复杂度 O(n)，空间复杂度 O(n)。

### 4. 关键思维点

- **栈中存下标而不是值**：因为我们需要计算天数差（下标差），温度可以通过下标查到
- **为什么是单调递减栈**：我们要找"下一个更大的"，所以栈中维护递减序列，一旦遇到更大的元素就触发弹出
- **每个元素最多入栈出栈各一次**：所以虽然有 while 循环嵌套在 for 循环里，总时间仍然是 O(n)
- **栈中剩余元素**：遍历结束后栈中还有元素，说明它们后面没有更大的温度，答案保持 0

### 5. 代码实现（Python）

```python
def dailyTemperatures(temperatures: list[int]) -> list[int]:
    n = len(temperatures)
    ans = [0] * n           # 初始化答案数组，默认 0（没找到更高温度）
    stack = []              # 单调栈，存下标，栈底到栈顶对应的温度递减

    for i in range(n):
        # 当栈不为空，且当前温度 > 栈顶下标对应的温度
        while stack and temperatures[i] > temperatures[stack[-1]]:
            prev_idx = stack.pop()          # 弹出栈顶下标
            ans[prev_idx] = i - prev_idx    # 等待天数 = 当前下标 - 栈顶下标
        stack.append(i)     # 当前下标入栈（等待未来的更高温度）

    return ans
```

### 6. 套路总结

**单调栈识别信号**：题目问"下一个更大/更小的元素"或"左边/右边第一个比我大/小的元素"。

**单调栈模板**：

```python
stack = []
for i in range(n):
    while stack and 比较条件(nums[i], nums[stack[-1]]):
        idx = stack.pop()
        # 处理 idx 位置的结果
    stack.append(i)
```

- 找"下一个更大" -> **单调递减栈**（栈顶最小）
- 找"下一个更小" -> **单调递增栈**（栈顶最大）

## 题目5：84. 柱状图中最大的矩形

### 1. 核心考点

- **单调栈的进阶应用**——同时求左右两侧第一个更小元素
- 理解"以每根柱子为高度的最大矩形"的思路
- 哨兵技巧简化边界处理

### 2. 解题思路（给新手讲）

给你一个柱状图，每根柱子的宽度都是 1，高度由数组 `heights` 给出。要求找出柱状图中能勾勒出的**最大矩形面积**。

比如 `heights = [2,1,5,6,2,3]`，最大矩形面积是 10（由高度 5 和 6 组成的矩形，宽 2、高 5）。

**核心思路**：对于每根柱子，如果以它的高度为矩形的高，那这个矩形最多能向左右**扩展到哪里**？

```
heights = [2, 1, 5, 6, 2, 3]

以每根柱子为高，找出它能扩展的最大宽度：

柱子0 (h=2): 左边没有更矮的（到边界），右边第一个更矮的是柱子1(h=1)
             宽度 = 1，面积 = 2 * 1 = 2

柱子1 (h=1): 左边没有更矮的，右边也没有更矮的
             宽度 = 6，面积 = 1 * 6 = 6

柱子2 (h=5): 左边第一个更矮的是柱子1(h=1)，右边第一个更矮的是柱子4(h=2)
             宽度 = 4-1-1 = 2，面积 = 5 * 2 = 10  ← 最大！

柱子3 (h=6): 左边第一个更矮的是柱子2(h=5)，右边第一个更矮的是柱子4(h=2)
             宽度 = 4-2-1 = 1，面积 = 6 * 1 = 6

柱子4 (h=2): 左边第一个更矮的是柱子1(h=1)，右边没有更矮的
             宽度 = 6-1-1 = 4，面积 = 2 * 4 = 8

柱子5 (h=3): 左边第一个更矮的是柱子4(h=2)，右边没有更矮的
             宽度 = 6-4-1 = 1，面积 = 3 * 1 = 3
```

所以关键问题变成了：**对每根柱子，找它左右两边第一个比它矮的柱子**。这正是单调栈的拿手好戏！

### 3. 从暴力到优化的推导

#### 暴力解法

对每根柱子，分别向左和向右扩展，直到遇到更矮的柱子：

```python
def largestRectangleArea(heights):
    n = len(heights)
    max_area = 0
    for i in range(n):                  # 枚举每根柱子作为高
        h = heights[i]
        left = i
        while left > 0 and heights[left - 1] >= h:  # 向左扩展
            left -= 1
        right = i
        while right < n - 1 and heights[right + 1] >= h:  # 向右扩展
            right += 1
        max_area = max(max_area, h * (right - left + 1))
    return max_area
```

时间复杂度 O(n^2)，最坏情况所有柱子等高，每根柱子都要扩展到两端。

#### 优化思路

暴力解法对每根柱子都独立地左右扩展，做了大量重复工作。

我们需要对**每根柱子**找到：
1. **左边第一个比它矮**的柱子位置（记为 `left[i]`）
2. **右边第一个比它矮**的柱子位置（记为 `right[i]`）

然后面积 = `heights[i] * (right[i] - left[i] - 1)`。

这两个问题都可以用**单调递增栈**一次遍历解决！

单调递增栈的工作过程：

```
heights = [2, 1, 5, 6, 2, 3]
加上哨兵: [0, 2, 1, 5, 6, 2, 3, 0]  (首尾各加一个高度0的虚拟柱子)
下标:      0  1  2  3  4  5  6  7

遍历过程（栈中存下标，对应高度递增）：

i=0, h=0: 栈空，入栈                        栈: [0]
i=1, h=2: 2>0，入栈                         栈: [0, 1]
i=2, h=1: 1<2，弹出1(h=2)
           面积 = 2 * (2-0-1) = 2
           1>0？不弹了，入栈                  栈: [0, 2]
i=3, h=5: 5>1，入栈                         栈: [0, 2, 3]
i=4, h=6: 6>5，入栈                         栈: [0, 2, 3, 4]
i=5, h=2: 2<6，弹出4(h=6)
           面积 = 6 * (5-3-1) = 6
           2<5，弹出3(h=5)
           面积 = 5 * (5-2-1) = 10  ← 最大！
           2>1，入栈                          栈: [0, 2, 5]
i=6, h=3: 3>2，入栈                         栈: [0, 2, 5, 6]
i=7, h=0: 0<3，弹出6(h=3)
           面积 = 3 * (7-5-1) = 3
           0<2，弹出5(h=2)
           面积 = 2 * (7-2-1) = 8
           0<1，弹出2(h=1)
           面积 = 1 * (7-0-1) = 6
           0=0，不弹了                        栈: [0]

最大面积 = 10 ✓
```

**为什么弹出时能确定左右边界？**
- 当柱子 `j` 被弹出时，当前柱子 `i` 就是它**右边第一个比它矮**的
- 弹出后的新栈顶就是它**左边第一个比它矮**的
- 所以宽度 = `i - 新栈顶 - 1`

#### 最优解法

单调递增栈 + 哨兵，一次遍历，时间复杂度 O(n)，空间复杂度 O(n)。

### 4. 关键思维点

- **以每根柱子为高度**：把问题转化为"对每根柱子，找它能扩展的最大宽度"
- **单调递增栈**：维护从栈底到栈顶递增的高度序列。遇到更矮的柱子时，栈顶柱子的右边界就确定了
- **弹出时同时确定左右边界**：右边界是当前柱子 `i`，左边界是新栈顶
- **哨兵技巧**：在首尾各加一个高度为 0 的柱子
  - 左边的 0：保证栈永远不会弹空（所有柱子都能找到左边界）
  - 右边的 0：保证遍历结束时栈中所有柱子都会被弹出处理
- **和"每日温度"的区别**：每日温度找"下一个更大"用递减栈；本题找"两侧更小"用递增栈

### 5. 代码实现（Python）

```python
def largestRectangleArea(heights: list[int]) -> int:
    # 哨兵技巧：首尾各加一个高度为 0 的柱子
    # 左哨兵保证栈不会弹空，右哨兵保证所有柱子最终都会被处理
    heights = [0] + heights + [0]
    n = len(heights)
    stack = []              # 单调递增栈，存下标
    max_area = 0

    for i in range(n):
        # 当前柱子比栈顶矮，栈顶柱子的右边界确定了
        while stack and heights[i] < heights[stack[-1]]:
            h = heights[stack.pop()]       # 弹出的柱子高度就是矩形的高
            w = i - stack[-1] - 1          # 宽度 = 右边界 - 左边界 - 1
            max_area = max(max_area, h * w)
        stack.append(i)     # 当前下标入栈

    return max_area
```

### 6. 套路总结

**柱状图最大矩形问题**的核心套路：
1. 枚举每根柱子作为矩形的高
2. 用单调递增栈找每根柱子左右两边第一个比它矮的柱子
3. 宽度 = 右边界 - 左边界 - 1
4. 使用哨兵简化边界处理

这个套路可以扩展到：接雨水（42题）、最大矩形（85题，逐行转化为柱状图问题）等。

**单调递增 vs 单调递减的选择**：
- 找左/右**第一个更小**的元素 -> 单调递增栈
- 找左/右**第一个更大**的元素 -> 单调递减栈

## 二、单调栈专题总结

### 什么时候用单调栈？

看到以下关键词就要想到单调栈：
- "下一个更大/更小的元素"
- "左/右边第一个比我大/小的元素"
- "能扩展到的最远范围"
- "柱状图"、"接雨水"类的几何问题

### 单调栈通用模板

```python
def monotonic_stack(nums):
    n = len(nums)
    result = [默认值] * n
    stack = []  # 存下标

    for i in range(n):
        while stack and 触发条件(nums[i], nums[stack[-1]]):
            idx = stack.pop()
            # 在这里处理 idx 位置的结果
            # nums[i] 是右边界
            # stack[-1]（新栈顶）是左边界
        stack.append(i)
```

### 单调递增 vs 单调递减

| 目标 | 栈的单调性 | 触发弹出条件 | 典型题目 |
|------|-----------|-------------|----------|
| 下一个**更大**元素 | 递减栈（栈顶最小） | `nums[i] > nums[stack[-1]]` | 739. 每日温度 |
| 下一个**更小**元素 | 递增栈（栈顶最大） | `nums[i] < nums[stack[-1]]` | 84. 柱状图最大矩形 |

### 哨兵技巧

在数组首尾添加**极值哨兵**，可以避免处理边界条件：
- 头部哨兵：防止栈弹空
- 尾部哨兵：确保所有元素最终都被处理

```python
# 柱状图问题的哨兵用法
heights = [0] + heights + [0]  # 首尾添加高度为 0 的哨兵
```

## 四、栈的本质理解

栈的核心能力是**"记住过去、回退到过去"**：

1. **括号匹配**——记住之前的左括号，遇到右括号时回退匹配
2. **最小栈**——记住每个时刻的最小值，弹出元素后回退到之前的最小值
3. **字符串解码**——记住外层的上下文，处理完内层后回退拼接
4. **单调栈**——记住"还没解决的问题"，遇到答案时一次性解决

理解了这个本质，遇到新的栈问题就不会慌了——只要问自己：**我需要"记住"什么？什么时候需要"回退"？**
