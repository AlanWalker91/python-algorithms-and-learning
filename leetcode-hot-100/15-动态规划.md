# 动态规划 - LeetCode 热题 100

> 动态规划(Dynamic Programming, DP)的核心思想：**把大问题拆成小问题，记住小问题的答案，避免重复计算。**
>
> 三步走：
> 1. **定义状态** -- `dp[i]` 代表什么？
> 2. **找状态转移方程** -- `dp[i]` 和之前的状态有什么关系？
> 3. **确定初始条件和边界** -- 最小的子问题答案是什么？

## 题目1：70. 爬楼梯

### 1. 核心考点

- 最基础的动态规划入门题
- 斐波那契数列的变体
- 理解「当前状态由前面的状态推导而来」

### 2. 解题思路（给新手讲）

想象你站在楼梯底部，要爬到第 n 阶。每次你只能选择爬 1 阶或者 2 阶。

关键问题：**到达第 n 阶，你的"最后一步"是什么？**

只有两种可能：
- 从第 n-1 阶爬 1 步上来
- 从第 n-2 阶爬 2 步上来

所以，到达第 n 阶的方法数 = 到达第 n-1 阶的方法数 + 到达第 n-2 阶的方法数。

这就是经典的斐波那契数列！

**状态定义：** `dp[i]` 表示爬到第 i 阶的方法总数。

**状态转移方程：** `dp[i] = dp[i-1] + dp[i-2]`

**初始条件：** `dp[0] = 1`（站在原地，一种方式），`dp[1] = 1`（爬1阶，只有一种方式）。

### 3. 从暴力到优化的推导

#### 暴力解法

用递归穷举所有可能。每一步要么爬1阶，要么爬2阶，形成一棵二叉递归树。

```python
def climbStairs(n: int) -> int:
    if n <= 1:
        return 1
    return climbStairs(n - 1) + climbStairs(n - 2)
```

时间复杂度：O(2^n)，大量重复计算（比如 climbStairs(3) 会被算很多次）。

#### 优化思路

暴力递归中，同一个子问题被反复求解。用一个数组把算过的结果记下来（记忆化），或者直接从底往上推（递推）。

#### 最优解法

由于 `dp[i]` 只依赖 `dp[i-1]` 和 `dp[i-2]`，我们甚至不需要数组，只需要两个变量。

```python
def climbStairs(n: int) -> int:
    # 特殊情况：0阶或1阶直接返回
    if n <= 1:
        return 1
    # prev 代表 dp[i-2]，curr 代表 dp[i-1]
    prev, curr = 1, 1
    # 从第2阶开始递推到第n阶
    for i in range(2, n + 1):
        # 当前阶的方法数 = 前一阶 + 前两阶
        prev, curr = curr, prev + curr
    # curr 就是 dp[n]
    return curr
```

时间复杂度：O(n)，空间复杂度：O(1)。

### 4. 关键思维点

- **"最后一步"思维法**：想清楚到达当前状态的最后一步是什么，就能找到状态转移方程。
- 这道题的本质就是斐波那契数列，`dp[i] = dp[i-1] + dp[i-2]`。
- 空间优化技巧：当 dp[i] 只依赖前几个状态时，可以用滚动变量替代整个数组。

### 5. 代码实现（Python）

```python
class Solution:
    def climbStairs(self, n: int) -> int:
        # 边界：0阶和1阶各只有1种方法
        if n <= 1:
            return 1
        # prev = dp[i-2], curr = dp[i-1]
        # 初始化：dp[0] = 1, dp[1] = 1
        prev, curr = 1, 1
        # 从第2阶推导到第n阶
        for i in range(2, n + 1):
            # dp[i] = dp[i-1] + dp[i-2]
            # 同时更新 prev 和 curr
            prev, curr = curr, prev + curr
        return curr
```

### 6. 套路总结

| 项目 | 内容 |
|------|------|
| 题型 | 计数型 DP（问"有几种方式"） |
| 状态定义 | `dp[i]` = 到达第 i 阶的方法数 |
| 转移方程 | `dp[i] = dp[i-1] + dp[i-2]` |
| 空间优化 | 滚动变量，O(1) 空间 |
| 类似题目 | 斐波那契数列、跳台阶、组合问题 |

## 题目2：118. 杨辉三角

### 1. 核心考点

- 二维 DP 的入门题
- 理解「当前行的每个元素由上一行的两个元素决定」
- 边界处理（每行首尾为1）

### 2. 解题思路（给新手讲）

杨辉三角长这样：

```
    1
   1 1
  1 2 1
 1 3 3 1
1 4 6 4 1
```

规律非常明确：
- 每行的第一个和最后一个元素都是 1。
- 中间的每个元素 = 上一行同位置的左边元素 + 上一行同位置元素。

用公式表示：`triangle[i][j] = triangle[i-1][j-1] + triangle[i-1][j]`

**状态定义：** `dp[i][j]` 表示第 i 行第 j 列的值。

**状态转移方程：** `dp[i][j] = dp[i-1][j-1] + dp[i-1][j]`（当 `0 < j < i` 时）

**边界条件：** `dp[i][0] = dp[i][i] = 1`（每行首尾为1）

### 3. 从暴力到优化的推导

#### 暴力解法

其实这道题本身就是用 DP 思想直接求解，没有比递推更暴力的做法了。递归写法虽然可行，但没有必要。

#### 优化思路

直接逐行构建，每行基于上一行的结果计算。由于每行只依赖上一行，空间上其实可以只保留上一行（但本题要求返回整个三角形，所以需要全部存储）。

#### 最优解法

```python
def generate(numRows: int) -> list:
    triangle = []
    for i in range(numRows):
        # 创建当前行，初始化全为1
        row = [1] * (i + 1)
        # 填充中间元素
        for j in range(1, i):
            row[j] = triangle[i - 1][j - 1] + triangle[i - 1][j]
        triangle.append(row)
    return triangle
```

时间复杂度：O(n^2)，空间复杂度：O(n^2)（存储整个三角形）。

### 4. 关键思维点

- 每行的长度等于行号+1（第0行1个元素，第1行2个元素...）。
- 先把整行初始化为1，然后只需要填充中间部分，省去了边界判断。
- 这道题是二维 DP 的最简单形式，理解了它就能理解更复杂的二维 DP。

### 5. 代码实现（Python）

```python
class Solution:
    def generate(self, numRows: int) -> list:
        # 结果列表，存储整个三角形
        triangle = []
        for i in range(numRows):
            # 第 i 行有 i+1 个元素，先全部初始化为 1
            row = [1] * (i + 1)
            # 从第 2 行开始，中间的元素需要计算
            # j 的范围是 [1, i-1]，首尾的 1 不需要动
            for j in range(1, i):
                # 当前元素 = 上一行左上角 + 上一行正上方
                row[j] = triangle[i - 1][j - 1] + triangle[i - 1][j]
            # 将当前行加入三角形
            triangle.append(row)
        return triangle
```

### 6. 套路总结

| 项目 | 内容 |
|------|------|
| 题型 | 构造型 DP / 模拟 |
| 状态定义 | `dp[i][j]` = 第 i 行第 j 个元素的值 |
| 转移方程 | `dp[i][j] = dp[i-1][j-1] + dp[i-1][j]` |
| 边界 | 每行首尾为 1 |
| 技巧 | 先初始化全1，再填充中间 |

## 题目3：198. 打家劫舍

### 1. 核心考点

- 经典的「选或不选」DP 模型
- 不能选相邻元素的约束条件
- 理解状态转移中的"取舍"

### 2. 解题思路（给新手讲）

你是一个小偷，沿着一排房子偷东西。每个房子里有一定金额的钱。**但是相邻的两间房子不能同时偷**（会触发警报）。问你最多能偷多少钱？

对于每间房子，你面临一个选择：**偷还是不偷？**

- **偷第 i 间**：那第 i-1 间就不能偷，收益 = 第 i 间的钱 + 前 i-2 间能偷到的最多的钱
- **不偷第 i 间**：收益 = 前 i-1 间能偷到的最多的钱

两种选择取最大值！

**状态定义：** `dp[i]` 表示考虑前 i 间房子，能偷到的最大金额。

**状态转移方程：** `dp[i] = max(dp[i-1], dp[i-2] + nums[i])`

- `dp[i-1]`：不偷第 i 间
- `dp[i-2] + nums[i]`：偷第 i 间

**初始条件：** `dp[0] = nums[0]`，`dp[1] = max(nums[0], nums[1])`

### 3. 从暴力到优化的推导

#### 暴力解法

对每间房子做「偷或不偷」的决策，用递归穷举所有组合。

```python
def rob(nums, i):
    if i < 0:
        return 0
    # 要么不偷第 i 间，要么偷第 i 间（跳过 i-1）
    return max(rob(nums, i - 1), rob(nums, i - 2) + nums[i])
```

时间复杂度：O(2^n)，存在大量重叠子问题。

#### 优化思路

加入记忆化，或者改为自底向上的递推。`dp[i]` 只依赖 `dp[i-1]` 和 `dp[i-2]`，可以用滚动变量优化空间。

#### 最优解法

```python
def rob(nums: list) -> int:
    if not nums:
        return 0
    if len(nums) == 1:
        return nums[0]
    # prev = dp[i-2], curr = dp[i-1]
    prev, curr = 0, 0
    for num in nums:
        # 偷当前房子(prev + num) vs 不偷当前房子(curr)
        prev, curr = curr, max(curr, prev + num)
    return curr
```

时间复杂度：O(n)，空间复杂度：O(1)。

### 4. 关键思维点

- **"偷与不偷"的选择模型**：这是 DP 中最常见的模型之一。面对每个元素，考虑「选它」和「不选它」两种情况。
- 关键约束是"不能相邻"，这意味着选了第 i 个，就只能从 i-2 转移过来。
- 滚动变量的写法 `prev, curr = curr, max(curr, prev + num)` 非常精炼，需要理解同步赋值的含义。

### 5. 代码实现（Python）

```python
class Solution:
    def rob(self, nums: list) -> int:
        # 特殊情况处理
        if not nums:
            return 0
        if len(nums) == 1:
            return nums[0]
        # prev 相当于 dp[i-2]，curr 相当于 dp[i-1]
        # 初始时都为 0（还没考虑任何房子）
        prev, curr = 0, 0
        for num in nums:
            # temp 保存当前的最优解
            # 选择1：不偷当前房子，保持 curr
            # 选择2：偷当前房子，prev + num（跳过上一间）
            temp = max(curr, prev + num)
            # 滚动更新
            prev = curr
            curr = temp
        # curr 就是考虑所有房子后的最大收益
        return curr
```

### 6. 套路总结

| 项目 | 内容 |
|------|------|
| 题型 | 选择型 DP（选或不选） |
| 状态定义 | `dp[i]` = 前 i 间房子能偷的最大金额 |
| 转移方程 | `dp[i] = max(dp[i-1], dp[i-2] + nums[i])` |
| 核心约束 | 不能选相邻元素 |
| 空间优化 | 滚动变量，O(1) 空间 |

## 题目4：279. 完全平方数

### 1. 核心考点

- 完全背包问题的变体
- 理解"最少个数"问题的 DP 建模
- BFS 也能解，但 DP 更经典

### 2. 解题思路（给新手讲）

给你一个正整数 n，找出若干个完全平方数（1, 4, 9, 16, ...）使得它们的和等于 n，要求使用的完全平方数个数最少。

比如 n = 12，答案是 3，因为 12 = 4 + 4 + 4。

怎么思考？想象你要凑出金额 n，你手里有面值为 1, 4, 9, 16, ... 的"硬币"，每种硬币可以无限使用，问最少用几枚硬币凑出 n。

这不就是"零钱兑换"嘛！

**状态定义：** `dp[i]` 表示组成数字 i 所需的最少完全平方数个数。

**状态转移方程：** `dp[i] = min(dp[i - j*j] + 1)`，其中 `j*j <= i`

含义：从 i 中减去一个完全平方数 j*j，剩下的部分 `i - j*j` 需要 `dp[i - j*j]` 个完全平方数，加上当前这一个，共 `dp[i - j*j] + 1` 个。遍历所有可能的 j，取最小值。

**初始条件：** `dp[0] = 0`（0 不需要任何完全平方数）

### 3. 从暴力到优化的推导

#### 暴力解法

递归尝试所有可能的完全平方数组合：

```python
def numSquares(n):
    if n == 0:
        return 0
    result = n  # 最差情况：全用1
    j = 1
    while j * j <= n:
        result = min(result, numSquares(n - j * j) + 1)
        j += 1
    return result
```

时间复杂度极高，有大量重复计算。

#### 优化思路

自底向上递推，从 `dp[0]` 开始填表到 `dp[n]`。每个 `dp[i]` 遍历所有不超过 i 的完全平方数。

#### 最优解法

```python
def numSquares(n: int) -> int:
    # dp[i] 表示组成 i 的最少完全平方数个数
    dp = [0] * (n + 1)
    for i in range(1, n + 1):
        # 最差情况：全用 1，需要 i 个
        dp[i] = i
        j = 1
        while j * j <= i:
            # 尝试减去每个完全平方数 j*j
            dp[i] = min(dp[i], dp[i - j * j] + 1)
            j += 1
    return dp[n]
```

时间复杂度：O(n * sqrt(n))，空间复杂度：O(n)。

### 4. 关键思维点

- 本题本质是**完全背包问题**：物品是所有完全平方数，每个物品可以无限使用，求装满背包的最少物品数。
- 初始化 `dp[i] = i` 很重要（最坏情况全用1来凑）。
- 内层循环只需要遍历到 `j*j <= i`，因为再大的完全平方数减去后会变成负数。

### 5. 代码实现（Python）

```python
class Solution:
    def numSquares(self, n: int) -> int:
        # dp[i]: 和为 i 的最少完全平方数个数
        # 初始化为 0，后面逐个计算
        dp = [0] * (n + 1)
        for i in range(1, n + 1):
            # 最坏情况：全部用 1 来凑，需要 i 个 1
            dp[i] = i
            # 尝试每个完全平方数 j*j
            j = 1
            while j * j <= i:
                # 如果用了 j*j 这个完全平方数
                # 剩下 i - j*j 需要 dp[i - j*j] 个
                # 总共需要 dp[i - j*j] + 1 个
                dp[i] = min(dp[i], dp[i - j * j] + 1)
                j += 1
        return dp[n]
```

### 6. 套路总结

| 项目 | 内容 |
|------|------|
| 题型 | 完全背包（最少物品数） |
| 状态定义 | `dp[i]` = 和为 i 的最少完全平方数个数 |
| 转移方程 | `dp[i] = min(dp[i - j*j] + 1)` |
| 初始化 | `dp[0] = 0`，其余为 i（最坏值） |
| 本质 | 完全背包求最小值 |

## 题目5：322. 零钱兑换

### 1. 核心考点

- 完全背包问题的经典应用
- 理解"最少硬币数"的 DP 建模
- 处理"凑不出来"的情况

### 2. 解题思路（给新手讲）

给你不同面额的硬币 coins 和一个总金额 amount。每种硬币可以无限使用。问凑成 amount 最少需要多少枚硬币？如果凑不出来，返回 -1。

比如 coins = [1, 2, 5]，amount = 11，答案是 3（5 + 5 + 1）。

思考方式：要凑出金额 i，我可以在前一步凑出金额 `i - coin` 的基础上，再加一枚面值为 coin 的硬币。遍历所有硬币面值，取最少的那个。

**状态定义：** `dp[i]` 表示凑成金额 i 所需的最少硬币数。

**状态转移方程：** `dp[i] = min(dp[i - coin] + 1)`，其中 coin 遍历 coins 数组，且 `i - coin >= 0`

**初始条件：** `dp[0] = 0`（凑0元不需要硬币），其他初始化为 `amount + 1`（表示不可达，因为硬币最小面值为1，最多用 amount 枚）。

### 3. 从暴力到优化的推导

#### 暴力解法

递归尝试每种硬币，穷举所有可能：

```python
def coinChange(coins, amount):
    if amount == 0:
        return 0
    if amount < 0:
        return -1
    result = float('inf')
    for coin in coins:
        sub = coinChange(coins, amount - coin)
        if sub != -1:
            result = min(result, sub + 1)
    return result if result != float('inf') else -1
```

时间复杂度：指数级，大量重叠子问题。

#### 优化思路

和完全平方数一样，自底向上递推。关键是初始化：用一个"不可能"的大值表示"凑不出来"。

#### 最优解法

```python
def coinChange(coins: list, amount: int) -> int:
    # 初始化：amount + 1 是一个不可能的大值
    dp = [amount + 1] * (amount + 1)
    dp[0] = 0  # 凑0元需要0枚硬币
    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i:
                dp[i] = min(dp[i], dp[i - coin] + 1)
    # 如果 dp[amount] 仍然是初始值，说明凑不出来
    return dp[amount] if dp[amount] <= amount else -1
```

时间复杂度：O(amount * len(coins))，空间复杂度：O(amount)。

### 4. 关键思维点

- 初始化为 `amount + 1` 是一个巧妙的技巧：它比任何合法答案都大，方便用 `min` 更新，最后还能用来判断是否可达。
- 本题和279题（完全平方数）几乎是同一个模型，只是"硬币面值"不同。
- 完全背包的标志：每种硬币可以无限使用。

### 5. 代码实现（Python）

```python
class Solution:
    def coinChange(self, coins: list, amount: int) -> int:
        # dp[i]: 凑成金额 i 的最少硬币数
        # 初始化为 amount + 1（一个不可能达到的大值）
        dp = [amount + 1] * (amount + 1)
        # 凑 0 元需要 0 枚硬币
        dp[0] = 0
        # 自底向上，从金额 1 推到 amount
        for i in range(1, amount + 1):
            # 尝试每种硬币
            for coin in coins:
                # 当前硬币面值不能超过要凑的金额
                if coin <= i:
                    # 用了这枚硬币后，还需要凑 i-coin
                    # 总硬币数 = dp[i-coin] + 1
                    dp[i] = min(dp[i], dp[i - coin] + 1)
        # 判断是否能凑出 amount
        # 如果 dp[amount] 还是初始的大值，说明凑不出
        return dp[amount] if dp[amount] <= amount else -1
```

### 6. 套路总结

| 项目 | 内容 |
|------|------|
| 题型 | 完全背包（最少物品数） |
| 状态定义 | `dp[i]` = 凑成金额 i 的最少硬币数 |
| 转移方程 | `dp[i] = min(dp[i - coin] + 1)` |
| 初始化 | `dp[0] = 0`，其余为 `amount + 1` |
| 注意点 | 需要处理凑不出来返回 -1 的情况 |

## 题目6：139. 单词拆分

### 1. 核心考点

- 字符串 DP
- 子串匹配与动态规划结合
- 理解"可行性"DP（结果是 True/False）

### 2. 解题思路（给新手讲）

给你一个字符串 s 和一个单词字典 wordDict，判断 s 是否可以被拆分成字典中的一个或多个单词。

比如 s = "leetcode"，wordDict = ["leet", "code"]，答案是 True，因为 "leetcode" 可以拆分为 "leet" + "code"。

思考方式：对于字符串 s 的前 i 个字符 `s[0:i]`，如果存在一个位置 j，使得：
1. `s[0:j]` 可以被拆分（即 `dp[j] = True`）
2. `s[j:i]` 是字典中的一个单词

那么 `s[0:i]` 就可以被拆分，即 `dp[i] = True`。

**状态定义：** `dp[i]` 表示字符串 s 的前 i 个字符 `s[0:i]` 是否可以被拆分成字典中的单词。

**状态转移方程：** `dp[i] = any(dp[j] and s[j:i] in wordDict for j in range(i))`

**初始条件：** `dp[0] = True`（空字符串可以被"拆分"）

### 3. 从暴力到优化的推导

#### 暴力解法

用回溯法尝试所有可能的拆分方式：

```python
def wordBreak(s, wordDict):
    if not s:
        return True
    for word in wordDict:
        if s.startswith(word):
            if wordBreak(s[len(word):], wordDict):
                return True
    return False
```

时间复杂度：指数级，因为可能有大量重叠子问题。

#### 优化思路

把字典转成 set 加速查找。用 DP 从左到右扫描，对于每个位置 i，检查是否存在一个合法的切割点 j。

#### 最优解法

```python
def wordBreak(s: str, wordDict: list) -> bool:
    word_set = set(wordDict)
    n = len(s)
    dp = [False] * (n + 1)
    dp[0] = True  # 空串可拆分
    for i in range(1, n + 1):
        for j in range(i):
            if dp[j] and s[j:i] in word_set:
                dp[i] = True
                break  # 找到一种拆法就够了
    return dp[n]
```

时间复杂度：O(n^2 * m)（m 为单词平均长度，用于切片比较），空间复杂度：O(n)。

### 4. 关键思维点

- **"在哪里切一刀"的思维**：对于每个位置 i，枚举所有可能的切割点 j，判断左半部分是否合法且右半部分是否在字典中。
- 字典转 set 是关键优化，否则每次查找都是 O(m*k)（k 为字典大小）。
- `dp[0] = True` 是精妙的设定：它代表空字符串是合法的，这样当整个前缀 `s[0:i]` 就是字典中的一个单词时，`dp[0] and s[0:i] in word_set` 可以正确返回 True。
- 找到一种合法拆分就 `break`，不需要继续检查。

### 5. 代码实现（Python）

```python
class Solution:
    def wordBreak(self, s: str, wordDict: list) -> bool:
        # 将字典转为集合，加速查找 O(1)
        word_set = set(wordDict)
        n = len(s)
        # dp[i]: s 的前 i 个字符能否被拆分
        dp = [False] * (n + 1)
        # 空字符串认为可以拆分（基础情况）
        dp[0] = True
        # 外层循环：枚举右端点 i
        for i in range(1, n + 1):
            # 内层循环：枚举切割点 j
            for j in range(i):
                # 如果前 j 个字符可以拆分
                # 并且 s[j:i] 是字典中的单词
                if dp[j] and s[j:i] in word_set:
                    dp[i] = True
                    break  # 找到一种方案即可
        # dp[n] 表示整个字符串能否被拆分
        return dp[n]
```

### 6. 套路总结

| 项目 | 内容 |
|------|------|
| 题型 | 可行性 DP（True/False） |
| 状态定义 | `dp[i]` = s 的前 i 个字符能否被字典中的单词拆分 |
| 转移方程 | `dp[i] = any(dp[j] and s[j:i] in wordDict)` |
| 技巧 | 字典转 set 加速查找 |
| 思维模式 | "在哪里切一刀" |

## 题目7：300. 最长递增子序列

### 1. 核心考点

- 经典子序列 DP
- 理解"子序列"和"子数组"的区别（子序列不要求连续）
- O(n^2) 的 DP 解法和 O(n log n) 的贪心+二分解法

### 2. 解题思路（给新手讲）

给你一个整数数组 nums，找到其中最长严格递增子序列的长度。子序列可以不连续，但要保持相对顺序。

比如 nums = [10, 9, 2, 5, 3, 7, 101, 18]，最长递增子序列是 [2, 3, 7, 101] 或 [2, 3, 7, 18]，长度为 4。

DP 思路：对于每个元素 nums[i]，看看它前面有哪些比它小的元素 nums[j]（j < i），nums[i] 可以接在以 nums[j] 结尾的递增子序列后面，形成更长的递增子序列。

**状态定义：** `dp[i]` 表示以 nums[i] 结尾的最长递增子序列的长度。

**状态转移方程：** `dp[i] = max(dp[j] + 1)`，其中 `j < i` 且 `nums[j] < nums[i]`

含义：遍历 i 之前的所有元素 j，如果 nums[j] < nums[i]，说明 nums[i] 可以接在以 nums[j] 结尾的子序列后面，长度 +1。取所有这样的 j 中最长的那个。

**初始条件：** `dp[i] = 1`（每个元素自身构成长度为1的递增子序列）

### 3. 从暴力到优化的推导

#### 暴力解法

枚举所有子序列（2^n 个），检查哪些是递增的，取最长的。时间复杂度 O(n * 2^n)。

#### 优化思路

用 DP，每个位置只需要看前面的所有位置，O(n^2)。还可以用贪心+二分优化到 O(n log n)。

#### 最优解法

**O(n^2) DP 解法（更好理解）：**

```python
def lengthOfLIS(nums: list) -> int:
    n = len(nums)
    dp = [1] * n  # 每个元素自成子序列，长度为1
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)
```

**O(n log n) 贪心+二分解法（更优）：**

```python
import bisect

def lengthOfLIS(nums: list) -> int:
    # tails[i] 表示长度为 i+1 的递增子序列的最小末尾元素
    tails = []
    for num in nums:
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)  # num 比所有末尾都大，扩展子序列
        else:
            tails[pos] = num  # 替换，让末尾尽可能小
    return len(tails)
```

### 4. 关键思维点

- **状态定义的细节**：`dp[i]` 是"以 nums[i] 结尾"的 LIS 长度，不是"前 i 个元素的 LIS 长度"。这个区别很重要！因为必须固定结尾才能做状态转移。
- 最终答案是 `max(dp)`，不是 `dp[n-1]`，因为最长递增子序列不一定以最后一个元素结尾。
- O(n log n) 解法的核心思想：维护一个 tails 数组，tails[i] 表示长度为 i+1 的递增子序列的最小末尾元素。末尾越小，后面越有机会接上更多元素。

### 5. 代码实现（Python）

```python
class Solution:
    def lengthOfLIS(self, nums: list) -> int:
        if not nums:
            return 0
        n = len(nums)
        # dp[i]: 以 nums[i] 结尾的最长递增子序列长度
        # 初始化为 1：每个元素自己就是长度为 1 的子序列
        dp = [1] * n
        # 外层循环：枚举当前元素 nums[i]
        for i in range(1, n):
            # 内层循环：检查 i 之前的每个元素
            for j in range(i):
                # 如果 nums[j] < nums[i]
                # nums[i] 可以接在以 nums[j] 结尾的子序列后面
                if nums[j] < nums[i]:
                    # 更新 dp[i] 为更大的值
                    dp[i] = max(dp[i], dp[j] + 1)
        # 答案是 dp 数组中的最大值
        # 因为最长递增子序列不一定以哪个元素结尾
        return max(dp)
```

**O(n log n) 贪心+二分版本：**

```python
class Solution:
    def lengthOfLIS(self, nums: list) -> int:
        import bisect
        # tails 数组：tails[i] 是长度为 i+1 的递增子序列的最小末尾
        tails = []
        for num in nums:
            # 在 tails 中找到第一个 >= num 的位置
            pos = bisect.bisect_left(tails, num)
            if pos == len(tails):
                # num 比 tails 中所有元素都大
                # 可以扩展最长递增子序列
                tails.append(num)
            else:
                # 替换 tails[pos]，让末尾更小
                # 这样后面更有可能接上更长的子序列
                tails[pos] = num
        # tails 的长度就是最长递增子序列的长度
        return len(tails)
```

### 6. 套路总结

| 项目 | 内容 |
|------|------|
| 题型 | 子序列 DP |
| 状态定义 | `dp[i]` = 以 nums[i] **结尾**的 LIS 长度 |
| 转移方程 | `dp[i] = max(dp[j] + 1)` 其中 `nums[j] < nums[i]` |
| 答案 | `max(dp)` 而不是 `dp[-1]` |
| 进阶 | 贪心+二分 O(n log n) |

## 题目8：152. 乘积最大子数组

### 1. 核心考点

- 子数组（连续）DP
- **同时维护最大值和最小值**的技巧
- 负数乘以负数变正数的特殊处理

### 2. 解题思路（给新手讲）

给你一个整数数组 nums，找出乘积最大的连续子数组，返回该子数组的最大乘积。

这道题和"最大子数组和"（53题）类似，但乘法有一个坑：**负负得正**！

比如 [-2, 3, -4]，前面累积的最小乘积 -6，乘以 -4 变成 24，反而是最大的！

所以我们不能只记录最大值，还必须同时记录最小值。因为当前最小值（负数）遇到负数后，可能翻身变成最大值。

**状态定义：**
- `max_val`：以当前元素结尾的子数组的最大乘积
- `min_val`：以当前元素结尾的子数组的最小乘积

**状态转移：**
```
新的 max_val = max(nums[i], max_val * nums[i], min_val * nums[i])
新的 min_val = min(nums[i], max_val * nums[i], min_val * nums[i])
```

每个位置有三种选择：
1. 从自己重新开始（nums[i]）
2. 接在之前的最大乘积后面（max_val * nums[i]）
3. 接在之前的最小乘积后面（min_val * nums[i]，负负得正的情况）

### 3. 从暴力到优化的推导

#### 暴力解法

枚举所有子数组，计算乘积，取最大值。

```python
def maxProduct(nums):
    result = nums[0]
    n = len(nums)
    for i in range(n):
        prod = 1
        for j in range(i, n):
            prod *= nums[j]
            result = max(result, prod)
    return result
```

时间复杂度：O(n^2)。

#### 优化思路

用 DP，关键洞察：乘法中负数的存在导致最大值和最小值可能互换。所以需要同时追踪最大值和最小值。

#### 最优解法

```python
def maxProduct(nums: list) -> int:
    max_val = min_val = result = nums[0]
    for i in range(1, len(nums)):
        # 如果当前数是负数，最大和最小会互换
        candidates = (nums[i], max_val * nums[i], min_val * nums[i])
        max_val = max(candidates)
        min_val = min(candidates)
        result = max(result, max_val)
    return result
```

时间复杂度：O(n)，空间复杂度：O(1)。

### 4. 关键思维点

- **同时维护最大值和最小值**是这道题的核心技巧。遇到负数时，之前的最小值可能变成最大值。
- 与"最大子数组和"的区别：加法中负数一定拖后腿，但乘法中两个负数相乘会变正。
- 三个候选值取 max/min：不只是"接着前面"或"重新开始"，还要考虑"最小值翻转"。
- 注意：更新 max_val 和 min_val 时，要用的是上一步的值，不能先更新 max_val 再用新的 max_val 计算 min_val。

### 5. 代码实现（Python）

```python
class Solution:
    def maxProduct(self, nums: list) -> int:
        # 初始化：第一个元素自成子数组
        max_val = nums[0]  # 以当前位置结尾的最大乘积
        min_val = nums[0]  # 以当前位置结尾的最小乘积
        result = nums[0]   # 全局最大乘积
        for i in range(1, len(nums)):
            # 当前元素有三种选择：
            # 1. 独立开始新的子数组：nums[i]
            # 2. 接在最大乘积后面：max_val * nums[i]
            # 3. 接在最小乘积后面：min_val * nums[i]（负负得正）
            # 先用临时变量保存，避免更新顺序问题
            temp_max = max(nums[i], max_val * nums[i], min_val * nums[i])
            temp_min = min(nums[i], max_val * nums[i], min_val * nums[i])
            # 更新最大值和最小值
            max_val = temp_max
            min_val = temp_min
            # 更新全局最大乘积
            result = max(result, max_val)
        return result
```

### 6. 套路总结

| 项目 | 内容 |
|------|------|
| 题型 | 子数组 DP（连续） |
| 状态定义 | 同时维护以当前元素结尾的最大乘积和最小乘积 |
| 转移方程 | `max_val = max(num, max_val*num, min_val*num)` |
| 核心技巧 | 负负得正 -> 必须同时跟踪最大和最小 |
| 注意 | 更新时用临时变量防止覆盖 |

## 题目9：416. 分割等和子集

### 1. 核心考点

- 0-1 背包问题的经典变体
- 将问题转化为"能否恰好装满容量为 target 的背包"
- 一维 DP 数组 + 逆序遍历

### 2. 解题思路（给新手讲）

给你一个只包含正整数的数组 nums，判断能否将其分成两个子集，使两个子集的元素之和相等。

比如 nums = [1, 5, 11, 5]，可以分成 [1, 5, 5] 和 [11]，两组都是 11，返回 True。

**关键转化**：如果数组总和为 sum，要分成两个相等的子集，每个子集的和必须是 sum/2。如果 sum 是奇数，直接返回 False。

问题转化为：**从数组中选出若干元素，使它们的和恰好等于 sum/2。**

这就是经典的 0-1 背包问题！每个元素要么选，要么不选，背包容量为 sum/2。

**状态定义：** `dp[j]` 表示是否能从数组中选出若干元素，使和恰好为 j。

**状态转移方程：** `dp[j] = dp[j] or dp[j - nums[i]]`

- `dp[j]`：不选 nums[i]，和为 j 是否可达
- `dp[j - nums[i]]`：选了 nums[i]，剩余 j - nums[i] 是否可达

**初始条件：** `dp[0] = True`（和为0总是可以的，不选任何元素即可）

### 3. 从暴力到优化的推导

#### 暴力解法

枚举所有子集（2^n 种），检查是否有一个子集的和为 sum/2。

```python
def canPartition(nums):
    total = sum(nums)
    if total % 2 != 0:
        return False
    target = total // 2
    # 递归枚举：每个数选或不选
    def dfs(index, remaining):
        if remaining == 0:
            return True
        if index >= len(nums) or remaining < 0:
            return False
        return dfs(index + 1, remaining - nums[index]) or dfs(index + 1, remaining)
    return dfs(0, target)
```

时间复杂度：O(2^n)。

#### 优化思路

用二维 DP：`dp[i][j]` 表示前 i 个元素能否凑出和为 j。但空间是 O(n * target)。

进一步优化：由于 `dp[i][j]` 只依赖 `dp[i-1][j]` 和 `dp[i-1][j-nums[i]]`（即上一行），可以压缩为一维数组。但必须**逆序遍历** j，否则会导致同一个元素被重复使用（变成完全背包）。

#### 最优解法

```python
def canPartition(nums: list) -> bool:
    total = sum(nums)
    if total % 2 != 0:
        return False
    target = total // 2
    dp = [False] * (target + 1)
    dp[0] = True
    for num in nums:
        # 逆序遍历！保证每个元素只用一次
        for j in range(target, num - 1, -1):
            dp[j] = dp[j] or dp[j - num]
    return dp[target]
```

时间复杂度：O(n * target)，空间复杂度：O(target)。

### 4. 关键思维点

- **问题转化**：等和分割 -> 0-1 背包（能否恰好装满 sum/2）。这步转化是解题的关键。
- **逆序遍历**：在一维 DP 中，必须从大到小遍历 j。如果正序遍历，同一轮中更新 dp[j] 时用到的 dp[j-num] 已经被当前轮更新过了，相当于 num 被用了多次。逆序则保证 dp[j-num] 还是上一轮的值。
- 提前剪枝：总和为奇数直接返回 False；如果数组中最大值超过 target，也返回 False。

### 5. 代码实现（Python）

```python
class Solution:
    def canPartition(self, nums: list) -> bool:
        total = sum(nums)
        # 总和为奇数，不可能平分
        if total % 2 != 0:
            return False
        target = total // 2
        # 剪枝：最大元素超过 target，不可能
        if max(nums) > target:
            return False
        # dp[j]: 能否从数组中选出若干元素，使和恰好为 j
        dp = [False] * (target + 1)
        # 和为 0 总是可以的（不选任何元素）
        dp[0] = True
        # 遍历每个元素
        for num in nums:
            # 关键：逆序遍历！从 target 到 num
            # 逆序保证每个元素在每轮中只被使用一次（0-1 背包）
            # 如果正序，dp[j-num] 可能已经在本轮被更新，导致重复使用
            for j in range(target, num - 1, -1):
                # 选 num：dp[j-num] 为 True 说明去掉 num 后的和可达
                # 不选 num：dp[j] 保持原值
                dp[j] = dp[j] or dp[j - num]
        return dp[target]
```

### 6. 套路总结

| 项目 | 内容 |
|------|------|
| 题型 | 0-1 背包（可行性） |
| 关键转化 | 等和分割 -> 能否凑出 sum/2 |
| 状态定义 | `dp[j]` = 能否选出元素使和为 j |
| 转移方程 | `dp[j] = dp[j] or dp[j - num]` |
| 核心技巧 | 一维数组 + 逆序遍历（0-1 背包标志性操作） |

## 题目10：32. 最长有效括号

### 1. 核心考点

- 字符串 DP 中的高难度题
- 状态定义需要仔细设计
- 也可以用栈解决，但 DP 解法更锻炼思维

### 2. 解题思路（给新手讲）

给你一个只包含 `(` 和 `)` 的字符串，找出其中最长的有效（格式正确且连续）括号子串的长度。

比如 s = "(()"，最长有效括号是 "()"，长度 2。s = ")()())"，最长是 "()()"，长度 4。

DP 思路：考虑以第 i 个字符结尾的最长有效括号子串。

- 如果 `s[i] = '('`：以左括号结尾不可能构成有效括号，`dp[i] = 0`。
- 如果 `s[i] = ')'`：需要找到和它匹配的左括号。

情况 1：`s[i-1] = '('` -> `...()` 的形式
- 这是最简单的情况，`dp[i] = dp[i-2] + 2`

情况 2：`s[i-1] = ')'` -> `...))` 的形式
- 以 i-1 结尾的最长有效括号长度是 `dp[i-1]`
- 那么和 `s[i]` 这个 `)` 匹配的 `(` 应该在位置 `i - dp[i-1] - 1`
- 如果 `s[i - dp[i-1] - 1] = '('`，那么匹配成功：`dp[i] = dp[i-1] + 2 + dp[i - dp[i-1] - 2]`

最后那个 `dp[i - dp[i-1] - 2]` 是什么？是匹配位置再往前的有效括号长度，需要拼接上去。

**状态定义：** `dp[i]` 表示以第 i 个字符结尾的最长有效括号子串的长度。

### 3. 从暴力到优化的推导

#### 暴力解法

枚举所有子串，检查每个子串是否是有效括号。

```python
def longestValidParentheses(s):
    def isValid(sub):
        count = 0
        for c in sub:
            if c == '(':
                count += 1
            else:
                count -= 1
            if count < 0:
                return False
        return count == 0

    max_len = 0
    n = len(s)
    for i in range(n):
        for j in range(i + 2, n + 1, 2):  # 有效括号长度必是偶数
            if isValid(s[i:j]):
                max_len = max(max_len, j - i)
    return max_len
```

时间复杂度：O(n^3)。

#### 优化思路

DP 的关键在于精确定义状态和处理两种情况。也可以用栈来解，用栈更直观但 DP 更考验功力。

#### 最优解法

```python
def longestValidParentheses(s: str) -> int:
    n = len(s)
    if n < 2:
        return 0
    dp = [0] * n
    max_len = 0
    for i in range(1, n):
        if s[i] == ')':
            if s[i - 1] == '(':
                # 情况1：...()
                dp[i] = (dp[i - 2] if i >= 2 else 0) + 2
            elif i - dp[i - 1] - 1 >= 0 and s[i - dp[i - 1] - 1] == '(':
                # 情况2：...))，且找到了匹配的 (
                dp[i] = dp[i - 1] + 2
                if i - dp[i - 1] - 2 >= 0:
                    dp[i] += dp[i - dp[i - 1] - 2]
            max_len = max(max_len, dp[i])
    return max_len
```

时间复杂度：O(n)，空间复杂度：O(n)。

### 4. 关键思维点

- **状态定义**：`dp[i]` 是"以第 i 个字符结尾的最长有效括号长度"。以 `(` 结尾的 dp 值一定是 0。
- **两种情况**要清楚区分：
  - `...()` 直接配对，加上前面的有效长度。
  - `...))` 需要跳过内部已经配对的部分（dp[i-1]），找到外层的 `(`。
- 情况2的难点：`i - dp[i-1] - 1` 这个下标的理解。`dp[i-1]` 是以 i-1 结尾的有效括号长度，跳过这段有效括号后，如果前面还有一个 `(`，就能和当前的 `)` 匹配。
- 拼接前面的有效括号：匹配成功后，还要加上匹配位置再往前的有效括号长度（`dp[i - dp[i-1] - 2]`），因为它们可能也是连续的有效括号。

### 5. 代码实现（Python）

```python
class Solution:
    def longestValidParentheses(self, s: str) -> int:
        n = len(s)
        if n < 2:
            return 0
        # dp[i]: 以 s[i] 结尾的最长有效括号子串的长度
        dp = [0] * n
        max_len = 0
        # 从第 1 个字符开始（第 0 个字符单独不可能构成有效括号）
        for i in range(1, n):
            # 只有 s[i] == ')' 时才可能构成有效括号
            # s[i] == '(' 时，dp[i] = 0，已经初始化好了
            if s[i] == ')':
                # 情况 1: s[i-1] == '('，即 ...() 的形式
                if s[i - 1] == '(':
                    # 新匹配了一对括号，长度 +2
                    # 再加上 s[i-2] 位置结尾的有效长度（如果存在）
                    dp[i] = (dp[i - 2] if i >= 2 else 0) + 2

                # 情况 2: s[i-1] == ')'，即 ...)) 的形式
                # 需要找到和 s[i] 匹配的 '(' 的位置
                # 跳过以 s[i-1] 结尾的有效括号，看前面是否有 '('
                elif i - dp[i - 1] - 1 >= 0 and s[i - dp[i - 1] - 1] == '(':
                    # 匹配成功！
                    # 长度 = 内部有效括号长度 dp[i-1] + 新匹配的 2
                    dp[i] = dp[i - 1] + 2
                    # 再拼上匹配位置前面的有效括号长度
                    if i - dp[i - 1] - 2 >= 0:
                        dp[i] += dp[i - dp[i - 1] - 2]

                # 更新全局最大值
                max_len = max(max_len, dp[i])
        return max_len
```

**附：栈解法（更直观，推荐对比理解）：**

```python
class Solution:
    def longestValidParentheses(self, s: str) -> int:
        # 栈中存储下标，栈底始终保持"最后一个没有被匹配的右括号下标"
        stack = [-1]  # -1 作为初始的"基准点"
        max_len = 0
        for i in range(len(s)):
            if s[i] == '(':
                # 左括号：下标入栈，等待匹配
                stack.append(i)
            else:
                # 右括号：弹出栈顶
                stack.pop()
                if not stack:
                    # 栈空了，说明当前右括号无法匹配
                    # 它成为新的基准点
                    stack.append(i)
                else:
                    # 栈不空，当前有效括号长度 = 当前下标 - 栈顶下标
                    max_len = max(max_len, i - stack[-1])
        return max_len
```

## 动态规划总结与套路归纳

### 一、DP 的四大步骤

1. **定义状态**：`dp[i]` 或 `dp[i][j]` 代表什么？（最难的一步）
2. **状态转移方程**：`dp[i]` 和之前的状态有什么关系？
3. **初始条件**：最小子问题的答案是什么？
4. **计算方向**：从小到大递推，还是从大到小？

### 二、常见 DP 模型速查

| 模型 | 典型题目 | 状态定义 | 特点 |
|------|----------|----------|------|
| 线性 DP | 爬楼梯、打家劫舍 | `dp[i]` = 前 i 个元素的最优解 | 从前往后递推 |
| 子序列 DP | 最长递增子序列 | `dp[i]` = 以第 i 个元素**结尾**的最优解 | 答案取 max(dp) |
| 子数组 DP | 乘积最大子数组 | 以当前元素结尾的最值 | 连续约束 |
| 背包 DP | 零钱兑换、分割等和子集 | `dp[j]` = 容量为 j 的背包的最优解 | 0-1/完全背包 |
| 字符串 DP | 单词拆分、最长有效括号 | `dp[i]` = 前 i 个字符的某种属性 | 枚举切割点 |

### 三、空间优化技巧

- **滚动变量**：当 dp[i] 只依赖 dp[i-1] 和 dp[i-2] 时，用两个变量代替数组。（爬楼梯、打家劫舍）
- **一维数组压缩**：二维 DP 只依赖上一行时，可以压缩为一维。（分割等和子集）
- **逆序遍历**：0-1 背包的一维优化中，必须逆序遍历保证每个物品只用一次。

### 四、常见思维模式

1. **"最后一步"法**：到达当前状态的最后一步是什么？（爬楼梯）
2. **"选或不选"法**：每个元素选还是不选？（打家劫舍、分割等和子集）
3. **"在哪里切一刀"法**：在字符串/数组的哪个位置划分？（单词拆分）
4. **"接在谁后面"法**：当前元素可以接在哪些元素后面？（最长递增子序列）

### 五、面试高频考点

按面试出现频率排序：
1. 爬楼梯（入门，必会）
2. 打家劫舍（选或不选模型）
3. 零钱兑换（完全背包）
4. 最长递增子序列（子序列 DP + 二分优化）
5. 单词拆分（字符串 DP）
6. 分割等和子集（0-1 背包）
7. 最长有效括号（高难度，考验功力）
