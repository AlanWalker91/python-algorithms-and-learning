# 第4章：递归

> **本章导读**：递归是程序员思维的一次重要升级。初学者往往觉得递归"绕"，是因为习惯了一步步告诉计算机"怎么做"（命令式思维），而递归要求你描述"问题是什么"（声明式思维）——把大问题表达成更小的同类问题。
>
> **学习路径**：建立直觉（类比） → 掌握三要素（写出能跑的递归）→ 理解调用栈（看懂执行过程）→ 识别重叠子问题（引入记忆化）→ 动态规划（终极优化）
>
> **核心问题**：递归为什么有用？——因为很多问题（树、图、分治）本身就是递归定义的，用递归描述它们比用循环更自然、更清晰。

---

## 4.0 关键术语速查

| 术语 | 英文 | 定义 |
|------|------|------|
| 递归 | Recursion | 函数在执行过程中直接或间接调用自身 |
| 基本情况 | Base Case | 可以直接返回答案、无需继续递归的最小情况，是递归的终止条件 |
| 递归情况 | Recursive Case | 将当前问题转化为规模更小的同类子问题 |
| 调用栈 | Call Stack | 内存中存储函数调用信息的栈结构，每次函数调用压入一帧，返回时弹出 |
| 栈帧 | Stack Frame | 调用栈中每次函数调用占据的一段内存，保存局部变量、参数和返回地址 |
| 栈溢出 | Stack Overflow | 递归层次过深，调用栈超出内存限制，程序崩溃 |
| 回溯 | Backtracking | 递归中当某条路径失败时，撤销当前选择、返回上一步尝试其他路径 |
| 记忆化 | Memoization | 用字典/数组缓存已计算的结果，避免重复计算，是自顶向下的动态规划 |
| 动态规划 | Dynamic Programming | 将大问题分解为子问题，先解决小的再推导大的（自底向上填表） |
| 重叠子问题 | Overlapping Subproblems | 同一个子问题在递归中被重复计算多次，是引入记忆化的信号 |
| 最优子结构 | Optimal Substructure | 原问题的最优解由子问题的最优解构成，是动态规划适用的前提 |
| 状态转移方程 | State Transition | 描述 dp[i] 如何从更小的 dp[j] 推导而来的数学关系式 |
| 尾递归 | Tail Recursion | 递归调用是函数体的最后一个操作，部分语言可优化为 O(1) 空间 |
| 分治 | Divide and Conquer | 将问题分为若干子问题分别解决后合并结果，归并排序是典型 |

## 4.1 为什么需要递归？从一个例子说起

### 先看迭代怎么做

```python
# 计算目录的总文件大小（包含所有子目录）

# 如果只有一层目录，迭代很简单：
def get_size_flat(files):
    total = 0
    for size in files:
        total += size
    return total

# 但现实中目录是嵌套的：
directory = {
    'report.pdf': 200,
    'images': {          # 子目录
        'photo.jpg': 500,
        'icons': {       # 子子目录
            'logo.png': 50
        }
    },
    'notes.txt': 10
}
```

用迭代处理任意深度的嵌套目录，需要手动维护一个栈模拟遍历，代码很复杂。

**而用递归，只需三行核心逻辑**：

```python
def get_total_size(item):
    """
    递归计算目录/文件的总大小
    
    关键洞察：
    - 文件：大小就是它本身（基本情况，直接返回）
    - 目录：大小 = 所有子项大小之和（递归情况）
    
    这就是递归的威力：用相同的逻辑处理任意层级的嵌套
    """
    if isinstance(item, int):        # 基本情况：是文件（数字），直接返回大小
        return item
    
    # 递归情况：是目录（字典），对每个子项递归求和
    return sum(get_total_size(sub) for sub in item.values())

print(get_total_size(directory))    # 760（200+500+50+10）
```

> **递归的本质价值**：当问题结构本身是嵌套/层级的（目录、树、图），递归让代码的结构和问题的结构一一对应，代码即逻辑。

## 4.2 递归三要素（写出正确递归的必要条件）

任何正确的递归函数都必须满足三个要素，缺一不可：

```
要素1：基本情况（Base Case）
  → 能直接返回答案，不再调用自身
  → 没有基本情况 = 无限递归 = 程序崩溃

要素2：递归情况（Recursive Case）
  → 把当前问题转化为规模更小的同类子问题
  → 调用自身时参数必须"更小"或"更简单"

要素3：向基本情况收敛
  → 每次递归调用，问题规模必须在缩小
  → 保证最终一定能触达基本情况，终止递归
```

### 从阶乘彻底理解三要素

```python
def factorial(n):
    """
    计算 n 的阶乘（n! = n × (n-1) × ... × 1）
    
    数学定义（递归式）：
        0! = 1            ← 基本情况
        n! = n × (n-1)!  ← 递归情况（n>0）
    
    这个数学定义本身就是递归的，代码只是把它翻译成Python。
    """
    # ── 要素1：基本情况 ──────────────────────────────────────
    if n == 0 or n == 1:    # 0! = 1，1! = 1，直接返回
        return 1
    
    # ── 要素2 & 3：递归情况，n 每次减 1，向基本情况收敛 ────────
    return n * factorial(n - 1)   # 问题从 n 缩小到 n-1


# ═══ 执行流程追踪（n=4）═══
#
# factorial(4) 被调用           → 压栈帧 f(4)
#   需要 factorial(3)           → 压栈帧 f(3)
#     需要 factorial(2)         → 压栈帧 f(2)
#       需要 factorial(1)       → 压栈帧 f(1)
#         n==1，返回 1          ← 触底！开始回溯
#       f(2) 收到 1，返回 2×1=2 ← 弹栈帧 f(2)
#     f(3) 收到 2，返回 3×2=6   ← 弹栈帧 f(3)
#   f(4) 收到 6，返回 4×6=24   ← 弹栈帧 f(4)
#
# 最终结果：24
```

### 常见错误：忘记基本情况

```python
# ❌ 错误示范：没有基本情况，无限递归
def bad_factorial(n):
    return n * bad_factorial(n - 1)   # 永远不会停止
# RecursionError: maximum recursion depth exceeded

# ❌ 错误示范：基本情况永远无法触达
def bad_factorial_2(n):
    if n == 0:
        return 1
    return n * bad_factorial_2(n + 1)   # n 在增大，永不到达 0！
```

## 4.3 调用栈：递归的执行引擎

### 什么是调用栈？

每次函数调用，操作系统都会在内存的"调用栈"区域压入一个**栈帧（Stack Frame）**，
记录这次调用的：
- 函数参数
- 局部变量
- 返回地址（调用结束后回到哪里）

函数返回时，对应的栈帧被弹出。

```
factorial(3) 的调用栈变化过程：

阶段1：一路向下（压栈）
┌─────────────────┐  ← 栈顶
│ factorial(1)    │    参数：n=1
│ 等待执行中...   │    局部变量：无
├─────────────────┤
│ factorial(2)    │    参数：n=2
│ 等待 f(1) 返回  │    已知：result = 2 × ?
├─────────────────┤
│ factorial(3)    │    参数：n=3
│ 等待 f(2) 返回  │    已知：result = 3 × ?
└─────────────────┘  ← 栈底（最早调用的函数）

阶段2：触底后回溯（弹栈）
factorial(1) 返回 1  → 弹出 f(1) 的栈帧
factorial(2) 得到1，计算 2×1=2，返回 2 → 弹出 f(2)
factorial(3) 得到2，计算 3×2=6，返回 6 → 弹出 f(3)
```

### Python 的递归深度限制

```python
import sys

# 查看当前递归深度限制（默认约1000）
print(sys.getrecursionlimit())   # 1000

# 如果递归层次超出限制
def infinite(n):
    return infinite(n + 1)
# infinite(0)  → RecursionError: maximum recursion depth exceeded

# 可以提高限制（但这是治标不治本，超深递归应该改用迭代）
sys.setrecursionlimit(10000)
```

### 递归 vs 迭代：本质对比

```python
# 同一个问题的两种思路

# ── 迭代版（命令式：告诉计算机"每步怎么做"）──
def factorial_iter(n):
    """
    用循环显式控制每一步
    空间 O(1)：只有一个变量 result，不受 n 影响
    """
    result = 1
    for i in range(2, n + 1):
        result *= i
    return result

# ── 递归版（声明式：描述"问题是什么"）──
def factorial_rec(n):
    """
    用函数定义本身表达问题的结构
    空间 O(n)：每层递归占一个栈帧，共 n 层
    """
    if n <= 1:
        return 1
    return n * factorial_rec(n - 1)


# ── 选择建议 ──────────────────────────────────
# 用递归的场景：
#   ✅ 问题天然是递归结构（树、图、分形）
#   ✅ 代码简洁性比性能更重要
#   ✅ 递归深度有限（< 1000层）
#
# 用迭代的场景：
#   ✅ 追求空间效率（O(1) vs O(n)）
#   ✅ 递归深度很深（大数据量的链表/图）
#   ✅ 语言不支持尾调用优化（Python就是）
```

## 4.4 ⭐ 经典案例1：进制转换

### 思路拆解

```
问题：把十进制整数 n 转换为 base 进制字符串
递归洞察：n 的 base 进制表示 = 高位部分 + 最低位
          高位部分 = (n // base) 的 base 进制表示（更小的同类问题！）
          最低位   = digits[n % base]

举例：convert(13, 2) = ?
13 % 2 = 1  ← 这是二进制的最低位（个位）
13 // 2 = 6 ← 去掉最低位后，递归处理 6

6 % 2 = 0   ← 第2位
6 // 2 = 3  ← 递归处理 3

3 % 2 = 1   ← 第3位
3 // 2 = 1  ← 递归处理 1

1 < 2       ← 触底！基本情况，直接返回 '1'

回溯拼接：'1' + '1' + '0' + '1' = '1101' ✅
（注意：递归先处理高位，后处理低位，回溯时高位在前）
```

```python
def convert_base(n, base):
    """
    十进制转 base 进制
    
    时间复杂度：O(log_base(n))，每次缩小 base 倍
    空间复杂度：O(log_base(n))，调用栈深度
    """
    digits = "0123456789ABCDEF"   # 最高支持16进制
    
    # 基本情况：n 只有一位，直接返回对应字符
    if n < base:
        return digits[n]
    
    # 递归情况：
    # ① 先递归处理"去掉最低位"的更大部分（高位）
    # ② 再拼接当前最低位
    # 字符串拼接的顺序自然保证了高位在前
    return convert_base(n // base, base) + digits[n % base]

# 验证
print(convert_base(13, 2))    # '1101'   = 8+4+0+1 ✅
print(convert_base(255, 16))  # 'FF'     = 15×16+15 ✅
print(convert_base(42, 8))    # '52'     = 5×8+2=42 ✅
```

## 4.5 ⭐ 经典案例2：汉诺塔

### 为什么汉诺塔是递归的完美例子？

汉诺塔问题之所以著名，是因为：
- **用迭代解决极其复杂**（需要判断奇偶、追踪状态）
- **用递归只需5行核心代码**，而且逻辑完全清晰

这展示了递归在表达能力上的压倒性优势。

### 递归分解的关键洞察

```
问题：把 n 个盘从柱子 A 移到柱子 C（B 是辅助柱）
      规则：大盘不能放在小盘上，每次只能移动一个盘

递归洞察：
  n 个盘的问题 可以分解为：
    ① 把上面 n-1 个盘从 A 移到 B（把C当辅助）→ 同类子问题，规模 n-1
    ② 把最大的盘从 A 移到 C                  → 直接操作，O(1)
    ③ 把 n-1 个盘从 B 移到 C（把A当辅助）→ 同类子问题，规模 n-1

n=1 时直接移动  ← 基本情况

关键：你不需要知道"n-1个盘怎么移"，只需相信递归能搞定它！
      这种"信任递归"的思维方式，是写递归的核心心法。
```

```python
def hanoi(n, source, target, auxiliary):
    """
    汉诺塔递归解法
    
    参数：
        n         ：要移动的盘数
        source    ：起始柱（盘子当前在这里）
        target    ：目标柱（盘子要到这里）
        auxiliary ：辅助柱（可以暂时放盘子）
    
    移动次数：2ⁿ - 1 次
        n=1：1次
        n=2：3次
        n=3：7次
        n=10：1023次
        n=64：18万亿亿次（即使每秒移10亿次，也需要585年）
    """
    # 基本情况：只有1个盘，直接从 source 移到 target
    if n == 1:
        print(f"  移动盘 1：{source} ──▶ {target}")
        return
    
    # 步骤①：把上面 n-1 个盘 从 source 借助 target 移到 auxiliary
    hanoi(n - 1, source, auxiliary, target)
    
    # 步骤②：把最大的盘（第 n 个）从 source 直接移到 target
    print(f"  移动盘 {n}：{source} ──▶ {target}")
    
    # 步骤③：把 n-1 个盘 从 auxiliary 借助 source 移到 target
    hanoi(n - 1, auxiliary, target, source)


# ═══ n=3 的完整执行过程 ═══
print("=== 汉诺塔 n=3 ===")
hanoi(3, 'A', 'C', 'B')
# 移动盘 1：A ──▶ C
# 移动盘 2：A ──▶ B
# 移动盘 1：C ──▶ B
# 移动盘 3：A ──▶ C    ← 最大盘就位
# 移动盘 1：B ──▶ A
# 移动盘 2：B ──▶ C
# 移动盘 1：A ──▶ C
# 共 7 = 2³-1 次

# ═══ 复杂度分析 ═══
# 设 T(n) = n个盘需要的移动次数
# T(1) = 1
# T(n) = 2 × T(n-1) + 1  （两次规模n-1的递归 + 1次直接移动）
# 求解：T(n) = 2ⁿ - 1  → O(2ⁿ)
# 这是指数级复杂度，不可避免（每个盘都必须被移动）
```

## 4.6 ⭐ 回溯：递归的"撤销"能力

### 什么是回溯？

回溯（Backtracking）= 递归 + 撤销选择。

当我们在递归中做了某个"选择"，若发现这条路走不通，就撤销该选择，返回上一步，尝试其他可能。

```
思路框架：
def backtrack(状态, 选择列表):
    if 满足结束条件:
        记录结果
        return
    
    for 选择 in 选择列表:
        做选择（修改状态）
        backtrack(新状态, 剩余选择)
        撤销选择（恢复状态）    ← 这就是"回溯"
```

### 迷宫探索——回溯的直观例子

```python
def solve_maze(maze, row, col, path):
    """
    迷宫递归 + 回溯求解
    
    maze：二维列表
        ' '=通路，'#'=墙壁，'E'=出口，'.'=已访问
    row, col：当前位置
    path：当前走过的路径（坐标列表）
    
    回溯过程：
    走到此路不通 → 恢复格子标记 → 返回上一层 → 尝试其他方向
    """
    rows, cols = len(maze), len(maze[0])
    
    # ── 基本情况1：越界 ──────────────────────────────────────
    if not (0 <= row < rows and 0 <= col < cols):
        return False
    
    cell = maze[row][col]
    
    # ── 基本情况2：碰墙或已走过 ──────────────────────────────
    if cell == '#' or cell == '.':
        return False
    
    # ── 基本情况3：找到出口！────────────────────────────────
    if cell == 'E':
        path.append((row, col))
        return True
    
    # ── 做选择：标记当前格为已访问，加入路径 ──────────────────
    maze[row][col] = '.'
    path.append((row, col))
    
    # ── 递归：向四个方向尝试 ──────────────────────────────────
    for dr, dc in [(-1,0),(1,0),(0,-1),(0,1)]:   # 上下左右
        if solve_maze(maze, row + dr, col + dc, path):
            return True                            # 某方向找到出口，成功返回
    
    # ── 撤销选择（回溯）：四个方向都不通，恢复状态 ────────────
    maze[row][col] = ' '    # 恢复格子
    path.pop()              # 从路径中移除
    return False            # 告知上层此路不通


# 测试
maze = [
    ['#','#','#','#','#'],
    ['#',' ',' ','#','#'],
    ['#','#',' ','#','#'],
    ['#','#',' ',' ','E'],
    ['#','#','#','#','#'],
]
path = []
if solve_maze(maze, 1, 1, path):
    print("找到路径：", path)
    # [(1,1), (1,2), (2,2), (3,2), (3,3), (3,4)]
else:
    print("无解")
```

## 4.7 ⭐⭐ 动态规划：从递归到 DP 的完整演进

动态规划（DP）是"有记忆的递归"。学习它最好的方式，是亲眼看见同一个问题从暴力递归一步步优化到 DP 的过程。

### 以斐波那契数列为例

**斐波那契定义**：F(0)=0，F(1)=1，F(n) = F(n-1) + F(n-2)

---

#### 阶段1：朴素递归（正确但极慢）

```python
def fib_naive(n):
    """
    朴素递归：直接翻译数学定义
    
    时间复杂度：O(2ⁿ)  → 指数级！
    空间复杂度：O(n)    → 调用栈深度
    
    n=40 时，大约需要计算 10 亿次，要等几秒
    n=50 时，几乎无法在合理时间内完成
    """
    if n <= 1:           # 基本情况
        return n
    return fib_naive(n-1) + fib_naive(n-2)   # 递归情况
```

**问题在哪里？——重叠子问题**

```
fib(5) 的调用树（只展示部分）：

                  fib(5)
               ╱         ╲
           fib(4)          fib(3)    ← fib(3)被计算了2次
          ╱     ╲         ╱    ╲
      fib(3)  fib(2)  fib(2)  fib(1) ← fib(2)被计算了3次
      ╱  ╲    ╱  ╲
  fib(2) fib(1) fib(1) fib(0)        ← fib(2)又出现了

fib(3) 被重复计算：2次
fib(2) 被重复计算：3次
fib(1) 被重复计算：5次
n越大，重复越多，呈指数级爆炸
```

---

#### 阶段2：记忆化递归（自顶向下 DP）

```python
def fib_memo(n, memo=None):
    """
    记忆化递归（自顶向下动态规划）
    
    核心优化：计算结果存入 memo，下次直接查表
    相当于：把指数级的重复计算，变成每个子问题只算一次
    
    时间复杂度：O(n)   → 每个子问题只算1次，共 n+1 个子问题
    空间复杂度：O(n)   → memo 字典 + 调用栈
    """
    if memo is None:
        memo = {}          # 用字典做缓存（备忘录）
    
    # 先查缓存：如果算过了，直接返回，O(1)
    if n in memo:
        return memo[n]
    
    # 基本情况
    if n <= 1:
        return n
    
    # 递归计算，并存入缓存
    result = fib_memo(n-1, memo) + fib_memo(n-2, memo)
    memo[n] = result       # ← 关键：存入缓存，避免重复计算
    return result

# Python 内置装饰器方案（更简洁）
from functools import lru_cache

@lru_cache(maxsize=None)   # maxsize=None 表示无限缓存
def fib_lru(n):
    if n <= 1:
        return n
    return fib_lru(n-1) + fib_lru(n-2)
# lru_cache 自动做记忆化，效果和上面完全相同
```

#### 阶段3：自底向上 DP（最优）

```python
def fib_dp(n):
    """
    自底向上动态规划（填表法）
    
    核心思想：
    不从 fib(n) 往下递归，而是从 fib(0), fib(1) 开始往上推
    建一张表，每个格子只填一次，从小到大填满
    
    时间复杂度：O(n)
    空间复杂度：O(n)  → 存储整个 dp 数组
    
    执行流程（n=6）：
    dp = [0, 1, ?, ?, ?, ?, ?]
    dp[2] = dp[1]+dp[0] = 1+0 = 1
    dp[3] = dp[2]+dp[1] = 1+1 = 2
    dp[4] = dp[3]+dp[2] = 2+1 = 3
    dp[5] = dp[4]+dp[3] = 3+2 = 5
    dp[6] = dp[5]+dp[4] = 5+3 = 8
    返回 dp[6] = 8
    """
    if n <= 1:
        return n
    
    dp = [0] * (n + 1)    # 状态数组，dp[i] 表示 fib(i) 的值
    dp[0] = 0             # 初始化基本情况
    dp[1] = 1
    
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]   # 状态转移方程
    
    return dp[n]

def fib_optimized(n):
    """
    空间优化版：O(1) 空间
    
    观察：计算 dp[i] 只需要 dp[i-1] 和 dp[i-2]，
    不需要保存整个数组，用两个变量滚动即可
    """
    if n <= 1:
        return n
    prev2, prev1 = 0, 1       # prev2=fib(i-2)，prev1=fib(i-1)
    for _ in range(2, n + 1):
        curr = prev1 + prev2   # 计算 fib(i)
        prev2 = prev1          # 滚动：prev2 前进
        prev1 = curr           # 滚动：prev1 前进
    return prev1
```

| 记忆化递归 | O(n) | O(n) | 微秒级 | 代码可读性好 |

### 动态规划两大核心特征

```
特征1：重叠子问题（Overlapping Subproblems）
  现象：同一个子问题被重复计算多次
  信号：你在朴素递归里发现同一参数被多次调用
  解决：记忆化（用空间换时间）

特征2：最优子结构（Optimal Substructure）
  含义：原问题的最优解包含子问题的最优解
  意义：可以从子问题的最优解推出原问题的最优解
  检验：子问题解组合后能得到原问题解吗？

两个特征都具备 → 适合用动态规划
```

### 面试题1：爬楼梯（LeetCode 70，Easy）

**题目**：爬 n 级楼梯，每次可以爬 1 或 2 级，共有多少种不同的方法？

**分析过程**（面试时要说出这个思路）：

```
到达第 n 级，最后一步只有两种可能：
  - 从第 n-1 级爬1步上来
  - 从第 n-2 级爬2步上来

所以：ways(n) = ways(n-1) + ways(n-2)

这就是斐波那契！

基本情况：
  ways(1) = 1  （只有一种：1步）
  ways(2) = 2  （两种：1+1 或 2）
```

```python
def climb_stairs(n):
    """
    爬楼梯：DP 解法
    
    状态定义：dp[i] = 到达第 i 级的方法数
    状态转移：dp[i] = dp[i-1] + dp[i-2]
    
    时间：O(n)，空间：O(1)
    """
    if n <= 2:
        return n
    
    prev2, prev1 = 1, 2   # dp[1]=1, dp[2]=2
    for i in range(3, n + 1):
        curr = prev1 + prev2
        prev2, prev1 = prev1, curr
    return prev1

print(climb_stairs(5))   # 8（1+1+1+1+1, 1+1+1+2, 1+1+2+1, ...）

# 验证：
# n=1: 1种 [1]
# n=2: 2种 [1,1] [2]
# n=3: 3种 [1,1,1] [1,2] [2,1]
# n=4: 5种
# n=5: 8种
# 恰好是斐波那契数列！
```

### 面试题2：零钱兑换（LeetCode 322，Medium）

**题目**：给定硬币面值 `coins` 和目标金额 `amount`，求凑出 `amount` 所需的最少硬币数，若无法凑出返回 -1。

**暴力递归 → 记忆化 → DP 完整演进**：

```python
# ── 暴力递归（理解思路）──────────────────────────────────────
def coin_change_naive(coins, amount):
    """
    暴力递归：对每种硬币，尝试"用它"或"不用它"
    时间：O(amount^len(coins))，超时！
    """
    if amount == 0:
        return 0                          # 基本情况：刚好凑满
    if amount < 0:
        return float('inf')              # 基本情况：超出，无效
    
    min_count = float('inf')
    for coin in coins:
        # 尝试用一枚面值为 coin 的硬币，剩余 amount-coin 递归处理
        sub = coin_change_naive(coins, amount - coin)
        if sub != float('inf'):
            min_count = min(min_count, sub + 1)  # +1 表示用了这枚硬币
    
    return min_count


# ── 自底向上 DP（最优解）────────────────────────────────────
def coin_change_dp(coins, amount):
    """
    自底向上动态规划
    
    状态定义：dp[i] = 凑出金额 i 所需的最少硬币数
    状态转移：dp[i] = min(dp[i - coin] + 1) for coin in coins if coin <= i
    初始值  ：dp[0] = 0（凑出0不需要硬币），其余 = 无穷大（初始不可达）
    
    执行流程（coins=[1,2,5], amount=6）：
    
    dp = [0, inf, inf, inf, inf, inf, inf]  初始
    
    i=1: 试 coin=1: dp[1-1]+1 = dp[0]+1 = 1  → dp[1]=1
    i=2: 试 coin=1: dp[1]+1=2；试 coin=2: dp[0]+1=1  → dp[2]=1（用一枚2分）
    i=3: 试 coin=1: dp[2]+1=2；试 coin=2: dp[1]+1=2  → dp[3]=2（1+2 或 1+2）
    i=4: 试 coin=2: dp[2]+1=2  → dp[4]=2（2+2）
    i=5: 试 coin=5: dp[0]+1=1  → dp[5]=1（直接用一枚5分）
    i=6: 试 coin=1: dp[5]+1=2；试 coin=2: dp[4]+1=3；试 coin=5: dp[1]+1=2 → dp[6]=2
    
    返回 dp[6] = 2  → 5+1 或 2+2+2? 不对，是 5+1=2枚 ✅
    
    时间：O(amount × len(coins))
    空间：O(amount)
    """
    INF = float('inf')
    dp = [INF] * (amount + 1)
    dp[0] = 0                      # 凑出0分：0枚硬币
    
    for i in range(1, amount + 1): # 从小金额到大金额，逐一填表
        for coin in coins:
            if coin <= i and dp[i - coin] != INF:
                # 尝试用一枚 coin：从 dp[i-coin] 多用一枚
                dp[i] = min(dp[i], dp[i - coin] + 1)
    
    return dp[amount] if dp[amount] != INF else -1


# 测试
print(coin_change_dp([1, 2, 5], 11))   # 3  → 5+5+1
print(coin_change_dp([2], 3))           # -1 → 无法凑出
print(coin_change_dp([1, 5, 10, 25], 36))  # 3 → 25+10+1
```

> **⚠️ 面试注意点**：
> 1. 贪心不一定正确！coins=[1,3,4], amount=6 时贪心选 4+1+1=3枚，而 3+3=2枚更少
> 2. 初始化 `dp[0]=0`，其余设为无穷大（表示"尚未可达"）
> 3. 状态转移时要判断 `dp[i-coin] != INF`，否则 INF+1 仍是 INF（但Python中 `inf+1==inf`，实际不会溢出，但逻辑上要注意）

### 面试题3：最大子数组和（LeetCode 53，Medium）

**题目**：给定整数数组，找出和最大的连续子数组，返回其最大和。

```python
def max_subarray(nums):
    """
    Kadane 算法（动态规划经典）
    
    暴力思路：枚举所有子数组，计算每个子数组的和，O(n²) 或 O(n³)
    
    DP 优化思路：
    状态定义：dp[i] = 以 nums[i] 结尾的最大子数组和
    状态转移：
        dp[i] = nums[i] + max(dp[i-1], 0)
        含义：要么把 nums[i] 加到前面的子数组（若前面的和为正）
              要么以 nums[i] 重新开始（若前面的和为负或零）
    
    举例：nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]
    
    i=0: dp[0]=-2  (只能是-2自己)
    i=1: dp[1]=1   (-2<0，重新开始，取1)
    i=2: dp[2]=-2  (1+(-3)=-2，比重新开始-3好)
    i=3: dp[3]=4   (-2<0，重新开始，取4)
    i=4: dp[4]=3   (4+(-1)=3)
    i=5: dp[5]=5   (3+2=5)
    i=6: dp[6]=6   (5+1=6)  ← 最大值在这里
    i=7: dp[7]=1   (6+(-5)=1)
    i=8: dp[8]=5   (1+4=5)
    
    答案：max(dp) = 6，对应子数组 [4,-1,2,1]
    
    时间：O(n)，空间：O(1)（只需一个变量记录当前状态）
    """
    # 空间优化：用 current_sum 代替整个 dp 数组
    current_sum = nums[0]    # 以 nums[0] 结尾的最大子数组和
    max_sum = nums[0]        # 全局最大值
    
    for i in range(1, len(nums)):
        # 状态转移：nums[i] 加入还是另起炉灶？
        current_sum = nums[i] + max(current_sum, 0)
        max_sum = max(max_sum, current_sum)
    
    return max_sum

print(max_subarray([-2,1,-3,4,-1,2,1,-5,4]))   # 6
print(max_subarray([-1, -2, -3]))               # -1（全负时取最大的负数）
```

### 面试题4：杨辉三角（LeetCode 118，Easy）

**题目**：生成杨辉三角的前 n 行。

```python
def generate_pascals_triangle(num_rows):
    """
    杨辉三角：经典 DP 填表
    
    规律（递推关系）：
    第 i 行第 j 列 = 第 i-1 行第 j-1 列 + 第 i-1 行第 j 列
    两端始终为 1
    
    直观理解：
    第1行：[1]
    第2行：[1, 1]
    第3行：[1, 2, 1]        2 = 1+1
    第4行：[1, 3, 3, 1]     3 = 1+2, 3 = 2+1
    第5行：[1, 4, 6, 4, 1]  4=1+3, 6=3+3, 4=3+1
    
    时间：O(n²)，空间：O(n²)
    """
    triangle = []
    
    for i in range(num_rows):
        # 每行初始化为全1（两端固定为1）
        row = [1] * (i + 1)
        
        # 填中间的值（j 从1到 i-1）
        for j in range(1, i):
            # 当前值 = 上一行左边 + 上一行右边
            row[j] = triangle[i-1][j-1] + triangle[i-1][j]
        
        triangle.append(row)
    
    return triangle

result = generate_pascals_triangle(5)
for row in result:
    print(row)
# [1]
# [1, 1]
# [1, 2, 1]
# [1, 3, 3, 1]
# [1, 4, 6, 4, 1]
```

### 面试题5：不同路径（LeetCode 62，Medium）

**题目**：m×n 网格，机器人从左上角走到右下角，只能向右或向下，共有多少种路径？

```python
def unique_paths(m, n):
    """
    网格路径计数：二维 DP
    
    状态定义：dp[i][j] = 到达 (i,j) 的路径数
    状态转移：dp[i][j] = dp[i-1][j] + dp[i][j-1]
              含义：只能从上方或左方到达 (i,j)
    初始值  ：第一行和第一列全为 1（只有一种走法）
    
    执行流程（m=3, n=3）：
    
    初始化第一行和第一列为1：
    dp = [[1, 1, 1],
          [1, ?, ?],
          [1, ?, ?]]
    
    填表：
    dp[1][1] = dp[0][1]+dp[1][0] = 1+1 = 2
    dp[1][2] = dp[0][2]+dp[1][1] = 1+2 = 3
    dp[2][1] = dp[1][1]+dp[2][0] = 2+1 = 3
    dp[2][2] = dp[1][2]+dp[2][1] = 3+3 = 6
    
    结果：dp[2][2] = 6 ✅
    
    时间：O(m×n)，空间：O(m×n)，可优化到 O(n)
    """
    # 创建 m×n 的 DP 表，全初始化为 0
    dp = [[0] * n for _ in range(m)]
    
    # 初始化第一行（只能向右走，只有1种路径）
    for j in range(n):
        dp[0][j] = 1
    
    # 初始化第一列（只能向下走，只有1种路径）
    for i in range(m):
        dp[i][0] = 1
    
    # 填表（从 (1,1) 开始，按行填）
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = dp[i-1][j] + dp[i][j-1]   # 从上 + 从左
    
    return dp[m-1][n-1]


# 空间优化版：O(n) 空间（只保留一行）
def unique_paths_optimized(m, n):
    """
    用滚动数组优化空间
    每次更新一行，上一行的值在更新时直接覆盖
    """
    dp = [1] * n            # 初始为全1（相当于第一行）
    for i in range(1, m):   # 从第二行开始
        for j in range(1, n):
            dp[j] += dp[j-1]   # dp[j]（来自上方）+ dp[j-1]（来自左方）
    return dp[n-1]

print(unique_paths(3, 3))             # 6
print(unique_paths_optimized(3, 7))   # 28
```

## 4.9 本章要点总结

### 核心概念映射表

| 概念 | 定义 | 关键特征 | 类比 |
|------|------|----------|------|
| **递归** | 函数调用自身 | 必须有终止条件 | 俄罗斯套娃 |
| **基本情况** | 直接返回答案，不再递归 | 递归的"出口" | 套娃最小的那个 |
| **调用栈** | 存储每次调用信息的系统栈 | 深度 = 递归层数 | 饭店叫号排队 |
| **回溯** | 递归 + 撤销选择 | 走不通就退回来 | 迷宫探索 |
| **记忆化** | 缓存已算过的结果 | 空间换时间 | 备忘录 |
| **动态规划** | 从小到大填表推导 | 避免重复，保证最优 | 拼图（先拼小块）|

### 三要素速查

| 要素 | 含义 | 缺失后果 |
|------|------|---------|
| 基本情况（Base Case） | 直接返回，不再递归 | 无限递归，栈溢出 |
| 递归情况（Recursive Case）| 转化为更小的子问题 | 函数无法推进 |
| 向基本情况收敛 | 问题规模每次缩小 | 永远触达不了基本情况 |

### 动态规划解题步骤

```
第1步：确认是否有重叠子问题（画递归树，看有无重复分支）
第2步：定义状态（dp[i] 或 dp[i][j] 的含义）
第3步：写出状态转移方程（dp[i] 如何从更小的状态推导）
第4步：确定初始值（基本情况的值）
第5步：确定计算顺序（保证计算 dp[i] 时，依赖的值都已算好）
第6步：（可选）空间优化（滚动数组等）
```

### 面试高频考点

| 题目 | 核心思路 | 时间 | 空间 | 难度 |
|------|---------|------|------|------|
| 爬楼梯 LC70 | dp[i]=dp[i-1]+dp[i-2] | O(n) | O(1) | Easy |
| 零钱兑换 LC322 | 完全背包变体，dp[i]=min(dp[i-coin]+1) | O(n×k) | O(n) | Medium |
| 最大子数组和 LC53 | Kadane：current=nums[i]+max(current,0) | O(n) | O(1) | Medium |
| 杨辉三角 LC118 | 经典填表，row[j]=上行[j-1]+上行[j] | O(n²) | O(n²) | Easy |
| 不同路径 LC62 | dp[i][j]=dp[i-1][j]+dp[i][j-1] | O(m×n) | O(n) | Medium |

### 递归 vs 迭代 vs DP 选择指南

```
遇到新问题，按以下顺序考虑：

1. 问题能分解为同类子问题？
   是 → 考虑递归
   否 → 用迭代/贪心/其他

2. 递归中存在重叠子问题？
   是 → 加记忆化（Memoization）
   否 → 朴素递归即可

3. 子问题之间有依赖顺序？
   是 → 改用自底向上 DP（填表法）
   
4. DP 状态只依赖相邻状态？
   是 → 滚动数组优化空间到 O(1)
```

### 常见陷阱（坑点）

| 陷阱 | 说明 | 解决方法 |
|------|------|---------|
| 忘写基本情况 | 无限递归，直接崩溃 | 写递归第一步先想基本情况 |
| 基本情况不全 | 某些输入触达不了基本情况 | 列举所有边界，逐一验证 |
| Python 递归深度限制 | 默认约1000层 | 大数据用迭代/DP，或 sys.setrecursionlimit() |
| 记忆化用可变默认参数 | `def f(memo={})` 跨调用共享 | 用 `memo=None` 然后内部初始化 |
| DP 初始值设错 | 整个表都错 | 仔细验证基本情况（dp[0]、第一行/列）|
| 贪心替代DP | 局部最优 ≠ 全局最优 | 先想能否举出贪心失败的反例 |

---

> **给初学者的学习建议**：
> 1. **写递归前先说"假设"**：假设 `f(n-1)` 已经能正确返回，那 `f(n)` 怎么利用它得到答案？先建立这个信心，再写代码。
> 2. **画递归树**：遇到不理解的递归，把调用树画出来，重叠的子问题一目了然，也能清楚看到指数级爆炸的原因。
> 3. **DP 三步走**：先写暴力递归（确保正确）→ 加 `@lru_cache` 记忆化（确保效率）→ 改写为自底向上（确保空间最优）。
> 4. **不要死背状态转移方程**：每道 DP 题的方程都是从"当前状态怎么来"的逻辑推导而来，理解推导过程比背公式重要得多。
