## 四个阶段概览

### 阶段 1：建立网络整体框架（1–2 周）

| 维度 | 内容 |
|------|------|
| **核心目标** | 在脑中建立"数据从 A 到 B 经过了什么"的完整画面 |
| **关键知识** | OSI 七层 / TCP-IP 四层、每层的职责与协议、数据封装与解封装、IP 地址 / 端口 / MAC 地址的角色 |
| **预期成果** | 能在白板上画出分层模型，并讲清一个 HTTP 请求从浏览器到服务器的完整旅程 |
| **验收标准** | 给一个不懂网络的朋友讲明白"你打开百度时发生了什么"，全程不卡壳 |

### 阶段 2：核心协议深入（2–3 周）

| 维度 | 内容 |
|------|------|
| **核心目标** | 深入理解面试高频协议的工作原理和设计思想 |
| **关键知识** | TCP（三次握手、四次挥手、状态机、滑动窗口、拥塞控制、粘包）、UDP（对比 TCP、适用场景）、HTTP/HTTPS（报文结构、方法、状态码、Cookie/Session/Token、TLS 握手）、DNS（解析流程、记录类型、递归与迭代） |
| **预期成果** | 能解释每个协议"为什么这样设计"，而不仅仅是"它是什么" |
| **验收标准** | 随机抽一道 TCP/HTTP 面试题，60 秒内给出结构化回答 |

### 阶段 3：实践能力（1–2 周）

| 维度 | 内容 |
|------|------|
| **核心目标** | 用工具验证理论，建立"看到现象能解释原因"的能力 |
| **关键知识** | Wireshark 抓包分析、tcpdump 命令行抓包、curl 调试接口、Python socket 编程 |
| **预期成果** | 能用 Wireshark 抓到三次握手并逐字段解释；能用 socket 写出 TCP 客户端/服务端 |
| **验收标准** | 给一段抓包截图，能在 2 分钟内分析出问题所在 |

### 阶段 4：面试强化（1 周）

| 维度 | 内容 |
|------|------|
| **核心目标** | 将知识转化为面试中的"可输出能力" |
| **关键知识** | 20+ 高频面试题及回答框架、1 分钟讲清楚核心概念的能力、结合测试开发场景的实际案例 |
| **预期成果** | 每道高频题都有自己的"标准回答"，能脱口而出 |
| **验收标准** | 模拟面试中网络部分得到"满意"评价 |

## 学习原则

1. **先建框架，再填细节** —— 不要一上来就背 TCP 状态转换图，先搞清楚它在整个网络中的位置
2. **理论 + 实践交替** —— 每学一个协议就用工具抓包验证，印象深 10 倍
3. **以面试倒推学习** —— 每个知识点问自己"如果面试官问这个，我能答出什么"
4. **输出倒逼输入** —— 学完一个模块就给别人（或对着镜子）讲一遍，讲不清楚的就是没学透
5. **重 TCP 和 HTTP** —— 测试开发面试中这两个占 70% 以上的网络题目，优先搞透
