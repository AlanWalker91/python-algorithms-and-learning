## 必学内容

### 1.1 OSI 七层模型

```
层号    层名           职责                        关键协议/设备         传输单元
────────────────────────────────────────────────────────────────────────────────
 7     应用层          为用户程序提供网络服务       HTTP, FTP, SMTP, DNS  数据/报文
 6     表示层          数据格式转换、加密、压缩     SSL/TLS, JPEG, ASCII  数据
 5     会话层          建立/管理/终止会话           RPC, NetBIOS          数据
 4     传输层          端到端可靠/不可靠传输        TCP, UDP              段(Segment)
 3     网络层          路由选择、IP 寻址            IP, ICMP, ARP         包(Packet)
 2     数据链路层      帧的封装、MAC 寻址           Ethernet, PPP         帧(Frame)
 1     物理层          比特流传输                   光纤, 网线, Wi-Fi     比特(Bit)
```

**记忆口诀（从上到下）**：应、表、会、传、网、数、物

**面试要点**：OSI 是理论参考模型，实际使用的是 TCP/IP 四层模型。面试时先说 TCP/IP 四层，再补充 OSI 对应关系，显得既实用又有深度。

### 1.2 TCP/IP 四层模型（重点）

```
TCP/IP 四层        对应 OSI 层        核心协议
──────────────────────────────────────────────
应用层              5 + 6 + 7         HTTP, HTTPS, FTP, DNS, SMTP, SSH
传输层              4                 TCP, UDP
网络层（网际层）     3                 IP, ICMP, ARP, RARP
网络接口层          1 + 2             Ethernet, Wi-Fi, PPP
```

### 1.3 数据封装与解封装

这是理解网络通信的关键——数据在每一层被加上头部（有时还有尾部），就像套信封：

```
发送方（自顶向下封装）：

  应用层        [HTTP 数据]
                    │
  传输层        [TCP头][HTTP 数据]                    → 称为"段 Segment"
                    │
  网络层        [IP头][TCP头][HTTP 数据]              → 称为"包 Packet"
                    │
  数据链路层    [MAC头][IP头][TCP头][HTTP 数据][FCS]  → 称为"帧 Frame"
                    │
  物理层        010110110100101...                   → 比特流

接收方（自底向上解封装）：反过来，逐层剥掉头部
```

### 1.4 关键概念辨析

#### IP 地址 vs MAC 地址 vs 端口

| 概念 | 层级 | 作用 | 类比 |
|------|------|------|------|
| **MAC 地址** | 数据链路层 | 标识网卡，局域网内寻址 | 身份证号（出厂固定） |
| **IP 地址** | 网络层 | 标识网络中的设备，跨网络寻址 | 家庭住址（可变） |
| **端口号** | 传输层 | 标识设备上的具体应用程序 | 门牌号 / 房间号 |

**面试常问**：为什么有了 MAC 地址还需要 IP 地址？
> MAC 地址是平面结构，无法做层级路由；IP 地址有网络号 + 主机号的层级结构，路由器靠它做转发决策。MAC 只能在同一个局域网内直接通信，跨网段必须靠 IP。

#### IPv4 地址

```
格式：点分十进制    192.168.1.100
实际：32 位二进制   11000000.10101000.00000001.01100100

私有地址段（不会出现在公网）：
  A 类：10.0.0.0     ~ 10.255.255.255     (10.0.0.0/8)
  B 类：172.16.0.0    ~ 172.31.255.255     (172.16.0.0/12)
  C 类：192.168.0.0   ~ 192.168.255.255    (192.168.0.0/16)

特殊地址：
  127.0.0.1    → 本机回环（localhost）
  0.0.0.0      → 所有网络接口 / 未指定地址
  255.255.255.255 → 广播地址
```

#### 子网掩码与 CIDR

```
IP 地址：    192.168.1.100
子网掩码：   255.255.255.0     (二进制：11111111.11111111.11111111.00000000)
CIDR 写法：  192.168.1.100/24  (/24 表示前 24 位是网络号)

网络地址 = IP & 掩码 = 192.168.1.0    （标识子网）
主机地址 = IP & ~掩码 = 0.0.0.100      （标识子网内主机）

同一子网的设备可以直接通信，不同子网必须经过路由器。
```

### 任务 2：追踪一次真实请求

```bash
# 1. DNS 解析
nslookup www.baidu.com
# 观察：返回了哪些 IP？有几条记录？

# 2. 网络连通性
ping www.baidu.com
# 观察：TTL 是多少？延迟多大？

# 3. 路由追踪
traceroute www.baidu.com    # Linux/Mac
# 或
tracert www.baidu.com       # Windows
# 观察：经过了几跳？哪些是内网 IP，哪些是公网？

# 4. 查看本机网络配置
ifconfig    # Linux/Mac
ipconfig    # Windows
# 找出：你的 IP、子网掩码、默认网关、MAC 地址
```

### 任务 3：Python 验证 DNS 解析

```python
import socket

# 域名 → IP
ip = socket.gethostbyname('www.baidu.com')
print(f"百度 IP: {ip}")

# 获取详细地址信息
infos = socket.getaddrinfo('www.baidu.com', 80)
for info in infos:
    family, socktype, proto, canonname, sockaddr = info
    print(f"  协议族={family}, 类型={socktype}, 地址={sockaddr}")

# 查看本机主机名
print(f"本机主机名: {socket.gethostname()}")
print(f"本机IP: {socket.gethostbyname(socket.gethostname())}")
```

## "浏览器输入 URL 到页面显示"完整流程

这是面试最高频的综合题，必须烂熟于心：

```
用户输入 www.example.com 并按回车

Step 1: URL 解析
   浏览器判断是 URL 还是搜索词 → 补全协议 https://

Step 2: DNS 解析（域名 → IP）
   查找顺序：
   ① 浏览器 DNS 缓存
   ② 操作系统 DNS 缓存
   ③ 本地 hosts 文件
   ④ 本地 DNS 服务器（递归查询）
   ⑤ 根域名服务器 → 顶级域服务器 → 权威域名服务器（迭代查询）
   最终得到：93.184.216.34

Step 3: TCP 三次握手
   客户端                        服务器 93.184.216.34:443
      ── SYN (seq=x) ──────────>
      <── SYN+ACK (seq=y, ack=x+1)
      ── ACK (ack=y+1) ────────>
   连接建立

Step 4: TLS 握手（如果是 HTTPS）
   协商加密算法 → 交换证书 → 生成对称密钥
   此后所有通信加密传输

Step 5: 发送 HTTP 请求
   GET / HTTP/1.1
   Host: www.example.com
   User-Agent: Chrome/120
   Accept: text/html
   ...

Step 6: 服务器处理请求
   Web 服务器(Nginx) → 应用服务器(Django/Spring) → 数据库
   → 生成 HTML 响应

Step 7: 返回 HTTP 响应
   HTTP/1.1 200 OK
   Content-Type: text/html
   Content-Length: 1234
   ...
   <html>...</html>

Step 8: 浏览器解析渲染
   解析 HTML → 构建 DOM 树
   解析 CSS  → 构建 CSSOM 树
   合成渲染树 → 布局(Layout) → 绘制(Paint) → 合成(Composite)
   遇到 <script> → 下载并执行 JS（可能阻塞渲染）
   遇到 <img> 等 → 异步加载资源

Step 9: TCP 四次挥手（连接关闭）
   如果是 keep-alive 则保持连接复用
```
